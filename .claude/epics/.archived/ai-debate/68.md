## Technical Details

### TestModel pattern (Context7-verified)

```python
from pydantic_ai import models
from pydantic_ai.models.test import TestModel

models.ALLOW_MODEL_REQUESTS = False  # module-level guard

@pytest.mark.asyncio
async def test_bull_valid_output() -> None:
    with bull_agent.override(model=TestModel()):
        result = await bull_agent.run("Analyze AAPL", deps=mock_deps)
    assert isinstance(result.output, AgentResponse)
```

### Custom output for controlled tests

```python
test_model = TestModel(custom_output_args={
    "agent_name": "bull",
    "direction": "bullish",
    "confidence": 0.75,
    "argument": "Strong momentum.",
    "key_points": ["RSI up"],
    "risks_cited": ["Earnings risk"],
    "contracts_referenced": ["AAPL 190C 2026-04-18"],
    "model_used": "test",
})
```

### Orchestrator mock pattern

Mock `agent.run` to control behavior:
- Success: return mock result with valid output
- Timeout: raise `TimeoutError`
- Connection error: raise `httpx.ConnectError`
- Validation failure: raise `UnexpectedModelBehavior`

### Fixtures needed

- `mock_market_context`: valid `MarketContext` with realistic AAPL data
- `mock_ticker_score`: valid `TickerScore` with `IndicatorSignals`
- `mock_contract`: valid `OptionContract` with computed Greeks
- `mock_quote`: valid `Quote`
- `mock_ticker_info`: valid `TickerInfo`
