# Task: OHLCV candle integrity validators + OptionGreeks theta/rho NaN defense

## Description

Add Pydantic validators to the `OHLCV` model to reject malformed candles at construction
time, and add `math.isfinite()` validators to `OptionGreeks.theta` and `.rho` to close
the NaN/Inf gap identified in the PRD.

Currently `OHLCV` has zero validators — any values pass through to indicators and pricing.
`OptionGreeks` validates `delta` (range), `gamma`/`vega` (finite + non-negative) but
`theta` and `rho` have no `math.isfinite()` defense.

## Acceptance Criteria

- [ ] `OHLCV.open`, `.high`, `.low`, `.close`, `.adjusted_close`: `field_validator` rejects `<= Decimal("0")` and non-finite (`Decimal("Inf")`, `Decimal("NaN")`, `Decimal("-Inf")`)
- [ ] `OHLCV.volume`: `field_validator` rejects `< 0`
- [ ] `OHLCV` `model_validator(mode="after")` returning `Self`: rejects `high < low`, verifies `low <= open <= high` and `low <= close <= high`
- [ ] `OptionGreeks.theta`: `field_validator` with `math.isfinite()` check (rejects NaN/Inf, allows negative values — theta is normally negative)
- [ ] `OptionGreeks.rho`: `field_validator` with `math.isfinite()` check (rejects NaN/Inf, allows either sign)
- [ ] All existing tests pass (fix any fixtures constructing OHLCV with zero/invalid prices)
- [ ] ~15-20 new tests covering positive, negative, and edge cases for each validator

### Implementation Notes

**OHLCV price validator pattern** (Decimal fields):
```python
@field_validator("open", "high", "low", "close", "adjusted_close")
@classmethod
def validate_price_positive(cls, v: Decimal) -> Decimal:
    if not v.is_finite() or v <= Decimal("0"):
        raise ValueError(f"price must be finite and > 0, got {v}")
    return v
```

**OHLCV candle consistency** (`model_validator(mode="after")` on frozen model):
```python
@model_validator(mode="after")
def validate_candle_consistency(self) -> Self:
    if self.high < self.low:
        raise ValueError(f"high ({self.high}) must be >= low ({self.low})")
    # ... similar for open/close within high-low range
    return self
```

**OptionGreeks theta/rho** — match existing `gamma`/`vega` pattern but allow negative:
```python
@field_validator("theta", "rho")
@classmethod
def validate_finite(cls, v: float) -> float:
    if not math.isfinite(v):
        raise ValueError(f"must be finite, got {v}")
    return v
```

### Key Considerations
- `OHLCV` is `frozen=True` — `model_validator(mode="after")` works for validation without mutation (Context7-verified)
- Decimal fields: use `v.is_finite()` (Decimal method), not `math.isfinite()` (which works on float)
- `volume` is `int` — just check `>= 0`, no finiteness check needed
- Theta is normally negative (time decay costs money) — do NOT add `>= 0` check
- Rho can be either sign — only check finiteness
