# Task: Write unit tests for expanded MarketContext

## Description
Write ~8 unit tests covering the new `MarketContext` fields, `render_context_block()`
conditional rendering, `contract_mid` Decimal serialization, and `build_market_context()`
population of new fields. Tests must use the project's existing fixtures and patterns.

## Acceptance Criteria
- [ ] Test: Construct `MarketContext` with ALL new fields populated ŌĆö all values accessible
- [ ] Test: Construct `MarketContext` with ONLY defaults for new fields ŌĆö backward compatibility
- [ ] Test: `render_context_block()` includes non-None indicators (ADX, SMA ALIGNMENT, etc.)
- [ ] Test: `render_context_block()` omits None indicators (no "None" text in output)
- [ ] Test: `render_context_block()` omits non-finite indicators (NaN/Inf produce no line)
- [ ] Test: `contract_mid` Decimal serialization round-trip via `model_dump_json()` / `model_validate_json()`
- [ ] Test: `build_market_context()` populates new fields from `TickerScore` and `OptionContract`
- [ ] Test: `build_market_context()` uses defaults when contracts list is empty
- [ ] All tests pass: `uv run pytest tests/ -v`
- [ ] `ruff check`, `ruff format`, `mypy --strict` all pass

## Technical Details
- **Test files**: Add tests to existing test files or create new focused test files
  - `tests/unit/models/test_analysis.py` ŌĆö MarketContext field tests, serialization
  - `tests/unit/agents/test_parsing.py` or `tests/unit/agents/test_orchestrator.py` ŌĆö render + build tests
- Follow existing test patterns:
  - `pytest.mark.asyncio` for async tests (if testing `build_market_context`)
  - Build fixtures from real model constructors, not mocks
  - Use `Decimal("...")` string construction for price fields
  - Assert exact field values, not just "is not None"
- For `render_context_block()` tests:
  - Assert specific substrings present/absent in output
  - Test with `adx=75.0` -> output contains "ADX: 75.0"
  - Test with `adx=None` -> output does NOT contain "ADX"
  - Test with `adx=float("nan")` -> output does NOT contain "ADX"
- For `build_market_context()` tests:
  - Build real `TickerScore`, `Quote`, `TickerInfo`, `OptionContract` fixtures
  - Assert new fields match expected source values
  - Test empty contracts -> Greek fields are None, contract_mid is None
- For serialization round-trip:
  - `ctx = MarketContext(contract_mid=Decimal("5.25"), ...)`
  - `json_str = ctx.model_dump_json()`
  - `restored = MarketContext.model_validate_json(json_str)`
  - `assert restored.contract_mid == Decimal("5.25")`
