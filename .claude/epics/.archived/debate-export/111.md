# Task: Add CLI export flags to debate command

## Description
Add `--export` and `--export-dir` options to the `debate` command in `src/options_arena/cli/commands.py`.
After the debate completes and renders to terminal, if `--export` is set, call `export_debate_to_file()`
to write the result to disk. Create the output directory if it doesn't exist.

## Acceptance Criteria
- [ ] `--export` option added: accepts "md" or "pdf" (or None to skip export)
- [ ] `--export-dir` option added: defaults to `"./reports"`
- [ ] Export runs after terminal rendering, not instead of it
- [ ] Output filename follows pattern: `debate_{TICKER}_{YYYY-MM-DD}.{fmt}`
- [ ] Output directory created automatically (`mkdir(parents=True, exist_ok=True)`)
- [ ] Success message printed: `Exported: {path}` (green, via `err_console`)
- [ ] Error handling: catch `ImportError` for PDF and show user-friendly message
- [ ] Error handling: catch `OSError` for file write failures
- [ ] Passes `ruff check`, `ruff format`, and `mypy --strict`

## Technical Details
- **File**: `src/options_arena/cli/commands.py`
- **Import**: `from options_arena.reporting import export_debate_to_file` (lazy, inside if-block)
- **Placement**: After debate result is rendered to terminal, before function returns
- **Current debate command signature** (lines ~192-201):
