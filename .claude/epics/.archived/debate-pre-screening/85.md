# Task: Write tests for pre-screening gate and config expansion

## Description

Write ~8 unit tests covering the `should_debate()` pure function, `DebateConfig` validator
for `min_debate_score`, and env var overrides for the new config fields.

### Test file
Add tests to the existing test structure. Likely locations:
- `tests/agents/test_orchestrator.py` — for `should_debate()` tests
- `tests/models/test_config.py` — for `DebateConfig` validator and env var tests

If these files don't exist, create them following existing test patterns in the `tests/` directory.

### Required tests (~8)

**`should_debate()` logic (5 tests):**
1. Returns `False` for `direction=SignalDirection.NEUTRAL` (any score)
2. Returns `False` for score below `min_debate_score` (e.g., score=20, threshold=30)
3. Returns `True` for `BULLISH` + score above threshold (e.g., score=50, threshold=30)
4. Edge: score exactly at `min_debate_score` returns `True` (inclusive — `>=` not `>`)
5. Edge: score `0.0` with non-NEUTRAL direction returns `False` (below default threshold of 30)

**`DebateConfig` validators (1 test):**
6. Validator rejects `min_debate_score` values > 100, < 0, NaN, Inf (use `pytest.raises(ValidationError)`)

**Env var overrides (2 tests):**
7. `ARENA_DEBATE__MIN_DEBATE_SCORE=50.0` overrides default via `AppSettings()`
8. `ARENA_DEBATE__ENABLE_VOLATILITY_AGENT=true` overrides default via `AppSettings()`

### Test patterns
- Use `monkeypatch.setenv()` for env var tests
- Use `pytest.raises(ValidationError)` for validator rejection tests
- Import `should_debate` from `options_arena.agents.orchestrator`
- Create minimal `TickerScore` and `DebateConfig` fixtures with only required fields
- Follow existing test naming: `test_<function>_<scenario>`
