# Task: Config models with AppSettings

## Description
Implement `config.py` with `ScanConfig`, `PricingConfig`, `ServiceConfig` (all plain `BaseModel`) and `AppSettings` (sole `BaseSettings` subclass). Uses pydantic-settings v2 with `SettingsConfigDict` for env var overrides.

## Acceptance Criteria
- [ ] `src/options_arena/models/config.py` contains `ScanConfig`, `PricingConfig`, `ServiceConfig`, `AppSettings`
- [ ] `ScanConfig(BaseModel)` — fields: `top_n=50`, `min_score=0.0`, `min_price=10.0`, `min_dollar_volume=10_000_000.0`, `ohlcv_min_bars=200`, `adx_trend_threshold=15.0`, `rsi_overbought=70.0`, `rsi_oversold=30.0`
- [ ] `PricingConfig(BaseModel)` — fields per FR-M5.1: risk_free_rate_fallback, delta ranges, DTE range, OI/volume mins, spread/IV solver params
- [ ] `ServiceConfig(BaseModel)` — fields per FR-M5.1: timeouts, rate limiting, cache TTLs, ollama host/model
- [ ] `AppSettings(BaseSettings)` — sole BaseSettings subclass, `SettingsConfigDict(env_prefix="ARENA_", env_nested_delimiter="__")`
- [ ] `AppSettings()` with no args succeeds — all defaults are production-ready
- [ ] Env override works: `ARENA_SCAN__TOP_N=30` → `settings.scan.top_n == 30`
- [ ] Sub-configs are NOT `BaseSettings` — only plain `BaseModel`
- [ ] Passes `ruff check` and `mypy --strict`

## Technical Details
- Import `BaseSettings`, `SettingsConfigDict` from `pydantic_settings`
- Import `BaseModel` from `pydantic` for sub-configs
- Source priority (Context7-verified): init kwargs > env vars > field defaults
- No `.env` file in MVP — can add `env_file=".env"` later without model changes
- CLI creates `AppSettings()`, passes `settings.scan` / `settings.pricing` / `settings.service` to modules
- Modules accept their config slice, never the full `AppSettings`

## Dependencies
- Issue #3 (pydantic-settings must be installed)
- Does NOT depend on enums — no enum imports needed
