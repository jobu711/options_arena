# Task: Project scaffold & tooling configuration

## Description
Initialize the `options_arena` project from scratch using `uv`. Create `pyproject.toml` with all runtime and dev dependencies, configure ruff, mypy, and pytest, and scaffold the directory structure with empty `__init__.py` files.

## Acceptance Criteria
- [ ] `uv init` creates the project with `pyproject.toml`
- [ ] All runtime dependencies added via `uv add`: pydantic, pandas, numpy, aiosqlite, yfinance, httpx, scipy, pydantic-ai, ollama, typer, rich, pydantic-settings
- [ ] All dev dependencies added via `uv add --dev`: ruff, mypy, pytest, pytest-asyncio, pytest-cov, pandas-stubs, scipy-stubs
- [ ] ruff config: target Python 3.13, line length 99, rules E/F/I/UP/B/SIM/ANN
- [ ] mypy config: `strict = true`, `warn_return_any = true`, `warn_unused_configs = true`
- [ ] pytest config: asyncio mode enabled
- [ ] Directory structure created:
  - `src/options_arena/__init__.py` (with `__version__ = "0.1.0"`)
  - `src/options_arena/models/`
  - `src/options_arena/utils/`
  - `src/options_arena/services/`
  - `src/options_arena/indicators/`
  - `src/options_arena/pricing/`
  - `src/options_arena/scoring/`
  - `src/options_arena/scan/`
  - `tests/`
  - `tests/unit/`
  - `tests/unit/models/`
- [ ] `uv run python -c "import options_arena"` succeeds

## Technical Details
- Use `uv init` for project creation, then `uv add` for each dependency
- Source layout: `src/options_arena/` (src-based)
- Build backend: Hatchling
- All tool configs in `pyproject.toml` (no separate config files)
- This task creates the foundation all other tasks build on
