# Task: Unit tests — data models, validation & serialization (~105 tests)

## Description
Write comprehensive unit tests for all data models: construction, frozen behavior, computed fields, field validation, defaults, and JSON serialization roundtrips. This is the largest test task and covers the bulk of the ~150 test target.

## Acceptance Criteria

- [ ] OHLCV: happy path construction, frozen enforcement, Decimal precision
- [ ] Quote: happy path, frozen, timestamp with UTC

- [ ] TickerInfo: `dividend_rate` and `trailing_dividend_rate` accept `None`
- [ ] All three: JSON roundtrip `model_validate_json(m.model_dump_json()) == m`

### Options Tests (`tests/unit/models/test_options.py`, ~30 tests)

- [ ] OptionGreeks: delta out of [-1, 1] raises ValidationError
- [ ] OptionGreeks: negative gamma raises ValidationError
- [ ] OptionGreeks: negative vega raises ValidationError
- [ ] OptionContract: happy path with all fields

- [ ] OptionContract: `mid` computed field = (bid + ask) / Decimal("2")
- [ ] OptionContract: `spread` computed field = ask - bid

- [ ] All tests pass: `uv run pytest tests/unit/models/ -v`

## Technical Details
- Use `pytest.raises(ValidationError)` for validation boundary tests
- Use `datetime.date.today()` awareness for DTE tests (compute expected value)
- Decimal construction always from string: `Decimal("1.05")`
- JSON roundtrip pattern: `Model.model_validate_json(m.model_dump_json()) == m`
- Test frozen with: assign attribute → catch `ValidationError`
- Use fixtures for commonly reused model instances
