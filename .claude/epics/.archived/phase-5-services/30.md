# Task: Retry & Rate Limiting Infrastructure

## Description
Build the foundational infrastructure modules for the services layer: `helpers.py` (retry logic + safe type conversions) and `rate_limiter.py` (token bucket + async semaphore). These are shared by all service modules and have no external dependencies beyond the existing exception hierarchy.

## Acceptance Criteria
- [ ] `fetch_with_retry()` implements exponential backoff (1s тЖТ 2s тЖТ 4s тЖТ 8s тЖТ 16s, capped at `max_delay`)
- [ ] `fetch_with_retry()` accepts a zero-arg async factory (not a coroutine) for re-invocation on each retry
- [ ] `fetch_with_retry()` raises the last exception after `max_retries` exhausted
- [ ] `fetch_with_retry()` logs each retry at WARNING level
- [ ] `safe_decimal()` converts values to `Decimal` safely, returns `None` on failure
- [ ] `safe_int()` converts values to `int` safely, returns `None` on failure
- [ ] `safe_float()` converts values to `float` safely, rejects `NaN`/`┬▒inf` via `math.isfinite()`, returns `None` on failure
- [ ] All safe converters log conversion failures at DEBUG level
- [ ] `RateLimiter` class with token bucket + `asyncio.Semaphore` dual-layer rate control
- [ ] `RateLimiter` supports async context manager (`async with limiter:`)
- [ ] Token bucket refills at configurable `rate` per second up to burst capacity
- [ ] Semaphore limits concurrent in-flight requests to `max_concurrent`
- [ ] ~24 tests covering retry success/exhaustion, backoff timing, safe conversions, token depletion, semaphore limiting, concurrent access

## Technical Details
- **Files to create**: `src/options_arena/services/helpers.py`, `src/options_arena/services/rate_limiter.py`
