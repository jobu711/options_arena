# Task: Health Service

## Description
Build `health.py` — pre-flight health checks for all external dependencies (yfinance, FRED, Ollama, CBOE). Returns typed `HealthStatus` models with latency measurements and error details. Standalone module with no dependencies on other service modules.

## Acceptance Criteria
- [ ] `HealthService` class with DI constructor (`ServiceConfig`)
- [ ] `check_yfinance()` → `HealthStatus` — tests by fetching SPY quote
- [ ] `check_fred()` → `HealthStatus` — HTTP HEAD to FRED API endpoint
- [ ] `check_ollama()` → `HealthStatus` — `GET {config.ollama_host}/api/tags`, verifies configured model available
- [ ] `check_cboe()` → `HealthStatus` — checks CBOE CSV download endpoint
- [ ] `check_all()` → `list[HealthStatus]` — runs all checks concurrently via `asyncio.gather(return_exceptions=True)`
- [ ] Each check measures wall-clock latency via `time.monotonic()`
- [ ] Each check returns `HealthStatus(available=True/False, latency_ms=..., error=...)`
- [ ] One failed check never blocks the others in `check_all()`
- [ ] `close()` closes httpx client
- [ ] ~10 tests covering each service up/down, latency recording, concurrent execution, partial failures

## Technical Details
- **Files to create**: `src/options_arena/services/health.py`
- **Test files**: `tests/unit/services/test_health.py`
- Uses `httpx.AsyncClient` for FRED, Ollama, CBOE checks
- Uses `yfinance` (via `asyncio.to_thread()`) for yfinance check
- Latency: `start = time.monotonic()` ... `latency_ms = (time.monotonic() - start) * 1000`
- `HealthStatus` model already exists from Phase 1
