# Task: Cache Layer

## Description
Build the two-tier caching system (`cache.py`): in-memory LRU for hot data (quotes, chains) and persistent SQLite for cold data (OHLCV, fundamentals). TTLs adjust based on whether US options markets are currently open.

## Acceptance Criteria
- [ ] `ServiceCache` class with DI constructor accepting `ServiceConfig` and optional `db_path`
- [ ] `get()` returns `bytes | None`, checks in-memory first, then SQLite, respects TTL
- [ ] `set()` stores value with TTL; short-TTL data (quotes, chains) stays in-memory only
- [ ] `invalidate()` removes key from both tiers
- [ ] `clear()` removes all cached data from both tiers
- [ ] `close()` closes SQLite connection cleanly
- [ ] `ttl_for()` returns market-hours-aware TTL based on data type string
- [ ] `is_market_hours()` correctly detects 9:30–16:00 ET, Mon–Fri
- [ ] Named TTL constants (no magic numbers): `TTL_OHLCV`, `TTL_CHAIN_MARKET`, `TTL_CHAIN_AFTER`, `TTL_QUOTE_MARKET`, `TTL_QUOTE_AFTER`, `TTL_FUNDAMENTALS`, `TTL_REFERENCE`, `TTL_FAILURE`
- [ ] In-memory tier: LRU eviction when entry count exceeds configurable max (default 1000)
- [ ] In-memory tier: lazy expiry check on `get()`, periodic sweep every 100 `get()` calls
- [ ] SQLite tier: `aiosqlite` with WAL mode, table `service_cache(key TEXT PRIMARY KEY, value BLOB, expires_at REAL)`
- [ ] Cache key format: `{source}:{type}:{ticker}:{params}`
- [ ] ~15 tests covering set/get, TTL expiry, LRU eviction, market-hours TTL selection, SQLite persistence, invalidation, failure caching

## Technical Details
- **Files to create**: `src/options_arena/services/cache.py`
- **Test files**: `tests/unit/services/test_cache.py`
- Uses `zoneinfo.ZoneInfo("America/New_York")` for market hours — never naive datetimes
- Serialization is caller's responsibility (e.g., `model.model_dump_json().encode()`)
- SQLite lazy deletion: expired rows cleaned up on access, not proactively
- `TTL_OHLCV = 0` means permanent — historical data is immutable
