# Task: Integration tests + verification gate

## Description
Write integration tests that exercise the full `ScanPipeline` end-to-end with mock services, verify edge cases (cancellation, empty universe, all failures), confirm progress callback invocation order, and pass the full verification gate (ruff + pytest + mypy --strict).

## Acceptance Criteria
- [ ] Full pipeline end-to-end test with mock services (all 4 phases, happy path)
- [ ] Cancelled mid-pipeline returns partial `ScanResult` with `cancelled=True`
- [ ] `ScanResult.phases_completed` reflects actual progress (0–4)
- [ ] Empty universe (0 tickers) → graceful empty result (no crash)
- [ ] All tickers fail OHLCV → graceful empty result (no crash)
- [ ] `ProgressCallback` invocation order: `UNIVERSE` → `SCORING` → `OPTIONS` → `PERSIST`
- [ ] Re-export verification: all public names importable from `options_arena.scan`
- [ ] ~8 integration tests written and passing
- [ ] Full verification gate passes: `uv run ruff check . --fix && uv run ruff format . && uv run pytest tests/ -v && uv run mypy src/ --strict`
- [ ] Total test count: ~905 existing + ~95 new ≈ ~1000

## Technical Details
- Integration tests go in `tests/integration/scan/test_scan_pipeline.py`
- Use mock services (not real API calls) — construct with `unittest.mock.AsyncMock` or custom fakes
- `RecordingCallback` (from Task #48 tests) records `(phase, current, total)` tuples for order verification
- Edge cases to test:
  - Cancel token set before Phase 1 → `phases_completed=0`, `cancelled=True`
  - Cancel token set between Phase 2 and Phase 3 → `phases_completed=2`, `cancelled=True`
  - `UniverseService.fetch_optionable_tickers()` returns empty list → empty `ScanResult`
  - `MarketDataService.fetch_batch_ohlcv()` returns all failures → empty scoring, empty options
  - `FredService.fetch_risk_free_rate()` returns fallback (it never raises)
- Re-export test: `from options_arena.scan import CancellationToken, ProgressCallback, ScanPhase, ScanPipeline, ScanResult`
