# Task: Repository Queries — Accuracy, Calibration, Weight CRUD

## Description

Add 4 new async query methods to `data/repository.py` for agent accuracy tracking,
confidence calibration, and auto-tune weight persistence. These methods JOIN
`agent_predictions` with `contract_outcomes` via `recommended_contract_id` and use
T+10 (`holding_days=10`) as the canonical evaluation period.

## Acceptance Criteria

- [ ] `get_agent_accuracy(window_days: int | None) -> list[AgentAccuracyReport]` — JOIN + GROUP BY, 10-sample minimum, optional time window
- [ ] `get_agent_calibration(agent_name: str | None) -> AgentCalibrationData` — 5-bucket binning with per-agent or aggregate view
- [ ] `get_latest_auto_tune_weights() -> list[AgentWeightsComparison]` — most recent weight set from `auto_tune_weights` table
- [ ] `save_auto_tune_weights(weights: list[AgentWeightsComparison], window_days: int) -> None` — persist computed weights
- [ ] Direction accuracy: `BULLISH` correct when `stock_return_pct > 0`, `BEARISH` correct when `stock_return_pct < 0`
- [ ] Brier score: `AVG((confidence - outcome_binary)^2)` where `outcome_binary` = 1 if direction correct, 0 otherwise
- [ ] 10-sample minimum enforced — agents with < 10 matched outcomes excluded from results
- [ ] NULL `recommended_contract_id` rows gracefully excluded (no crash)
- [ ] All returned models are typed (`AgentAccuracyReport`, `AgentCalibrationData`, `AgentWeightsComparison`)

## Technical Details

### JOIN Path
```sql
agent_predictions ap
JOIN contract_outcomes co ON ap.recommended_contract_id = co.recommended_contract_id
WHERE co.holding_days = 10
```

### Direction Hit Logic (in SQL)
```sql
CASE
    WHEN ap.direction = 'BULLISH' AND co.stock_return_pct > 0 THEN 1
    WHEN ap.direction = 'BEARISH' AND co.stock_return_pct < 0 THEN 1
    ELSE 0
END AS direction_correct
```

### get_agent_accuracy SQL sketch
```sql
SELECT
    ap.agent_name,
    AVG(direction_correct) AS direction_hit_rate,
    AVG(ap.confidence) AS mean_confidence,
    AVG(POWER(ap.confidence - direction_correct, 2)) AS brier_score,
    COUNT(*) AS sample_size
FROM agent_predictions ap
JOIN contract_outcomes co ON ap.recommended_contract_id = co.recommended_contract_id
WHERE co.holding_days = 10
  AND ap.direction IS NOT NULL
  [AND ap.created_at >= datetime('now', '-{window_days} days')]
GROUP BY ap.agent_name
HAVING COUNT(*) >= 10
```

### get_agent_calibration SQL sketch
5 buckets: `[0.0, 0.2)`, `[0.2, 0.4)`, `[0.4, 0.6)`, `[0.6, 0.8)`, `[0.8, 1.0]`.
Use CASE WHEN to bin confidence values, compute mean confidence and actual hit rate per bucket.

### Edge Cases
- All agents have < 10 outcomes → empty accuracy list
- `recommended_contract_id` is NULL → row excluded from JOIN
- `stock_return_pct` is exactly 0.0 → direction_correct = 0 (neutral)
- Window of 0 days → effectively empty
- Confidence exactly at bucket boundary (0.2, 0.4, etc.)
