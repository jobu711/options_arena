# Task: Integration Tests — End-to-End Calibration Pipeline

## Description

Write integration tests that verify the full calibration pipeline: seeded predictions →
outcome collection → accuracy queries → weight computation → orchestrator integration.
Also validate cross-task interactions and ensure backward compatibility of the full debate
flow with auto-tune disabled and enabled.

## Acceptance Criteria

- [ ] End-to-end test: seed predictions + outcomes → `get_agent_accuracy()` → `compute_auto_tune_weights()` → `synthesize_verdict(vote_weights=...)` → valid `ExtendedTradeThesis`
- [ ] Backward compatibility test: `run_debate()` with `auto_tune_weights=False` produces identical results to current behavior
- [ ] Auto-tune enabled test: `run_debate()` with `auto_tune_weights=True` loads and uses weights
- [ ] Weight constraint validation: computed weights satisfy sum=0.85, floor/cap, risk=0.0
- [ ] CLI + API integration: verify CLI commands and API endpoints return consistent data from same DB
- [ ] 10-sample guard integration: verify pipeline correctly excludes agents with insufficient data
- [ ] All tests use in-memory SQLite with migrations applied

## Technical Details

### Test Strategy
Integration tests use in-memory SQLite (`Database(":memory:")`) with all migrations applied.
Seed realistic data: 6 agents × 15 predictions each = 90 predictions, with matching
`contract_outcomes` records. Some agents get < 10 outcomes to test the minimum sample guard.

1. **Prediction → Outcome JOIN**: Verify the FK path works with realistic data
2. **Accuracy → Weights**: Verify computed weights reflect accuracy differences
3. **Weights → Verdict**: Verify different weights produce different verdict confidence
4. **Config toggle**: Verify `auto_tune_weights=False` vs `True` behavior difference

### Edge Cases
- Zero outcomes for all agents → no accuracy data, weight computation returns manual
- All agents perfectly accurate (Brier=0.0) → equal weights, sum=0.85
- All agents wrong (Brier=1.0) → floor weights, renormalized
- Mixed: some accurate, some not → weights reflect the difference

### Mocking Strategy
- `Database(":memory:")` — real SQLite, no mocks for persistence
- `AsyncMock` for LLM calls in `run_debate()` (use `TestModel` or mock agent outputs)
- Real `compute_auto_tune_weights()` — no mocking the weight function
- Real repository queries — no mocking the SQL
