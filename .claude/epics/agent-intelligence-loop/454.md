# Task: auto_tune_weights() orchestration function

## Description
Add `auto_tune_weights()` async function to `agents/orchestrator.py` that connects the existing dead-code primitives: `get_agent_accuracy()` -> `compute_auto_tune_weights()` -> build `AgentWeightsComparison` list -> optionally `save_auto_tune_weights()`. This is the missing glue that activates the intelligence loop.

## Acceptance Criteria
- [ ] `auto_tune_weights(repo, window_days=90, dry_run=False) -> list[AgentWeightsComparison]` added to `agents/orchestrator.py`
- [ ] Function calls `repo.get_agent_accuracy(window_days=window_days)`
- [ ] Function calls `compute_auto_tune_weights(accuracy)` (existing pure function)
- [ ] Function builds `AgentWeightsComparison` for each agent with manual weight from `AGENT_VOTE_WEIGHTS`
- [ ] When `dry_run=False`, calls `repo.save_auto_tune_weights(comparisons, window_days=window_days)`
- [ ] When `dry_run=True`, skips persistence
- [ ] Function re-exported from `agents/__init__.py`
- [ ] Unit tests with mocked repository

## Technical Details
- Lives in `agents/orchestrator.py` next to `compute_auto_tune_weights()` (~line 912)
- Imports: uses `AGENT_VOTE_WEIGHTS` (same file), `AgentWeightsComparison` from models, `Repository` type for annotation
- `brier_score` and `sample_size` extracted from matching `AgentAccuracyReport` by `agent_name`
- Architecture boundary: `agents/` can access `models/` and `data/` per boundary table

### Edge Cases
- Empty accuracy data: returns empty list (no agents met threshold)
- Agent in accuracy but not in AGENT_VOTE_WEIGHTS: manual_weight defaults to 0.0
- Agent with brier_score=None (< 10 samples): passed through
