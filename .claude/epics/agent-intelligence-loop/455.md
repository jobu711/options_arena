# Task: outcomes auto-tune CLI subcommand

## Description
Add `outcomes auto-tune [--dry-run] [--window 90]` subcommand to `cli/outcomes.py`. Calls `auto_tune_weights()` from task 454, renders a Rich table showing per-agent comparison (manual vs tuned weight, delta, Brier score, sample count), and prints status message.

## Acceptance Criteria
- [ ] `@outcomes_app.command("auto-tune")` added to `cli/outcomes.py`
- [ ] `--dry-run` flag (default False): shows weights without persisting
- [ ] `--window` option (default 90, range 1-365): lookback window in days
- [ ] Rich table columns: Agent | Manual | Tuned | Delta | Brier | Samples
- [ ] Delta column shows `tuned - manual` with green (positive) / red (negative) styling
- [ ] Dry run prints "[DRY RUN] Weights not saved."
- [ ] Non-dry-run prints "Weights saved. Next debate will use tuned weights."
- [ ] Handles empty results gracefully (no outcomes yet)
- [ ] Follows existing CLI patterns (sync command + asyncio.run, AppSettings, Database lifecycle)

## Technical Details
- Pattern: copy `agent_weights_cmd` / `_agent_weights_async` structure from same file
- Import `auto_tune_weights` from `agents.orchestrator` (or `agents` package re-export)
- Delta = `auto_weight - manual_weight`, format as `+0.034` / `-0.028`
- Color: `Text(delta_str, style="green")` if positive, `"red"` if negative, dim if zero
- Brier: format to 3 decimals, `"--"` if None
- DB lifecycle: create `Database`, `connect()`, create `Repository`, call function, `close()` in `finally`
