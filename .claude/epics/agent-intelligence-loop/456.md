# Task: Auto-tune API endpoints

## Description
Add two endpoints to `api/routes/analytics.py`: `POST /api/analytics/weights/auto-tune` to trigger weight computation (with optional dry_run and window query params), and `GET /api/analytics/weights/history` to retrieve historical weight snapshots for the evolution chart.

## Acceptance Criteria
- [ ] `POST /api/analytics/weights/auto-tune` endpoint added
- [ ] Accepts `window: int = Query(90, ge=1, le=365)` and `dry_run: bool = Query(False)`
- [ ] Returns `list[AgentWeightsComparison]`
- [ ] `GET /api/analytics/weights/history` endpoint added
- [ ] Accepts `limit: int = Query(20, ge=1, le=100)`
- [ ] Returns `list[WeightSnapshot]`
- [ ] Both use existing `Depends(get_repo)` pattern
- [ ] Unit tests with mocked dependencies

## Technical Details
- Add to existing `router` in `api/routes/analytics.py`
- POST endpoint calls `auto_tune_weights(repo, window_days=window, dry_run=dry_run)` from agents
- GET endpoint calls `repo.get_weight_history(limit=limit)`
- Import `auto_tune_weights` from `agents` package
- Import `WeightSnapshot` from `models` package
- Follow existing endpoint patterns (e.g., `get_agent_weights` at line 174)
