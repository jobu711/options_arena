# Task: Weight Tuning tab (types, store, panel, page integration)

## Description
Build the Weight Tuning tab on the Analytics page. Includes TypeScript interfaces, Pinia store additions, `WeightTuningPanel.vue` component, and AnalyticsPage tab integration. The panel shows a comparison table (manual vs tuned weights), an Auto-Tune trigger button, and a weight evolution line chart.

## Acceptance Criteria
- [ ] `web/src/types/weights.ts` created with `AgentWeight` and `WeightSnapshot` TS interfaces
- [ ] Pinia store (in `backtest.ts` or new `weights.ts`) with state, fetch, and trigger actions
- [ ] `web/src/components/analytics/WeightTuningPanel.vue` created
- [ ] Panel has: comparison DataTable, "Auto-Tune" button, weight evolution Chart
- [ ] Empty state shown when no tuned weights exist ("No tuned weights yet. Click Auto-Tune.")
- [ ] Auto-Tune button POSTs to `/api/analytics/weights/auto-tune`, refreshes data on success
- [ ] History chart fetches from `/api/analytics/weights/history`, renders line chart (one line per agent)
- [ ] Loading and error states handled
- [ ] Toast notification on auto-tune success/failure
- [ ] Tab added to `AnalyticsPage.vue` with lazy loading on tab change

## Technical Details

### TypeScript interfaces (`web/src/types/weights.ts`)
```typescript
export interface AgentWeight {
  agent_name: string
  manual_weight: number
  auto_weight: number
  brier_score: number | null
  sample_size: number
}

export interface WeightSnapshot {
  computed_at: string  // ISO datetime
  window_days: number
  weights: AgentWeight[]
}
```

### Store additions
- `weights: AgentWeight[]` state
- `weightHistory: WeightSnapshot[]` state
- `fetchWeights()` action -> `GET /api/analytics/agent-weights`
- `fetchWeightHistory()` action -> `GET /api/analytics/weights/history`
- `triggerAutoTune(window?: number)` action -> `POST /api/analytics/weights/auto-tune`
- Loading flags per action

### Panel layout
- **DataTable** (PrimeVue): columns for Agent, Manual, Tuned, Delta, Brier, Samples
- Delta column: computed as `auto_weight - manual_weight`, color-coded green/red
- **Button** (PrimeVue): "Auto-Tune" with loading state during POST
- **Chart** (PrimeVue Chart component / Chart.js): line chart with one dataset per agent, x-axis = computed_at dates, y-axis = weight value
- Color palette: match existing analytics chart colors

### AnalyticsPage integration
- Add tab entry to the tab list
- Lazy load panel on tab activation (same pattern as other analytics tabs)
