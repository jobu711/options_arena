# Task: Orchestrator and fallback

## Description

Create `agents/orchestrator.py` with `run_debate()` (sequential agent coordination, error
handling, persistence), `build_market_context()` (maps scan pipeline output to `MarketContext`),
and the data-driven fallback synthesis logic. Update `agents/__init__.py` re-exports.

The orchestrator follows the FRED-style "never-raises" pattern: debate failure returns a
data-driven fallback, never crashes the caller.

## Technical Details

### run_debate() flow (FR-D2, FR-D4)

```python
async def run_debate(...) -> DebateResult:
    context = build_market_context(ticker_score, quote, ticker_info, contracts)
    model = build_ollama_model(config)
    start = time.monotonic()
    try:
        # 1. Bull
        bull_deps = DebateDeps(context=context, ticker_score=ticker_score, contracts=contracts)
        bull_result = await asyncio.wait_for(bull_agent.run(..., model=model, deps=bull_deps), timeout=config.ollama_timeout)
        # 2. Bear (receives bull argument)
        bear_deps = DebateDeps(..., opponent_argument=bull_result.output.argument)
        bear_result = await asyncio.wait_for(bear_agent.run(..., model=model, deps=bear_deps), timeout=config.ollama_timeout)
        # 3. Risk (receives both)
        risk_deps = DebateDeps(..., bull_response=bull_result.output, bear_response=bear_result.output)
        risk_result = await asyncio.wait_for(risk_agent.run(..., model=model, deps=risk_deps), timeout=config.ollama_timeout)
        # Accumulate usage
        total_usage = bull_result.usage() + bear_result.usage() + risk_result.usage()
        duration_ms = int((time.monotonic() - start) * 1000)
        result = DebateResult(context, bull_result.output, bear_result.output, risk_result.output, total_usage, duration_ms, is_fallback=False)
    except (...):
        result = _build_fallback_result(context, ticker_score, contracts, config, start)
    # Persist if repository provided
    if repository:
        await repository.save_debate(...)
    return result
```

### build_market_context() mapping (FR-D11)

- `TickerScore.signals.rsi` → `rsi_14` (use 0.0 if None)
- `Quote.price` → `current_price`
- `TickerInfo.fifty_two_week_high/low` → `price_52w_high/low`
- First contract → `target_strike`, `target_delta`, `dte_target`
- `ExerciseStyle.AMERICAN` (always for US equity)
- Options-specific indicators (`iv_rank`, `iv_percentile`, etc.) default to 0.0 if None

### Data-driven fallback (FR-D5)

- Bull `AgentResponse`: argument cites composite score + positive indicators, confidence = `score/100 * 0.3`
- Bear `AgentResponse`: argument cites risk factors + negative indicators, confidence = `(100 - score)/100 * 0.3`
- `TradeThesis`: confidence fixed at `config.fallback_confidence`, summary includes "Data-driven analysis (AI unavailable)"
- `is_fallback = True` triggers yellow warning banner in CLI
