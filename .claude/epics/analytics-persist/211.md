# Task: Contract and Normalization Persistence

## Description

Wire the analytics models from Task 210 into the data layer and scan pipeline. Add Repository methods for saving/getting recommended contracts and normalization stats. Modify the scan pipeline to capture entry stock prices in Phase 3 and persist contracts + normalization metadata in Phase 4. Add `compute_normalization_stats()` to the scoring module.

## Technical Details

### `scan/models.py` Changes

Add `entry_prices` field to `OptionsResult`:

```python
class OptionsResult:
    # ... existing fields ...
    entry_prices: dict[str, Decimal] = {}  # ticker → spot price at scan time
```

This is NOT frozen (OptionsResult allows mutation). Default empty dict.

### `scan/pipeline.py` Changes

**Phase 3 (`_process_ticker_options` or equivalent):**
- After fetching `ticker_info`, capture `ticker_info.current_price` into `options_result.entry_prices[ticker]`
- This spot price is already available — `_process_ticker_options` fetches `ticker_info` for `dividend_yield` and `current_price`

**Phase 4 (`_phase_persist`):**
- After saving `ScanRun` and `TickerScore`, build `RecommendedContract` list:
  - Iterate `options_result.recommendations` (dict[str, list[OptionContract]])
  - For each ticker's contracts, look up `entry_prices[ticker]` for `entry_stock_price`
  - Compute `entry_mid` from contract's `mid` computed field
  - Get `direction` and `composite_score` from corresponding `TickerScore`
  - Set `risk_free_rate` from `options_result.risk_free_rate`
  - Set `created_at` to current UTC time
- Call `repo.save_recommended_contracts(scan_id, contracts)`
- Build `NormalizationStats` list from pipeline state (computed in Phase 2)
- Call `repo.save_normalization_stats(scan_id, stats)`

### `scoring/normalization.py` Changes

Add `compute_normalization_stats()`:

```python
def compute_normalization_stats(
    raw_signals: dict[str, IndicatorSignals],
) -> list[NormalizationStats]:
    """Compute per-indicator distribution metadata from raw signals.

    For each indicator field on IndicatorSignals, collects all non-None values
    across all tickers and computes min, max, median, mean, std_dev, p25, p75.
    """
```

- Uses `get_active_indicators()` (already exists) to identify which indicators have data
- For each active indicator, collect values from `raw_signals`, compute stats via numpy
- Returns list of `NormalizationStats` (without `id`, `scan_run_id`, `created_at` — those are set at persist time)
- Actually, `scan_run_id` and `created_at` need to be set at persist time. So the function returns stats without those fields. The pipeline will need to set them before persisting. Consider either:
  - Having `compute_normalization_stats()` return a simpler intermediate type
  - Or having `NormalizationStats` accept `scan_run_id=0` and `created_at=UTC.now()` as defaults that get overwritten

**Decision:** Return `NormalizationStats` with `scan_run_id=0` (placeholder) and `created_at=datetime.now(UTC)`. The pipeline sets the real `scan_run_id` at persist time. Since `NormalizationStats` is frozen, the pipeline builds new instances with the correct `scan_run_id`.

### `data/repository.py` Changes

Add 5 methods grouped under `# --- Analytics: Contracts & Normalization ---`:

1. `save_recommended_contracts(scan_id: int, contracts: list[RecommendedContract]) -> None`
   - Use `executemany()` for batch insert
   - Store Decimal as `str(value)`, floats directly
   - `await db.commit()`

2. `get_contracts_for_scan(scan_id: int) -> list[RecommendedContract]`
   - SELECT * WHERE scan_run_id = ?
   - Use `_row_to_recommended_contract()` converter

3. `get_contracts_for_ticker(ticker: str, limit: int = 50) -> list[RecommendedContract]`
   - SELECT * WHERE ticker = ? ORDER BY created_at DESC LIMIT ?
   - Returns most recent contracts first

4. `save_normalization_stats(scan_id: int, stats: list[NormalizationStats]) -> None`
   - Use `executemany()`
   - `await db.commit()`

5. `get_normalization_stats(scan_id: int) -> list[NormalizationStats]`
   - SELECT * WHERE scan_run_id = ?
   - Use `_row_to_normalization_stats()` converter

### Edge Cases
- Empty recommendations dict (no contracts recommended for any ticker)
- Ticker with entry price but no contracts (passed liquidity filter but no options matched)
- All Greek fields None (contract where Greeks computation failed)
- Duplicate contract insert (same scan, ticker, strike, expiry — should raise IntegrityError)
- Single-ticker normalization (min == max)
- All-None indicator across all tickers (should be excluded from stats)
