# Task: Add conditional ModelSettings for extended thinking in orchestrator

## Description

Add conditional `ModelSettings` construction in the orchestrator for Anthropic extended thinking. When `config.provider == LLMProvider.ANTHROPIC` and `config.enable_extended_thinking` is true, build `ModelSettings` with `temperature=1.0` (Anthropic requirement) and `anthropic_thinking` dict. Two call sites need updating (v1 `_run_agents` and v2 `_run_agents_v2`).

## Acceptance Criteria

- [ ] When `provider=ANTHROPIC` + `enable_extended_thinking=True`: `ModelSettings` has `temperature=1.0` and `anthropic_thinking={"type": "enabled", "budget_tokens": config.thinking_budget_tokens}`
- [ ] When `provider=ANTHROPIC` + `enable_extended_thinking=False`: standard `ModelSettings(temperature=config.temperature)`
- [ ] When `provider=GROQ`: standard `ModelSettings(temperature=config.temperature)` (unchanged)
- [ ] Both v1 and v2 call sites updated consistently
- [ ] Existing orchestrator tests pass (no regressions)

### Implementation Notes
- Read `agents/CLAUDE.md` before editing
- Import `LLMProvider` from `options_arena.models`
- Extract a helper function to avoid duplicating the conditional at both call sites:
  ```python
  def _build_model_settings(config: DebateConfig) -> ModelSettings:
      if config.provider == LLMProvider.ANTHROPIC and config.enable_extended_thinking:
          return ModelSettings(
              temperature=1.0,
              anthropic_thinking={
                  "type": "enabled",
                  "budget_tokens": config.thinking_budget_tokens,
              },
          )
      return ModelSettings(temperature=config.temperature)
  ```
- Replace both `settings = ModelSettings(temperature=config.temperature)` lines with `settings = _build_model_settings(config)`
- The `model` variable is already `Model` type — no changes to model construction
- `anthropic_thinking` is a dict passed via `ModelSettings` — verify field name via Context7 if unsure
