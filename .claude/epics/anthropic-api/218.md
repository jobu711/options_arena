# Task: Add check_anthropic() health check to HealthService

## Description

Add `check_anthropic()` method to `HealthService` in `services/health.py`, following the exact `check_groq()` pattern. Resolves API key from `ServiceConfig.anthropic_api_key > ANTHROPIC_API_KEY env > None`. Hits `https://api.anthropic.com/v1/models` with `x-api-key` header. Include in `check_all()` results.

## Acceptance Criteria

- [ ] `check_anthropic()` returns `HealthStatus(service_name="anthropic", ...)`
- [ ] API key resolved: `ServiceConfig.anthropic_api_key > ANTHROPIC_API_KEY env > None`
- [ ] No key → `available=False, error="no API key configured"`
- [ ] 401 response → `available=False, error="invalid API key"`
- [ ] 200 response → `available=True` with latency
- [ ] Exception → `available=False, error=str(exc)` with latency
- [ ] `check_all()` includes Anthropic in results
- [ ] Never raises — all exceptions caught and returned as `HealthStatus`
