# Task: WeightSnapshot model + weight history query

## Description
Add `WeightSnapshot` Pydantic model to `models/analytics.py` and `get_weight_history()` method to `data/_debate.py`. The model wraps a single auto-tune snapshot (timestamp + window + per-agent comparisons). The repo method groups `auto_tune_weights` rows by `created_at` DESC, reconstructing snapshots, and returns the N most recent.

## Acceptance Criteria
- [ ] `WeightSnapshot` model added to `models/analytics.py` with `frozen=True`
- [ ] `WeightSnapshot` has fields: `computed_at: datetime` (UTC-validated), `window_days: int`, `weights: list[AgentWeightsComparison]`
- [ ] `WeightSnapshot` re-exported from `models/__init__.py` and `models/analytics.py`
- [ ] `get_weight_history(limit: int = 20) -> list[WeightSnapshot]` added to `DebateMixin` in `data/_debate.py`
- [ ] Query groups `auto_tune_weights` rows by `created_at`, orders DESC, limits to N snapshots
- [ ] Each snapshot includes `window_days` from the DB row
- [ ] Returns empty list when no auto-tune weights exist
- [ ] Unit tests for model validation and repo method

## Technical Details
- `WeightSnapshot` follows existing `AgentWeightsComparison` patterns in `analytics.py`
- UTC validator on `computed_at` field (same pattern as other datetime fields in the codebase)
- `window_days` validator: `>= 1`
- `weights` validator: non-empty list
- Repo query: `SELECT DISTINCT created_at, window_days FROM auto_tune_weights ORDER BY created_at DESC LIMIT ?`, then for each snapshot fetch its agent rows
- Alternative efficient approach: single query fetching all rows for top N timestamps, then group in Python
