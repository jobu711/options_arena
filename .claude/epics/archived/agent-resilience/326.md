# Task: Add tests for zero-enrichment agent execution

## Description

Add tests verifying that Flow and Fundamental agents run with zero enrichment data, produce valid output, and are counted in agreement scores. Tests should use `TestModel` (no real API calls) and create `MarketContext` fixtures with all 11 enrichment fields set to None.

## Acceptance Criteria

- [ ] Test: Flow agent produces valid `FlowThesis` with zero-enrichment `MarketContext`
- [ ] Test: Fundamental agent produces valid `FundamentalThesis` with zero-enrichment `MarketContext`
- [ ] Test: `render_context_block()` includes data-availability note when `enrichment_ratio() == 0.0`
- [ ] Test: `render_context_block()` does NOT include note when enrichment data is present
- [ ] Test: `compute_agreement_score()` works correctly with 6 agent directions (including flow + fundamental)
- [ ] All tests use `TestModel` override — no real Groq API calls
- [ ] All tests pass with `uv run pytest tests/ -v`

## Technical Details

### Zero-Enrichment MarketContext Fixture
Create a fixture with all enrichment fields explicitly None:
```python
pe_ratio=None, forward_pe=None, peg_ratio=None, price_to_book=None,
debt_to_equity=None, revenue_growth=None, profit_margin=None,
net_call_premium=None, net_put_premium=None, options_put_call_ratio=None,
news_sentiment=None
```
Scan-derived fields (put_call_ratio, max_pain_distance, iv_rank, next_earnings, etc.) should be populated normally.
