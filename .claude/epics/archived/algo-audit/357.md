# Task: Backoff jitter in fetch_with_retry

## Description

Add random jitter to the exponential backoff delay in `fetch_with_retry()` in
`services/helpers.py` (FR-7). Jitter prevents synchronized retry storms when multiple
concurrent requests fail at the same time.

## Acceptance Criteria

- [ ] Delay formula includes `* (0.5 + random.random() * 0.5)` jitter factor
- [ ] Jitter range is 50%-100% of the base delay (never increases beyond original max)
- [ ] Existing retry behavior (max attempts, backoff base) unchanged
- [ ] 2 new tests pass

## Technical Details

### Change in `services/helpers.py`
```python
import random

# In fetch_with_retry(), change delay calculation:
delay = min(base_delay * (2 ** attempt), max_delay) * (0.5 + random.random() * 0.5)
```

### Files affected
- `src/options_arena/services/helpers.py` — add jitter to delay calculation
