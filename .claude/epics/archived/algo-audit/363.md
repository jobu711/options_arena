# Task: Retry-outside-limiter helper + call site migration

## Technical Details

### New helper function
```python
async def fetch_with_limiter_retry(
    fn: Callable[..., Awaitable[T]],
    *args: Any,
    limiter: RateLimiter,
    max_attempts: int = 3,
    base_delay: float = 1.0,
    max_delay: float = 16.0,
    **kwargs: Any,
) -> T:
    """Retry with rate limiter — releases semaphore during backoff sleep."""
    last_error: Exception | None = None
    for attempt in range(max_attempts):
        async with limiter:  # acquire per attempt
            try:
                return await fn(*args, **kwargs)
            except Exception as e:
                last_error = e
        # Released limiter before sleeping
        if attempt < max_attempts - 1:
            delay = min(base_delay * (2 ** attempt), max_delay) * (0.5 + random.random() * 0.5)
            await asyncio.sleep(delay)
    raise last_error  # type: ignore[misc]
```

### Call site migration pattern
```python
# Before (in market_data.py):
async with self._limiter:
    result = await fetch_with_retry(self._fetch_quote_impl, ticker)

# After:
result = await fetch_with_limiter_retry(
    self._fetch_quote_impl, ticker, limiter=self._limiter
)
```

### Edge Cases
- Function succeeds on last attempt → returns result
- Limiter at capacity → waits for slot, then retries
- All attempts fail → raises last exception (not first)
- Concurrent calls → semaphore properly shared
