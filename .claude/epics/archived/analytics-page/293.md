# Task: Create AnalyticsPage.vue shell with data loading and Collect Outcomes

## Description
Create the main `AnalyticsPage.vue` page component — thin orchestrator that loads analytics data from 5 API endpoints, manages loading/error/empty states, and provides a "Collect Outcomes" button. Sub-components will be slots/placeholders initially (completed in Tasks 295-296). Page follows `DashboardPage.vue` pattern: `<script setup lang="ts">`, `onMounted()` data fetch, `Promise.all()` for parallel loads, `useToast()` for notifications.

## Acceptance Criteria
- [ ] `web/src/pages/AnalyticsPage.vue` renders at `/analytics` route (route already registered)
- [ ] Page loads `PerformanceSummary` on mount via `GET /api/analytics/summary?lookback_days=30`
- [ ] If `total_contracts > 0`, parallel-fetches win-rate, score-calibration, holding-period, delta-performance
- [ ] Three-tier empty state hierarchy:
  - `total_contracts === 0` → "No recommendations yet. Run a scan to start building analytics." with link to /scan
  - `total_with_outcomes === 0` → "N recommendations but no outcomes. Click Collect Outcomes."
  - Per-panel empty when data array is empty
- [ ] "Collect Outcomes" button triggers `POST /api/analytics/collect-outcomes`
- [ ] Button shows loading spinner during POST
- [ ] Success toast shows outcomes collected count
- [ ] 409 response shows warning toast "Operation in progress"
- [ ] After successful collection, refetches all analytics data
- [ ] Lookback period state managed via `ref<number>(30)` with options: 7, 14, 30, 60, 90
- [ ] Page is < 150 lines (thin orchestrator — data display delegated to sub-components)
- [ ] `<script setup lang="ts">` with Composition API only
- [ ] All API calls via `useApi` composable (`api<T>()`)
- [ ] `npm run type-check` passes
