## Technical Details

### New Models (all `frozen=True`)

1. **EquityCurvePoint** — `date: datetime.date`, `cumulative_return_pct: float`, `trade_count: int`
2. **DrawdownPoint** — `date: datetime.date`, `drawdown_pct: float`, `peak_value: float`
3. **SectorPerformanceResult** — `sector: str`, `total: int`, `win_rate_pct: float`, `avg_return_pct: float`
4. **DTEBucketResult** — `dte_min: int`, `dte_max: int`, `total: int`, `win_rate_pct: float`, `avg_return_pct: float`
5. **IVRankBucketResult** — `iv_min: float`, `iv_max: float`, `total: int`, `win_rate_pct: float`, `avg_return_pct: float`
6. **GreeksDecompositionResult** — `group_key: str`, `delta_pnl: float`, `residual_pnl: float`, `total_pnl: float`, `count: int`
7. **HoldingPeriodComparison** — `holding_days: int`, `direction: str`, `avg_return: float`, `median_return: float`, `win_rate: float`, `sharpe_like: float`, `max_loss: float`, `count: int`

### Pattern
Follow existing models like `WinRateResult`, `ScoreCalibrationBucket`. All use `ConfigDict(frozen=True)`, `field_validator` for `math.isfinite()` on every float field.

### Migration 029
```sql
CREATE INDEX IF NOT EXISTS idx_co_holding_days ON contract_outcomes(holding_days);
CREATE INDEX IF NOT EXISTS idx_co_collected_at ON contract_outcomes(collected_at);

CREATE INDEX IF NOT EXISTS idx_rc_market_iv ON recommended_contracts(market_iv);
CREATE INDEX IF NOT EXISTS idx_rc_direction_created ON recommended_contracts(direction, created_at);
```

### Edge Cases
- NaN in any float field: rejected by `math.isfinite()` validator
- Inf in any float field: rejected by `math.isfinite()` validator
- Zero trade count: valid (empty bucket)
- Negative returns: valid (losses are negative)
- Mutation attempt on frozen model: raises `ValidationError`
