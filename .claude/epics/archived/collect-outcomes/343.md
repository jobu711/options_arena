# Task: Extend WebSocket bridge and inject outcome collection into scan background task

## Description

Add `outcomes_collected: int = 0` keyword parameter to `WebSocketProgressBridge.complete()` and inject automatic outcome collection into `_run_scan_background()` after a successful non-cancelled scan. Write unit tests covering both changes.

## Acceptance Criteria

- [ ] `WebSocketProgressBridge.complete()` accepts `outcomes_collected: int = 0` keyword parameter
- [ ] The `outcomes_collected` value is included in the queue payload dict
- [ ] Existing callers (debate routes, error paths) work unchanged due to default `0`
- [ ] `_run_scan_background()` calls `OutcomeCollector.collect_outcomes()` after successful non-cancelled scan
- [ ] `OutcomeCollector` is constructed from `request.app.state` matching the `deps.py` pattern
- [ ] `outcomes_collected=len(outcomes)` is passed to `bridge.complete()`
- [ ] Outcome collection is skipped when `result.cancelled` is `True`
- [ ] A failed `collect_outcomes()` does not crash the scan background task (never-raises contract)
- [ ] All new and existing tests pass
- [ ] `ruff check` and `mypy --strict` pass

## Technical Details

### File 1: `src/options_arena/api/ws.py` (line 62)

Change `complete()` signature and payload:

```python
def complete(self, scan_id: int, *, cancelled: bool, outcomes_collected: int = 0) -> None:
    """Signal scan completion."""
    self.queue.put_nowait({
        "type": "complete",
        "scan_id": scan_id,
        "cancelled": cancelled,
        "outcomes_collected": outcomes_collected,
    })
```

### File 2: `src/options_arena/api/routes/scan.py` (lines 78-82)

After `pipeline.run()` succeeds, before `bridge.complete()`:

```python
# Collect outcomes for previous scans (never-raises contract)
outcomes_count = 0
if not result.cancelled:
    collector = OutcomeCollector(
        config=request.app.state.settings.analytics,
        repository=request.app.state.repo,
        market_data=request.app.state.market_data,
        options_data=request.app.state.options_data,
    )
    outcomes = await collector.collect_outcomes()
    outcomes_count = len(outcomes)

actual_id = result.scan_run.id if result.scan_run.id is not None else scan_id
bridge.complete(actual_id, cancelled=result.cancelled, outcomes_collected=outcomes_count)
```

Add `OutcomeCollector` import at the top of the file.

### Edge Cases
- `collect_outcomes()` returns empty list → `outcomes_collected=0` in payload
- `result.cancelled=True` → no `OutcomeCollector` constructed at all
- Multiple callers of `bridge.complete()` without the new kwarg still work (default=0)

### Mocking Strategy
- Mock `OutcomeCollector` class and its `collect_outcomes()` return value
- Mock `request.app.state` with appropriate service attributes

- Mock `ScanPipeline.run()` to return a `ScanResult` with known `cancelled` and `scan_run.id`
