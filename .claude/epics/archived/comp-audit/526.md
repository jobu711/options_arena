# Task 526: Multi-Methodology Valuation (FR-C6)

## Description

Implement a four-model valuation framework that computes composite fair value estimates
for equities using Owner Earnings DCF, 3-Stage DCF, EV/EBITDA Relative, and Residual
Income methodologies. Results are exposed as typed models, integrated into the debate
pipeline via `MarketContext`, and surfaced in the fundamental agent's prompt context.

Data sourced from `MarketContext.fd_*` fields (FinancialDatasets enrichment). When
insufficient data prevents a model from producing a value, its weight is redistributed
across the remaining models. All four failing yields `None` for the composite.

### New Enum

```python
# models/enums.py
class ValuationSignal(StrEnum):
    UNDERVALUED = "undervalued"       # margin_of_safety > 0.15
    FAIRLY_VALUED = "fairly_valued"   # -0.15 <= margin_of_safety <= 0.15
    OVERVALUED = "overvalued"         # margin_of_safety < -0.15
```

### New Models (`models/valuation.py`)

```python
class ValuationModelResult(BaseModel):
    model_config = ConfigDict(frozen=True)
    methodology: str           # "owner_earnings_dcf", "three_stage_dcf", etc.
    fair_value: float | None   # Per-share, None if data insufficient
    margin_of_safety: float | None  # (fair - price) / fair
    confidence: float          # [0.0, 1.0] data quality score
    data_quality_notes: list[str]
    # Validators: isfinite() on fair_value/margin_of_safety, confidence in [0,1]

class CompositeValuation(BaseModel):
    model_config = ConfigDict(frozen=True)
    ticker: str
    current_price: float
    composite_fair_value: float | None
    composite_margin_of_safety: float | None
    valuation_signal: ValuationSignal | None
    models: list[ValuationModelResult]
    weights_used: dict[str, float]
    computed_at: datetime      # UTC validated
    # Validators: isfinite() on price/fair_value/margin_of_safety, UTC on computed_at
```

### Valuation Models (`analysis/valuation.py`)

| Model | Weight | Key Inputs | Reference |
|-------|--------|------------|-----------|
| Owner Earnings DCF | 35% | net_income, D&A, capex | Damodaran |
| 3-Stage DCF | 35% | FCF, growth rate | Damodaran |
| EV/EBITDA Relative | 20% | ev_to_ebitda, sector avg | Damodaran |
| Residual Income | 10% | book_value, ROE | Hull |

- Owner Earnings: owner_earnings = net_income + D&A - capex. Discount stream at WACC with 25% haircut.
- 3-Stage DCF: high-growth (5yr) → transition (5yr) → terminal value. Scenario-weighted (bull/base/bear).
- EV/EBITDA Relative: compare ticker EV/EBITDA to sector average, derive implied fair value.
- Residual Income: excess ROE × book_value per share, discounted. 20% haircut.

Combiner: `compute_composite_valuation(ticker, current_price, fd_data, risk_free_rate, config) -> CompositeValuation`
— renormalizes weights across models that produced values. All four fail → None composite.

### MarketContext Additions

Six new `fd_*` fields:
- `fd_free_cash_flow: float | None` (absolute FCF, not yield)
- `fd_capex: float | None`
- `fd_depreciation_amortization: float | None`
- `fd_book_value_per_share: float | None`
- `fd_roe: float | None`
- `fd_shares_outstanding: float | None`

Three valuation summary fields:
- `valuation_signal: ValuationSignal | None = None`
- `valuation_margin_of_safety: float | None = None`
- `valuation_fair_value: float | None = None`

All added to `validate_optional_finite` field list and `financial_datasets_ratio()`.

### Edge Cases
- All four valuation models return None → composite None, signal None
- Single model produces negative fair value → treated as None
- Zero shares outstanding → owner earnings DCF returns None
- Extremely high growth rate → DCF clamps or flags
- NaN/Inf in any fd_* field → model skips gracefully
- Zero current price → no margin of safety computation

### Mocking Strategy
- No external API mocks needed — pure computation from MarketContext fields
