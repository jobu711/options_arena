# Task: Short interest end-to-end

## Description

Extract short interest data from yfinance, add it to `TickerInfo`, thread it through `MarketContext` to the debate agents, and render it in the agent context block. This is a vertical slice from data source → model → orchestrator → agent prompt.

## Acceptance Criteria

- [ ] `TickerInfo` has `short_ratio: float | None = None` and `short_pct_of_float: float | None = None` fields
- [ ] Both fields have `field_validator`: `math.isfinite(v)` and `v >= 0` (allow >1.0 for short_pct_of_float)
- [ ] `fetch_ticker_info()` extracts via `safe_float(info.get("shortRatio"))` and `safe_float(info.get("shortPercentOfFloat"))`
- [ ] `MarketContext` has `short_ratio: float | None = None`
- [ ] `build_market_context()` in orchestrator threads `ticker_info.short_ratio`
- [ ] `render_context_block()` renders SHORT RATIO in Fundamental Profile section via `_render_optional()`
- [ ] All new tests pass, all existing tests still pass
- [ ] `ruff check`, `mypy --strict` clean for modified files

## Technical Details

### Files to modify (5):
1. **`src/options_arena/models/market_data.py`** — Add 2 fields to `TickerInfo` after `fifty_two_week_low`. Add shared `field_validator("short_ratio", "short_pct_of_float")` with `math.isfinite()` + `v >= 0` guard. No upper bound on `short_pct_of_float` (squeezes can exceed 100%).

2. **`src/options_arena/services/market_data.py`** — In `fetch_ticker_info()` TickerInfo constructor call (~line 443-456), add:
   ```python
   short_ratio=safe_float(info.get("shortRatio")),
   short_pct_of_float=safe_float(info.get("shortPercentOfFloat")),
   ```

3. **`src/options_arena/models/analysis.py`** — Add `short_ratio: float | None = None` to `MarketContext`. Existing `validate_optional_finite` validator (if present) should cover it, or add to appropriate validator group.

4. **`src/options_arena/agents/orchestrator.py`** — In `build_market_context()` constructor call (~line 172), add `short_ratio=ticker_info.short_ratio`.

5. **`src/options_arena/agents/_parsing.py`** — In `render_context_block()` Fundamental Profile section (~line 443-457), add `_render_optional("SHORT RATIO", ctx.short_ratio, ".2f")`.

### Key considerations:
- `TickerInfo` is `frozen=True` — fields must be set at construction time
- `safe_float()` already rejects NaN/Inf — the field_validator is defense-in-depth
- No cache key versioning needed — old cached entries serve `None` for new fields
- MarketContext is stored as JSON blob (`market_context_json TEXT`) — no SQL migration
- `compute_short_interest()` in `indicators/fundamental.py` already exists but is NOT called here — that's for indicator normalization, not model population

### Edge Cases
- yfinance returns `None` for both fields → `TickerInfo` constructed with `None` defaults
- yfinance returns `0.0` → valid (some stocks have negligible short interest)
- `short_pct_of_float = 1.5` → valid (>100% of float shorted, e.g., GME squeeze)
- Old cached `TickerInfo` missing new fields → Pydantic defaults to `None` on deserialization

### Mocking Strategy
- yfinance: patch `_yf_call` to return info dict with/without `shortRatio`/`shortPercentOfFloat`
- No external calls needed — all unit-level
