# Task: Active-contract P&L via OptionsDataService DI

## Description

Inject `OptionsDataService` into `OutcomeCollector` so that active (non-expired) contracts can compute `exit_contract_mid`, `contract_return_pct`, and `is_winner` from live option chain data. Currently these are always `None` for active contracts. Update all 3 instantiation sites (deps.py, 2x CLI).

## Acceptance Criteria

- [ ] `OutcomeCollector.__init__()` accepts `options_data: OptionsDataService | None = None`
- [ ] `_process_active_contract()` fetches option chain when `options_data` is available
- [ ] Matching logic: finds contract by `strike == contract.strike and option_type == contract.option_type`
- [ ] Computes `exit_contract_mid = (bid + ask) / Decimal("2")`
- [ ] Computes `contract_return_pct` and `is_winner` from entry/exit mid
- [ ] `asyncio.wait_for(timeout=10)` wraps the `fetch_chain()` call
- [ ] `try/except Exception` → log warning, fall back to `None` (graceful degradation)
- [ ] Without `options_data`, behavior is identical to current (backward compatible)
- [ ] `api/deps.py` passes `request.app.state.options_data` to `OutcomeCollector()`
- [ ] CLI `outcomes.py` creates `OptionsDataService`, passes to collector, closes in `finally`
- [ ] All new tests pass, all existing tests still pass

## Technical Details

### Files to modify (3):

1. **`src/options_arena/services/outcome_collector.py`**
   - Add `options_data: OptionsDataService | None = None` as 4th param to `__init__()` (after `market_data`)
   - Store as `self._options_data`
   - In `_process_active_contract()` (~line 237-260), replace the hardcoded `None` block:
     ```python
     if self._options_data is not None:
         try:
             chain = await asyncio.wait_for(
                 self._options_data.fetch_chain(contract.ticker, contract.expiration),
                 timeout=10,
             )
             matching = next(
                 (c for c in chain
                  if c.strike == contract.strike and c.option_type == contract.option_type),
                 None,
             )
             if matching is not None and matching.bid is not None and matching.ask is not None:
                 exit_contract_bid = matching.bid
                 exit_contract_ask = matching.ask
                 exit_contract_mid = (matching.bid + matching.ask) / Decimal("2")
                 if contract.entry_mid and contract.entry_mid > 0:
                     contract_return = float(
                         (exit_contract_mid - contract.entry_mid) / contract.entry_mid * 100
                     )
                     is_winner = contract_return > 0
         except Exception:
             logger.warning("Failed to fetch option chain for %s %s", contract.ticker, contract.expiration)
     ```
   - Keep the existing stock-return-only logic as the fallback path

2. **`src/options_arena/api/deps.py`** (~line 75-81)

   - In `_outcomes_collect_async()`: create `options_data = OptionsDataService(settings.service, settings.pricing, cache, limiter)`, pass to collector, add `await options_data.close()` in `finally`

### Edge Cases
- `matching.bid` or `matching.ask` is `None` → skip P&L computation, return `None`
- `contract.entry_mid` is `None` or `0` → skip return computation
- Empty chain returned → `next(...)` returns `None`
- Chain has multiple matching strikes (shouldn't happen) → takes first match
- `fetch_chain` raises any exception type → caught and logged

### Mocking Strategy
- `OptionsDataService`: `AsyncMock()` with `fetch_chain = AsyncMock(return_value=[...])`
- `OptionContract` for chain results: construct typed model directly (not dict)
- `_market_today()`: patched as in existing tests
- Existing 23 tests should NOT be modified — they construct without `options_data` and must still pass
