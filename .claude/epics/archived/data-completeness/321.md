# Task: Indicator signal interpretation labels

## Description

Add colored interpretation labels (Overbought, Strong Trend, Compressed, etc.) next to indicator values in the TickerDrawer. Pure frontend change — a `classifySignal()` TypeScript function maps 6 indicator keys to threshold-based labels, rendered as PrimeVue `<Tag>` components.

## Acceptance Criteria

- [ ] `classifySignal(key: string, value: number)` function returns `{ label: string; severity: 'success' | 'danger' | 'warn' | 'info' } | null`
- [ ] 6 indicators classified per PRD thresholds (normalized 0-100 percentile values):
  - RSI: <30 Oversold (success), 30-70 Neutral, >70 Overbought (danger)
  - Stochastic RSI: <20 Oversold (success), 20-80 Neutral, >80 Overbought (danger)
  - ADX: <30 Weak (warn), 30-60 Moderate (info), >60 Strong Trend (success)
  - SMA Alignment: <40 Bearish (danger), 40-60 Neutral, >60 Bullish (success)
  - Relative Volume: <30 Low Vol (info), 30-70 Normal, >70 High Vol (warn)
  - BB Width: <20 Compressed (info), 20-80 Normal, >80 Expanded (warn)
- [ ] Indicators not in the table return `null` → no tag rendered
- [ ] PrimeVue `<Tag>` rendered inline next to the numeric value
- [ ] Signal grid CSS updated to accommodate the tag column
- [ ] `npm run build` succeeds with no TypeScript errors

2. **Update template** (~line 291-302): Add `<Tag>` after the value span:
   ```vue

     <span class="signal-name">{{ formatSignalName(String(key)) }}</span>
     <span class="signal-value mono">{{ formatSignalValue(val as number | null) }}</span>
     <Tag

     />
     <span v-else class="signal-tag-placeholder" />
   </div>
   ```
   Note: Consider computing the classification once per row to avoid triple-calling the function. Use a computed or inline `const`.

### Key considerations:
- PrimeVue `Tag` is already imported in TickerDrawer (line 9) — no new import needed
- Values are normalized 0-100 percentiles, NOT raw indicator values
- The function must be pure (no API calls, no store access)
- `display: contents` on `.signal-row` means grid children are direct grid items — adding a 3rd child per row requires updating `grid-template-columns`
- For unknown indicator keys, return `null` → no tag, placeholder maintains grid alignment

### Edge Cases
- `value` is `null` → no tag (handled by null check)
- `value` is exactly on a threshold boundary (e.g., RSI = 70.0) → should fall into the neutral range (use `>` not `>=` for upper thresholds)
- All signals are null → no tags rendered, grid still aligned
- Very long label text → Tag should not wrap (short labels only: "Overbought", "Weak", etc.)
