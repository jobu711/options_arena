foundational task — all other rebuttal tasks depend on these fields existing.

## Technical Details
- File: `src/options_arena/agents/_parsing.py`
- `bear_counter_argument` is injected into the bull's deps during rebuttal mode — contains
  the bear's `key_points` joined as text (not the full argument)
- `bull_rebuttal` on `DebateDeps` is for the risk agent to see the rebuttal output
- `bull_rebuttal` on `DebateResult` stores the rebuttal for CLI rendering and persistence
- Both are `AgentResponse | None` since AgentResponse is already imported in the file
- `bear_counter_argument` is `str | None` (plain text, not a model)
