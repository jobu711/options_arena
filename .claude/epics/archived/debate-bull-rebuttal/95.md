# Task: Convert bull agent to dynamic prompt with rebuttal mode

## Description
Convert the bull agent's system prompt from static (`@bull_agent.system_prompt`) to dynamic
(`@bull_agent.system_prompt(dynamic=True)`). When `ctx.deps.bear_counter_argument` is set,
append rebuttal instructions to the prompt. When it's None, return the same base prompt as
before (no behavioral change for the initial bull run).

## Acceptance Criteria
- [ ] `@bull_agent.system_prompt` replaced with `@bull_agent.system_prompt(dynamic=True)`
- [ ] Prompt function signature changes to `async def bull_dynamic_prompt(ctx: RunContext[DebateDeps]) -> str`
- [ ] `BULL_REBUTTAL_INSTRUCTIONS` constant added with `<<<BEAR_COUNTER>>>` delimiters
- [ ] When `ctx.deps.bear_counter_argument is None`: returns base prompt (identical to current)
- [ ] When `ctx.deps.bear_counter_argument is not None`: appends rebuttal instructions with bear's key points
- [ ] Rebuttal instructions tell the bull to address 2-3 strongest bear points in 3-5 sentences
- [ ] Version comment updated to `# VERSION: v2.1`

## Technical Details
- File: `src/options_arena/agents/bull.py`
- The dynamic prompt pattern is already used by bear.py and risk.py — follow same style
- `BULL_REBUTTAL_INSTRUCTIONS` uses `<<<BEAR_COUNTER>>>...<<<END_BEAR_COUNTER>>>` delimiters
  to prevent instruction bleed from LLM-generated text (same pattern as bear's `<<<BULL_ARGUMENT>>>`)
- The `.format(bear_key_points=ctx.deps.bear_counter_argument)` injects bear's key points
- No changes to `bull_agent` Agent() init or output_validator
