# Task: Add rebuttal phase to orchestrator

## Description
Add a conditional bull rebuttal phase to `_run_agents()` in the orchestrator. When
`config.enable_rebuttal` is True, run the bull agent a second time after the bear agent
completes, injecting the bear's `key_points` as `bear_counter_argument`. The rebuttal
runs before the volatility agent (if enabled) and before the risk agent.

Flow: Bull -> Bear -> **Bull Rebuttal (opt-in)** -> Volatility (opt-in) -> Risk

## Acceptance Criteria
- [ ] When `config.enable_rebuttal is True`: bull agent runs a second time after bear
- [ ] Rebuttal deps include `bear_counter_argument` = bear's key_points joined as bullet list
- [ ] Rebuttal output stored as `rebuttal_output: AgentResponse | None`
- [ ] Rebuttal usage accumulated into `total_usage`
- [ ] Risk agent deps include `bull_rebuttal=rebuttal_output` when rebuttal was run
- [ ] `DebateResult` includes `bull_rebuttal=rebuttal_output`
- [ ] When `config.enable_rebuttal is False` (default): no changes to existing flow
- [ ] Fallback results set `bull_rebuttal=None`
- [ ] Logging: "Running bull rebuttal for {ticker}" when rebuttal phase starts

## Technical Details
- File: `src/options_arena/agents/orchestrator.py`
- Insert rebuttal block between bear agent and volatility agent sections
- Build `bear_key_points` from `bear_output.key_points`: `"\n".join(f"- {p}" for p in ...)`
- Create `rebuttal_deps = DebateDeps(context=context, ticker_score=ticker_score,
  contracts=contracts, bear_counter_argument=bear_key_points)`
- Wrap in `asyncio.wait_for(bull_agent.run(...), timeout=per_agent_timeout)`
- Same `model`, `model_settings` as other agents
- Prompt message: `f"Rebut the bear's counterarguments for {context.ticker}.\n\n{context_text}"`
- Update `_build_fallback_result` and `_build_screening_fallback` to include `bull_rebuttal=None`
