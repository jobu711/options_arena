# Task: Domain context renderers + prompt calibration

## Description

Implement FR-1 (domain-partitioned context renderers) and FR-3 (remove COMPOSITE SCORE anchors from PROMPT_RULES_APPENDIX) in `agents/_parsing.py`. This is Wave 1 foundation — all other tasks depend on these renderers existing.

Each Phase 1 agent receives only its domain-specific fields plus a shared identity block. The original `render_context_block()` stays unchanged for Risk, Contrarian, persistence, and display.

## Acceptance Criteria

- [ ] `_render_identity_block(ctx: MarketContext) -> list[str]` helper returns shared fields (TICKER, PRICE, 52W HIGH/LOW, SECTOR, DTE, TARGET STRIKE, TARGET DELTA, EXERCISE, DIV YIELD, NEXT EARNINGS + warning)
- [ ] `render_trend_context(ctx: MarketContext) -> str` includes identity + RSI(14), MACD, ADX, SMA ALIGNMENT, STOCHASTIC RSI, REL VOLUME, RSI DIVERGENCE, dim_trend
- [ ] `render_volatility_context(ctx: MarketContext) -> str` includes identity + IV RANK, IV PERCENTILE, ATM IV 30D, BB WIDTH, ATR%, VOL REGIME, IV-HV SPREAD, SKEW RATIO, VIX TERM STRUCTURE, EXPECTED MOVE, EXPECTED MOVE RATIO, VEGA, VOMMA, dim_iv_vol, dim_hv_vol
- [ ] `render_flow_context(ctx: MarketContext) -> str` includes identity + PUT/CALL RATIO, MAX PAIN DISTANCE %, GEX, UNUSUAL ACTIVITY SCORE, NET CALL/PUT PREMIUM, OPTIONS PUT/CALL RATIO, REL VOLUME, dim_flow, dim_microstructure
- [ ] `render_fundamental_context(ctx: MarketContext) -> str` includes identity + P/E, FORWARD P/E, PEG, P/B, DEBT/EQUITY, REVENUE GROWTH, PROFIT MARGIN, SHORT RATIO, SHORT % OF FLOAT, ANALYST fields, INSIDER fields, INSTITUTIONAL OWNERSHIP, NEWS SENTIMENT, dim_fundamental
- [ ] No domain renderer includes COMPOSITE SCORE, DIRECTION, or DIRECTION CONFIDENCE
- [ ] `render_context_block()` is unchanged
- [ ] PROMPT_RULES_APPENDIX: removed COMPOSITE SCORE anchors, replaced with domain-neutral calibration language
- [ ] All 4 renderers re-exported from `agents/__init__.py`
- [ ] `ruff check` + `ruff format` clean
- [ ] `mypy --strict` passes

## Technical Details

### Files Modified
- `src/options_arena/agents/_parsing.py` — add `_render_identity_block()` + 4 domain renderers, update `PROMPT_RULES_APPENDIX`
- `src/options_arena/agents/__init__.py` — re-export 4 new renderers

### Implementation Approach
1. Read `agents/CLAUDE.md` before starting
2. Add `_render_identity_block(ctx: MarketContext) -> list[str]` — shared fields for all domain renderers
3. Add 4 `render_*_context()` functions, each calling `_render_identity_block()` + domain fields via `_render_optional()`
4. Update `PROMPT_RULES_APPENDIX`: remove 2 COMPOSITE SCORE lines, add 4 domain-neutral calibration lines
5. Re-export from `__init__.py`

### Key Considerations
- MACD IS included in `render_trend_context()` — it's a genuine indicator from `indicators/trend.py`, not a scan conclusion (resolved in research)
- Only COMPOSITE SCORE, DIRECTION, and DIRECTION CONFIDENCE are excluded
- Each renderer uses existing `_render_optional()` helper for None/NaN guarding
- `render_context_block()` must remain untouched — Risk, Contrarian, persistence, display depend on it
