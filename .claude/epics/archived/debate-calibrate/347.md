# Task: Orchestrator wiring + log-odds pooling

## Description

Implement FR-2 (partitioned context wiring in orchestrator) and FR-4 (log-odds pooling replaces linear averaging) in `agents/orchestrator.py`. This connects the domain renderers from Task 1 to Phase 1 agents and replaces the mathematically flawed linear confidence averaging with log-odds pooling (Bordley 1982).

## Acceptance Criteria

- [ ] `_run_v2_agents()` generates 4 domain-specific context strings + 1 full context string
- [ ] Trend agent receives `render_trend_context()` output
- [ ] Volatility agent receives `render_volatility_context()` output
- [ ] Flow agent receives `render_flow_context()` output
- [ ] Fundamental agent receives `render_fundamental_context()` output
- [ ] Risk agent (Phase 2) receives full `render_context_block()` output
- [ ] Contrarian agent (Phase 3) receives full `render_context_block()` output
- [ ] `_log_odds_pool(probabilities, weights) -> float` pure function added
- [ ] Linear weighted average in `synthesize_verdict()` replaced with `_log_odds_pool()`
- [ ] `"risk": 0.15` removed from `AGENT_VOTE_WEIGHTS` (confirmed dead code)
- [ ] Agreement-based confidence capping preserved unchanged after log-odds
- [ ] Three agents at 0.9 produce combined > 0.95
- [ ] `ruff check` + `ruff format` clean
- [ ] `mypy --strict` passes

## Technical Details

### Files Modified
- `src/options_arena/agents/orchestrator.py` — wiring + `_log_odds_pool()` + remove dead weight

### Implementation Approach
1. Read `agents/CLAUDE.md` before starting
2. Import 4 domain renderers from `agents._parsing`
3. In `_run_v2_agents()`: replace single `context_text = render_context_block(context)` with 5 context strings
4. Map each Phase 1 agent to its domain context when calling `.run()`
5. Add `_log_odds_pool(probabilities: list[float], weights: list[float]) -> float` pure function
6. Replace linear averaging block in `synthesize_verdict()` with `_log_odds_pool()` call
7. Remove `"risk": 0.15` from `AGENT_VOTE_WEIGHTS`
8. Preserve existing agreement-based confidence cap (`agreement < 0.4 -> cap at 0.4`)

### Key Considerations
- `_log_odds_pool()` must clamp inputs to [0.01, 0.99] to prevent `log(0)` / `log(inf)`
- Risk weight removal is safe — research confirmed it's dead code (RiskAssessment passed separately, never enters confidence loop)
- The `else:` fallback when no probabilities exist should use `config.fallback_confidence`
- Must understand how Phase 1 agents receive context_text currently (the prompt string in `.run()`)
