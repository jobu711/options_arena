# Task: Ensemble diversity — model fields, vol direction, entropy, gate

## Description

Implement FR-5 (vote entropy), FR-6 (volatility direction), and FR-7 (relaxed contrarian gate). These three independent enhancements increase ensemble diversity and signal quality. FR-5 adds measurement infrastructure, FR-6 adds a new directional signal, and FR-7 ensures the contrarian runs in most debates.

## Acceptance Criteria

- [ ] `ExtendedTradeThesis` gains `ensemble_entropy: float | None = None` with `math.isfinite()` validator
- [ ] `VolatilityThesis` gains `direction: SignalDirection = SignalDirection.NEUTRAL` (backward-compatible)
- [ ] Existing serialized `VolatilityThesis` without `direction` field deserializes correctly
- [ ] Existing serialized `ExtendedTradeThesis` without `ensemble_entropy` deserializes correctly
- [ ] `_vote_entropy(agent_directions)` pure function added to orchestrator
- [ ] Entropy computed in `synthesize_verdict()` and set on `ExtendedTradeThesis`
- [ ] `isinstance(output, VolatilityThesis): continue` skip removed from direction voting
- [ ] Volatility included in `agent_directions` dict
- [ ] Volatility prompt updated with IV regime calibration anchors for directional output
- [ ] `build_cleaned_volatility_thesis()` passes `direction=output.direction`
- [ ] Contrarian gate changed from `phase1_failures < 2` to `phase1_failures < 3`
- [ ] `ruff check` + `ruff format` clean
- [ ] `mypy --strict` passes

## Technical Details

### Files Modified
- `src/options_arena/models/analysis.py` — add `ensemble_entropy` to `ExtendedTradeThesis`, add `direction` to `VolatilityThesis`
- `src/options_arena/agents/orchestrator.py` — add `_vote_entropy()`, wire entropy, remove vol skip, change gate
- `src/options_arena/agents/_parsing.py` — update `build_cleaned_volatility_thesis()` to pass `direction`
- `src/options_arena/agents/prompts/` — update volatility prompt with IV regime calibration

### Implementation Approach

**Model Changes (`models/analysis.py`):**
1. Add `ensemble_entropy: float | None = None` to `ExtendedTradeThesis` with `@field_validator` using `math.isfinite()`
2. Add `direction: SignalDirection = SignalDirection.NEUTRAL` to `VolatilityThesis`

**Orchestrator Changes (`agents/orchestrator.py`):**
3. Add `_vote_entropy(agent_directions: dict[str, SignalDirection]) -> float` pure function (Shannon entropy of vote distribution)
4. In `synthesize_verdict()`: remove `isinstance(output, VolatilityThesis): continue` skip from direction voting
5. In `synthesize_verdict()`: compute `entropy = _vote_entropy(agent_directions)`, set on `ExtendedTradeThesis`
6. In `_run_v2_agents()`: change `phase1_failures < 2` to `phase1_failures < 3`

**Parsing Changes (`agents/_parsing.py`):**
7. In `build_cleaned_volatility_thesis()`: add `direction=output.direction` to constructor call

**Prompt Changes:**
8. Update volatility prompt with directional output instructions + IV regime calibration anchors:
   - IV rank < 25 → lean underpriced, direction bullish
   - IV rank 25-75 → lean fair, direction neutral
   - IV rank > 75 → lean overpriced, direction bearish

### Key Considerations
- `VolatilityThesis` is `frozen=True` — must use constructor, not field assignment
- Entropy validator must guard `math.isfinite()` (matching existing `agent_agreement_score` pattern)
- Entropy range: 0.0 (unanimous) to ~1.585 (equal 3-way split of BULLISH/BEARISH/NEUTRAL)
- Calibration anchors are defaults the LLM can override with reasoning
