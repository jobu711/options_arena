# Task: Live validation — run debates, measure success criteria

## Description

Post-merge validation: run 50+ debates across diverse tickers and measure success criteria against the targets defined in the PRD. This validates that information partitioning and log-odds pooling actually improve ensemble diversity in production.

## Acceptance Criteria

- [ ] 50+ debates run across diverse tickers (mix of bullish/bearish scan directions, multiple sectors)
- [ ] Unanimous agreement rate measured (target: < 50%, current: 100%)
- [ ] Mean ensemble entropy measured (target: > 0.4, current: 0.0)
- [ ] Trend echo rate measured (target: < 60%, current: 80%)
- [ ] Fundamental confidence std measured (target: > 0.10, current: 0.000)
- [ ] Contrarian execution rate measured (target: > 80%, current: 20%)
- [ ] Per-agent token count measured (target: < 2,000, current: ~2,600)
- [ ] Log-odds confidence vs win rates compared (calibration curve data)
- [ ] Results documented

## Technical Details

### Validation Protocol
1. Run a scan to identify diverse tickers: `options-arena scan --sector "Technology" --sector "Healthcare" --sector "Financials"`
2. Run batch debate on top results: `options-arena debate --batch --top 50`
3. After T+1 outcomes available: `options-arena outcomes collect`
4. Query analytics: `options-arena outcomes summary`
5. Compute metrics from database:
   - Unanimous rate: count debates where all agent directions match / total
   - Mean entropy: average `ensemble_entropy` from `verdict_json`
   - Trend echo: count debates where trend.direction == scan.direction / total
   - Fundamental std: std of fundamental confidence across debates
   - Contrarian execution: count debates where contrarian ran / total
   - Token count: average from debate records

### Key Considerations
- Requires `GROQ_API_KEY` to be set (live LLM calls)
- Rate limiting may slow batch — use sequential mode
- This is manual validation, not automated testing
- Results feed into the PRD's success criteria evaluation
- If metrics don't meet targets, document what needs adjustment

### Metrics Collection
```sql
-- Unanimous rate
SELECT COUNT(*) as total,
       SUM(CASE WHEN json_extract(verdict_json, '$.agent_agreement_score') = 1.0 THEN 1 ELSE 0 END) as unanimous
FROM ai_theses WHERE created_at > '{implementation_date}';

-- Mean entropy
SELECT AVG(json_extract(verdict_json, '$.ensemble_entropy')) as mean_entropy
FROM ai_theses WHERE created_at > '{implementation_date}';

-- Agent predictions
SELECT agent_name, direction, COUNT(*) as count
FROM agent_predictions
GROUP BY agent_name, direction;
```
