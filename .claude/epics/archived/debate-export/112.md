# Task: Add batch export support

## Description
When the debate command is invoked with `--batch --export md` (or pdf), export one file per
ticker plus a combined summary file. This task extends the CLI export integration from Task 111
to handle the batch case. Each ticker gets its own export file, and a summary file aggregates
all verdicts.

**Note**: This task is also blocked by Epic 6 (batch mode). If batch mode is not yet implemented,
this task should create the export-side logic and wire it into the batch flow when available.

## Acceptance Criteria
- [ ] Per-ticker export: each ticker in batch produces `debate_{TICKER}_{DATE}.{fmt}`
- [ ] Combined summary file: `debate_summary_{DATE}.{fmt}` with all verdicts
- [ ] Summary includes: ticker, direction, confidence, strategy for each debate
- [ ] Export directory created automatically
- [ ] Handles partial failures gracefully (some tickers fail, others still export)
- [ ] Passes `ruff check`, `ruff format`, and `mypy --strict`

## Technical Details
- **Files**: `src/options_arena/cli/commands.py`, possibly `src/options_arena/reporting/debate_export.py`
- **New function** (in `debate_export.py`):
  ```python
  def export_batch_summary(results: list[DebateResult], fmt: str = "md") -> str:
      """Generate combined summary of multiple debate results."""
  ```
- **Batch flow**: After all debates complete, iterate results and call `export_debate_to_file()`
  for each, then generate and write the summary file
- **Error handling**: Use `asyncio.gather(return_exceptions=True)` pattern — log failures,
  continue with successful results

## Dependencies
- [ ] Task 111 must be complete (single-ticker export)
- [ ] Epic 6 (batch mode) must provide the batch execution framework
