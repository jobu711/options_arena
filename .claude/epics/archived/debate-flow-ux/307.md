# Task: Add scan_run_id to API schema and consensus fields to TS interface

## Description
Expose the existing `scan_run_id` column through the debate API response and add 5 missing
fields to the TypeScript `DebateResult` interface. The backend already computes and persists
all data — this task makes it available to the frontend. This is the foundation task that
unblocks Tasks 308 and 309.

## Acceptance Criteria
- [ ] `DebateResultDetail` schema in `schemas.py` includes `scan_run_id: int | None = None`
- [ ] `get_debate()` in `routes/debate.py` populates `scan_run_id` from `row.scan_run_id`
- [ ] `DebateResult` interface in `web/src/types/debate.ts` includes all 5 new optional fields
- [ ] Existing tests still pass (no regressions)
- [ ] TypeScript compiles without errors (`npm run type-check`)

## Technical Details

### Backend (2 lines)

1. **`src/options_arena/api/schemas.py`** — Add field to `DebateResultDetail`:
   ```python
   scan_run_id: int | None = None
   ```

2. **`src/options_arena/api/routes/debate.py`** — In `get_debate()` response construction,
   add `scan_run_id=row.scan_run_id,` alongside existing fields.

### Frontend (5 lines)

**`web/src/types/debate.ts`** — Add to `DebateResult` interface:
```typescript
contrarian_dissent?: string | null
agent_agreement_score?: number | null
dissenting_agents?: string[]
agents_completed?: number | null
scan_run_id?: number | null
```
