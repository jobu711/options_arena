# Task: Add pre-screening and future-feature config fields to DebateConfig

## Description

Add three new fields to `DebateConfig` in `src/options_arena/models/config.py`:

1. `min_debate_score: float = 30.0` — Skip debate when composite score is below this threshold.
2. `enable_volatility_agent: bool = False` — Opt-in toggle for Epic 4 (Volatility Agent).
3. `enable_rebuttal: bool = False` — Opt-in toggle for Epic 5 (Bull Rebuttal Round).

Add a `field_validator` for `min_debate_score` that enforces:
- `math.isfinite(v)` — reject NaN and Inf
- `0.0 <= v <= 100.0` — composite scores are 0-100

The two boolean fields need no validators.

## Acceptance Criteria
- [ ] `DebateConfig.min_debate_score` defaults to `30.0`
- [ ] `DebateConfig.enable_volatility_agent` defaults to `False`
- [ ] `DebateConfig.enable_rebuttal` defaults to `False`
- [ ] Validator rejects `min_debate_score` values > 100, < 0, NaN, and Inf
- [ ] Existing `DebateConfig()` construction (no args) still works
- [ ] `ruff check` and `mypy --strict` pass

## Technical Details
- File: `src/options_arena/models/config.py`

- Validator pattern: use `@field_validator("min_debate_score")` with `@classmethod`, matching existing validators in the class
- Bool fields are plain typed fields — no validator needed
- These fields are consumed by Epic 3 (screening) and pre-provisioned for Epics 4 and 5
