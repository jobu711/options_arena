# Task: Add CLI rendering for volatility output

## Description
Add a cyan-bordered Rich panel to `cli/rendering.py` that displays the Volatility Agent's output when `DebateResult.vol_response` is set. The panel should show IV assessment, recommended strategy, strategy rationale, target IV entry/exit levels, suggested strikes, and key volatility factors. When `vol_response` is None, no panel is rendered.

## Acceptance Criteria
- [ ] New `render_volatility_panel(thesis: VolatilityThesis) -> Panel` function in `rendering.py`
- [ ] Panel has cyan border (distinguishing it from bull=green, bear=red, risk=yellow)
- [ ] Panel title: "Volatility Analysis" or similar
- [ ] Displays: IV assessment, confidence, recommended strategy, strategy rationale
- [ ] Displays: target IV entry/exit (when not None), suggested strikes, key vol factors
- [ ] `markup=False` on any Text objects containing agent-generated content (prevent bracket crashes)
- [ ] Debate rendering flow conditionally includes vol panel when `vol_response is not None`
- [ ] When `vol_response is None`, no vol panel appears (backward compatible)

## Technical Details
- File: `src/options_arena/cli/rendering.py` — add `render_volatility_panel()` function
- Follow existing patterns for bull/bear/risk panels (if they exist) or the scan results table pattern
- Use `rich.panel.Panel`, `rich.text.Text` for content
- Confidence displayed as percentage: `f"{thesis.confidence:.0%}"`
- Strategy displayed as `thesis.recommended_strategy.value` if not None, else "None recommended"
- Suggested strikes as comma-separated list
- Key vol factors as bulleted list
