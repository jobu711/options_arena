### Debate Protocol Flow
```
Phase 1 (parallel, configurable parallelism):
  - Trend Agent → AgentResponse
  - Volatility Agent → VolatilityThesis (existing)
  - Flow Agent → FlowThesis (from Task 003)
  - Fundamental Agent → FundamentalThesis (from Task 004)

Phase 2 (sequential):
  - Risk Agent(Phase 1 outputs) → RiskAssessment

Phase 3 (sequential):
  - Contrarian Agent(all 5 outputs + DimensionalScores) → ContrarianThesis

Phase 4 (algorithmic, no LLM):
  - synthesize_verdict() → ExtendedTradeThesis
```

### Verdict Synthesis
- Extract direction from each agent output
- Compute agreement score (fraction agreeing with majority)
- Weighted vote using `AGENT_VOTE_WEIGHTS`
- Cap confidence at 0.50 when agreement < 0.4
- Preserve contrarian dissent in `ExtendedTradeThesis`
- Track `agents_completed: int` and `dissenting_agents: list[str]`

### Key Constraints
- Read `agents/CLAUDE.md` and `agents/prompts/CLAUDE.md` before modifying
- Module-level `Agent[Deps, OutputType]` instances — no classes
- `model=None` at init, actual model passed at `agent.run(model=...)` time
- `@agent.output_validator` delegates to shared helpers
- `retries=2`, `model_settings=ModelSettings(extra_body={"num_ctx": 8192})`
- Per-agent timeout: `config.agent_timeout` (60s default)
- Orchestrator never raises — catches errors → data-driven fallback
- Bull/Bear agents kept for backward compatibility but replaced by Trend Agent in new protocol
- `asyncio.gather(*tasks, return_exceptions=True)` for Phase 1
