### Integration Wiring

**Scan Pipeline Phase 2 additions:**
```
1. fetch_universe_data() — once per scan (cached)
2. For each ticker:
   a. IV analytics (13 indicators) — requires option chain + IV surface
   b. Flow analytics (5 indicators) — requires option chain + gamma
   c. Trend extensions (3 indicators) — requires OHLCV
   d. Fundamental (5 indicators) — requires ticker info + earnings
   e. Regime (7 indicators) — requires universe data + ticker OHLCV
   f. Second-order Greeks (3) — requires pricing params
   g. Extended options (4) — requires chain + pricing
3. compute_dimensional_scores() per ticker
4. compute_direction_signal() per ticker
```

**Debate Integration:**
```
1. MarketContext extended with DimensionalScores
2. Orchestrator uses 6-agent protocol (or fallback to 3-agent if config flag)
3. ExtendedTradeThesis returned with contrarian dissent + agreement score
4. CLI renders expanded table with dimensional scores row
5. API returns full ExtendedTradeThesis JSON
```

### Key Constraints
- Read `scan/CLAUDE.md`, `api/CLAUDE.md`, `cli/CLAUDE.md` before modifying
- `scan/` orchestrates but never calls `pricing/` directly — that's `scoring/contracts.py`'s job
- `scan/` never calls `print()` — logging only
- API routes use `Depends()` DI pattern
- CLI renders with Rich tables — use `math.isfinite()` guards before formatting
- Universe data fetch: use `asyncio.gather()` for batch, cache with existing 2-tier caching
- Backward compatibility flag: `DebateConfig.use_legacy_protocol: bool = False` (3-agent fallback)
