# Task: Implement markdown export function

## Description
Create the `export_debate_markdown()` pure function in `src/options_arena/reporting/debate_export.py`.
This function takes a `DebateResult` dataclass and returns a structured markdown string containing
all debate panels (Bull, Bear, optional Volatility, optional Rebuttal, Verdict).

Also update `src/options_arena/reporting/__init__.py` with `__all__` re-exports.

## Acceptance Criteria
- [ ] `export_debate_markdown(result: DebateResult) -> str` implemented as a pure function
- [ ] Markdown includes header with ticker, date, duration, model name, fallback status
- [ ] Bull Case section with confidence, argument, key points, risks cited
- [ ] Bear Case section with same structure
- [ ] Volatility Assessment section rendered only when `vol_response` is not None
- [ ] Bull Rebuttal section rendered only when `bull_rebuttal` is not None
- [ ] Verdict section with direction, confidence, strategy, summary, risk assessment
- [ ] Disclaimer footer always present
- [ ] `__init__.py` updated with `__all__` re-exports for public functions
- [ ] Passes `ruff check`, `ruff format`, and `mypy --strict`

## Technical Details
- **File**: `src/options_arena/reporting/debate_export.py`
- **Input**: `DebateResult` dataclass from `agents/_parsing.py`
- **Output**: `str` (structured markdown)
- **Key models accessed**:
  - `DebateResult.bull_response` / `bear_response` -> `AgentResponse` (agent_name, direction, confidence, argument, key_points, risks_cited, model_used)
  - `DebateResult.thesis` -> `TradeThesis` (ticker, direction, confidence, summary, risk_assessment, recommended_strategy)
  - `DebateResult.vol_response` -> `VolatilityThesis | None`
  - `DebateResult.bull_rebuttal` -> `AgentResponse | None`
  - `DebateResult.duration_ms` -> `int` (convert to seconds for display)
  - `DebateResult.is_fallback` -> `bool`
- **Architecture boundary**: `reporting/` can access `models/` only. Import `DebateResult` from `agents/_parsing` (it's a dataclass, not an agent).
- Use f-strings for templating (no templating library needed)
- Follow markdown format from epic spec exactly
