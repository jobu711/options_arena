# Task: Implement liquidity multiplier in select_by_delta()

## Description
Modify `select_by_delta()` in `scoring/contracts.py` to incorporate a per-contract
liquidity score multiplier into the sort key. The multiplier adjusts `delta_distance`
so liquid contracts are preferred when delta distances are close, while delta proximity
remains the dominant factor.

## Acceptance Criteria
- [ ] Module-level constants: `_SPREAD_WEIGHT = 0.7`, `_OI_WEIGHT = 0.3`
- [ ] `_compute_liquidity_score()` helper computes score from contract spread and OI
- [ ] `effective_distance = delta_distance / max(liq_score, 0.01)` replaces raw distance
- [ ] Sort key uses `(effective_distance, contract.strike)` instead of `(delta_distance, contract.strike)`
- [ ] Delta proximity still dominates: contract at 0.01 delta distance with poor liquidity beats 0.10 with perfect liquidity
- [ ] `max_spread_pct` from `PricingConfig` used as denominator (already available in scope)
- [ ] Zero or negative mid → `spread_pct` treated as 0.0 (no spread contribution)
- [ ] Zero OI → OI component is 0.0

## Technical Details
- File: `src/options_arena/scoring/contracts.py`
- Function: `select_by_delta()` (lines ~304-365)
- Current sort key: `lambda t: (t[1], t[0].strike)` where `t[1]` is delta distance

### Liquidity Score Formula
```python
_SPREAD_WEIGHT: float = 0.7
_OI_WEIGHT: float = 0.3

def _compute_liquidity_score(
    contract: OptionContract, max_spread_pct: float
) -> float:
    """Compute 0-1 liquidity score. Higher = more liquid."""
    mid = float(contract.mid)
    if mid > 0:
        spread_pct = float(contract.spread) / mid
        spread_component = max(1.0 - spread_pct / max_spread_pct, 0.0)
    else:
        spread_component = 0.0
    oi_component = min(math.log10(contract.open_interest + 1) / 4.0, 1.0)
    return spread_component * _SPREAD_WEIGHT + oi_component * _OI_WEIGHT
```

### Sort Key Change
```python
# Before
sorted_pairs = sorted(candidates, key=lambda t: (t[1], t[0].strike))

# After
def _sort_key(pair: tuple[OptionContract, float]) -> tuple[float, Decimal]:
    contract, delta_dist = pair
    liq = _compute_liquidity_score(contract, config.max_spread_pct)
    effective = delta_dist / max(liq, 0.01)
    return (effective, contract.strike)

sorted_pairs = sorted(candidates, key=_sort_key)
```

### Edge Cases
- Contract with bid=ask (zero spread) → spread_component = 1.0
- Contract with spread > max_spread_pct → clamped to 0.0
- Contract with OI = 0 → OI component = 0.0
- Contract with OI = 10,000 (log10=4) → OI component = 1.0
- All candidates have identical liquidity → falls back to raw delta distance
