# Task: Unit tests — computation functions and scoring integration

## Description
Write integration-level unit tests that verify the end-to-end flow of liquidity
indicators through the scoring pipeline: computation → merge → normalization → composite
score impact. Also tests backward compatibility when fields are `None`.

## Acceptance Criteria
- [ ] Test that `compute_phase3_indicators()` returns both new fields
- [ ] Test normalization: `chain_spread_pct` is inverted (higher raw → lower normalized)
- [ ] Test normalization: `chain_oi_depth` is NOT inverted (higher raw → higher normalized)
- [ ] Test composite score changes when liquidity fields are populated vs `None`
- [ ] Test backward compat: pre-liquidity `IndicatorSignals` JSON → new fields are `None`
- [ ] Test composite score unchanged when new fields are `None`

## Technical Details
- Tests verify the full data flow, not individual functions (those are in Tasks 377, 379)
- Construct a small universe of `IndicatorSignals` with varying liquidity values
- Run `normalize_single_ticker()` and `composite_score()` to verify end-to-end
