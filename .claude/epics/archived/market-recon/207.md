# Task: IntelligenceService implementation

## Description
Create `IntelligenceService` in `services/intelligence.py` with 6 category-specific fetch methods plus a `fetch_intelligence()` aggregator. Mirrors the OpenBB service pattern exactly: class-based DI, never-raises contract, cache-first strategy, rate-limited yfinance wrapping via `asyncio.to_thread`. Re-export from `services/__init__.py`.

## Acceptance Criteria
- [ ] `IntelligenceService.__init__(config, cache, limiter)` — class-based DI
- [ ] `fetch_analyst_targets(ticker) -> AnalystSnapshot | None` — wraps `get_analyst_price_targets()` + `get_recommendations()`
- [ ] `fetch_analyst_activity(ticker) -> AnalystActivitySnapshot | None` — wraps `get_upgrades_downgrades()`
- [ ] `fetch_insider_activity(ticker) -> InsiderSnapshot | None` — wraps `get_insider_transactions()`
- [ ] `fetch_institutional(ticker) -> InstitutionalSnapshot | None` — wraps `get_institutional_holders()` + `get_major_holders()`
- [ ] `fetch_news_headlines(ticker) -> list[str] | None` — wraps `get_news(count=10)`, extracts titles
- [ ] `fetch_intelligence(ticker, price) -> IntelligencePackage | None` — aggregator calling all above
- [ ] `close() -> None` — explicit cleanup
- [ ] Every method follows 6-step pattern: config gate → cache check → rate-limited yfinance call → map to model → cache result → outer except returns None
- [ ] Uses `safe_float()`, `safe_int()` from `services/helpers.py`
- [ ] Uses `_yf_call` pattern: `asyncio.to_thread(fn, *args)` + `asyncio.wait_for(timeout)`
- [ ] All yfinance column names match PRD section 4 (camelCase, index-based dates, etc.)
- [ ] Cache keys: `intel:analyst:{ticker}`, `intel:activity:{ticker}`, etc.
- [ ] Re-exported from `services/__init__.py`
- [ ] Ruff, mypy, full test suite pass

### Critical yfinance parsing (from PRD, verified via live testing)
- `get_recommendations()`: Filter `period == "0m"` row; columns: `strongBuy`, `buy`, `hold`, `sell`, `strongSell` (camelCase)
- `get_upgrades_downgrades()`: Date in **index** `GradeDate`; `Action` values: `up`/`down`/`init`/`main`/`reit` → `ACTION_MAP`
- `get_insider_transactions()`: `Transaction` column always empty → parse from `Text`; column `Insider` not `Filer`, `Position` not `Relationship`, `Start Date` not `Date`
- `get_major_holders()`: Single `Value` column, indexed by string keys like `insidersPercentHeld`
- `get_institutional_holders()`: Column `pctHeld` (float), NOT `% Out`
- `get_news(count=N)`: `list[dict]`, title at `item["content"]["title"]`

### Reusable code
- `safe_float()`, `safe_int()` from `services/helpers.py`
- `ServiceCache.get()/set()` from `services/cache.py`
- `RateLimiter` context manager from `services/rate_limiter.py`
- `_yf_call` pattern from `services/market_data.py`

### Edge Cases
- Empty DataFrame from yfinance → return None
- yfinance timeout → return None (never raise)
- NaN values in yfinance output → safe_float returns None
- Ticker with no analyst coverage → AnalystSnapshot with zeroed counts
- Cache serialization roundtrip preserves all fields
- Multiple rapid calls → cache hit on second call

### Mocking Strategy
- Mock `asyncio.to_thread` to return pre-built DataFrames/dicts for each yfinance method
- Real `ServiceCache` with short TTL for cache tests
- Real `RateLimiter` (fast in tests)
- Integration tests: real yfinance calls with `@pytest.mark.integration`
