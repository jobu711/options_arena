# Task: Repository CRUD methods for ticker_metadata

## Description
Add 6 CRUD methods to `data/repository.py` for the `ticker_metadata` table: `upsert_ticker_metadata()`, `upsert_ticker_metadata_batch()`, `get_ticker_metadata()`, `get_all_ticker_metadata()`, `get_stale_tickers()`, `get_metadata_coverage()`. Follow the existing watchlist/themes patterns.

## Acceptance Criteria
- [ ] `upsert_ticker_metadata(metadata: TickerMetadata) -> None` — INSERT OR REPLACE single row
- [ ] `upsert_ticker_metadata_batch(items: list[TickerMetadata], *, commit: bool = True) -> None` — batch upsert via executemany
- [ ] `get_ticker_metadata(ticker: str) -> TickerMetadata | None` — lookup by ticker (uppercased)
- [ ] `get_all_ticker_metadata() -> list[TickerMetadata]` — return all rows
- [ ] `get_stale_tickers(max_age_days: int = 30) -> list[str]` — tickers older than max_age or not in table
- [ ] `get_metadata_coverage() -> MetadataCoverage` — return total, with_sector count, with_industry_group count, coverage ratio
- [ ] Static `_row_to_ticker_metadata(row: Row) -> TickerMetadata` helper
- [ ] New `MetadataCoverage` typed model (can be a simple frozen BaseModel in `models/metadata.py`)

## Technical Details
- Use `INSERT OR REPLACE INTO ticker_metadata` (project convention from themes pattern)
- Named row access: `row["ticker"]`, `row["sector"]`, etc.
- Enum reconstruction: `GICSSector(row["sector"]) if row["sector"] else None`
- `ticker.upper()` normalization on insert
- `commit: bool = True` parameter on batch method for atomic operations
- `get_stale_tickers` compares `updated_at` against `datetime.now(UTC) - timedelta(days=max_age_days)`
- `get_metadata_coverage` uses SQL `COUNT(*)` and `COUNT(sector)` for efficiency
