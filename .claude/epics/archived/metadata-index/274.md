# Task: API endpoints — index trigger + metadata stats

## Description
Add two API endpoints to `api/routes/universe.py`: `POST /api/universe/index` (trigger bulk indexing as background task with operation lock) and `GET /api/universe/metadata/stats` (return metadata coverage statistics). Add `MetadataStats` response schema to `api/schemas.py`.

## Acceptance Criteria
- [ ] `POST /api/universe/index` triggers bulk indexing as a background task
- [ ] Returns `202 Accepted` with task ID when started
- [ ] Returns `409 Conflict` if operation lock is already held (scan/debate/index running)
- [ ] Uses the same `asyncio.Lock` as scan and batch debate (operation mutex)
- [ ] `GET /api/universe/metadata/stats` returns `MetadataStats` from `repo.get_metadata_coverage()`
- [ ] `MetadataStats` schema in `api/schemas.py`: `total: int`, `with_sector: int`, `with_industry_group: int`, `coverage: float`
- [ ] Both endpoints follow existing universe route patterns (rate limiting, deps injection)

## Technical Details

### POST /api/universe/index
- Acquire operation lock atomically (same pattern as scan start in `routes/scan.py`)
- Create background task via `asyncio.create_task()` — reuse same bulk index logic as CLI
- Lock released in background task's `finally` block
- Counter-based task ID (existing pattern)
- Accept optional query params: `force: bool = False`, `max_age: int = 30`

### GET /api/universe/metadata/stats
- Simple query endpoint — calls `repo.get_metadata_coverage()`
- Returns `MetadataStats` response schema
- No lock needed (read-only)

### MetadataStats Schema
```python
class MetadataStats(BaseModel):
    total: int
    with_sector: int
    with_industry_group: int
    coverage: float  # with_sector / total, 0.0 if total == 0
```

### Edge Cases
- Lock contention: POST while scan is running → 409
- Lock release after error: background task crashes → lock still released
- Coverage with zero total → coverage = 0.0 (not division by zero)
