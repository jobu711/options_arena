# Task: Integration tests + edge case coverage

## Description
Write integration tests that verify the end-to-end metadata index flow: bulk index → Phase 1 loading → Phase 3 write-back → coverage stats. Also verify all existing tests still pass with the new code paths.

## Acceptance Criteria
- [ ] Integration test: bulk index populates ticker_metadata table, Phase 1 loads it, extends sector_map
- [ ] Integration test: Phase 3 write-back persists metadata, subsequent Phase 1 reads it
- [ ] Integration test: S&P 500 CSV priority preserved when metadata conflicts
- [ ] Integration test: fail-open behavior — uncached tickers pass through filters
- [ ] All 3,799+ existing Python tests still pass
- [ ] ruff, mypy, and full test suite clean

## Technical Details

### Integration Test Strategy
- Use in-memory SQLite with migration 021 applied
- Create realistic test data: mix of S&P 500 tickers (with CSV data) and non-S&P 500 tickers (metadata-only)
- Verify data flows correctly through all layers: service → repository → pipeline → ticker_score

### Key Scenarios
1. **Full flow**: Index 5 tickers → run mock Phase 1 → verify sector_map extended
2. **Incremental enrichment**: Phase 3 processes ticker → next Phase 1 sees it in metadata
3. **Priority**: S&P 500 ticker has sector from CSV AND metadata → CSV wins
4. **Fail-open**: Filter by sector with uncached tickers → uncached pass through
5. **Stale refresh**: Index with max_age=0 → all tickers re-indexed

- Empty CBOE universe + empty metadata → Phase 1 works with S&P 500 data only
- Ticker in metadata but not in CBOE universe → ignored (metadata is a cache, not authoritative)
- Migration 021 applied to existing DB with data → no data loss in other tables
