## Technical Details

### SpreadAnalysis Fields
- `spread: OptionSpread` — the constructed spread (existing model)
- `net_premium: Decimal` — net cost/credit of the spread
- `max_profit: Decimal` — maximum profit potential
- `max_loss: Decimal` — maximum loss potential
- `breakevens: list[Decimal]` — breakeven price(s)
- `risk_reward_ratio: float` — max_profit / max_loss (NaN for undefined)
- `pop_estimate: float` — probability of profit [0, 1]
- `net_greeks: OptionGreeks | None` — aggregated Greeks across all legs
- `strategy_rationale: str` — human-readable reason for selection
- `iv_regime: VolRegime | None` — the IV regime that drove selection

### SpreadConfig Fields
- `vertical_width: int = 5` — strike width for vertical spreads
- `iron_condor_wing_width: int = 5` — wing width for iron condors
- `short_leg_delta: float = 0.30` — target delta for short legs
- `min_pop: float = 0.40` — minimum probability of profit to recommend
- `max_legs: int = 4` — maximum legs per strategy
- `enabled: bool = True` — master toggle for spread construction

### Implementation approach
- Follow frozen model pattern (`ConfigDict(frozen=True)`)
- Decimal fields with `field_serializer` (prevent float precision loss)
- `math.isfinite()` + range check on `pop_estimate`, `risk_reward_ratio`
- `pop_estimate` validator constrains to [0.0, 1.0]
- `risk_reward_ratio` uses `math.isfinite()` check (allow NaN for undefined ratios)
- `field_validator` on `breakevens` ensures non-empty list
- `VolRegime` import from `enums.py` (already exists)
