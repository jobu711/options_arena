## Technical Details

### Volatility Context Rendering (agents/_parsing.py)
Add a new block in `render_volatility_context()` after existing IV metrics:
```
--- VOL SURFACE ---
SKEW 25D:          -0.045
SMILE CURVATURE:   0.012
PROB ABOVE CURRENT: 52.3%
HV YANG-ZHANG:     28.5%
```

Use existing `_render_optional("SKEW 25D", ctx.skew_25d, ".3f")` pattern. Format probabilities as percentages (`".1%"`), HV as percentage, skew/curvature as decimal.

The `volatility-intelligence` epic will later append `IV VS SURFACE` and `SURFACE R²` lines to this same block.

### Second-Order Greeks Rendering
The section already exists in `_parsing.py` (lines ~966-976) rendering `target_vanna`, `target_charm`, `target_vomma`. Since Task 486 wires these values, they will now appear in output. Verify the existing rendering still works with populated (non-None) values.

### Prompt Updates
- **Volatility agent** (`agents/prompts/volatility.py`): Add interpretation guidance for vol surface metrics. Skew_25d < 0 indicates normal put skew (hedging demand); large negative values signal fear. Smile curvature indicates tail risk pricing. prob_above_current near 0.5 is neutral.
- **Risk agent** (`agents/prompts/risk.py`): Add guidance on interpreting vanna (P&L sensitivity to vol changes × price moves), charm (delta decay over time — important for DTE-sensitive positions), and vomma (vega convexity — large vomma means vega changes rapidly with vol).
