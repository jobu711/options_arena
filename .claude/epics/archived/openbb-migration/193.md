# Task: Model & Config Extensions

## Description

Extend `OptionContract` with three new fields (`bid_iv`, `ask_iv`, `greeks_source`) and extend `OpenBBConfig` with CBOE chain configuration fields. All new fields have `None`/`False` defaults for backward compatibility. Wire the existing `GreeksSource` enum into `OptionContract`.

- [ ] `OptionContract` has `bid_iv: float | None = None` with `math.isfinite() + >= 0` validator
- [ ] `OptionContract` has `ask_iv: float | None = None` with `math.isfinite() + >= 0` validator
- [ ] `OptionContract` has `greeks_source: GreeksSource | None = None`
- [ ] `OpenBBConfig` has `cboe_chains_enabled: bool = False`
- [ ] `OpenBBConfig` has `chains_cache_ttl: int = 60`
- [ ] `OpenBBConfig` has `chain_validation_mode: bool = False`
- [ ] All existing tests pass without modification (backward compatible defaults)
- [ ] JSON roundtrip preserves new fields
- [ ] `models/__init__.py` re-exports `GreeksSource` (verify — may already exist)

### Implementation Notes
- `bid_iv` / `ask_iv` validators follow the same pattern as `market_iv`:
  ```python
  @field_validator("bid_iv", "ask_iv")
  @classmethod
  def validate_bid_ask_iv(cls, v: float | None) -> float | None:
      if v is not None and (not math.isfinite(v) or v < 0.0):
          raise ValueError(f"IV must be finite and >= 0, got {v}")
      return v
  ```
- `greeks_source` uses existing `GreeksSource` enum from `enums.py` (already has `COMPUTED` and `MARKET`)
- New fields placed AFTER `greeks` in `OptionContract` to preserve field ordering
- `OptionContract` is `frozen=True` — no mutation concerns, just construction
- `OpenBBConfig` fields use simple defaults, no complex validators needed
