# Task: Provider Orchestration + Fallback

## Description

Update `OptionsDataService` to accept an `openbb_config` parameter, build a prioritized list of `ChainProvider` instances, and iterate them with fallback on `DataSourceUnavailableError`. CBOE is tried first (when enabled and available), yfinance as fallback.

## Acceptance Criteria

- [ ] `OptionsDataService.__init__()` gains `openbb_config: OpenBBConfig | None = None` parameter
- [ ] When `openbb_config.cboe_chains_enabled=True` and SDK available: providers = [CBOE, yfinance]
- [ ] When `openbb_config.cboe_chains_enabled=False` or None: providers = [yfinance only]
- [ ] When CBOE SDK not installed: providers = [yfinance only] (even if config enabled)
- [ ] `fetch_chain()` tries providers in order, falls back on `DataSourceUnavailableError`
- [ ] `fetch_expirations()` tries providers in order with same fallback
- [ ] All providers exhausted → raises `DataSourceUnavailableError`
- [ ] Fallback events logged at WARNING with provider name and reason
- [ ] `fetch_chain_all_expirations()` continues to work (delegates to `fetch_chain`)
- [ ] Existing constructor call sites (without `openbb_config`) still work (backward compatible)

## Technical Details

### Files Modified
- `src/options_arena/services/options_data.py`

### Implementation Approach

```python
class OptionsDataService:
    def __init__(
        self,
        config: ServiceConfig,
        pricing_config: PricingConfig,
        cache: ServiceCache,
        limiter: RateLimiter,
        openbb_config: OpenBBConfig | None = None,  # NEW
    ) -> None:
        self._providers: list[ChainProvider] = []
        if openbb_config and openbb_config.cboe_chains_enabled:
            cboe = CBOEChainProvider(openbb_config, cache, limiter)
            if cboe.available:
                self._providers.append(cboe)
                logger.info("CBOE chain provider registered")
            else:
                logger.warning("CBOE chains enabled but OpenBB SDK not installed")
        yf_provider = YFinanceChainProvider(config, pricing_config, cache, limiter)
        self._providers.append(yf_provider)

    async def fetch_chain(self, ticker: str, expiration: date) -> list[OptionContract]:
        last_error: DataSourceUnavailableError | None = None
        for provider in self._providers:
            try:
                return await provider.fetch_chain(ticker, expiration)
            except DataSourceUnavailableError as e:
                logger.warning("Provider %s failed for %s: %s", type(provider).__name__, ticker, e)
                last_error = e
        raise last_error or DataSourceUnavailableError("no providers available")
```

### Key Constraints
- `openbb_config` is optional with `None` default — existing code paths are unaffected
- Import `CBOEChainProvider` at module level (it uses guarded import internally, so import won't fail)
- `fetch_chain_all_expirations()` doesn't need changes — it calls `self.fetch_chain()` which now has fallback
- `close()` should close all providers that have close methods

### Edge Cases
- `openbb_config=None` → yfinance only (backward compat)
- `cboe_chains_enabled=True` but SDK not installed → yfinance only + warning log
- Both providers fail → raise last error
- CBOE succeeds → yfinance never called (short-circuit)

### Mocking Strategy
- Mock `CBOEChainProvider` class with controllable `available` property and `fetch_chain`/`fetch_expirations`
- Mock `YFinanceChainProvider` as backup
- Use `caplog` to verify WARNING logs on fallback
- Test constructor variants with different `openbb_config` values
