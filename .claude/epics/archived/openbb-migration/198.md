# Task: DI Wiring + Health Check + Validation Mode

### Health Check Extension
```python
async def check_cboe_chains(self) -> HealthStatus:
    """Test CBOE chain endpoint with a known ticker (AAPL)."""
    if not self._openbb_config or not self._openbb_config.cboe_chains_enabled:
        return HealthStatus(
            service_name="cboe_chains",
            available=False,
            error="CBOE chains disabled",
            checked_at=datetime.now(UTC),
        )
    provider = CBOEChainProvider(self._openbb_config, self._cache, self._limiter)
    if not provider.available:
        return HealthStatus(
            service_name="cboe_chains",
            available=False,
            error="OpenBB SDK not installed",
            checked_at=datetime.now(UTC),
        )
    # Try fetching AAPL expirations as probe
    start = time.monotonic()
    try:
        await provider.fetch_expirations("AAPL")
        latency = (time.monotonic() - start) * 1000
        return HealthStatus(service_name="cboe_chains", available=True, latency_ms=latency, checked_at=datetime.now(UTC))
    except Exception as e:
        latency = (time.monotonic() - start) * 1000
        return HealthStatus(service_name="cboe_chains", available=False, latency_ms=latency, error=str(e), checked_at=datetime.now(UTC))
```

### Parallel Validation Mode
When `chain_validation_mode=True` in `OptionsDataService.fetch_chain()`:
1. Fetch from primary provider (CBOE) as normal
2. Also fetch from yfinance in parallel (background)
3. Compare: strike coverage overlap, IV differences, Greeks differences
4. Log comparison at INFO level
5. Return primary provider's result (validation is observational only)
