# Task: API lifecycle + DI + schema + serialization

## Description

Wire OpenBBService into the FastAPI application lifecycle, add a DI provider, extend `DebateResultDetail` schema with flat enrichment fields, and extract enrichment data from `row.market_context` in the `get_debate()` route.

## Acceptance Criteria

- [ ] `lifespan()` in `app.py` creates `OpenBBService(settings.openbb, cache, limiter)` when `settings.openbb.enabled` is True, stores on `app.state.openbb`
- [ ] `lifespan()` stores `None` on `app.state.openbb` when config disabled
- [ ] `lifespan()` closes OpenBBService at shutdown (guarded by `is not None`)
- [ ] `deps.py` has `get_openbb(request: Request) -> OpenBBService | None` provider
- [ ] `DebateResultDetail` in `schemas.py` has 12 flat optional enrichment fields + `enrichment_ratio`
- [ ] `get_debate()` route extracts enrichment fields from `row.market_context` into response
- [ ] All existing API tests pass unchanged
