# Task: CLI debate OpenBB wiring

## Description

Wire OpenBBService into CLI debate commands: create/close in `_debate_async()` and `_batch_async()`, fetch enrichment in `_debate_single()`, and add `--no-openbb` flag to the `debate` command.

## Acceptance Criteria

- [ ] `_debate_async()` creates `OpenBBService(settings.openbb, cache, limiter)` when `settings.openbb.enabled` and `no_openbb` is False
- [ ] `_batch_async()` creates OpenBBService with same guards
- [ ] Both close OpenBBService in `finally` blocks (guarded by `is not None`)
- [ ] `_debate_single()` accepts `openbb_svc: OpenBBService | None = None` param
- [ ] `_debate_single()` fetches enrichment via `asyncio.gather()` when service provided
- [ ] `_debate_single()` passes `fundamentals=`, `flow=`, `sentiment=` to `run_debate_v2()`
- [ ] `debate` command has `--no-openbb` flag (default False)
- [ ] When `--no-openbb` is True, service not created, all enrichment is `None`
- [ ] When OpenBB config disabled, service not created (same behavior)

### Edge Cases
- `--no-openbb` flag: service not created, enrichment all `None`
- Config disabled: same behavior as `--no-openbb`
- Service created but all fetches return `None`: debate proceeds normally
- Exception during debate: service still closed in `finally`
