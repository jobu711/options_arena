# Task: Market data models (OHLCV, Quote, TickerInfo)

## Description
Implement `market_data.py` with three Pydantic v2 models: `OHLCV` (historical price bars), `Quote` (real-time snapshot), and `TickerInfo` (fundamental data including dividend yield with provenance tracking).

## Acceptance Criteria
- [ ] `src/options_arena/models/market_data.py` contains `OHLCV`, `Quote`, `TickerInfo`
- [ ] `OHLCV`: frozen=True, fields: `ticker: str`, `date: date`, `open: Decimal`, `high: Decimal`, `low: Decimal`, `close: Decimal`, `volume: int`, `adjusted_close: Decimal`
- [ ] `Quote`: frozen=True, fields: `ticker: str`, `price: Decimal`, `bid: Decimal`, `ask: Decimal`, `volume: int`, `timestamp: datetime`
- [ ] `TickerInfo`: frozen=True, fields per models/CLAUDE.md spec including:
  - `dividend_yield: float = 0.0` (never None, decimal fraction)
  - `dividend_source: DividendSource = DividendSource.NONE`
  - `dividend_rate: float | None = None` (audit field)
  - `trailing_dividend_rate: float | None = None` (audit field)
  - `market_cap_tier: MarketCapTier | None = None`
- [ ] All `Decimal` fields have `field_serializer` converting to `str`
- [ ] `Decimal` fields constructed from strings: `Decimal("185.50")`
- [ ] Passes `ruff check` and `mypy --strict`

## Technical Details
- `frozen=True` via `model_config = ConfigDict(frozen=True)` on OHLCV, Quote, TickerInfo
- Import enums from `options_arena.models.enums`
- `dividend_yield` default `0.0` means pricing engine always gets a float — no None handling
- `dividend_source` tracks which waterfall tier produced the value
- `dividend_rate` and `trailing_dividend_rate` are audit-only, not used in pricing
- All timestamps must use UTC (`datetime` with timezone info)
