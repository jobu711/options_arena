# Task: BSM unit tests

## Description
Write comprehensive unit tests for `src/options_arena/pricing/bsm.py` in
`tests/unit/pricing/test_bsm.py`. ~40 tests covering put-call parity, known reference
values, boundary conditions, Greeks sign correctness, and IV round-trip verification.

These tests validate FR-P1 and establish the correctness baseline that BAW tests
(FR-P4, FR-P5) will build upon.

## Acceptance Criteria
- [ ] `tests/unit/pricing/__init__.py` created (empty)
- [ ] `tests/unit/pricing/test_bsm.py` created with ~40 tests
- [ ] **Put-call parity**: `C - P ≈ S*e^(-qT) - K*e^(-rT)` verified across parameter grid
- [ ] **Known values**: at least 3 Hull-textbook-level reference prices verified (parametrized)
- [ ] **Boundary T→0**: price converges to intrinsic value for call and put
- [ ] **Boundary sigma→0**: price equals discounted intrinsic value
- [ ] **ATM approximation**: ATM call ≈ `0.4 * S * sigma * sqrt(T)` for small T, q=0, r≈0
- [ ] **Greeks sign correctness**: delta ∈ [0,1] calls / [-1,0] puts, gamma > 0, vega > 0, theta < 0 (typical)
- [ ] **IV round-trip**: `bsm_price(bsm_iv(price)) ≈ price` within solver tolerance
- [ ] **IV non-convergence**: `ValueError` raised for impossible market prices
- [ ] **Parametrized** across `OptionType.CALL` / `OptionType.PUT`
- [ ] **Parametrized** across multiple `(S, K, T, r, q, sigma)` tuples
- [ ] All tests use `pytest.approx(rel=1e-6)` for float comparisons
- [ ] No mocking — pure math functions, deterministic output
- [ ] `uv run pytest tests/unit/pricing/test_bsm.py -v` passes

### Test categories (~40 tests total)

1. **Pricing correctness** (~8 tests)
   - Known reference values (parametrized)
   - Put-call parity across all parameter sets
   - ATM approximation

2. **Boundary conditions** (~8 tests)
   - T=0 → intrinsic (call and put, ITM and OTM)
   - sigma=0 → discounted intrinsic
   - Deep ITM and deep OTM

3. **Greeks correctness** (~12 tests)
   - Sign correctness for all 5 Greeks (call + put = 10)
   - `pricing_model` field is `PricingModel.BSM`
   - Gamma > 0 for both call and put

4. **IV solver** (~8 tests)
   - Round-trip for multiple parameter sets
   - Convergence with yfinance-style initial guess
   - Convergence with default initial guess (0.30)
   - Non-convergence raises ValueError
   - Respects PricingConfig tolerance

5. **Vega standalone** (~4 tests)
   - Matches vega from `bsm_greeks`
   - Positive for both call and put
   - Parametrized across moneyness
