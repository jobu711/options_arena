# Task: Implement BAW American pricing module

## Description
Implement `src/options_arena/pricing/american.py` — the Barone-Adesi-Whaley analytical
approximation for American option pricing with finite-difference Greeks and a `brentq`-based
IV solver. This is the core differentiator of the project: correct American options pricing
instead of misapplying European BSM to U.S. equity options.

Covers PRD requirements **FR-P2**, **FR-P4** (identity), **FR-P5** (dominance).

## Technical Details

### Functions to implement

1. **`american_price`** — BAW analytical approximation:
   - Compute BSM European price as base
   - Compute auxiliary parameters: `M = 2*r/sigma²`, `N_param = 2*(r-q)/sigma²`, `K_param = 1 - e^(-rT)`, `q1`, `q2`
   - Find critical stock price `S*` (calls) or `S**` (puts) via Newton-Raphson on boundary condition
   - Add early exercise premium: `A2 * (S/S*)^q2` for calls, `A1 * (S/S**)^q1` for puts
   - Immediate exercise when `S >= S*` (call) or `S <= S**` (put)

2. **`american_greeks`** — Finite-difference bump-and-reprice:
   - Bump sizes: `dS = 0.01 * S`, `dT = 1/365`, `dSigma = 0.14`, `dR = 0.14`
   - Delta: centered difference on S
   - Gamma: second derivative on S
   - Theta: `(price(T-dT) - price(T)) / dT`, forward difference fallback when `T - dT <= 0`
   - Vega: centered difference on sigma
   - Rho: centered difference on r
   - Returns `OptionGreeks(pricing_model=PricingModel.BAW, ...)`

3. **`american_iv`** — Bracket-based IV solver:
   - Objective: `f(sigma) = american_price(..., sigma, ...) - market_price`
   - `scipy.optimize.brentq(f, 1e-6, 5.0, xtol=config.iv_solver_tol, maxiter=config.iv_solver_max_iter)`
   - Raises `ValueError` with diagnostic info if bracket doesn't contain root
   - Log warning for edge cases

### Critical design decisions
- **BAW calls `bsm_price` internally** — the European price is the base component
- **NO Newton-Raphson for IV** — BAW has no analytical vega w.r.t. IV. `brentq` is bracket-based, guaranteed convergent, needs no derivative
- **Finite-difference Greeks** — BAW has no closed-form Greeks. All via bump-and-reprice
- **T=0 guard for theta** — when `T < dT`, switch to forward difference to avoid `T - dT <= 0`
