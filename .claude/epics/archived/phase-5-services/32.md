# Task: Universe Service

## Description
Build `universe.py` — fetches the optionable ticker universe from CBOE, classifies tickers by S&P 500 membership and GICS sector (via Wikipedia), and assigns `MarketCapTier`. Two external data sources: CBOE CSV (optionable universe) and Wikipedia table (S&P 500 constituents).

## Acceptance Criteria
- [ ] `UniverseService` class with DI constructor (`ServiceConfig`, `ServiceCache`, `RateLimiter`)
- [ ] `fetch_optionable_tickers()`: CBOE CSV → `list[str]`
  - Downloads CBOE directory CSV
  - Extracts ticker symbols
  - Filters out index symbols containing `^`, `$`, `/`
  - Strips whitespace, deduplicates
  - Caches for 24 hours
- [ ] `fetch_sp500_constituents()`: Wikipedia → `dict[str, str]` (ticker → sector)
  - Uses `pd.read_html(url, attrs={"id": "constituents"})` (Context7-verified)
  - Validates `{"Symbol", "GICS Sector"} <= set(df.columns)` at parse time
  - Translates tickers: `.` → `-` (BRK.B → BRK-B)
  - Raises `InsufficientDataError` on schema drift
  - Caches for 24 hours
- [ ] `classify_market_cap()`: `int | None` → `MarketCapTier | None`
  - Mega ≥ 200B, Large ≥ 10B, Mid ≥ 2B, Small ≥ 300M, Micro < 300M
- [ ] ~15 tests covering all paths

## Technical Details
- **Files to create**: `src/options_arena/services/universe.py`
- **Test files**: `tests/unit/services/test_universe.py`
- S&P 500 URL: `https://en.wikipedia.org/wiki/List_of_S%26P_500_companies`
- `SP500_REQUIRED_COLUMNS = {"Symbol", "GICS Sector"}`
- CBOE download via `httpx.AsyncClient` or `pd.read_html`/`pd.read_csv`
- Wikipedia parsing via `asyncio.to_thread(pd.read_html, ...)` (sync call wrapped)
- Market cap thresholds match those in epic spec
