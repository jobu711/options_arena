# Task: Options Data Service

## Description
Build `options_data.py` — fetches option chains from yfinance, applies basic liquidity filters, and converts to `list[OptionContract]`. Passes `impliedVolatility` through as `market_iv`. Does NOT compute Greeks — that is `pricing/dispatch.py`'s job.

## Acceptance Criteria
- [ ] `OptionsDataService` class with DI constructor (`ServiceConfig`, `PricingConfig`, `ServiceCache`, `RateLimiter`)
- [ ] `fetch_expirations()`: available dates → `list[date]` sorted ascending
- [ ] `fetch_chain()`: chain for specific expiration → `list[OptionContract]`
- [ ] Column mapping: yfinance camelCase columns → `OptionContract` fields
- [ ] `strike`, `last`, `bid`, `ask` converted to `Decimal(str(value))`
- [ ] `volume`, `open_interest` converted via `safe_int(value) or 0`
- [ ] `impliedVolatility` → `market_iv` as `float` (no re-annualization)
- [ ] `exercise_style = ExerciseStyle.AMERICAN` for all contracts
- [ ] `greeks = None` for all contracts (yfinance provides NO Greeks)
- [ ] Liquidity filter: OI >= `PricingConfig.min_oi`, volume >= `PricingConfig.min_volume`
- [ ] Both-zero bid/ask rejection (truly dead contracts)
- [ ] Zero-bid exemption: bid=0/ask>0 passes through
- [ ] `fetch_chain_all_expirations()`: concurrent fetch for all dates
- [ ] ~15 tests covering all paths

## Technical Details
- **Files to create**: `src/options_arena/services/options_data.py`
- **Test files**: `tests/unit/services/test_options_data.py`
- yfinance chain columns (Context7-verified): `contractSymbol`, `lastTradeDate`, `strike`, `lastPrice`, `bid`, `ask`, `change`, `percentChange`, `volume`, `openInterest`, `impliedVolatility`, `inTheMoney`, `contractSize`, `currency`
- `_passes_liquidity_filter()` is a pure function (testable independently)
- `option_type` determined by which DataFrame (`.calls` vs `.puts`)
- `ticker` and `expiration` passed as arguments, not from yfinance data
