# Task: Implement repository.py — typed CRUD for ScanRun and TickerScore

## Description
Write `src/options_arena/data/repository.py` from scratch. The `Repository` class provides typed CRUD operations for `ScanRun` and `TickerScore` models. Every method accepts and returns Pydantic models — never raw dicts, tuples, or Row objects. `IndicatorSignals` is serialized to/from JSON via Pydantic's `model_dump_json()` / `model_validate_json()`.

Write tests in `tests/unit/data/test_repository.py`.

- [ ] `Repository` class with DI constructor: `__init__(self, db: Database)`
- [ ] Accesses connection via `Database` (not by holding its own aiosqlite ref)
- [ ] Full type annotations, `mypy --strict` passes
- [ ] `logging` module only (DEBUG for queries)

### Write Operations
- [ ] `async def save_scan_run(self, scan_run: ScanRun) -> int` — INSERT, return `lastrowid`
  - Serializes `started_at`/`completed_at` via `.isoformat()` (handles `None` for completed_at)
  - Serializes `preset` via `.value` (StrEnum string)

## Technical Details
- Imports from `models/`: `ScanRun`, `TickerScore`, `IndicatorSignals`, `ScanPreset`, `SignalDirection`
- Uses `aiosqlite.Row` for named column access (set by Database on connection)
- `ScanRun` is `frozen=True` — cannot mutate after construction. Reconstruct with `ScanRun(id=row["id"], ...)`
- `save_scan_run` does NOT set `scan_run.id` (frozen model) — returns `int` for caller to use
- Parameterized queries only — never f-strings or `.format()` in SQL
- `executemany` for batch inserts (more efficient than individual inserts in a loop)
