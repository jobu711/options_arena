# Task: scan Command + Rendering

## Description

Implement the `scan` command — the primary CLI entry point that runs the full 4-phase
pipeline (universe → scoring → options → persist). Includes option parsing (`--preset`,
`--top-n`, `--min-score`, `--sectors`), `_scan_async()` with full service lifecycle,
`RichProgressCallback` integration, SIGINT handling, Rich results table rendering,
summary stats, and regulatory disclaimer.

This is the largest task (~150 lines) and the integration point for everything else.

## Acceptance Criteria

- [ ] `scan` command with `--preset` (ScanPreset), `--top-n`, `--min-score`, `--sectors` options
- [ ] CLI args override `AppSettings` defaults (CLI > env > field defaults)
- [ ] `_scan_async()` creates all infrastructure: AppSettings, ServiceCache, RateLimiter, Database, Repository, all 4 services, ScanPipeline
- [ ] SIGINT handler registered inside async function, restored in `finally`
- [ ] RichProgressCallback connected to pipeline via `Progress` context manager
- [ ] `render_scan_table(result: ScanResult)` with trading-convention styling (green/red/yellow direction, formatted prices/Greeks)
- [ ] Summary line: "N tickers scanned, M scored, K recommendations in X.Xs"
- [ ] Disclaimer printed after every results table
- [ ] ALL services closed in `finally` block (no leaked connections)
- [ ] Cancelled scan displays partial results if available
- [ ] 5 unit tests
- [ ] ruff + mypy --strict pass

### Implementation Notes

#### Command Options
| Option | Type | Default | Source |
|--------|------|---------|--------|
| `--preset` | `ScanPreset` | `sp500` | StrEnum auto-choices |
| `--top-n` | `int` | `50` | From `ScanConfig.top_n` |
| `--min-score` | `float` | `0.0` | From `ScanConfig.min_score` |
| `--sectors` | `str \| None` | `None` | Comma-separated GICS |

#### Service Lifecycle Flow
1. `AppSettings()` → apply CLI overrides
2. Create infra: `ServiceCache`, `RateLimiter`, `Database` (connect), `Repository`
3. Create services: `MarketDataService`, `OptionsDataService`, `FredService`, `UniverseService`
4. Create `ScanPipeline(settings, market_data, options_data, fred, universe_svc, repo)`
5. Register SIGINT handler with `CancellationToken`
6. `with Progress(...) as progress:` → `RichProgressCallback(progress)`
7. `await pipeline.run(preset, token, callback)` → `ScanResult`
8. Render table + summary + disclaimer
9. `finally`: close all services, database, restore SIGINT handler

#### Scan Results Table Columns
| Column | Justify | Style | Format |
|--------|---------|-------|--------|
| Ticker | left | bold white | `AAPL` |
| Score | right | cyan | `87.3` (1 dec) |
| Direction | center | green/red/yellow | `BULLISH` |
| Type | center | — | `CALL`/`PUT` |
| Strike | right | — | `$185.00` (2 dec) |
| Exp | right | — | `2026-04-17` |
| DTE | right | — | `53` |
| Delta | right | — | `0.3512` (4 dec) |
| IV | right | — | `32.1%` (1 dec) |
| Bid/Ask | right | — | `$2.45/$2.65` |

Tickers with no recommendation show `--` in contract columns.
