# Task: Logging + Entry Point

## Description

Create the CLI package skeleton with Typer app, dual-handler logging configuration
(`RichHandler` for console + `RotatingFileHandler` for file), `@app.callback()` with
`--verbose` flag, and `pyproject.toml` entry point. This is the foundation task that
all other CLI tasks depend on.

## Acceptance Criteria

- [ ] `src/options_arena/cli/__init__.py` re-exports `app` with `__all__`
- [ ] `src/options_arena/cli/app.py` has `app = typer.Typer(...)` and `configure_logging()`
- [ ] `configure_logging(verbose)` creates RichHandler (stderr, `markup=False`, `show_path=False`) + RotatingFileHandler (5MB x 3)
- [ ] `@app.callback()` runs `configure_logging()` before any command
- [ ] `pyproject.toml` has `[project.scripts] options-arena = "options_arena.cli:app"`
- [ ] `uv run options-arena --help` prints help text
- [ ] Log file created in `logs/` directory
- [ ] Noisy loggers (`aiosqlite`, `httpx`, `httpcore`, `yfinance`) suppressed to WARNING
- [ ] 5 unit tests in `tests/unit/cli/test_logging.py`
- [ ] ruff + mypy --strict pass on new files

## Technical Details

### Files Created
- `src/options_arena/cli/__init__.py` ‚Äî re-export `app`
- `src/options_arena/cli/app.py` ‚Äî Typer app, `configure_logging()`, `@app.callback()`

### Files Modified
- `pyproject.toml` ‚Äî add `[project.scripts]` entry

### Implementation Notes
- `RichHandler(markup=False)` ‚Äî prevents `[AAPL]` bracket crashes
- `RichHandler(show_path=False)` ‚Äî keeps terminal clean
- `Console(stderr=True)` on RichHandler ‚Äî separates logging from data output
- `root.handlers.clear()` before adding handlers ‚Äî prevents duplicates on re-entry
- Never use `logging.basicConfig()` ‚Äî conflicts with manual setup
- Root logger level = DEBUG, console handler level = INFO (or DEBUG if `--verbose`)
- File handler level = always DEBUG regardless of `--verbose`
- Suppressed loggers: `aiosqlite`, `httpx`, `httpcore`, `yfinance` set to WARNING
- `LOG_DIR = Path("logs")`, `LOG_FILE = LOG_DIR / "options_arena.log"`
- File format: `%(asctime)s | %(levelname)-8s | %(name)s | %(message)s`

### Tests (~5)
- `test_configure_logging_creates_two_handlers` ‚Äî verify RichHandler + RotatingFileHandler
- `test_noisy_loggers_suppressed` ‚Äî all 4 noisy loggers at WARNING+
- `test_verbose_lowers_console_handler` ‚Äî `--verbose` sets console to DEBUG
- `test_log_directory_created` ‚Äî `logs/` dir created if missing
- `test_handlers_cleared_on_reentry` ‚Äî calling twice doesn't duplicate handlers
