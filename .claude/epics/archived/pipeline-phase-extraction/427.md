# Task: Slim pipeline.py to orchestrator with thin wrappers

## Description

Transform `pipeline.py` from a 1,362 LOC god-class into a ~170 LOC orchestrator. Remove the
extracted phase method bodies and module-level helpers, replacing them with:
1. Thin delegation wrappers on `ScanPipeline` (preserves ~110+ test call sites)

## Technical Details

### Thin Wrapper Pattern

```python
# Preserves backward compat for ~110+ direct test calls
async def _phase_universe(
    self, preset: ScanPreset, progress: ProgressCallback
) -> UniverseResult:
    return await run_universe_phase(
        preset, progress,
        universe=self._universe, market_data=self._market_data,
        repository=self._repository, scan_config=self._settings.scan,
    )

async def _phase_scoring(
    self, universe_result: UniverseResult, progress: ProgressCallback
) -> ScoringResult:
    return await run_scoring_phase(
        universe_result, progress,
        scan_config=self._settings.scan,
    )

async def _phase_options(
    self,
    scoring_result: ScoringResult,
    universe_result: UniverseResult,
    progress: ProgressCallback,
) -> OptionsResult:
    return await run_options_phase(
        scoring_result, universe_result, progress,
        fred=self._fred, market_data=self._market_data,
        options_data=self._options_data, repository=self._repository,
        scan_config=self._settings.scan, pricing_config=self._settings.pricing,
    )

async def _phase_persist(self, ...) -> ScanResult:
    return await run_persist_phase(...)

async def _process_ticker_options(self, ...) -> ...:
    return await process_ticker_options(...)
```

### Monkey-Patching Compatibility

Tests that monkey-patch `pipeline._phase_scoring` (5 instances across 4 files) will still
work because the thin wrapper methods exist on the `ScanPipeline` instance. The monkey-patch
replaces the wrapper, and `run()` won't call the wrapper (it calls the standalone function
directly). However, tests that monkey-patch AND then call `pipeline.run()` need careful
verification — check if any tests rely on `run()` going through the wrappers.

**IMPORTANT**: If any tests monkey-patch a wrapper AND call `run()`, the `run()` method
must also go through the wrappers (not directly to standalone functions) to preserve this
pattern. Verify by checking test patterns before deciding.
