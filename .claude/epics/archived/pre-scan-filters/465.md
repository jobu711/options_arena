# Task: Create filter models

## Description
Create `models/filters.py` with four frozen Pydantic v2 models: `UniverseFilters`, `ScoringFilters`, `OptionsFilters`, and `ScanFilterSpec`. These compose into a single object describing all pre-scan filter criteria. Re-export from `models/__init__.py`. This is the foundation task — no existing code changes, purely additive.

## Acceptance Criteria
- [ ] `models/filters.py` exists with all 4 models
- [ ] All models have `ConfigDict(frozen=True)`
- [ ] All numeric validators use `math.isfinite()` guard
- [ ] Cross-field validators: `min_dte <= max_dte`, delta primary range ⊂ fallback range, `min_price <= max_price`
- [ ] `custom_tickers` validator: uppercase, strip, TICKER_RE check, dedup, max 200
- [ ] `sectors` validator reuses `normalize_sectors` pattern from `ScanConfig`
- [ ] `industry_groups` validator reuses `normalize_industry_groups` pattern
- [ ] `market_cap_tiers` validator reuses `deduplicate_market_cap_tiers` pattern
- [ ] `min_score`: `0.0 <= v <= 100.0` + `isfinite()`
- [ ] `min_direction_confidence`: `0.0 <= v <= 1.0` + `isfinite()`
- [ ] `ScanFilterSpec()` with no args produces valid defaults matching current `ScanConfig`/`PricingConfig` defaults
- [ ] All 4 models re-exported from `models/__init__.py` and added to `__all__`
- [ ] ~40 unit tests pass

## Technical Details

### Models to create in `src/options_arena/models/filters.py`:

**UniverseFilters** (Phase 1 — cheap filters before/during OHLCV fetch):
- `preset: ScanPreset = ScanPreset.SP500`
- `sectors: list[GICSSector] = []`
- `industry_groups: list[GICSIndustryGroup] = []`
- `custom_tickers: list[str] = []`
- `market_cap_tiers: list[MarketCapTier] = []`
- `ohlcv_min_bars: int = 200`
- `min_price: float = 10.0`
- `max_price: float | None = None`

**ScoringFilters** (post-Phase 2 — applied after indicators + composite scoring):
- `direction_filter: SignalDirection | None = None`
- `min_score: float = 0.0`
- `min_direction_confidence: float = 0.0`

**OptionsFilters** (Phase 3 — applied during options chain processing):
- `top_n: int = 50`
- `min_dollar_volume: float = 10_000_000.0`
- `min_dte: int = 30`
- `max_dte: int = 365`
- `exclude_near_earnings_days: int | None = None`
- `min_iv_rank: float | None = None`
- `min_oi: int = 100`
- `min_volume: int = 1`
- `max_spread_pct: float = 30.0`
- `delta_primary_min: float = 0.20`
- `delta_primary_max: float = 0.50`
- `delta_fallback_min: float = 0.10`
- `delta_fallback_max: float = 0.80`

**ScanFilterSpec** (composite):
- `universe: UniverseFilters = UniverseFilters()`
- `scoring: ScoringFilters = ScoringFilters()`
- `options: OptionsFilters = OptionsFilters()`

### Validators to implement:
- Reuse `TICKER_RE` from `config.py` for custom_tickers validation
- Reuse `SECTOR_ALIASES`, `INDUSTRY_GROUP_ALIASES` for normalization
- `model_validator(mode="after")` for cross-field checks on `OptionsFilters`:
  - `min_dte <= max_dte`
  - `delta_primary_min >= delta_fallback_min`
  - `delta_primary_max <= delta_fallback_max`
  - `delta_primary_min <= delta_primary_max`
  - `delta_fallback_min <= delta_fallback_max`
- `model_validator(mode="after")` for cross-field check on `UniverseFilters`:
  - `min_price <= max_price` (when both set)
- `validate_all_finite` model_validator on each model (NaN/Inf defense)

### Files affected:
- `src/options_arena/models/filters.py` (NEW)
- `src/options_arena/models/__init__.py` (add re-exports)
