# Task: Architecture Hardening

## Description

Add missing model validators for data integrity, API rate limiting via `slowapi`, optional
structured JSON logging via `python-json-logger`, and optimize Phase 3 scan pipeline batching
with semaphore-bounded concurrency.

**Audit findings**: AUDIT-019, AUDIT-004, AUDIT-012, AUDIT-021

## Acceptance Criteria

### Model Validators (AUDIT-019)
- [ ] `Quote`: `field_validator` on `price`, `bid`, `ask` — positivity + `math.isfinite()`
- [ ] `OptionContract`: `field_validator` on `strike`, `bid`, `ask`, `last` — non-negative +
      `math.isfinite()`; `volume`, `open_interest` — non-negative integers
- [ ] `WatchlistTickerDetail` and `HistoryPoint`: `composite_score` constrained to `[0, 100]`
- [ ] `ScanConfig`: `top_n` minimum 1, `ohlcv_min_bars` minimum 5 (or reasonable lower bound)
- [ ] Invalid values raise `ValidationError` with clear messages
- [ ] Existing data construction in tests/services updated if needed to pass new validators

### API Rate Limiting (AUDIT-004)
- [ ] `uv add slowapi` — add dependency
- [ ] Rate limiter middleware added to FastAPI app in `api/app.py`
- [ ] Per-IP limits: generous for loopback (e.g., 60/minute for most endpoints)
- [ ] Scan and debate endpoints: stricter limits (e.g., 5/minute) since they're expensive
- [ ] 429 Too Many Requests response with `Retry-After` header
- [ ] Rate limiting does not conflict with operation mutex (lock is for concurrency, rate limit
      is for abuse prevention)

### Structured Logging (AUDIT-012)
- [ ] `uv add python-json-logger` — add dependency
- [ ] New config option: `LogConfig.json_mode: bool = False` in `models/config.py`
- [ ] When `json_mode=True`, file handler uses `pythonjsonlogger.JsonFormatter`
- [ ] `scan_id` and `ticker` fields injected via `logging.LoggerAdapter` or `extra={}` in
      scan pipeline and debate orchestrator
- [ ] Text mode (default) unchanged — no regression in terminal output
- [ ] JSON log lines parseable by standard log aggregators (each line is valid JSON)

### Phase 3 Batch Optimization (AUDIT-021)
- [ ] Replace sequential batch loop in scan Phase 3 with semaphore-bounded `asyncio.gather()`
- [ ] Semaphore size configurable via `ScanConfig.options_concurrency` (default: 5)
- [ ] All top-N tickers submitted at once, semaphore limits concurrent option chain fetches
- [ ] Error isolation preserved: one ticker failure doesn't abort others
- [ ] Performance improvement measurable (or at minimum, no regression)

## Technical Details

### Model Validators Pattern
```python
@field_validator("price")
@classmethod
def _validate_price(cls, v: float) -> float:
    if not math.isfinite(v):
        raise ValueError("price must be finite")
    if v <= 0:
        raise ValueError("price must be positive")
    return v
```
Note: Some models may use `frozen=True` — validators still work on frozen models (they run
before freezing). Check existing validators to avoid conflicts.

### slowapi Integration
```python
from slowapi import Limiter
from slowapi.util import get_remote_address

limiter = Limiter(key_func=get_remote_address)
app.state.limiter = limiter
app.add_exception_handler(RateLimitExceeded, _rate_limit_exceeded_handler)

@app.get("/api/scan")
@limiter.limit("5/minute")
async def scan(...): ...
```

### Semaphore-Bounded Gather
```python
sem = asyncio.Semaphore(config.options_concurrency)

async def _fetch_with_sem(ticker: str) -> TickerResult | Exception:
    async with sem:
        return await _fetch_option_chain(ticker)

results = await asyncio.gather(
    *[_fetch_with_sem(t) for t in top_n_tickers],
    return_exceptions=True,
)
```
