## Technical Details

### Domain-Specific Context Labels Per Agent
- **Flow**: PUT/CALL RATIO, VOLUME, unusual activity signals, OI changes
- **Fundamental**: P/E RATIO, DIVIDEND YIELD, sector valuation, earnings proximity
- **Risk v2**: All Phase 1 outputs (trend, volatility, flow, fundamental), position sizing,
  hedging strategies, max loss scenarios. Output schema is `TradeThesis` (not `AgentResponse`).

### Risk v2 Example Schema
```json
{
  "direction": "bullish",
  "confidence": 0.50,
  "summary": "ACME presents a moderate bullish case with ADX at 32 confirming ...",
  "risk_assessment": "Max loss limited to $2.15 per contract. Position size ...",
  "strategy": "long_call",
  "max_loss_per_contract": 2.15,
  "position_size_pct": 2.0,
  "hedge_recommendation": "Consider protective put at 140 strike ...",
  "model_used": "llama-3.3-70b-versatile"
}
```

### Key Constraints
- Risk v2 output is `TradeThesis`, not `AgentResponse` — example must match that schema
- Risk v1 (fallback) does NOT get a few-shot — it's legacy
- Groq/Llama-optimized formatting: JSON blocks
- Must leave headroom in 8192 context window
