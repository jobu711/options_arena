# Task: API dimensional filtering and TypeScript types

## Description

Extend the `GET /api/scan/{id}/scores` endpoint with query parameters for dimensional score filtering. Add 8 new filter params and extend sort options to support `direction_confidence`. Extend `POST /api/scan` with pre-scan filter fields on `ScanRequest`. Update TypeScript `TickerScore` interface to include the 3 new fields (`dimensional_scores`, `direction_confidence`, `market_regime`). All filtering is server-side in the existing Python list comprehension chain — no SQL WHERE clauses needed.

## Acceptance Criteria

- [ ] `GET /api/scan/{id}/scores` accepts 8 new query params: `min_confidence`, `market_regime`, `min_trend`, `min_iv_vol`, `min_flow`, `min_risk`, `max_earnings_days`, `min_earnings_days`
- [ ] Sort extended to support `direction_confidence` as a sort field
- [ ] Server-side filter chain in `get_scores()` applies all dimensional filters
- [ ] `POST /api/scan` `ScanRequest` accepts `market_cap_tiers` and `exclude_near_earnings_days`
- [ ] TypeScript `TickerScore` interface includes `dimensional_scores?`, `direction_confidence?`, `market_regime?`
- [ ] API response for new scans includes dimensional data on each TickerScore
- [ ] Filters with None/omitted values are ignored (backward compatible)
- [ ] All existing API tests pass

## Technical Details

### API Route Extension (`api/routes/scan.py`)
- **`get_scores()` endpoint**: Add `Query()` parameters:
  - `min_confidence: float | None = None` — filter `direction_confidence >= min_confidence`
  - `market_regime: MarketRegime | None = None` — filter exact match
  - `min_trend: float | None = None` — filter `dimensional_scores.trend >= min_trend`
  - `min_iv_vol: float | None = None` — filter `dimensional_scores.iv_vol >= min_iv_vol`
  - `min_flow: float | None = None` — filter `dimensional_scores.flow >= min_flow`
  - `min_risk: float | None = None` — filter `dimensional_scores.risk >= min_risk`
  - `max_earnings_days: int | None = None` — exclude tickers with earnings > N days
  - `min_earnings_days: int | None = None` — only tickers with earnings >= N days
- Extend existing list comprehension filter chain (~lines 195-206) with new checks
- Each filter skips if the value is None (backward compatible)
- Handle `dimensional_scores is None` gracefully (pre-migration data passes through or is excluded based on filter)

### Sort Extension
- Add `direction_confidence` to the valid sort fields
- Handle None values in sort (push to end)

### ScanRequest Schema (`api/schemas.py`)
- Add `market_cap_tiers: list[MarketCapTier] = []` with validator for deduplication
- Add `exclude_near_earnings_days: int | None = None`
- These are passed through to `ScanConfig` for pipeline Phase 1 filtering (Task 4 wires them)

### TypeScript Types (`web/src/types/scan.ts`)
- Extend `TickerScore` interface:
```typescript
dimensional_scores?: {
  trend: number | null
  iv_vol: number | null
  hv_vol: number | null
  flow: number | null
  microstructure: number | null
  fundamental: number | null
  regime: number | null
  risk: number | null
} | null
direction_confidence?: number | null
market_regime?: 'trending' | 'mean_reverting' | 'volatile' | 'crisis' | null
```

### Edge Cases
- All filters set simultaneously → AND composition
- Pre-migration tickers with `dimensional_scores=None` → excluded by dimensional filters, included by non-dimensional filters
- `min_confidence=0.0` → includes everything with confidence, excludes None
- `market_regime` filter on ticker with `None` regime → excluded
- Sort by `direction_confidence` with mixed None values → Nones sort to end
- Empty result after filtering → valid empty paginated response
