# Task: Regime banner and dimensional score display

## Description

Add visual context to scan results: a color-coded market regime banner above results, direction confidence as a sortable DataTable column, expandable rows showing 8 dimensional score bar segments, and a dimensional score breakdown section in the TickerDrawer. The regime banner shows the dominant (mode) classification across all scored tickers with strategy hint text.

## Acceptance Criteria

- [ ] `RegimeBanner.vue` displays the dominant market regime above scan results
- [ ] Banner is color-coded: green (TRENDING), blue (MEAN_REVERTING), orange (VOLATILE), red (CRISIS)
- [ ] Banner includes strategy hint text (e.g., "Volatile — favor premium selling")
- [ ] Regime derived as mode (most common) of all `market_regime` values in current results
- [ ] Direction confidence shown as a sortable percentage column in DataTable
- [ ] DataTable rows expandable to show 8 horizontal bar segments for dimensional scores
- [ ] Bars color-coded: red (<30), yellow (30-60), green (>60)
- [ ] TickerDrawer sidebar includes dimensional score breakdown section
- [ ] Pre-migration tickers (null dimensional data) display "--" gracefully
- [ ] All existing E2E tests pass

## Technical Details

### RegimeBanner.vue (new component)
- Receives `scores: TickerScore[]` as prop
- Computes mode of `market_regime` across all tickers (exclude null)
- If no regime data (all null), hide banner
- Color scheme:
  - TRENDING: green background (#10b981), "Trending — favor directional plays"
  - MEAN_REVERTING: blue (#3b82f6), "Mean Reverting — favor range-bound strategies"
  - VOLATILE: orange (#f59e0b), "Volatile — favor premium selling"
  - CRISIS: red (#ef4444), "Crisis — reduce position sizes, favor hedges"
- PrimeVue `Message` or custom styled banner

### Direction Confidence Column
- Add column to DataTable: `direction_confidence` formatted as percentage (e.g., "73%")
- Sortable: click header to sort ascending/descending
- Handle null: display "--" for pre-migration data
- Column should be optional/toggleable (don't clutter default view)

### Expandable Rows — Dimensional Score Bars
- PrimeVue DataTable `rowExpansion` slot
- 8 horizontal bars, one per dimensional family:
  - trend, iv_vol, hv_vol, flow, microstructure, fundamental, regime, risk
- Each bar: label on left, filled bar 0-100, numeric value on right
- Color coding:
  - score < 30: red (#ef4444)
  - 30 <= score < 60: yellow (#f59e0b)
  - score >= 60: green (#10b981)
  - null: gray (#6b7280), display "--"
- Compact layout: all 8 bars visible without scrolling

### TickerDrawer Dimensional Section
- Add "Dimensional Scores" section below existing content in the drawer
- Same 8-bar layout as expandable rows but in vertical list format
- Include direction confidence with visual indicator
- Include market regime badge

### ScanResultsPage.vue Updates
- Import and place `RegimeBanner` above DataTable
- Add expandable row template to DataTable
- Add direction_confidence column
- Wire regime computation (mode of all visible scores)

### Files Affected
- `web/src/components/RegimeBanner.vue` (new, ~60 lines)
- `web/src/components/DimensionalScoreBars.vue` (new, ~100 lines — shared between table and drawer)
- `web/src/pages/ScanResultsPage.vue` (add banner, column, expandable rows)
- `web/src/components/TickerDrawer.vue` (add dimensional section)

### Edge Cases
- All tickers have same regime → banner shows that regime
- Tie between regimes (e.g., 50% trending, 50% volatile) → pick first alphabetically or first encountered
- All tickers have null regime → banner hidden
- Expandable row with all-null dimensional scores → 8 gray "--" bars
- Direction confidence of exactly 0 → "0%", not "--"
- Direction confidence of exactly 1.0 → "100%"
- Very long ticker list → regime mode computation performance (should be trivial for 5K)
