# Task: Create FilterSummaryChips.vue component

## Description
Create a new `web/src/components/scan/FilterSummaryChips.vue` component that displays active (non-default) scan filters as removable chips. Each chip shows a human-readable label and an X button to clear that filter. A "Clear All" link appears when 2+ chips are active.

## Acceptance Criteria
- [ ] Generates chips from `PreScanFilterPayload` for all non-default filter values
- [ ] Each chip has a close (X) button that emits `clear-filter` with the filter key
- [ ] "Clear All" link appears when 2+ chips are active, emits `clear-all`
- [ ] Chips hidden when no active filters
- [ ] Disabled state dims chips and disables interaction
- [ ] Chip label formatting matches PRD spec exactly (see below)

### Chip Generation Logic
| Filter | Condition (non-default) | Label |
|--------|------------------------|-------|
| `direction_filter` | not null | "Bullish Only" / "Bearish Only" / "Neutral Only" |
| `market_cap_tiers` | length > 0 | "Mega, Large" (comma-separated, capitalized) |
| `min_iv_rank` | not null | "IV Rank > {N}%" |
| `min_score` | not null | "Score > {N}" |
| `exclude_near_earnings_days` | not null | "Exclude {N}d Earnings" |
| `min_price` + `max_price` | either set | "$50 - $200" / "Min $50" / "Max $200" |
| `min_dte` + `max_dte` | either set | "DTE 14d - 90d" / "DTE > 14d" / "DTE < 90d" |
| `sectorCount` | > 0 | "{N} Sectors" |
| `industryGroupCount` | > 0 | "{N} Industry Groups" |

### Implementation
- Use a `computed` that builds a `{ key: string, label: string }[]` array from props
- Price range: combine min/max into single chip if both set
- DTE range: combine min/max into single chip if both set
- Sector/industry counts passed as separate props (not inside `filters`)
