# Task: Migrate MarketDataService to ServiceBase

## Description
Migrate `MarketDataService` to inherit from `ServiceBase[ServiceConfig]`. This is one of
the two complex migrations: MarketDataService has 6 cached methods, its own `_yf_call`
implementation, retry usage, and is the most used service in the codebase. Its standalone
`_yf_call` method gets deleted in favor of `ServiceBase._yf_call`. Constructor signature
must remain backward-compatible. All existing tests must pass without modification.

## Acceptance Criteria
- [ ] `MarketDataService` inherits `ServiceBase[ServiceConfig]`
- [ ] `MarketDataService.__init__` calls `super().__init__(config, cache, limiter)`
- [ ] Standalone `_yf_call` method deleted — uses inherited `ServiceBase._yf_call`
- [ ] 6 cache blocks replaced with `self._cached_fetch()` calls (some with `deserializer` callback)
- [ ] Retry calls use `self._retried_fetch()`
- [ ] `close()` NOT overridden (current impl is log-only, base default no-op is better)
- [ ] Constructor signature unchanged — backward compatible
- [ ] All existing tests pass unchanged
- [ ] No consumer code changes
- [ ] Lint, format, and type check pass

## Technical Details

### MarketDataService Migration
- Config type: `ServiceConfig` (has `yfinance_timeout`)
- 6 cached methods with varying serde patterns:
  - Some use direct `model_validate_json` → standard `_cached_fetch`
  - Some use custom `_deserialize_*()` helpers → `_cached_fetch` with `deserializer` callback
  - Some use `json.loads()` + loop `model_validate()` for lists → `deserializer` callback
- Has its own `_yf_call` implementation (functionally identical to base) → DELETE
- Uses `self._config.yfinance_timeout` → passes as explicit `timeout=self._config.yfinance_timeout`
- Uses retry pattern → delegate to `self._retried_fetch()`
- Current `close()` is log-only → remove override, let base default no-op handle it

### Key Risk
- Most used service — thorough test verification essential
- 6 cache blocks with different serde patterns — careful conversion needed
- Existing `_yf_call` callers must switch to `self._yf_call(fn, *args, timeout=...)`
