# Task: Backend service — BatchQuote model + fetch_batch_daily_changes()

## Description

Add a `fetch_batch_daily_changes()` method to `MarketDataService` that uses `yf.download()`
to fetch 2-day OHLCV for ~500 tickers in a single batch call, computes daily % change, and
returns typed `list[BatchQuote]`. This is a new yfinance pattern for the codebase (existing
code uses per-ticker `yf.Ticker().history()`). Include 5-minute server-side caching via
existing `ServiceCache`.

## Acceptance Criteria

- [ ] `BatchQuote` frozen Pydantic model defined in `market_data.py` with fields: `ticker: str`, `price: float`, `change_pct: float | None`, `volume: int`
- [ ] `isfinite()` field validator on `price` and `change_pct`
- [ ] `fetch_batch_daily_changes(tickers: list[str]) -> list[BatchQuote]` method on `MarketDataService`
- [ ] Uses `yf.download(tickers, period="2d", group_by="ticker", progress=False, threads=True, timeout=30)` via `asyncio.to_thread()` + `asyncio.wait_for()`
- [ ] Computes `(today_close - prev_close) / prev_close * 100` per ticker
- [ ] Skips tickers with missing/NaN data — partial results acceptable
- [ ] Caches full result via `ServiceCache` with `"heatmap"` data type, TTL 5 min
- [ ] All new unit tests passing
- [ ] `ruff check` + `mypy --strict` clean

## Technical Details

### Files to Modify
- `src/options_arena/services/market_data.py` — Add `BatchQuote` model + `fetch_batch_daily_changes()` method

### Implementation Notes
- `BatchQuote` follows `TickerOHLCVResult` pattern (frozen model in same file)
- Price as `float` (display-only, no P&L computation — architecture decision #3)
- `yf.download()` with `group_by="ticker"` returns MultiIndex DataFrame: access `data[ticker]["Close"]`
- Per-ticker extraction wrapped in `try/except KeyError` for tickers that failed
- `math.isfinite()` check on computed `change_pct` before constructing model
- Cache key: `"yf:heatmap:sp500"` (or hash of sorted ticker list)
- May need to add `"heatmap"` case to `ServiceCache.ttl_for()` match statement if it uses named types

### Context7-Verified API

### Edge Cases
- Empty ticker list: returns `[]`
- All tickers fail in yf.download: returns `[]`
- Zero prev_close (division by zero): skip ticker (Inf rejected by isfinite)
- Single ticker: still uses `yf.download()` (not `yf.Ticker`)
- yfinance timeout: raises within `asyncio.wait_for`, returns cached or empty

### Mocking Strategy
- Mock `yf.download` via `unittest.mock.patch("yfinance.download")` returning synthetic DataFrame
- Mock `ServiceCache.get()` / `ServiceCache.set()` for cache tests
- Build test DataFrames with `pd.DataFrame` + MultiIndex columns matching `group_by="ticker"` structure
