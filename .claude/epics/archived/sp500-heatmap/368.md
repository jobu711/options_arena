# Task: Frontend data layer ŌĆö HeatmapTicker type + Pinia heatmap store

## Description

Create the frontend data layer for the heatmap: a `HeatmapTicker` TypeScript interface in
`types/scan.ts` and a `heatmap.ts` Pinia setup store following the `health.ts` auto-refresh
pattern. The store fetches from `GET /api/market/heatmap`, manages loading/error state, and
auto-refreshes every 5 minutes.

- [ ] `HeatmapTicker` interface in `web/src/types/scan.ts` with all fields matching API schema
- [ ] `HeatmapTicker` re-exported from `web/src/types/index.ts`

- [ ] Store exposes: `tickers`, `loading`, `error`, `lastUpdated`, `fetchHeatmap()`, `startAutoRefresh()`, `stopAutoRefresh()`
- [ ] Auto-refresh interval: 5 minutes (300,000ms)
- [ ] Error handling: catches API errors, sets `error` ref, doesn't throw
- [ ] `npm run type-check` and `npm run lint` pass

export interface HeatmapTicker {
  ticker: string

### Store Pattern (follows health.ts)
```typescript
export const useHeatmapStore = defineStore('heatmap', () => {
  const tickers = ref<HeatmapTicker[]>([])
  const loading = ref(false)
  const error = ref<string | null>(null)
  const lastUpdated = ref<Date | null>(null)

  let intervalId: ReturnType<typeof setInterval> | null = null

  async function fetchHeatmap(): Promise<void> { ... }
  function startAutoRefresh(intervalMs = 5 * 60 * 1000): void { ... }
  function stopAutoRefresh(): void { ... }

  return { tickers, loading, error, lastUpdated, fetchHeatmap, startAutoRefresh, stopAutoRefresh }
})
```

### Key Conventions
- Use `api<HeatmapTicker[]>('/api/market/heatmap')` from `useApi` composable
- `loading` set true before fetch, false in `finally`
- `error` set to message string on catch, cleared on successful fetch
- `lastUpdated` set to `new Date()` on successful fetch
- `stopAutoRefresh()` clears interval and sets `intervalId = null`
