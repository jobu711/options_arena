# Task: Frontend component — MarketHeatmap.vue with squarify treemap

## Description

Create the `MarketHeatmap.vue` component that renders a Finviz-style treemap heatmap using
HTML divs. Implements a squarify algorithm in TypeScript (zero npm deps), two-level layout
(sectors → tickers), color encoding by daily % change, hover tooltips, and click-to-navigate.
Uses data from the Pinia heatmap store.

- [ ] `web/src/components/MarketHeatmap.vue` created with `<script setup lang="ts">`
- [ ] Squarify treemap algorithm implemented in TypeScript (~80 lines, no npm deps)
- [ ] Two-level layout: sectors (weighted by sum of ticker weights) → tickers within sectors
- [ ] Color gradient: red (#ef4444) ↔ neutral (#374151) ↔ green (#22c55e), ±5% = full saturation
- [ ] Ticker symbol displayed in each cell, font size scales with cell area

## Technical Details

### Files to Create
- `web/src/components/MarketHeatmap.vue`

### Squarify Algorithm
Standard squarify treemap algorithm. Input: `items: {weight: number}[]`, container `{x, y, w, h}`.
Output: `{x, y, width, height}[]` rectangles. Two-level application:

1. Group `HeatmapTicker[]` by `sector`, sum `market_cap_weight` per sector
2. Squarify sector weights into the full container → sector rectangles
3. For each sector, squarify its tickers within the sector rectangle → cell rectangles

### Font Scaling
- Cell area threshold for showing ticker symbol: ~2000px² (width * height)
- Cell area threshold for showing change %: ~4000px²
- Font size: `Math.max(8, Math.min(16, Math.sqrt(area) / 4))` or similar

### Tooltip
- Floating div with `position: fixed`, follows mouse via `@mousemove`
- Shows: ticker (bold), company name, sector, change %, price, volume
- `pointer-events: none`, `z-index: 50`
- Hidden on `@mouseleave`

### Component Props & Store Integration
- Component uses `useHeatmapStore()` directly (store-driven, no props)
- Calls `store.fetchHeatmap()` + `store.startAutoRefresh()` in `onMounted`
- Calls `store.stopAutoRefresh()` in `onUnmounted`
- Computed treemap layout recalculates when `store.tickers` changes

### PrimeVue Components
- Refresh button: PrimeVue `Button` with `icon="pi pi-refresh"` + `@click="store.fetchHeatmap()"`
- Loading: `Skeleton` component or custom skeleton div
- Section header uses existing dashboard section styling

## Test Plan

### Test Files
- No unit tests (squarify algorithm is well-known; visual verification for v1 per PRD out-of-scope)

### Edge Cases
- Zero tickers: show empty state message
- All tickers in one sector: single sector fills entire container
- All tickers have 0% change: all cells neutral gray
- Extreme changes (>10%): color clamped to full saturation
- Very small cells: ticker text hidden, but cell still rendered as colored block
- Null change_pct: cell rendered in neutral gray

### Visual Verification
- Mega-cap tickers (AAPL, MSFT, GOOGL) should visually dominate
- Sectors should be visually grouped with clear boundaries
- Color should be immediately readable (red = down, green = up)
