# Task: CLI rendering — 4 v2 agent panels + protocol-aware layout

## Description
Add 4 new Rich panel render functions for v2 agent outputs (flow, fundamental, risk_v2, contrarian) and modify `render_debate_panels()` to detect `debate_protocol` and render the appropriate layout. V1 path unchanged.

## Acceptance Criteria
- [ ] `render_flow_panel(flow: FlowThesis)` returns Rich Panel with bright_magenta border showing GEX, smart money, OI analysis, volume confirmation, key flow factors
- [ ] `render_fundamental_panel(fund: FundamentalThesis)` returns Rich Panel with bright_cyan border showing catalyst impact, earnings assessment, IV crush risk, key factors
- [ ] `render_risk_v2_panel(risk: RiskAssessment)` returns Rich Panel with bright_blue border showing risk level, PoP, max loss, key risks, mitigants, position size
- [ ] `render_contrarian_panel(contra: ContrarianThesis)` returns Rich Panel with yellow border showing dissent direction, primary challenge, overlooked risks, alternative scenario
- [ ] `render_debate_panels()` branches on `result.debate_protocol == "v2"` — v2 renders: Trend (green) → Flow → Fundamental → Volatility → Risk → Contrarian → Verdict
- [ ] V2 renders "TREND ANALYSIS" label instead of "BULL" for the trend agent
- [ ] V2 omits synthetic "BEAR" placeholder panel
- [ ] V1 debates render identically to before (zero regression)
- [ ] All panels use `Text()` constructor (not f-strings with markup) to avoid Rich bracket crash

## Technical Details
- File: `src/options_arena/cli/rendering.py`
- Add 4 new functions after existing `render_volatility_panel()` (~line 186)
- Modify `render_debate_panels()` (~line 189) to branch on `debate_protocol`
- Follow existing pattern: each function returns `Panel(Text(...), title="...", border_style="...")`
- Use `Text("\n".join(lines))` pattern — never `markup=True` (bracket crash)
