# Task: API layer — v2 schema fields, route parsing, persistence

## Description
Extend `DebateResultDetail` in `api/schemas.py` with 5 new optional fields. Update `get_debate()` in `api/routes/debate.py` to parse v2 JSON columns from DebateRow. Update `_run_debate_background()` to pass v2 fields to `save_debate()`.

## Acceptance Criteria
- [ ] `DebateResultDetail` has 5 new fields: `flow_response: dict[str, object] | None = None`, `fundamental_response: dict[str, object] | None = None`, `risk_v2_response: dict[str, object] | None = None`, `contrarian_response: dict[str, object] | None = None`, `debate_protocol: str | None = None`
- [ ] `get_debate()` deserializes v2 JSON columns: `json.loads(row.flow_json)` when not None
- [ ] `_run_debate_background()` passes v2 JSON to `save_debate()`: `flow_json=result.flow_response.model_dump_json() if result.flow_response else None`
- [ ] GET `/api/debate/{id}` returns v2 fields when present, None when absent
- [ ] V1 debates return None for all v2 fields (backward compat)
- [ ] `mypy --strict` passes

## Technical Details
- File: `src/options_arena/api/schemas.py` — add 5 fields to `DebateResultDetail` (~line 210)
- File: `src/options_arena/api/routes/debate.py`:
  - `get_debate()` (~line 509): parse v2 JSON columns after existing vol_response parsing
  - `_run_debate_background()` (~line 154): add v2 serialization to save_debate call
- Use `dict[str, object] | None` type (not model types) per PRD constraint — avoids circular imports
