# Task: API ScanRequest.sectors, GET /universe/sectors, enriched responses

## Description
Add sector filtering to the scan API request, create a new sectors endpoint, and verify that enriched TickerScore responses (with sector + company_name) flow through correctly.

## Acceptance Criteria
- [ ] `ScanRequest` in `schemas.py`: new `sectors: list[GICSSector] = []` field with `mode='before'` validator for alias normalization
- [ ] `POST /api/scan` accepts `{"preset": "sp500", "sectors": ["technology"]}` and normalizes to canonical GICS names
- [ ] `POST /api/scan` with invalid sector returns 422 with clear error
- [ ] New `SectorInfo` schema in `schemas.py`: `name: str`, `ticker_count: int`
- [ ] New `GET /api/universe/sectors` endpoint returns `list[SectorInfo]` with 11 sectors and accurate ticker counts
- [ ] `GET /api/scan/{id}/scores` responses include `sector` and `company_name` fields on each TickerScore (automatic — TickerScore is the response model)
- [ ] `UniverseStats` schema: add `etf_count: int` field
- [ ] All existing API tests pass

## Technical Details
- **schemas.py**: Add `sectors: list[GICSSector] = []` to `ScanRequest`. Add `mode='before'` `field_validator` that normalizes each entry via `SECTOR_ALIASES`. Add `SectorInfo(BaseModel)` with `name: str` and `ticker_count: int`. Add `etf_count: int = 0` to `UniverseStats`.
- **routes/scan.py**: Pass `request.sectors` to the scan pipeline via `ScanConfig.sectors`.
- **routes/universe.py**: New `GET /api/universe/sectors` route. Fetches `sp500_constituents` from `UniverseService`, groups by sector, returns `list[SectorInfo]`. Uses `Depends(get_universe)`.
- **routes/universe.py**: Update stats endpoint to include `etf_count` from `fetch_etf_tickers()` (or 0 if it fails).
- **TickerScore enrichment**: No API route changes needed — `GET /api/scan/{id}/scores` already returns `TickerScore` objects. The new `sector` and `company_name` fields are included automatically via Pydantic serialization.
