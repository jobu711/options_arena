# Task: Frontend sector column, company name in drawer, sector filter, ETF preset

## Description
Update the Vue 3 frontend to display sector data in scan results, show company name in the ticker drawer, add a sector MultiSelect filter to the scan page, and enable the ETF preset option.

## Acceptance Criteria
- [ ] TypeScript `TickerScore` interface: add `sector: string | null` and `company_name: string | null` fields
- [ ] Scan results DataTable: new **Sector** column after the Ticker column, displaying sector as a PrimeVue Tag/chip with color coding
- [ ] Sector column: tickers without sector data show `"—"`
- [ ] Sector column: sortable and filterable
- [ ] TickerDrawer: company full name displayed as a subtitle beneath the ticker symbol in the drawer header
- [ ] TickerDrawer: show `"—"` if `company_name` is null
- [ ] Scan page: PrimeVue MultiSelect component for sector filtering with `display="chip"` and `filter` enabled
- [ ] Scan page: selected sectors passed to `POST /api/scan` request body
- [ ] Scan page: ETF preset option visible in the preset selector dropdown
- [ ] Scan results header: shows active sector filter when sectors are selected
- [ ] All existing E2E tests pass

## Technical Details
- **TypeScript types** (`web/src/types/scan.ts`): Add `sector: string | null` and `company_name: string | null` to `TickerScore` interface.
- **ScanResultsPage.vue**: Add `<Column field="sector" header="Sector" sortable>` after the Ticker column. Use `<template #body>` with a PrimeVue `<Tag>` component. Map sectors to colors (e.g., Technology = blue, Health Care = green, Energy = orange). Null sector renders `"—"`.
- **TickerDrawer.vue**: In the drawer header section (near the ticker symbol display), add `<div class="text-sm text-surface-500">{{ score?.company_name ?? '—' }}</div>` beneath the ticker.
- **Scan page filter**: Add `<MultiSelect v-model="selectedSectors" display="chip" :options="sectorOptions" optionLabel="name" optionValue="value" filter placeholder="Filter by sector" />` above the scan results table. Fetch sector options from `GET /api/universe/sectors` on mount. Include `sectors` in the scan request payload.
- **Scan store** (`web/src/stores/scan.ts`): Add `selectedSectors` state. Pass to scan start action.
- **Preset selector**: The `ScanPreset` already includes `etfs`. Ensure the preset dropdown/selector in the scan page includes it as an option label (e.g., "ETFs").
- **Sector color mapping**: Create a simple `SECTOR_COLORS: Record<string, string>` map for PrimeVue Tag severity or custom CSS classes. Use PrimeVue's built-in severity props (`info`, `success`, `warning`, `danger`, `secondary`) or custom background colors.
