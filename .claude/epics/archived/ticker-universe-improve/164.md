## Technical Details
- **Pipeline tests**: Mock `UniverseService` to return known SP500 constituents with sectors. Mock `MarketDataService` for OHLCV. Run pipeline with `ScanConfig(sectors=[GICSSector.INFORMATION_TECHNOLOGY])` and verify only IT tickers in results.
- **Repository tests**: Use in-memory SQLite. Run migration. Save TickerScore with sector + company_name. Retrieve and verify fields roundtrip.
- **API tests**: Use `httpx.AsyncClient` with `ASGITransport`. Override deps with mocks. Test request validation, response shapes, error cases.
- **CLI tests**: Use `typer.testing.CliRunner`. Mock pipeline to avoid real API calls. Verify exit code and output content.
- **E2E tests** (Playwright): Add assertions to existing scan flow tests checking for sector column existence and drawer company name. Use API mocking (existing pattern in E2E tests).
- Follow existing test patterns in `tests/unit/` and `tests/e2e/`.
