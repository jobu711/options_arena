# Task: GICSSector enum, alias mapping, TickerScore enrichment, DB migration

## Description
Add the foundational models layer for sector filtering. This is the critical path — every other task depends on these types existing.

## Acceptance Criteria
- [ ] `GICSSector(StrEnum)` in `enums.py` with exactly 11 GICS sectors using canonical names (e.g., `INFORMATION_TECHNOLOGY = "Information Technology"`)
- [ ] `SECTOR_ALIASES: dict[str, GICSSector]` mapping in `enums.py` covering lowercase, hyphenated, underscored, and short-name variants (e.g., `"technology"` -> `INFORMATION_TECHNOLOGY`)
- [ ] `ScanConfig.sectors: list[GICSSector] = []` field in `config.py` with `mode='before'` validator that normalizes input strings via `SECTOR_ALIASES`
- [ ] `TickerScore.sector: GICSSector | None = None` field in `scan.py`
- [ ] `TickerScore.company_name: str | None = None` field in `scan.py`
- [ ] `GICSSector` re-exported from `models/__init__.py` and added to `__all__`
- [ ] New DB migration file adding nullable `sector TEXT` and `company_name TEXT` columns to `ticker_scores` table
- [ ] All existing tests pass (zero regressions)
- [ ] `ruff check`, `mypy --strict` pass

## Technical Details
- **enums.py**: Add `GICSSector` after existing enums. All 11 GICS sectors: Communication Services, Consumer Discretionary, Consumer Staples, Energy, Financials, Health Care, Industrials, Information Technology, Materials, Real Estate, Utilities. Add `SECTOR_ALIASES` module-level dict below the enum.
- **config.py**: Add `sectors` field to `ScanConfig`. The `mode='before'` validator should iterate the list, normalize each string via `SECTOR_ALIASES` (case-insensitive lookup after lowering + replacing hyphens/underscores with spaces), and return canonical `GICSSector` values. If a string doesn't match any alias or enum value, raise `ValueError` listing valid options.
- **scan.py**: Add two optional fields to `TickerScore`. `sector` uses `GICSSector | None`, `company_name` uses `str | None`. Both default to `None`.
- **Migration**: Next sequential migration file in `data/migrations/`. Use `ALTER TABLE ticker_scores ADD COLUMN sector TEXT; ALTER TABLE ticker_scores ADD COLUMN company_name TEXT;`. Check existing migrations for the naming convention.
- **__init__.py**: Add `GICSSector` to imports and `__all__`.
