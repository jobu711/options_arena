# Task: UniverseService ETF detection and sector filtering helpers

## Description
Implement ETF detection for the `ScanPreset.ETFS` preset and add a sector filtering helper to `UniverseService`. ETF detection cross-references yfinance `quoteType` against the CBOE optionable list.

## Acceptance Criteria
- [ ] `UniverseService.fetch_etf_tickers() -> list[str]` implemented with 24h cache TTL
- [ ] ETF detection uses yfinance `Ticker.info["quoteType"] == "ETF"` cross-referenced against CBOE optionable tickers
- [ ] Batch ETF detection uses `asyncio.gather(return_exceptions=True)` for fault isolation
- [ ] Returns sorted, deduplicated list of ETF ticker symbols
- [ ] `UniverseService.filter_by_sectors(tickers, sectors, sp500_map) -> list[str]` implemented as pure helper
- [ ] Sector filter uses OR logic (ticker matches if in any provided sector)
- [ ] Empty `sectors` list returns all tickers unfiltered
- [ ] All existing tests pass (zero regressions)

## Technical Details
- **fetch_etf_tickers()**: Check cache first (`_CACHE_KEY_ETF`). On miss: fetch CBOE optionable list (reuse existing `fetch_optionable_tickers()`), then batch-check a representative sample via yfinance `Ticker(symbol).info["quoteType"]` wrapped in `_yf_call`. Cache result for 24h. If yfinance is unreliable, fall back to a curated seed list of ~100 popular optionable ETFs (SPY, QQQ, IWM, etc.) intersected with CBOE list.
- **filter_by_sectors()**: Pure function. Takes `tickers: list[str]`, `sectors: list[GICSSector]`, `sp500_map: dict[str, GICSSector]`. Returns `[t for t in tickers if sp500_map.get(t) in sectors]`. If `sectors` is empty, return `tickers` unchanged.
- **sp500_map construction**: Add a helper `build_sector_map(constituents: list[SP500Constituent]) -> dict[str, GICSSector]` that maps ticker -> GICSSector. Handle Wikipedia sector strings that may have slight formatting differences vs canonical GICS names.
- Follow existing patterns: `_yf_call` for sync yfinance wrapping, `asyncio.wait_for` for timeouts, cache-first with `ServiceCache`.
