# Task: Pipeline sector filter injection and TickerScore enrichment

## Description
Wire sector filtering into the scan pipeline Phase 1, attach sector and company_name to TickerScore during the pipeline, and update the Repository to persist/retrieve the new fields.

## Acceptance Criteria
- [ ] Pipeline Phase 1 (`_phase_universe`) builds `sector_map: dict[str, GICSSector]` from existing `sp500_constituents`
- [ ] When `ScanConfig.sectors` is non-empty, Phase 1 filters universe tickers to only those matching the provided sectors (OR logic)
- [ ] `ScanPreset.ETFS` routes to `UniverseService.fetch_etf_tickers()` instead of falling back to FULL
- [ ] Phase 2/3: `TickerScore.sector` populated from `sector_map` for each scored ticker
- [ ] Phase 3: `TickerScore.company_name` populated from `TickerInfo.company_name` (already fetched via `fetch_ticker_info()`)
- [ ] `Repository.save_ticker_scores()` persists `sector` and `company_name` columns
- [ ] `Repository.get_scores_for_scan()` reads and populates `sector` and `company_name` from DB
- [ ] Sector filter logging: log count before/after filtering at INFO level
- [ ] All existing tests pass (zero regressions)

## Technical Details
- **Pipeline Phase 1**: After fetching `sp500_constituents`, build `sector_map = {c.ticker: GICSSector(c.sector) for c in constituents}`. If `config.sectors` is non-empty, filter `universe_tickers = [t for t in universe_tickers if sector_map.get(t) in config.sectors]`. Log filter results.
- **Pipeline Phase 1 (ETFS)**: Replace the no-op warning at pipeline.py:208 with actual call to `self._universe.fetch_etf_tickers()`.
- **Pipeline Phase 2/3**: After scoring, iterate scored tickers and set `ts.sector = sector_map.get(ts.ticker)`. In Phase 3, after `fetch_ticker_info()`, set `ts.company_name = ticker_info.company_name`.
- **Repository save**: Update `save_ticker_scores` INSERT statement to include `sector` and `company_name` columns. Serialize `GICSSector` to its string value.
- **Repository read**: Update `get_scores_for_scan` to read `sector` (deserialize to `GICSSector | None`) and `company_name` from rows.
- **UniverseResult model**: Consider adding `sector_map` to `UniverseResult` so it flows through the pipeline cleanly.
