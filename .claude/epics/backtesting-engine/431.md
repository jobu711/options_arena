# Task: 7 AnalyticsMixin queries + Greeks decomposition

## Description
Implement 7 new query methods in `data/_analytics.py` (AnalyticsMixin) for backtesting analytics. All queries use existing tables (`recommended_contracts`, `contract_outcomes`, `agent_predictions`, `ticker_metadata`). Returns frozen Pydantic models defined in Task 1.

## Acceptance Criteria
- [ ] 7 new query methods added to AnalyticsMixin in `_analytics.py`
- [ ] All methods return `list[FrozenModel]` (typed, never raw dicts)
- [ ] All SQL uses parameterized queries (no string formatting)
- [ ] Greeks decomposition: `delta_pnl = stock_return_pct * entry_delta` (negated for puts)
- [ ] Median computed in Python via `statistics.median()` (SQLite lacks built-in)
- [ ] Sharpe-like ratio computed in Python (mean/std of returns)
- [ ] All queries handle empty results gracefully (return empty list)
- [ ] 4-5 tests per query method with in-memory SQLite
- [ ] `mypy --strict` passes

## Technical Details

### New Methods in AnalyticsMixin

1. **`get_equity_curve(direction: str | None = None, period_days: int | None = None) -> list[EquityCurvePoint]`**
   - SQL: ORDER BY `created_at`, compute running cumulative return in Python
   - Filter by direction and/or lookback period

2. **`get_drawdown_series(period_days: int | None = None) -> list[DrawdownPoint]`**
   - Derived from equity curve: track peak, compute (current - peak) / peak
   - Returns series of drawdown percentages over time

3. **`get_win_rate_by_sector(holding_days: int = 20) -> list[SectorPerformanceResult]`**
   - JOIN `recommended_contracts` → `contract_outcomes` → `ticker_metadata`
   - GROUP BY sector, compute win rate (return > 0) and avg return
   - Filter NULL sectors

4. **`get_win_rate_by_dte_bucket(holding_days: int = 20) -> list[DTEBucketResult]`**
   - Compute DTE from `expiration - created_at`, bucket into (0-7, 7-14, 14-30, 30-60, 60+)
   - GROUP BY bucket, compute win rate and avg return

5. **`get_win_rate_by_iv_rank(holding_days: int = 20) -> list[IVRankBucketResult]`**
   - Bucket `market_iv` into quartiles (0-25, 25-50, 50-75, 75-100)
   - GROUP BY bucket, compute win rate and avg return

6. **`get_greeks_decomposition(holding_days: int = 20, groupby: str = "direction") -> list[GreeksDecompositionResult]`**
   - `delta_pnl = stock_return_pct * entry_delta` (negate for puts)
   - `residual_pnl = contract_return_pct - delta_pnl`
   - GROUP BY direction, sector, or time period

7. **`get_holding_period_comparison() -> list[HoldingPeriodComparison]`**
   - Enhanced version of existing `get_optimal_holding_period`
   - Adds: median return (Python), Sharpe-like ratio (mean/std), max loss
   - GROUP BY holding_days, direction

### Pattern
Follow existing analytics query pattern: parameterized SQL → `_fetchall()` → list comprehension → frozen model. See `get_win_rate_by_direction()` as template.

### Edge Cases
- Empty database: returns empty list (no crash)
- Single outcome: cumulative return equals single return
- All losses: win rate = 0%, negative avg return
- NULL sector/IV: excluded from results
- Put delta negation: `delta_pnl = stock_return * (-entry_delta)`
- Zero standard deviation: Sharpe-like returns 0.0 (not division by zero)

### Mocking Strategy
- Use real in-memory SQLite (fast, no mocking needed)
- Seed test data with typed Pydantic models, not raw dicts
- Use `pytest.approx(rel=1e-4)` for float comparisons
