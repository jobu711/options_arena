# Task: Vue analytics dashboard

## Description
Build the frontend analytics dashboard: install Chart.js, create a Pinia store (`useBacktestStore`), create `AnalyticsView.vue` with 5 PrimeVue tabs (Overview, Agents, Segments, Greeks, Holding), ~6 chart components, and add the `/analytics` route with navigation link. Uses existing PrimeVue dark theme and Composition API patterns.

## Acceptance Criteria
- [ ] Chart.js installed as dependency (`npm install chart.js` in `web/`)
- [ ] `useBacktestStore` Pinia store with async fetchers for all 7 backtest + 2 existing agent endpoints
- [ ] Store caches responses in refs, has filter state (direction, holding period, date range)
- [ ] `AnalyticsView.vue` with PrimeVue `TabView` (5 tabs)
- [ ] **Overview tab**: Equity curve line chart, summary cards (trades, win rate, avg return, max drawdown), drawdown chart
- [ ] **Agents tab**: Accuracy heatmap, confidence calibration scatter, agent comparison table (uses existing agent-accuracy + agent-calibration endpoints)
- [ ] **Segments tab**: Sector bar chart, DTE bucket bars, IV rank quartile bars
- [ ] **Greeks tab**: Stacked bar chart (delta vs residual) by direction/sector
- [ ] **Holding tab**: Period comparison table with highlighted best periods
- [ ] `/analytics` route added to Vue Router
- [ ] Navigation link added in sidebar/header
- [ ] Tabs lazy-load data on first activation
- [ ] Charts use PrimeVue Aura dark theme styling
- [ ] Responsive layout (works on desktop and tablet)
