# Task: Debate Trend + Earnings Alerts + Auto-Refresh

## Description

Add debate confidence trend visualization (Chart.js), earnings proximity alerts in
TickerDrawer, and auto-refresh polling on the dashboard. Three lightweight features
bundled into one stream — no `pipeline.py` changes, cleanest merge of all streams.

**Branch**: `feat/v2.1-trends-polish`

## Acceptance Criteria

### Debate Trend (Feature 5)
- [ ] Repository: `get_debate_trend_for_ticker(ticker, limit=20) -> list[DebateTrendPoint]`
      — queries `ai_theses`, parses `verdict_json`, returns chronological order
- [ ] `GET /api/debate/trend/{ticker}?limit=20` → `DebateTrendResponse` (404 if none)

### Key Implementation Notes
- `get_debate_trend_for_ticker` SQL: `SELECT id, created_at, verdict_json FROM ai_theses WHERE ticker = ? ORDER BY created_at ASC LIMIT ?`
  — parse `verdict_json` (JSON string) to extract `direction`, `confidence`, `bull_score`, `bear_score`, `is_fallback`
- Chart.js install: `cd web && npm install chart.js vue-chartjs`
- Auto-refresh: use `onMounted`/`onUnmounted` lifecycle hooks with `setInterval`/`clearInterval`
- Earnings countdown: compute `days = (next_earnings - today).days` in template
- No `pipeline.py` changes — this stream is the cleanest to merge first
