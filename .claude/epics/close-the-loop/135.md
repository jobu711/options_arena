# Task: Contract Persistence + Scan Diff

## Description

Persist recommended option contracts to SQLite so they survive across sessions, and add
scan diff capability to compare two scan runs. Covers repository methods, pipeline
integration, API endpoints, CLI flags, and Vue components.

**Branch**: `feat/v2.1-contracts-diff`

## Acceptance Criteria

### Contract Persistence
- [ ] Repository methods: `save_recommended_contracts()`, `get_recommended_contracts()`,
      `get_all_recommendations_for_scan()`
- [ ] Pipeline Phase 4 persists contracts after `save_ticker_scores()` (~5 lines)
- [ ] `GET /api/scan/{id}/scores/{ticker}` returns `TickerDetail.contracts` as
      `list[OptionContract]` (was `list[str]`, always empty)
- [ ] TickerDrawer in Web UI now shows actual recommended contracts

### Scan Diff
- [ ] Repository method: `get_scan_diff(old_scan_id, new_scan_id) -> ScanDiffResult`
      using LEFT JOIN on `ticker_scores`
- [ ] `GET /api/scan/{scan_id}/diff?compare_to={other_id}` returns `ScanDiffResult`
- [ ] CLI `--diff` flag on scan command: after scan completes, compares to previous
      scan and renders Rich table with added/removed/changed tickers
- [ ] Vue `ScanDiffView.vue`: dropdown to select comparison scan, color-coded table
      (green=added, red=removed, yellow=changed), score delta arrows

### Tests
- [ ] ~35 tests pass covering repository roundtrips, API endpoints, pipeline
      integration (mock repo), CLI output, Vue component rendering
