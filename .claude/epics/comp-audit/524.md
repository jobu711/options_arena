# 524 — Hurst Exponent Indicator (FR-C1)

## Description

Add Hurst exponent as a new technical indicator to the scoring pipeline. The Hurst
exponent (H) classifies a time series as trending (H > 0.5), mean-reverting (H < 0.5),
or random walk (H ≈ 0.5) using rescaled range (R/S) analysis (Mandelbrot & Wallis, 1969).
This enriches the regime detection capability of the composite scoring system, giving
directional options strategies an edge by quantifying persistence in price movements.

## Acceptance Criteria

1. `indicators/hurst.py` computes Hurst exponent from a pandas Series of close prices
2. Returns `float | None` — None when data is insufficient or result is unreliable
3. `IndicatorSignals` model includes `hurst_exponent: float | None = None`
4. `INDICATOR_WEIGHTS` includes `"hurst_exponent": (0.02, "regime")` with net-zero weight change
5. `DOMAIN_BOUNDS` includes `"hurst_exponent": (0.0, 1.0)`, NOT inverted
6. All existing tests pass — weight sum still == 1.0
7. New tests cover trending, mean-reverting, random walk, edge cases

## Technical Details

**Algorithm — Rescaled Range (R/S) Analysis:**
1. Compute log returns from close prices
2. For each lag in range(10, max_lag):
   - Subdivide returns into non-overlapping windows of size `lag`
   - For each window: cumulative deviations from mean, range R = max - min, std S
   - Average R/S across all windows for that lag
3. OLS regression of log(R/S) vs log(lag) — slope = H
4. Clamp H to [0, 1]
5. Return None if R² < 0.5 (unreliable fit)

**Weight Redistribution:**
- Add `"hurst_exponent": (0.02, "regime")`
- Reduce `"roc"` from 0.03 to 0.02
- Reduce `"put_call_ratio"` from 0.03 to 0.02
- Net change: +0.02 - 0.01 - 0.01 = 0. Sum remains 1.0.

**`tests/unit/indicators/test_hurst.py`:**

```python
import math
import numpy as np
import pandas as pd
import pytest
from options_arena.indicators import hurst_exponent


class TestHurstExponent:
    """Tests for Hurst exponent via R/S analysis."""

    def test_trending_series_high_hurst(self) -> None:
        """Cumulative sum of positive increments → H ≈ 0.8-0.9."""
        rng = np.random.default_rng(42)
        prices = pd.Series(np.cumsum(rng.uniform(0.5, 1.5, size=500)) + 100)
        h = hurst_exponent(prices)
        assert h is not None
        assert 0.7 <= h <= 1.0

    def test_mean_reverting_series_low_hurst(self) -> None:
        """Alternating series → H ≈ 0.2-0.4."""
        rng = np.random.default_rng(42)
        base = 100 + rng.normal(0, 0.5, size=500)
        alternating = np.where(np.arange(500) % 2 == 0, base + 1, base - 1)
        prices = pd.Series(np.abs(alternating))
        h = hurst_exponent(prices)
        assert h is not None
        assert 0.0 <= h <= 0.45

    def test_random_walk_mid_hurst(self) -> None:
        """White noise cumsum → H ≈ 0.5."""
        rng = np.random.default_rng(42)
        prices = pd.Series(np.cumsum(rng.standard_normal(1000)) + 200)
        h = hurst_exponent(prices)
        assert h is not None
        assert 0.35 <= h <= 0.65

    def test_insufficient_data_returns_none(self) -> None:
        """< 200 bars → None."""
        prices = pd.Series(range(100, 250))  # 150 bars
        assert hurst_exponent(prices, min_bars=200) is None

    def test_flat_prices_returns_none(self) -> None:
        """Constant prices → zero log returns → None."""
        prices = pd.Series([100.0] * 300)
        assert hurst_exponent(prices) is None

    def test_nan_in_input_handled(self) -> None:
        """NaN values in series → graceful handling (None or valid float)."""
        rng = np.random.default_rng(42)
        prices = pd.Series(np.cumsum(rng.standard_normal(500)) + 200)
        prices.iloc[50] = float("nan")
        result = hurst_exponent(prices)
        assert result is None or (isinstance(result, float) and math.isfinite(result))

    def test_r_squared_below_threshold_returns_none(self) -> None:
        """Very noisy R/S relationship → R² < 0.5 → None."""
        # Pathological series designed to break R/S scaling
        rng = np.random.default_rng(99)
        prices = pd.Series(np.exp(rng.uniform(-5, 5, size=300)))
        result = hurst_exponent(prices, r_squared_threshold=0.99)
        assert result is None

    def test_result_clamped_to_valid_range(self) -> None:
        """Result is always in [0, 1] or None."""
        rng = np.random.default_rng(42)
        prices = pd.Series(np.cumsum(rng.standard_normal(500)) + 200)
        h = hurst_exponent(prices)
        assert h is None or 0.0 <= h <= 1.0

    def test_result_is_finite(self) -> None:
        """Result must be finite float or None."""
        rng = np.random.default_rng(42)
        prices = pd.Series(np.cumsum(rng.standard_normal(500)) + 200)
        h = hurst_exponent(prices)
        assert h is None or math.isfinite(h)

    def test_minimum_bars_parameter(self) -> None:
        """Custom min_bars respected."""
        prices = pd.Series(range(100, 260))  # 160 bars
        assert hurst_exponent(prices, min_bars=200) is None
        assert hurst_exponent(prices, min_bars=100) is not None or True  # may still be None for R²
```

### Edge Cases

- Series with exactly `min_bars` length (boundary condition)
- Series with all identical values (zero variance)
- Series with NaN/Inf sprinkled in
- Very short max_lag (e.g., max_lag=5) — may produce unreliable results
- Extremely long series (10K+ bars) — performance check

### Mocking Strategy

- No mocking needed — `hurst_exponent()` is pure math on pandas Series
- Scoring integration tests use real constants (INDICATOR_WEIGHTS, DOMAIN_BOUNDS)
