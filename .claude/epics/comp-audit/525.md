# Task 525: Position Sizing Algorithm (FR-C5)

## Description

Implement a volatility-regime-aware position sizing algorithm that maps annualized IV to
allocation tiers with linear interpolation within tiers and an optional correlation penalty.
The algorithm replaces the existing unstructured `recommended_position_size: str | None`
field on `MarketContext` with structured fields (`position_size_pct`, `position_size_rationale`).
Position sizing results are integrated into debate agent context so agents can reference
concrete allocation guidance.

### New Model (`models/analysis.py`)

```python
class PositionSizeResult(BaseModel):
    model_config = ConfigDict(frozen=True)
    vol_regime_tier: int           # 1-4
    vol_regime_label: str          # "low", "moderate", "elevated", "extreme"
    annualized_iv: float
    base_allocation_pct: float     # [0.0, 1.0]
    correlation_adjustment: float  # [0.5, 1.0]
    final_allocation_pct: float    # base * adjustment
    rationale: str
```

Validators: `math.isfinite()` on all float fields, `vol_regime_tier` in `[1, 4]`,
`vol_regime_label` in `{"low", "moderate", "elevated", "extreme"}`.

### New Module (`analysis/position_sizing.py`)

```python
def compute_position_size(
    annualized_iv: float,
    correlation_with_portfolio: float | None = None,
    config: PositionSizingConfig | None = None,
) -> PositionSizeResult: ...
```

### Vol Tier Mapping

| Tier | IV Range | Base Allocation | Label |
|------|----------|----------------|-------|
| 1 | < 15% | 25% | low |
| 2 | 15-30% | 17.5% (linear interp from 25% to 17.5%) | moderate |
| 3 | 30-50% | 10% (linear interp from 17.5% to 10%) | elevated |
| 4 | >= 50% | 5% hard cap | extreme |

Linear interpolation within tiers: e.g., IV=22.5% (midpoint of 15-30%) yields allocation
midpoint of 17.5-25% = 21.25%.

### Correlation Adjustment

If `correlation_with_portfolio` is provided and exceeds `high_corr_threshold` (0.70),
multiply base allocation by `corr_penalty` (0.50). Otherwise adjustment is 1.0.

### NaN/Inf Handling

Non-finite IV values default to Tier 4 (5% allocation) — safest default.

- `position_size_rationale: str | None = None`

Add both to the `validate_optional_finite` list.

### Edge Cases

- IV exactly at tier boundary (15%, 30%, 50%) — should map to the higher tier
- IV = 0.0 — valid Tier 1
- Negative IV — should default to Tier 4 (treat as non-finite/invalid)
- Correlation exactly at threshold (0.70) — should NOT apply penalty (strictly greater than)
- All config values at zero — should not cause division by zero

### Mocking Strategy

- No external mocks needed — pure computation on float inputs
- `PositionSizingConfig` created directly in tests with custom values
- MarketContext field replacement may require updating test fixtures that reference `recommended_position_size`
