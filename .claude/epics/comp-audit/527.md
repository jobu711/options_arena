# Task 527: Portfolio Correlation Matrix (FR-C7)

## Description

Implement portfolio correlation analysis that computes pairwise Pearson correlation
coefficients between watchlist (or user-specified) tickers using log daily returns.
Results are exposed as typed models, served via a REST API endpoint, and accessible
through a CLI subcommand. The correlation data feeds into Task 525's position sizing
logic via `correlation_with_portfolio`.

Based on Markowitz (1952) portfolio theory — correlation is essential for understanding
diversification benefits and concentration risk across held positions.

## Technical Details

### New Models (`models/correlation.py`)

```python
class PairwiseCorrelation(BaseModel):
    model_config = ConfigDict(frozen=True)
    ticker_a: str
    ticker_b: str
    correlation: float         # Pearson [-1.0, 1.0]
    overlapping_days: int      # >= 0
    computed_at: datetime      # UTC validated
    # Validators: isfinite on correlation, UTC on computed_at

class CorrelationMatrix(BaseModel):
    model_config = ConfigDict(frozen=True)
    tickers: list[str]
    pairs: list[PairwiseCorrelation]
    computed_at: datetime      # UTC validated
    # Validators: UTC on computed_at
```

### Core Function (`analysis/correlation.py`)

```python
def compute_correlation_matrix(
    price_data: dict[str, pd.DataFrame],
    min_overlap: int = 30,
) -> CorrelationMatrix: ...
```

**Algorithm:**
1. For each ticker, extract "Close" column from its OHLCV DataFrame
2. Align all series on common dates (inner join on index)
3. Compute log daily returns: `ln(P_t / P_{t-1})`
4. For each unique pair `(i, j)` where `i < j`:
   - Count overlapping non-NaN days
   - If overlapping_days >= min_overlap: compute Pearson correlation
   - If overlapping_days < min_overlap: skip pair (NaN correlation)
5. Return `CorrelationMatrix` with all pairs and metadata

**Reference:** Markowitz, H. (1952). Portfolio Selection. *The Journal of Finance*.

### Edge Cases
- Empty `price_data` dict → empty `CorrelationMatrix`
- Single ticker → valid matrix with empty pairs
- Zero-variance series (constant price) → correlation undefined, handle as NaN/skip
- All pairs below `min_overlap` → all pairs skipped, empty pairs list
- NaN values in price columns → drop before computing returns
- Very short series (2 data points) → 1 return, likely below min_overlap
- Duplicate tickers in input → deduplicate before processing
