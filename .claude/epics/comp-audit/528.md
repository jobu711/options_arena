# 528 — Risk-Adjusted Performance Metrics (FR-C2)

## Description

Add risk-adjusted performance metrics (Sharpe ratio, Sortino ratio, max drawdown) to the
analytics pipeline. These are standard portfolio performance measures that allow users to
evaluate the quality of the system's recommendations beyond raw P&L. Computation lives in
`analysis/performance.py`, with a data query in `AnalyticsMixin`, a REST endpoint, and a
CLI subcommand.

## Acceptance Criteria

1. `RiskAdjustedMetrics` model in `models/analytics.py` with all fields validated
2. `compute_risk_adjusted_metrics()` produces correct Sharpe, Sortino, max drawdown
3. Returns None for ratios when < 30 trades
4. Database query in `AnalyticsMixin` fetches outcome data and returns typed model
5. API endpoint `GET /api/analytics/risk-metrics` returns JSON
6. CLI subcommand `outcomes risk-metrics` displays Rich table
7. All new + existing tests pass

## Technical Details

**Model — `RiskAdjustedMetrics(BaseModel)`:**
```python
class RiskAdjustedMetrics(BaseModel):
    model_config = ConfigDict(frozen=True)
    lookback_days: int
    total_trades: int
    sharpe_ratio: float | None
    sortino_ratio: float | None
    max_drawdown_pct: float | None
    max_drawdown_date: date | None
    annualized_return_pct: float | None
    risk_free_rate: float
```
All float fields get `math.isfinite()` validators.

**Formulas:**
- Sharpe = `sqrt(252 / avg_holding_days) * mean(excess_returns) / std(excess_returns)`
- Sortino = `sqrt(252 / avg_holding_days) * mean(excess_returns) / std(min(excess_returns, 0))`
- Max Drawdown = walk equity curve tracking peak, compute `(peak - trough) / peak`
- `excess_returns = returns - (risk_free_rate * holding_days / 365)`
- Returns None for ratios when `total_trades < 30` or `std == 0`

**Data Flow:**
1. `AnalyticsMixin.get_risk_adjusted_metrics(lookback_days=365)` queries `contract_outcomes`
2. Passes returns + holding days to `compute_risk_adjusted_metrics()`
3. API route calls repository method, returns model
4. CLI calls database, prints Rich table

```python
import math
from datetime import date
import pytest
from options_arena.analysis.performance import compute_risk_adjusted_metrics
from options_arena.models.analytics import RiskAdjustedMetrics


class TestComputeRiskAdjustedMetrics:
    """Tests for risk-adjusted performance metric computation."""

    def test_sharpe_known_returns(self) -> None:
        """Textbook example: known mean/std → predictable Sharpe."""
        # 50 trades, each +2% return, 5-day holding, rf=5%
        returns = [0.02] * 50
        holding_days = [5] * 50
        result = compute_risk_adjusted_metrics(returns, holding_days, risk_free_rate=0.0)
        assert result.sharpe_ratio is not None
        # With zero rf and constant returns, std ≈ 0 → None (or very high)
        # Use slightly varying returns for meaningful test
        returns = [0.02 + 0.524 * (i % 5) for i in range(50)]
        result = compute_risk_adjusted_metrics(returns, holding_days, risk_free_rate=0.0)
        assert result.sharpe_ratio is not None
        assert result.sharpe_ratio > 0

    def test_sortino_only_downside_deviation(self) -> None:
        """Sortino uses only negative excess returns for denominator."""
        # Mix of positive and negative returns
        returns = [0.05, -0.02, 0.03, -0.01, 0.04] * 10  # 50 trades
        holding_days = [5] * 50
        result = compute_risk_adjusted_metrics(returns, holding_days, risk_free_rate=0.0)
        assert result.sortino_ratio is not None
        assert result.sharpe_ratio is not None
        # Sortino >= Sharpe when there are positive returns (downside vol < total vol)
        assert result.sortino_ratio >= result.sharpe_ratio

    def test_max_drawdown_known_curve(self) -> None:
        """Known equity curve with identifiable drawdown."""
        # Up 10%, down 20%, up 5% → max dd from peak
        returns = [0.10, -0.20, 0.05] * 20  # 60 trades for min_trades
        holding_days = [5] * 60
        result = compute_risk_adjusted_metrics(returns, holding_days)
        assert result.max_drawdown_pct is not None
        assert result.max_drawdown_pct > 0  # Positive means drawdown occurred
        assert result.max_drawdown_date is not None

    def test_fewer_than_min_trades_returns_none_ratios(self) -> None:
        """< 30 trades → None for Sharpe and Sortino, but max drawdown still computed."""
        returns = [0.01] * 10
        holding_days = [5] * 10
        result = compute_risk_adjusted_metrics(returns, holding_days)
        assert result.sharpe_ratio is None
        assert result.sortino_ratio is None
        assert result.total_trades == 10

    def test_single_trade_returns_none(self) -> None:
        """Single trade → None for all ratios."""
        result = compute_risk_adjusted_metrics([0.05], [5])
        assert result.sharpe_ratio is None
        assert result.sortino_ratio is None
        assert result.total_trades == 1

    def test_zero_std_returns_none(self) -> None:
        """Constant returns → std = 0 → None for Sharpe/Sortino."""
        returns = [0.02] * 50
        holding_days = [5] * 50
        result = compute_risk_adjusted_metrics(returns, holding_days, risk_free_rate=0.0)
        assert result.sharpe_ratio is None  # std == 0

    def test_all_positive_excess_sortino_none(self) -> None:
        """All positive excess returns → downside deviation = 0 → Sortino None."""
        returns = [0.10] * 50  # All very positive
        holding_days = [5] * 50
        result = compute_risk_adjusted_metrics(returns, holding_days, risk_free_rate=0.0)
        assert result.sortino_ratio is None  # No downside deviation

    def test_nan_returns_rejected(self) -> None:
        """NaN in returns list → ValueError."""
        with pytest.raises(ValueError):
            compute_risk_adjusted_metrics([0.01, float("nan"), 0.02], [5, 5, 5])

    def test_inf_returns_rejected(self) -> None:
        """Inf in returns list → ValueError."""
        with pytest.raises(ValueError):
            compute_risk_adjusted_metrics([0.01, float("inf"), 0.02], [5, 5, 5])

    def test_frozen_model(self) -> None:
        """RiskAdjustedMetrics is immutable."""
        result = compute_risk_adjusted_metrics([0.01] * 50, [5] * 50)
        with pytest.raises(Exception):  # ValidationError for frozen model
            result.total_trades = 999  # type: ignore[misc]

    def test_json_roundtrip(self) -> None:
        """Model serializes to JSON and back."""
        result = compute_risk_adjusted_metrics([0.01, -0.02, 0.03] * 20, [5] * 60)
        json_str = result.model_dump_json()
        restored = RiskAdjustedMetrics.model_validate_json(json_str)
        assert restored == result
```

### Edge Cases

- Empty returns list → `total_trades=0`, all ratios None
- All returns identical → `std=0` → Sharpe/Sortino None
- All returns positive → Sortino downside deviation = 0 → None
- Single massive loss → max drawdown captures it
- Very large lookback (3650 days) → handles gracefully
- Holding days of 0 → guard against division by zero in annualization

### Mocking Strategy

- `test_performance.py`: No mocking — pure computation on in-memory lists
- `test_risk_metrics_query.py`: Mock `aiosqlite` connection with `AsyncMock`, inject sample rows
- API tests (if added): `TestClient` with mocked repository via dependency override
