# Task 530: Constraint Pre-Check (FR-C4)

## Description

Add a constraint pre-check system that validates recommended contracts against hard and
soft constraint rules before they enter the debate pipeline. Hard violations (expired,
DTE too short, OI too low, spread too wide) disqualify contracts entirely. Soft violations
(zero bid, volume too low) generate caution warnings. The constraint warnings are injected
into agent context so LLM agents are aware of problematic contracts and can avoid
recommending them.

### New Enums (`models/enums.py`)

```python
class ConstraintViolationType(StrEnum):
    EXPIRED = "expired"
    DTE_TOO_SHORT = "dte_too_short"
    OI_TOO_LOW = "oi_too_low"
    SPREAD_TOO_WIDE = "spread_too_wide"
    ZERO_BID = "zero_bid"
    VOLUME_TOO_LOW = "volume_too_low"


class ConstraintSeverity(StrEnum):
    HARD = "hard"
    SOFT = "soft"
```

### New Model (`models/analysis.py`)

```python
class ContractConstraint(BaseModel):
    model_config = ConfigDict(frozen=True)
    contract_label: str       # "AAPL 200C 2026-04-18"
    violation_type: ConstraintViolationType
    detail: str
    severity: ConstraintSeverity
```

### New Module (`agents/constraints.py`)

```python
def check_contract_constraints(
    contracts: list[OptionContract],
    filters: OptionsFilters,
) -> list[ContractConstraint]: ...

def render_constraint_warnings(violations: list[ContractConstraint]) -> str: ...
```

### Constraint Rules

| Rule | Condition | Severity |
|------|-----------|----------|
| Expired | `expiration < today` | hard |
| DTE too short | `dte < filters.min_dte` | hard |
| OI too low | `open_interest < filters.min_oi` | hard |
| Spread too wide | `spread_pct > filters.max_spread_pct` | hard |
| Zero bid | `bid == 0 and ask > 0` | soft |
| Volume too low | `volume < filters.min_volume` | soft |

### Render Format

```
<<<CONSTRAINT_WARNINGS>>>
DO NOT recommend these contracts (hard constraint violations):
- AAPL 200C 2026-04-18: Bid-ask spread 42% exceeds 30% maximum
EXERCISE CAUTION with these contracts (soft violations):
- AAPL 210C 2026-05-16: Zero bid price
<<<END_CONSTRAINT_WARNINGS>>>
```

### Integration Changes

- `agents/orchestrator.py` — in `run_debate()`, call `check_contract_constraints(contracts, filters)`
