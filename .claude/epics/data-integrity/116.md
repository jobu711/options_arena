# Task: Reject zero prices at service ingestion

## Description

Replace the `safe_decimal(price) or Decimal("0")` zero-fallback pattern in
`services/market_data.py` with validation that raises `TickerNotFoundError` when
price data is None or <= 0. Currently, a zero-price `Quote` flows into IV solvers,
scoring denominators, and `MarketContext` — corrupting everything downstream.

Bid/ask zero-fallback is kept because zero bid is legitimate for illiquid contracts
(zero-bid exemption already exists in `options_data.py`).

## Acceptance Criteria

- [ ] `fetch_quote()`: raises `TickerNotFoundError` when `price_raw` is None or `safe_decimal(price_raw)` is None or `<= Decimal("0")`
- [ ] `fetch_quote()`: bid/ask retain `safe_decimal(...) or Decimal("0")` fallback (legitimate zero)
- [ ] `fetch_ticker_info()`: raises `TickerNotFoundError` when `current_price_raw` is None or `safe_decimal(current_price_raw)` is None or `<= Decimal("0")`
- [ ] Error messages include the ticker symbol for debuggability
- [ ] Scan pipeline batch isolation preserved (one rejected ticker doesn't crash batch)
- [ ] All existing tests pass (fix mocks returning None/zero prices where needed)
- [ ] ~8-10 new tests

## Technical Details

### Files Modified
- `src/options_arena/services/market_data.py` — replace zero-fallback pattern
- `tests/services/test_market_data.py` — new rejection tests + fixture updates

### Implementation Pattern

In `fetch_quote()` (around line 330):
```python
# BEFORE:
price = safe_decimal(price_raw) or Decimal("0")

# AFTER:
price = safe_decimal(price_raw)
if price is None or price <= Decimal("0"):
    raise TickerNotFoundError(
        ticker, f"invalid price data: {price_raw!r}"
    )
```

In `fetch_ticker_info()` (around line 383):
```python
# BEFORE:
current_price = safe_decimal(current_price_raw) or Decimal("0")

# AFTER:
current_price = safe_decimal(current_price_raw)
if current_price is None or current_price <= Decimal("0"):
    raise TickerNotFoundError(
        ticker, f"invalid current price: {current_price_raw!r}"
    )
```

### Key Considerations
- `TickerNotFoundError` is imported from `utils/exceptions.py`
- Batch isolation: scan pipeline already catches `TickerNotFoundError` per-ticker via `asyncio.gather(return_exceptions=True)` — no scan-level changes needed
- `safe_decimal()` returns `Decimal | None` (never raises) — check both None and <= 0
- Don't change bid/ask handling — zero bid is a legitimate market condition
