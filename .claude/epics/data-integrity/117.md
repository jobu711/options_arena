# Task: MarketContext completeness validation + debate quality gate

## Description

Change MarketContext indicator fields from `float` to `float | None` so missing data
is represented as `None` (not silently `0.0`). Add a `completeness_ratio()` method.
Update `build_market_context()` to pass `None` instead of `0.0` for unavailable data.
Add a debate quality gate that triggers data-driven fallback when completeness < 60%.

context rendering, and potentially CLI display.

### Completeness Ratio Method
```python
def completeness_ratio(self) -> float:
    """Fraction of context fields that are populated (not None)."""
    checkable_fields = [
        self.iv_rank, self.iv_percentile, self.atm_iv_30d,
        self.rsi_14, self.put_call_ratio, self.adx,
        self.sma_alignment, self.bb_width, self.atr_pct,
        self.stochastic_rsi, self.relative_volume,
        self.target_gamma, self.target_theta, self.target_vega,
    ]
    populated = sum(1 for f in checkable_fields if f is not None)
    return populated / len(checkable_fields)
```

### Quality Gate in Orchestrator
```python
# Before bull agent invocation:
ratio = context.completeness_ratio()
if ratio < 0.6:
    logger.warning(
        "MarketContext completeness %.0f%% < 60%% for %s — using data-driven fallback",
        ratio * 100, context.ticker,
    )
    return _build_fallback_result(context, ticker_score, contracts, reason="insufficient data")
if ratio < 0.8:
    logger.warning(
        "MarketContext completeness %.0f%% < 80%% for %s — proceeding with caution",
        ratio * 100, context.ticker,
    )
```
