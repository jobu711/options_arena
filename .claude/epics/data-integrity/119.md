# Task: Test fixture audit and cleanup

## Description

Final sweep across the entire test suite after all validators are in place (tasks 1-4).
Identify and fix any remaining test fixtures that construct models with data that now
violates the new validators. Ensure all 1,402+ tests pass with zero regressions.

Each prior task fixes its own immediate test breakages, but cross-cutting fixtures
(shared conftest helpers, factory functions, parameterized test data) may slip through.
This task catches those.

## Acceptance Criteria

- [ ] Full test suite (`uv run pytest tests/ -v`) passes with 0 failures
- [ ] No test constructs `OHLCV` with zero or negative prices
- [ ] No test constructs `OptionGreeks` with NaN/Inf theta or rho
- [ ] No test constructs `Quote` or `TickerInfo` with `Decimal("0")` price (unless testing the rejection)
- [ ] No test constructs `MarketContext` with `0.0` for fields that are now `float | None` (unless explicitly testing the None path)
- [ ] Shared test fixtures/conftest helpers use valid representative data
- [ ] `uv run mypy src/ --strict` passes
- [ ] `uv run ruff check . --fix && uv run ruff format .` passes

## Technical Details

### Files to Audit
- `tests/conftest.py` — shared fixtures
- `tests/models/` — all model test files
- `tests/services/` — service test mocks
- `tests/agents/` — agent test fixtures (MarketContext construction)
- `tests/scoring/` — scoring test fixtures
- `tests/scan/` — scan pipeline test fixtures
- `tests/cli/` — CLI test fixtures

### Audit Process
1. Run `uv run pytest tests/ -v` and collect all failures
2. Categorize failures by root cause:
   - OHLCV validator rejection (zero/negative prices in fixtures)
   - OptionGreeks NaN/Inf in fixtures
   - Quote/TickerInfo zero-price in mocks
   - MarketContext type mismatch (`float` vs `float | None`)
3. Fix each fixture with valid representative data:
   - Prices: use realistic values like `Decimal("150.00")`
   - Greeks: use realistic values like `theta=-0.05`, `rho=0.02`
   - Indicators: use `None` for unavailable, realistic values otherwise
4. Re-run full suite to confirm zero failures

### Key Considerations
- Don't change test intent — if a test was verifying behavior with edge-case data, update the data to be edge-case but still valid (e.g., `Decimal("0.01")` instead of `Decimal("0")`)
- Shared conftest fixtures are highest priority — they affect the most tests
- Some tests may intentionally construct invalid models to test rejection — leave those alone (they should use `pytest.raises(ValidationError)`)
