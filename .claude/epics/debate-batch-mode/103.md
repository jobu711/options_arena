# Task: Add --batch and --batch-limit CLI flags with validation

## Description
Modify the `debate` command signature in `src/options_arena/cli/commands.py` to support
batch mode. The `ticker` argument becomes optional (`str | None`), and two new options
are added: `--batch` (bool) and `--batch-limit` (int, default 5).

Add mutual exclusivity validation: `--batch` requires no `ticker` argument; providing
a `ticker` without `--batch` works as before. Providing both `ticker` and `--batch`
is an error.

## Acceptance Criteria
- [ ] `ticker` parameter changes from required `str` to optional `str | None` (default `None`)
- [ ] `--batch` flag added (bool, default False)
- [ ] `--batch-limit` option added (int, default 5)
- [ ] Validation: `ticker` without `--batch` → single-ticker debate (existing behavior)
- [ ] Validation: `--batch` without `ticker` → batch mode (calls `_batch_async()`)
- [ ] Validation: `--batch` with `ticker` → error message and exit(1)
- [ ] Validation: neither `ticker` nor `--batch` → error message suggesting either path
- [ ] `--batch-limit` without `--batch` is ignored (no error)
- [ ] Help text updated for all options
- [ ] All existing tests still pass

## Technical Details

New command signature:
```python
@app.command()
def debate(
    ticker: str | None = typer.Argument(None, help="Ticker symbol (omit for --batch)"),
    batch: bool = typer.Option(False, "--batch", help="Debate top tickers from latest scan"),
    batch_limit: int = typer.Option(5, "--batch-limit", help="Max tickers in batch mode"),
    history: bool = typer.Option(False, "--history", help="Show past debates"),
    fallback_only: bool = typer.Option(False, "--fallback-only", help="Force data-driven path"),
) -> None:
```

Validation logic at the top of the function body:
```python
if batch and ticker is not None:
    err_console.print("[red]Cannot use --batch with a ticker argument.[/red]")
    raise typer.Exit(code=1)
if not batch and ticker is None:
    err_console.print("[red]Provide a TICKER or use --batch.[/red]")
    raise typer.Exit(code=1)
```

When `batch=True`, call `asyncio.run(_batch_async(batch_limit, fallback_only))`.
When `ticker` is provided, call `asyncio.run(_debate_async(ticker.upper(), history, fallback_only))`
as before.

Note: `--history` only applies to single-ticker mode. Using `--history` with `--batch`
should produce an error: "Cannot use --history with --batch."
