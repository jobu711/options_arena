## Technical Details

### Test file location
`tests/unit/cli/test_batch_debate.py` — new file dedicated to batch debate tests.

### Testing approach
- Use `typer.testing.CliRunner` for CLI invocation tests
- Mock `_batch_async` and `_debate_async` at the module level to test CLI routing
- Mock `Repository`, `MarketDataService`, `OptionsDataService`, `FredService` for
  integration-level batch tests
- Mock `run_debate` (from `options_arena.agents`) to avoid Ollama dependency
- Use `pytest-asyncio` for async function tests (`_debate_single`, `_batch_async`)

### Key test patterns from existing tests
```python
from typer.testing import CliRunner
from options_arena.cli import app

runner = CliRunner()

def test_batch_flag_without_ticker() -> None:
    """--batch without ticker should invoke batch mode."""
    with patch("options_arena.cli.commands._batch_async") as mock_batch:
        mock_batch.return_value = None
        result = runner.invoke(app, ["debate", "--batch"])
        assert result.exit_code == 0
        mock_batch.assert_called_once()
```

### Mocking strategy for service lifecycle test
Verify that `MarketDataService.close()`, `OptionsDataService.close()`, `FredService.close()`
are each called exactly once (not N times for N tickers).
