# Task: Implement should_debate() gate and screening fallback in orchestrator

## Description

Add a pure-function pre-debate gate to `src/options_arena/agents/orchestrator.py` that skips
the AI debate when the signal is too weak, saving 60-90s of Ollama CPU time per skipped ticker.

### 1. Add `should_debate()` pure function

```python
def should_debate(ticker_score: TickerScore, config: DebateConfig) -> bool:
    """Return False if signal is too weak for meaningful AI debate."""
    if ticker_score.direction == SignalDirection.NEUTRAL:
        return False
    if ticker_score.composite_score < config.min_debate_score:
        return False
    return True
```

- Pure function: no side effects, no I/O, no logging.
- Score comparison is strict `<` (score exactly at threshold returns `True`).

### 2. Add `_build_screening_fallback()`

A thin wrapper around `_build_fallback_result()` with a custom summary indicating
the debate was skipped due to weak signal:

```python
def _build_screening_fallback(
    context: MarketContext,
    ticker_score: TickerScore,
    contracts: list[OptionContract],
    config: DebateConfig,
    start_time: float,
) -> DebateResult:
    """Build fallback for tickers that fail pre-debate screening."""
    result = _build_fallback_result(context, ticker_score, contracts, config, start_time)
    # Override thesis summary with screening-specific message
    ...
```

The thesis summary should read:
`"Signal too weak for meaningful debate (composite: {score:.1f}/100, direction: {direction})."`

### 3. Wire into `run_debate()`

Add the screening check at the top of `run_debate()`, after `build_market_context()`:

```python
if not should_debate(ticker_score, config):
    logger.info("Skipping debate for %s: signal too weak", ticker_score.ticker)
    return _build_screening_fallback(context, ticker_score, contracts, config, start_time)
```

This short-circuits before any agent runs, returning immediately with `is_fallback=True`.
