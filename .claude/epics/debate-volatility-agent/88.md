# Task: Integrate Volatility Agent into orchestrator and update Risk agent

## Description
Add the conditional Volatility Agent phase to the orchestrator's `run_debate()` flow, running after Bear and before Risk. Guarded by `config.enable_volatility_agent` — when disabled (default), the flow is unchanged. Also update the Risk agent's dynamic prompt to inject `<<<VOL_CASE>>>...<<<END_VOL_CASE>>>` when `ctx.deps.vol_response` is present, so Risk can incorporate volatility analysis into its synthesis.

## Acceptance Criteria
- [ ] Orchestrator runs volatility phase after Bear, before Risk, when `config.enable_volatility_agent is True`
- [ ] When disabled (default), orchestrator flow is unchanged — zero overhead
- [ ] Volatility agent receives `DebateDeps` with `bull_response` and `bear_response` set
- [ ] `asyncio.wait_for()` wraps volatility agent call with `per_agent_timeout`
- [ ] `total_usage` accumulates volatility agent usage: `bull + bear + vol + risk`
- [ ] `DebateResult.vol_response` is set when volatility phase runs, `None` when skipped
- [ ] Volatility agent failure is caught by the never-raises pattern (fallback still works)
- [ ] Risk agent dynamic prompt includes `<<<VOL_CASE>>>...<<<END_VOL_CASE>>>` block when `ctx.deps.vol_response` is not None
- [ ] Risk prompt omits vol block when `vol_response` is None (backward compatible)

## Technical Details
- File: `src/options_arena/agents/orchestrator.py` — add volatility phase between bear and risk
- File: `src/options_arena/agents/risk.py` — update dynamic prompt to inject vol case
- Import `volatility_agent` from `agents/volatility.py` in orchestrator (orchestrator is the only file that imports from agent modules)
- Flow order: Bull -> Bear -> **Volatility** (if enabled) -> Risk
- Vol case injection in risk prompt: format `vol_response.iv_assessment`, `vol_response.strategy_rationale`, `vol_response.recommended_strategy` into a readable block
- `config.enable_volatility_agent` is a `bool` field on `DebateConfig` (added in Epic 3)
