# Task: Write comprehensive tests for Volatility Agent feature

## Description
Write ~14 unit tests covering all components of the Volatility Agent feature. This includes model tests, agent tests, orchestrator integration tests, risk prompt tests, and CLI rendering tests. All tests use `TestModel` (no real LLM calls) and follow existing test patterns from `tests/unit/agents/` and `tests/unit/models/`.

## Acceptance Criteria
- [ ] **VolatilityThesis model tests** (~4):
  - Construction with valid data
  - Frozen (raises on attribute reassignment)
  - Confidence validation (rejects < 0, > 1, NaN, inf)
  - JSON roundtrip: `model_validate_json(vt.model_dump_json()) == vt`
- [ ] **Volatility agent tests** (~3):
  - Agent with `TestModel` produces valid `VolatilityThesis`
  - Think-tag stripping: agent output with `<think>` tags gets cleaned
  - `build_cleaned_volatility_thesis()` handles all string fields
- [ ] **Orchestrator tests** (~3):
  - Vol enabled (`enable_volatility_agent=True`): 4-agent flow, `vol_response` populated
  - Vol disabled (default): 3-agent flow, `vol_response is None`
  - Vol agent failure: falls back gracefully, debate still produces result
- [ ] **Risk prompt tests** (~2):
  - `<<<VOL_CASE>>>` block present when `vol_response` is set
  - Vol block absent when `vol_response` is None
- [ ] **CLI rendering tests** (~2):
  - Vol panel rendered when `vol_response` is set
  - No vol panel when `vol_response` is None
- [ ] All tests use `models.ALLOW_MODEL_REQUESTS = False` guard
- [ ] All tests pass: `uv run pytest tests/ -v`
