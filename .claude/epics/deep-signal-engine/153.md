# Task: Multi-dimensional scoring engine

## Description

Replace the single composite score with multi-dimensional family sub-scores. Create `scoring/dimensional.py` to compute 8 per-family sub-scores using the existing weighted geometric mean formula scoped per family. Add 4 regime-adjusted weight profiles keyed to `MarketRegime`. Upgrade `scoring/direction.py` with continuous confidence gradient (`DirectionSignal`). Extend `scoring/composite.py` with weight redistribution.

Branch: `epic/dse-4-scoring-debate` (shared with Task 006)

## Acceptance Criteria

- [ ] `scoring/dimensional.py` created with `compute_dimensional_scores()` ‚Äî computes 8 family sub-scores from `IndicatorSignals`, gracefully skips `None` indicators
- [ ] `FAMILY_INDICATOR_MAP` constant mapping 8 families to their indicator field names
- [ ] `REGIME_WEIGHT_PROFILES` ‚Äî 4 weight profiles (trending, mean_reverting, volatile, crisis), each summing to 1.0
- [ ] `scoring/composite.py` extended with new default weights (trend 0.22, iv_vol 0.20, flow 0.18, etc.) and `apply_regime_weights()` function
- [ ] `scoring/direction.py` extended with `compute_direction_signal()` returning `DirectionSignal` (direction + continuous confidence + contributing_signals)
- [ ] Backward compatibility: existing `TickerScore.direction` unchanged; `DirectionSignal` is additional
- [ ] Regression test: same inputs with old weight distribution ‚Üí same composite score output
- [ ] Graceful `None` handling: family with all-None indicators ‚Üí family score is None; partial None ‚Üí skip and redistribute within family
- [ ] Unit tests for dimensional scoring, regime weights, direction confidence
- [ ] `ruff check`, `ruff format`, `mypy --strict` pass
- [ ] All existing tests pass unchanged
