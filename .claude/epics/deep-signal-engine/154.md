# Task: Foundation models, enums & config

## Description

Define all shared model interfaces, enums, and configuration extensions that the four parallel indicator/scoring/debate epics depend on. This is Phase 0 — it must merge to master before any other task starts. All changes are in `models/` only (plus tests). No business logic, no I/O.

Branch: `epic/dse-foundation`

### Key Constraints
- Read `models/CLAUDE.md` before modifying any model file
- Every `confidence` field gets `field_validator` constraining to `[0.0, 1.0]`
- Every `datetime` field gets UTC validator
- All new optional fields default to `None` (no breaking changes)
- `DimensionalScores` family scores use `float | None` (None when all indicators in family are missing)
- No business logic in models — just data shapes
