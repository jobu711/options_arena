# Task: Create devops-audit command with Phase 1 static analysis

## Description

Create the `.claude/commands/devops-audit.md` command file with the complete command
scaffold (frontmatter, role, context, task, instructions, constraints) and all Phase 1
static analysis checks — 29 agent coordination checks (categories A-F) plus 10
CI/config/hook checks (39 total). Phase 2 and Phase 3 are added in Task 002.

This is the bulk of the epic — the command file structure and the most detailed checks.

## Acceptance Criteria

- [ ] `.claude/commands/devops-audit.md` exists with valid frontmatter (`allowed-tools`, `description`)
- [ ] Command follows the `full-audit.md` structural pattern (`<role>`, `<context>`, `<task>`, `<instructions>`, `<constraints>`)
- [ ] Phase 1 contains all 29 agent coordination checks (A. Registry 1-5, B. Scope 6-10, C. Parallel Safety 11-15, D. Output Format 16-19, E. Coordination Efficiency 20-25, F. Change Detection 26-29)
- [ ] Phase 1 contains all 10 CI/config/hook checks (30-39)
- [ ] Each check has: number, name, what to read/grep, pass/fail criteria, severity level
- [ ] Checks reference correct file paths (`.claude/agents/*.md`, `CLAUDE.md`, `pyproject.toml`, etc.)
- [ ] Priority mapping (P1-P4) included for all Phase 1 findings

## Technical Details

### Command file structure
```
---
allowed-tools: Read, Glob, Grep, Bash, Write
description: "Audit CI/CD, agents, hooks, config, and devops gaps"
---

<role>...</role>
<context>...</context>
<task>...</task>
<instructions>
## Phase 1: Static Analysis
### A. Agent Registry & Inventory (checks 1-5)
### B. Scope Boundaries & Overlap (checks 6-10)
### C. Parallel Execution Safety (checks 11-15)
### D. Output Format Consistency (checks 16-19)
### E. Coordination Efficiency (checks 20-25)
### F. Change Detection (checks 26-29)
### G. CI/Config/Hook Checks (checks 30-39)
## Phase 2: (placeholder — added in Task 002)
## Phase 3: (placeholder — added in Task 002)
</instructions>
<constraints>...</constraints>
```

### Key patterns to follow from `full-audit.md`
- `allowed-tools` in frontmatter (no `Agent` tool — this command runs checks directly)
- `<role>` establishes the audit persona
- `<constraints>` section enforces read-only behavior and timeout discipline

### Check implementation pattern
Each check should specify:
1. **What to read** — specific Glob/Grep/Read calls
2. **What to validate** — the pass/fail condition
3. **Severity** — P1/P2/P3/P4 per the PRD priority mapping
4. **Finding format** — `[category] file:line — Description. Impact. Fix.`
