## Technical Details

### Phase 2: Dynamic Probes (checks 40-44)

Each probe runs as a background Bash command with a 30s timeout:

| # | Check | Command | Graceful skip |
|---|-------|---------|---------------|
| 40 | CI run status | `gh run list --limit 5` | Skip if `gh` not found |
| 41 | Dependency freshness | `uv lock --check` | Skip if fails (report stale lock) |
| 42 | Dependency security | `uv run pip-audit` | Skip if `pip-audit` not installed |
| 43 | External service health | `uv run options-arena health` | Skip if command fails |
| 44 | Test tier coverage | `uv run pytest --collect-only -q` + count by marker | Skip on error |

### Phase 3: Gap Analysis & Report (checks 45-49)

| # | Check | Logic |
|---|-------|-------|
| 45 | Missing practices | Flag absent: Dependabot/Renovate, SAST, Docker, release automation |
| 46 | Stale audit reports | Check `.claude/audits/*.md` timestamps — flag if >7 days old |
| 47 | Broken references | Verify all guides referenced in CLAUDE.md exist in `.claude/guides/` |
| 48 | Health check completeness | Cross-ref services used in code vs services in `health.py` |
| 49 | Report consolidation | Collect all findings from Phases 1-3, write AUDIT_DEVOPS.md |

### Report output format (AUDIT_DEVOPS.md)

```yaml
---
agent: devops-audit
status: COMPLETE
timestamp: {ISO 8601 UTC}
scope: agents, commands, CI, config, hooks, services, gaps
findings:
  critical: {count}
  high: {count}
  medium: {count}
  low: {count}
---
```

Followed by P1-P4 sections with findings in `[category] file:line — Description. Impact. Fix.` format.

### User-facing summary (displayed after report is written)

- Summary table with finding counts by severity
- If P1 > 0: recommend running `/fix-loop`
- Path to full report: `.claude/audits/AUDIT_DEVOPS.md`
