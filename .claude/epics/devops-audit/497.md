# Task: Smoke test and validate devops-audit command

## Description

Run `/devops-audit` on the live repo and validate:
1. Report is written to `.claude/audits/AUDIT_DEVOPS.md`
2. YAML preamble is valid and machine-parseable
3. At least one real finding is detected (e.g., stale audit reports, missing practices)
4. Report follows the P1-P4 format with correct finding format
5. Command completes in <3 minutes

If issues are found, fix them in the command file and re-run until the report is clean.

## Acceptance Criteria

- [ ] `/devops-audit` runs to completion without errors
- [ ] `.claude/audits/AUDIT_DEVOPS.md` is created with valid content
- [ ] YAML preamble has all required fields: agent, status, timestamp, scope, findings
- [ ] Finding counts in YAML preamble match the actual findings in the report body
- [ ] At least one real finding is detected
- [ ] Report sections P1-P4 are present (even if some are empty)
- [ ] Findings follow `[category] file:line — Description. Impact. Fix.` format
- [ ] Command completes in <3 minutes total

## Technical Details

### Known expected findings (should catch at least one)
- Stale audit reports (`.claude/audits/` files older than 7 days)
- Missing Dependabot/Renovate configuration
- Missing Docker/containerization
- Missing release automation workflow
- Possible agent scope gaps or stale coordination guide

### Validation steps
1. Run the command: `/devops-audit`
2. Read the output report: `.claude/audits/AUDIT_DEVOPS.md`
3. Parse the YAML preamble — verify all fields present
4. Count findings in each P section — verify matches preamble counts
5. Verify at least one finding has the correct format
6. If any issues: fix the command file, re-run

### Iteration loop
- If the command errors or produces malformed output, diagnose the issue in the
  command file (`.claude/commands/devops-audit.md`), fix it, and re-run
- Maximum 3 iterations before escalating to user
