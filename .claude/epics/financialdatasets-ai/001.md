# Task: Models + Config — FD response models and FinancialDatasetsConfig

## Description
Create the data foundation for the financialdatasets.ai integration: 4 frozen Pydantic response models in a new `models/financial_datasets.py` file, a `FinancialDatasetsConfig` nested config in `models/config.py`, and re-exports in `models/__init__.py`.

## Acceptance Criteria
- [ ] `FinancialMetricsData` frozen model with 21 optional float fields + `math.isfinite()` validators
- [ ] `IncomeStatementData` frozen model with 8 fields (revenue through shares_outstanding as `int | None`)
- [ ] `BalanceSheetData` frozen model with 8 fields (total_assets through current_liabilities)
- [ ] `FinancialDatasetsPackage` frozen aggregate model (ticker, metrics, income, balance_sheet, fetched_at with UTC validator)
- [ ] `FinancialDatasetsConfig(BaseModel)` with fields: `enabled: bool = True`, `api_key: str | None = None`, `base_url: str`, `request_timeout: float = 10.0`, `cache_ttl: int = 3600`
- [ ] `financial_datasets` field added to `AppSettings`
- [ ] All new models re-exported from `models/__init__.py`
- [ ] Env var `ARENA_FINANCIAL_DATASETS__API_KEY` works via nested delimiter

## Technical Details
- **New file**: `src/options_arena/models/financial_datasets.py`
- **Modified files**: `src/options_arena/models/config.py`, `src/options_arena/models/__init__.py`
- Template: follow `models/openbb.py` frozen model pattern with `ConfigDict(frozen=True)`
- All float fields: `float | None = None` with `@field_validator` using `math.isfinite()`
- Revenue/income fields that can be negative (net_income, operating_income): no `>= 0` constraint, just `isfinite()`
- `FinancialDatasetsConfig` follows `OpenBBConfig` pattern exactly — `BaseModel` not `BaseSettings`
- `base_url` default: `"https://api.financialdatasets.ai"`
- `FinancialDatasetsPackage.fetched_at`: UTC datetime with standard UTC validator

### Edge Cases
- NaN/Inf on every float field: rejected by `isfinite()` validator
- Negative financial values (net_income, operating_income): allowed (losses are real)
- Naive datetime on `fetched_at`: rejected by UTC validator
- Empty string `api_key`: treated as set (service will fail on auth, not our problem)
