# Task: FinancialDatasetsService — httpx service with cache and rate limiting

## Description
Create the `FinancialDatasetsService` in `services/financial_datasets.py` following the OpenBBService/FRED patterns: DI constructor, httpx AsyncClient, never-raises contract, cache-first strategy, rate limiting. Provides 3 endpoint methods + `fetch_package()` parallel gather + `close()`.

## Acceptance Criteria
- [ ] `FinancialDatasetsService(config, cache, limiter)` class with DI constructor
- [ ] `fetch_financial_metrics(ticker) -> FinancialMetricsData | None` method
- [ ] `fetch_income_statement(ticker) -> IncomeStatementData | None` method
- [ ] `fetch_balance_sheet(ticker) -> BalanceSheetData | None` method
- [ ] `fetch_package(ticker) -> FinancialDatasetsPackage | None` — parallel `asyncio.gather` of above 3
- [ ] `close()` method calls `self._client.aclose()`
- [ ] Never-raises: all methods catch exceptions, log WARNING, return None
- [ ] Cache-first: check cache before API call, store result with TTL on success
- [ ] Rate-limited: `async with self._limiter:` before each API call
- [ ] Config-gated: no-op (return None) when `config.enabled is False` or `config.api_key is None`
- [ ] httpx AsyncClient with `X-API-KEY` header and configurable timeout
- [ ] Cache keys: `"fd:metrics:{ticker}"`, `"fd:income:{ticker}"`, `"fd:balance:{ticker}"`
- [ ] Re-exported from `services/__init__.py`

## Technical Details
- **New file**: `src/options_arena/services/financial_datasets.py`
- **Modified file**: `src/options_arena/services/__init__.py`
- Template: `services/openbb_service.py` (class structure), `services/fred.py` (httpx setup)
- API endpoints (per PRD research):
  - `GET /api/v1/financial-metrics?ticker={ticker}&period=ttm&limit=1`
  - `GET /api/v1/income-statements?ticker={ticker}&period=ttm&limit=1`
  - `GET /api/v1/balance-sheets?ticker={ticker}&period=ttm&limit=1`
- Response parsing: JSON `{"financial_metrics": [...]}`, `{"income_statements": [...]}`, `{"balance_sheets": [...]}`
- `fetch_package()` wraps all 3 in `asyncio.gather(*tasks, return_exceptions=True)` — individual failures don't block others
- `FinancialDatasetsPackage` only created when at least one component succeeds
- Timeout: `asyncio.wait_for(coro, timeout=config.request_timeout)`
- Logging: `logger = logging.getLogger(__name__)`

### Edge Cases
- Empty response array `{"financial_metrics": []}`: return None
- Malformed JSON: caught by exception handler, return None
- HTTP 401/403 (bad API key): return None, log warning
- Partial package (2 of 3 succeed): still return package with None for failed component
- All 3 fail in gather: return None

### Mocking Strategy
- `httpx.AsyncClient` mocked via `unittest.mock.patch` or `AsyncMock`
- `ServiceCache`: `MagicMock(get=AsyncMock(return_value=None), set=AsyncMock())`
- `RateLimiter`: `MagicMock(__aenter__=AsyncMock(), __aexit__=AsyncMock())`
- No real API calls
