# Task: Integration wiring — orchestrator, API, CLI, health check

## Description
Wire the FinancialDatasetsService through all integration points: orchestrator (`build_market_context` + `run_debate`), API (lifespan, deps, debate route), CLI (debate command), and health check. Implements FD > OpenBB > None priority for 7 overlapping fields.

## Acceptance Criteria
- [ ] `build_market_context()` accepts `fd_package: FinancialDatasetsPackage | None = None` parameter
- [ ] FD fields mapped to `MarketContext.fd_*` fields from package components
- [ ] 7 overlapping fields use FD > OpenBB > None priority: `pe_ratio`, `forward_pe`, `peg_ratio`, `price_to_book`, `debt_to_equity`, `revenue_growth`, `profit_margin`
- [ ] `run_debate()` accepts and passes through `fd_package` parameter
- [ ] API `lifespan()` creates `FinancialDatasetsService` when config enabled and API key set
- [ ] API `get_financial_datasets()` dependency in `deps.py`
- [ ] API debate route fetches FD package and passes to `run_debate()`
- [ ] CLI debate command creates service, fetches package, passes to `run_debate()`, closes in `finally`
- [ ] `check_financial_datasets()` health check method added
- [ ] Health check added to `check_all()` when config enabled
- [ ] All new parameters are optional — existing flows work unchanged with `fd_package=None`

### Edge Cases
- `fd_package=None`: all fd_* fields remain None, existing behavior unchanged
- FD package with partial data (metrics present, income None): only available fields mapped
- Both FD and OpenBB provide same field: FD wins (priority rule)
- Neither provides field: remains None
- Service creation fails: debate proceeds without FD data (never-raises)
