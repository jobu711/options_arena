# Task: Add model fields for chain_spread_pct and chain_oi_depth

## Description
Add two new float fields to `IndicatorSignals` in `models/scan.py`: `chain_spread_pct`
(OI-weighted average bid-ask spread as percentage points) and `chain_oi_depth` (log10 of
total open interest). Both are `float | None = None` — the existing `_normalize_non_finite`
model_validator auto-sanitizes NaN/Inf for free.

## Technical Details
- File: `src/options_arena/models/scan.py`
- Add fields in the DSE Risk section (around line ~105)
- `chain_spread_pct` — percentage points (0.0–30.0), NOT ratio. Higher = worse liquidity
- `chain_oi_depth` — log10(total_oi + 1), range 0.0–6.0. Higher = better depth
- The `_normalize_non_finite` model_validator (mode="before") already replaces non-finite
  floats with `None` — new fields inherit this automatically
