# Task: Compute chain_spread_pct and chain_oi_depth in Phase 3

## Description
Add computation logic for both liquidity indicators inside `compute_phase3_indicators()`
in `scan/indicators.py`. Reuse the existing `_contracts_to_dataframe()` helper which
already provides `bid`, `ask`, and `openInterest` columns.

## Acceptance Criteria
- [ ] `chain_spread_pct` computed as OI-weighted average of `(spread / mid) * 100` across all contracts
- [ ] `chain_oi_depth` computed as `log10(total_oi + 1)` across all contracts
- [ ] Both values returned in the signals dict from `compute_phase3_indicators()`
- [ ] Zero-mid contracts excluded from `chain_spread_pct` average (avoid division by zero)
- [ ] Empty chain returns `None` for both fields
- [ ] Single-contract chain works correctly

## Technical Details
- File: `src/options_arena/scan/indicators.py`
- Function: `compute_phase3_indicators()` (line ~411)
- Reuse `_contracts_to_dataframe(contracts)` (line ~747) — already builds DataFrame with
  `bid`, `ask`, `openInterest` columns
- `chain_spread_pct` formula:
  ```python
  spread = chain_df["ask"] - chain_df["bid"]
  mid = (chain_df["ask"] + chain_df["bid"]) / 2
  spread_pct = (spread / mid) * 100  # percentage points
  # OI-weighted average (exclude zero-mid rows)
  oi = chain_df["openInterest"]
  chain_spread_pct = (spread_pct * oi).sum() / oi.sum()
  ```
- `chain_oi_depth` formula:
  ```python
  import math
  total_oi = chain_df["openInterest"].sum()
  chain_oi_depth = math.log10(total_oi + 1)
  ```
- Guard: if DataFrame is empty or all OI is zero, return `None` for both
- Use `math.isfinite()` on computed values before returning
