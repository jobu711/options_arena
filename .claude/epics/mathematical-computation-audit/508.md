# Task: Reference Data — Academic Known-Values JSON + QuantLib Baseline Generator

## Description
Compile academic known-values JSON fixtures from Hull (2018), BAW (1987), and standard textbook references for all mathematical functions. Build the QuantLib baseline generator tool that produces cross-validation data. This provides the oracle data that correctness tests compare against.

## Acceptance Criteria
- [ ] `tests/audit/reference_data/pricing_known_values.json` — BSM + BAW academic references (Hull Table 15.3, BAW 1987 Table 1, etc.)
- [ ] `tests/audit/reference_data/indicator_known_values.json` — textbook indicator references (RSI, MACD, Bollinger Bands, etc.)
- [ ] `tests/audit/reference_data/scoring_known_values.json` — scoring algorithm references
- [ ] `tests/audit/reference_data/orchestration_known_values.json` — log-odds pooling, Shannon entropy references
- [ ] `tools/generate_quantlib_baselines.py` — generates `quantlib_baselines.json` from QuantLib (dev-only)
- [ ] `tests/audit/reference_data/quantlib_baselines.json` — pre-generated QuantLib cross-validation data (100+ parameter combos per function)
- [ ] Schema validation test for all JSON fixtures
- [ ] Each fixture entry has `source` field citing the academic reference

## Technical Details
- Academic known-values come from: Hull "Options, Futures, and Other Derivatives" (2018), Barone-Adesi & Whaley (1987) paper Table 1, Wilder (1978) for RSI, Appel (1979) for MACD, Bollinger (2001) for BB
- QuantLib baseline generator is a standalone script — NOT part of the test suite
- Generator iterates over a parameter grid: S ∈ {50, 100, 200}, K ∈ {80, 100, 120}, T ∈ {0.083, 0.25, 0.5, 1.0}, r ∈ {0.02, 0.05, 0.10}, q ∈ {0, 0.02}, σ ∈ {0.1, 0.2, 0.3, 0.5}
- For each combination, QuantLib computes: price (European + American), all first-order Greeks, second-order Greeks, IV round-trip
- Output JSON schema must match the `AuditFinding` tolerance fields
- If QuantLib unavailable on Python 3.13/Windows, provide instructions for Docker-based generation

   - WHY: academic citations required per test conventions (tests/CLAUDE.md: "cite source of expected values")

   - WHY: log-odds pooling (Bordley 1982), Shannon entropy have known analytical solutions for simple cases
5. Create `tools/generate_quantlib_baselines.py` — standalone script that uses QuantLib to generate cross-validation data
   - WHY: QuantLib is the gold standard for derivatives pricing; cross-validation catches transcription errors

   - WHY: tests load JSON at runtime — file must exist even if generated later

   - WHY: reference data corruption would silently invalidate all correctness tests
