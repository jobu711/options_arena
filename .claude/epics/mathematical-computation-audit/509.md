# Task: CLI Command + Report Generator + Coverage Meta-Test

## Description
Build the `options-arena audit math` CLI command that orchestrates pytest runs for each audit layer, the markdown report generator, and the coverage meta-test that verifies all 87 functions have tests in all layers.

## Acceptance Criteria
- [ ] `cli/audit.py` — `audit_app` Typer group with `math` command
- [ ] Flags: `--correctness`, `--stability`, `--performance`, `--report`, `--discover`
- [ ] Without flags, runs all 3 deterministic layers
- [ ] `tools/math_audit_report.py` — takes AuditReport model → markdown string
- [ ] Report includes: summary table, per-function status, findings with severity, performance comparison
- [ ] Coverage meta-test: scans MATH_FUNCTION_REGISTRY and asserts each function has tests in all 3 layers
- [ ] `audit_app` registered in `cli/app.py` via `add_typer`
- [ ] Command completes in <5s for report-only mode

## Technical Details
- CLI follows outcomes pattern: `audit_app = typer.Typer(help="...")`, registered via `app.add_typer(audit_app, name="audit")`
- `math` command is sync wrapper: `def math_audit() -> None: asyncio.run(_math_audit_async())`
- Command invokes pytest programmatically via `subprocess.run(["uv", "run", "pytest", "-m", marker, "--json-report", ...])`
- Report generator is a pure function: `def generate_math_audit_report(report: AuditReport) -> str`
- Coverage meta-test: iterates `MATH_FUNCTION_REGISTRY`, collects test node IDs from `pytest --co`, asserts coverage
- The `--discover` flag is handled in Task 511 — this task just validates the flag is accepted
