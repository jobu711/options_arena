# Task: Correctness Tests — All 87 Functions vs Academic + QuantLib Baselines

## Description
Write correctness tests for all 87 mathematical functions across pricing (17), indicators (53), scoring (16), and orchestration (5) modules. Each test compares function output against academic known-values and QuantLib cross-validation baselines using the tolerances defined in the PRD.

## Acceptance Criteria
- [ ] `test_pricing_correctness.py` — 17 pricing functions tested against Hull/BAW/QuantLib references
- [ ] `test_indicators_correctness.py` — 53 indicator functions tested against textbook + computed references
- [ ] `test_scoring_correctness.py` — 16 scoring functions tested against deterministic references
- [ ] `test_orchestration_correctness.py` — 5 orchestration functions tested against analytical solutions
- [ ] All tests decorated with `@pytest.mark.audit_correctness`
- [ ] Tolerances match PRD specification (prices: abs=0.01, delta/gamma: abs=0.506, etc.)
- [ ] Each test method has a docstring citing the reference source
- [ ] All 87 functions in MATH_FUNCTION_REGISTRY have at least one correctness test

## Technical Details
- Tests load reference data from `tests/audit/reference_data/*.json` fixtures
- Use `@pytest.mark.parametrize` to iterate over reference values per function
- Pricing tolerances: prices abs=0.01/rel=0.1%, delta/gamma abs=0.506/rel=0.5%, theta/vega/rho abs=0.01/rel=1.0%, vanna/charm/vomma abs=0.01/rel=2.0%, IV round-trip abs=0.0001/rel=0.1%
- Indicator tolerances: abs=0.01/rel=0.5%
- Scoring tolerances: composite abs=0.1/rel=1.0%
- Orchestration tolerances: log-odds abs=0.506/rel=0.1%
- Use `pytest.approx()` with appropriate tolerance per category
