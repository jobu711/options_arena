# Task: Stability Tests — Hypothesis + Extreme Inputs + NaN Injection

## Description
Write stability tests for all 87 mathematical functions using three techniques: Hypothesis property-based testing (random valid inputs with invariant assertions), predefined extreme input battery (stress values from PRD), and NaN/Inf injection (every input position). Ensures zero silent NaN propagation.

## Acceptance Criteria
- [ ] `test_pricing_stability.py` — Hypothesis strategies + extreme battery + NaN injection for 17 pricing functions
- [ ] `test_indicators_stability.py` — Hypothesis strategies + extreme battery + NaN injection for 53 indicator functions
- [ ] `test_scoring_stability.py` — Hypothesis strategies + extreme battery + NaN injection for 16 scoring functions
- [ ] `test_orchestration_stability.py` — Hypothesis strategies + extreme battery + NaN injection for 5 orchestration functions
- [ ] All tests decorated with `@pytest.mark.audit_stability`
- [ ] Hypothesis profiles: `ci` (100 examples), `thorough` (1000 examples)
- [ ] Zero silent NaN propagation: every function produces finite output or raises clean ValueError
- [ ] Monotonicity assertions: call price increases with S, put price decreases with S, vega >= 0
- [ ] Put-call parity holds for BSM within tolerance

## Technical Details
- Hypothesis strategies must generate inputs within domain bounds (S>0, K>0, T>0, 0<σ<5, etc.)
- Use `@given(...)` with `@settings(max_examples=...)` — profiles override at runtime
- Extreme input battery from PRD:
  - S: 0.01, 0.1, 1, 100, 10000, 100000
  - K: 0.01, 0.1, 1, 100, 10000, 100000
  - T: 1/365/24 (1hr), 1/365 (1day), 7/365 (1wk), 1.0, 5.0, 10.0
  - σ: 0.506, 0.01, 0.05, 0.5, 1.0, 3.0, 4.99
  - r: -0.05, 0, 0.506, 0.05, 0.15, 0.50
  - Moneyness: S/K = 0.01, 0.5, 0.95, 1.0, 1.05, 2.0, 10.0
- NaN/Inf injection: for each function, inject `float('nan')` and `float('inf')` into each parameter position. Assert either ValueError raised or documented fallback returned.
- Indicators need synthetic pandas DataFrames with varying lengths and value distributions

class TestBSMStability:
    @given(S=st.floats(0.01, 100000), K=st.floats(0.01, 100000), ...)

    @given(S=st.floats(0.01, 100000), ...)
    def test_put_call_parity(self, S, K, T, r, q, sigma) -> None:
        """Property: C - P = S*exp(-qT) - K*exp(-rT) within tolerance."""

    @pytest.mark.parametrize("S,K,T,r,q,sigma", EXTREME_PRICING_INPUTS)
    def test_bsm_extreme_inputs(self, S, K, T, r, q, sigma) -> None:
        """Extreme inputs produce finite output or clean error."""

    @pytest.mark.parametrize("position", range(6))
    def test_bsm_nan_injection(self, position) -> None:
        """NaN in any input position raises ValueError."""

### Edge Cases
- S = K exactly (ATM): delta should be ~0.5
- T approaching 0: option value approaches intrinsic
- σ approaching 0: option value approaches discounted intrinsic
- All identical prices in DataFrame: indicators should not crash
- Single-row DataFrame: should raise InsufficientDataError

- Hypothesis generates valid inputs; extreme battery uses predefined values
