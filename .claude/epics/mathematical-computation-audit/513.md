# Task: CI Integration — GitHub Actions PR Gate + Weekly Performance Schedule

## Description
Add mathematical audit tests to the GitHub Actions CI pipeline. Correctness and stability tests run on every PR (gate). Performance benchmarks run on a weekly schedule (informational). Configure markers, timeouts, and reporting.

## Acceptance Criteria
- [ ] `pytest -m "audit_correctness or audit_stability"` added to PR gate in `.github/workflows/ci.yml`
- [ ] Correctness + stability gate completes in <60s
- [ ] Weekly performance benchmark schedule added to `.github/workflows/nightly.yml` (or new `benchmark.yml`)
- [ ] Performance >20% regression flags warning annotation (not hard failure)
- [ ] Hypothesis seed database (`.hypothesis/`) committed for reproducibility
- [ ] pytest-benchmark baselines stored in repo for comparison
- [ ] CI passes with all audit tests green

## Technical Details
- PR gate: add a new step to the existing `ci.yml` test job: `uv run pytest -m "audit_correctness or audit_stability" -q`
- Weekly benchmark: add a `schedule` trigger to nightly.yml or create `benchmark.yml` with `cron: '0 6 * * 1'` (Monday 6am UTC)
- Benchmark step: `uv run pytest -m audit_performance --benchmark-enable --benchmark-autosave`
- Benchmark comparison: `--benchmark-compare` against stored baselines
- Regression detection: pytest-benchmark's `--benchmark-compare-fail=min:20%` as soft warning
- `.hypothesis/` directory: add to `.gitignore` exceptions or commit seed database
- Ensure audit dev dependencies (hypothesis, pytest-benchmark) are installed in CI
