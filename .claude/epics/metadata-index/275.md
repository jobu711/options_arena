# Task: TickerInfo.industry field + market_data extraction

## Description
Add `industry: str = "Unknown"` field to the existing `TickerInfo` frozen Pydantic model and extract `info.get("industry", "Unknown")` from yfinance in `market_data.py`'s `fetch_ticker_info()`. This is backward-compatible — existing cached JSON will deserialize correctly because the field has a default.

## Acceptance Criteria
- [ ] `TickerInfo` model has new `industry: str = "Unknown"` field
- [ ] `fetch_ticker_info()` in `services/market_data.py` populates `industry` from yfinance `info.get("industry", "Unknown")`
- [ ] Existing `TickerInfo` serialization cache deserializes correctly (backward compatible)
- [ ] Existing tests still pass without modification

## Technical Details
- Add field after `sector: str` in `TickerInfo` (models/market_data.py ~line 166)
- Add `industry=str(info.get("industry", "Unknown"))` to the `TickerInfo(...)` constructor call in `fetch_ticker_info()` (~line 443)
- Pydantic v2 with `model_validate_json()` accepts missing fields when a default exists — no cache invalidation needed
- The `_deserialize_ticker_info()` function uses `TickerInfo.model_validate_json()` which handles this

### Files Affected
- **Modified**: `src/options_arena/models/market_data.py` (add field)
- **Modified**: `src/options_arena/services/market_data.py` (extract industry from yfinance)
