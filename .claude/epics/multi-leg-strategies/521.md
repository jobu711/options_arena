# Task: Presentation Layer

## Description

## Acceptance Criteria
- [ ] API schemas include spread data in `DebateResultDetail` and `TickerDetail`
- [ ] CLI renders spread legs as a Rich table with P&L summary

- [ ] Frontend displays spread legs, P&L, breakevens (stretch goal)
- [ ] All new fields optional — backward compatible when no spread
- [ ] `mypy --strict` clean

class SpreadDetail(BaseModel):

### CLI Rendering
In `cli/rendering.py`, after thesis display:
```python
if spread:
    console.print("\n[bold]Strategy: {spread.spread_type}[/bold]")
    table = Table(title="Spread Legs")
    table.add_column("Side")
    table.add_column("Type")
    table.add_column("Strike")
    table.add_column("Exp")
    table.add_column("Bid")
    table.add_column("Ask")
    table.add_column("Delta")
    for leg in spread.legs:
        table.add_row(...)
    console.print(table)
    console.print(f"Net Premium: ${spread.net_premium}")
    console.print(f"Max Profit: ${spread.max_profit} | Max Loss: ${spread.max_loss}")
    console.print(f"PoP: {spread.pop:.1%} | Risk/Reward: {spread.risk_reward:.2f}")
```

### Edge Cases
- No spread (None): all presentation layers handle gracefully
- Spread with NaN risk_reward: display "--" or "N/A"
- Spread with unlimited max_profit (straddle): display "Unlimited"
- 4-leg iron condor: table shows all 4 legs
- Missing delta on a leg: display "--"
