# Task: Pipeline Integration

## Description
Wire HV estimators, second-order Greeks, and vol surface analytics into the scan pipeline. This is the integration task that connects the pure-math foundations (Tasks 1-3) and model extensions (Task 4) into the live pipeline flow. Includes: Phase 3 indicator computation, contract recommendation with second-order Greeks, vol surface computation, composite weight redistribution, and orchestrator MarketContext mapping.

## Acceptance Criteria
- [ ] `compute_phase3_indicators()` calls Yang-Zhang HV estimator with OHLC data
- [ ] `compute_phase3_indicators()` accepts optional `vol_result: VolSurfaceResult | None` and extracts metrics
- [ ] `process_ticker_options()` calls `compute_vol_surface()` before `compute_phase3_indicators()`
- [ ] `scoring/contracts.py` calls `option_second_order_greeks()` and constructs single OptionGreeks with all fields
- [ ] `_PHASE3_FIELDS` in `phase_options.py` includes new field names
- [ ] `INDICATOR_WEIGHTS` updated: +skew_25d(0.02) +smile_curvature(0.01), redistributed from existing
- [ ] Orchestrator maps new signals → MarketContext fields
- [ ] Existing scan pipeline tests pass (zero regressions)

## Technical Details

### Phase 3 Indicator Wiring (scan/indicators.py)
In `compute_phase3_indicators()`:
1. Add Yang-Zhang HV call: extract `open_series = df["open"]` alongside existing `close_series = df["close"]`. Call `compute_hv_yang_zhang(open_series, high_series, low_series, close_series)`. Set `signals.hv_yang_zhang = result`.
2. Add optional vol_result parameter. When provided, extract `signals.skew_25d = vol_result.skew_25d`, `signals.smile_curvature = vol_result.smile_curvature`, `signals.prob_above_current = vol_result.prob_above_current`. Also use `vol_result.atm_iv_30d` to replace existing `_extract_atm_iv_by_dte()` call when available.
3. All new indicator calls use isolated try/except — one failure never crashes pipeline.

### Vol Surface Computation (scan/phase_options.py)
In `process_ticker_options()`, BEFORE calling `compute_phase3_indicators()`:
1. Extract contract arrays from the option chain: strikes, IVs, DTEs, option_types
2. Call `compute_vol_surface(strikes, ivs, dtes, option_types, spot)`
3. Pass `vol_result` to `compute_phase3_indicators()`
4. This ensures surface-interpolated ATM IV replaces raw nearest-contract extraction

### Second-Order Greeks (scoring/contracts.py)
In `recommend_contracts()`, after computing first-order Greeks via `option_greeks()`:
1. Call `option_second_order_greeks(exercise_style, S, K, T, r, q, sigma, option_type)`
2. Get `SecondOrderGreeks(vanna, charm, vomma)`
3. Construct single `OptionGreeks(delta=..., gamma=..., ..., vanna=second.vanna, charm=second.charm, vomma=second.vomma)`
4. Wrap in try/except — if second-order fails, still return first-order Greeks (vanna/charm/vomma=None)

### Composite Weight Redistribution (scoring/normalization.py)
Add new entries to `INDICATOR_WEIGHTS`:
- `skew_25d: 0.02`
- `smile_curvature: 0.01`
Redistribute 0.03 total from:
- `put_call_ratio: 0.04 → 0.03` (-0.01)
- `max_pain_distance: 0.04 → 0.03` (-0.01)
- `chain_spread_pct: 0.04 → 0.03` (-0.01)
Import-time sum==1.0 guard catches drift.

Note: `hv_yang_zhang` and `prob_above_current` are context-only — NOT added to weights.

### _PHASE3_FIELDS Update (scan/phase_options.py)
Add `"hv_yang_zhang"`, `"skew_25d"`, `"smile_curvature"`, `"prob_above_current"` to the field tuple.

### Orchestrator Mapping (agents/orchestrator.py)
In the `MarketContext` construction:
```python

skew_25d=signals.skew_25d,
smile_curvature=signals.smile_curvature,

```
