# Task: Model Field Extensions + Migration

## Description
Add new fields to `IndicatorSignals`, `MarketContext`, and `RecommendedContract` models for HV estimators, vol surface metrics, and second-order Greeks. Create SQLite migration 032 to add vanna/charm/vomma columns to `recommended_contracts` table. This is the data-layer foundation for pipeline integration (Task 486).

## Technical Details

### IndicatorSignals (models/scan.py)
Add 4 new fields after existing options-specific fields. These are `float | None = None` with `math.isfinite()` validators. They are NOT normalized (0-100) — they are raw values that flow through to MarketContext for debate agent context, and optionally into composite scoring (skew_25d, smile_curvature get weights in Task 486).

### MarketContext (models/analysis.py)
Add 4 fields: `hv_yang_zhang: float | None = None`, `skew_25d: float | None = None`, `smile_curvature: float | None = None`, `prob_above_current: float | None = None`. Note: `target_vanna`, `target_charm`, `target_vomma` already exist on MarketContext (placeholder fields, currently always None). These will be populated by Task 486 pipeline wiring — no model change needed for second-order Greeks on MarketContext.

### RecommendedContract (models/analytics.py)
Add `vanna: float | None = None`, `charm: float | None = None`, `vomma: float | None = None`. These are flat columns matching the existing pattern of `delta`, `gamma`, `theta`, `vega`, `rho` on `RecommendedContract`.

### Migration (data/migrations/032_second_order_greeks.sql)
```sql
ALTER TABLE recommended_contracts ADD COLUMN vanna REAL;
ALTER TABLE recommended_contracts ADD COLUMN charm REAL;
ALTER TABLE recommended_contracts ADD COLUMN vomma REAL;
```
