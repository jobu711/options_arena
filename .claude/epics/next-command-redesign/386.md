# Task: Rewrite Phase 1 — fix AskUserQuestion with prose-based directives

## Description

Rewrite Phase 1 of `.claude/commands/pm/next.md` to fix the critical AskUserQuestion
rendering bug. The current implementation uses a JSON code fence to describe the tool
call parameters, which Claude interprets as documentation rather than an invocation
directive. Replace with prose-based directives following the pattern used by all other
working PM commands.

## Acceptance Criteria

- [ ] JSON code fence (lines 21-57) removed entirely
- [ ] Text-before-tool instruction ("Output this exact text") removed
- [ ] Negative constraints ("Do NOT run any bash commands...") removed
- [ ] Single positive directive leads: "Your first action is to call AskUserQuestion"
- [ ] 3 questions described as prose bullets with field names inline
- [ ] Phase 2 gated with clear ordering: "Phase 2 begins AFTER the user answers"
- [ ] `allowed-tools` frontmatter still includes `AskUserQuestion`

## Technical Details

### Current broken pattern (remove)
```markdown
1. Output this exact text to the user: "Let me ask..."
2. Immediately call AskUserQuestion with the 3 questions below.
3. Do NOT run any bash commands... before OR alongside this call.

Ask all 3 questions in ONE AskUserQuestion call. Use this exact structure:
\`\`\`json
{ "questions": [...] }
\`\`\`
```

### New pattern (write)
```markdown
Your first action is to call AskUserQuestion with these 3 questions:

- Question 1 (header: "Mode", single-select): "What kind of work fits your headspace right now?"
  - "Build new features" — New capabilities, endpoints, or UI
  - "Fix and harden" — Bugs, edge cases, test coverage
  - "Polish and refine" — UX, performance, code quality
  - "Surprise me" — Rank purely on project impact

- Question 2 (header: "Area", multi-select): "Which parts of the codebase do you want to work in?"
  - "Backend (Python)" — Models, services, scoring, pricing
  - "Frontend (Vue)" — Components, views, stores, styling
  - "AI agents" — Prompts, orchestration, LLM providers
  - "Infrastructure" — CI/CD, config, tooling, DevOps

- Question 3 (header: "Scope", single-select): "How much time do you want to invest?"
  - "Quick wins (hours)" — S-sized: a focused session or two
  - "Focused sprint (days)" — M-sized: a few days of work
  - "Deep project (week+)" — L/XL-sized: multi-day epics

Phase 2 begins AFTER the user answers all questions.
```

### Files affected
- `.claude/commands/pm/next.md` — Phase 1 section (lines 10-60)

### Key principles
- Lead with the tool call, not text output
- Prose bullets, not JSON code fences
- Positive framing only, no negative constraints
- Same 3 questions with identical content (Mode, Area, Scope)
