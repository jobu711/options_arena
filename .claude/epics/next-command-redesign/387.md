# Task: Rewrite Phase 2 — Agent-delegated context sweep with cross-referencing

## Description

Rewrite Phase 2 of `.claude/commands/pm/next.md` to delegate heavy context gathering to
an Explore subagent and add multi-source cross-referencing to eliminate false positives
(shipped features appearing in recommendations). The subagent performs git queries, PRD
scanning, and codebase verification, returning a compact summary to the main context.

## Acceptance Criteria

- [ ] 4 parallel git commands replaced with Agent tool delegation
- [ ] Subagent instructions include: `git log master --oneline -10`, `git branch --no-merged master`
- [ ] Subagent reads PRD frontmatter (first 10 lines only, not full content)
- [ ] Subagent reads `progress.md` "Recently Completed" section
- [ ] Subagent checks `.claude/epics/archived/` directory listing
- [ ] Subagent performs glob/grep to verify candidates aren't already implemented
- [ ] Cross-referencing algorithm documented: 3-source check (archived epics + progress.md + codebase)
- [ ] Subagent prompt caps output: "Return a summary under 30 lines"
- [ ] Ranking multiplier tables (Mode, Scope) preserved unchanged in main context
- [ ] Strategic vision analysis instructions included in subagent prompt

## Technical Details

### Current pattern (remove)
```markdown
Run in parallel:
\`\`\`bash
git log master --oneline --merges -15
git log master --oneline -20 --no-merges
git branch -a --no-merged master
git log master --oneline --since="14 days ago" --no-merges
\`\`\`

For each unmerged branch: `git log master..<branch> --oneline`
```

### New pattern (write)
Delegate to Explore subagent using prose-based Task directive:

```markdown
Spawn an Explore agent to gather project state silently:

Task:
  description: "Context sweep for /pm:next"
  subagent_type: "Explore"
  prompt: |
    Gather project state for development targeting. Return a compact summary.

    1. Run: git log master --oneline -10 (recent commits)
    2. Run: git branch --no-merged master (in-flight work)
    3. Read first 10 lines of each file in .claude/prds/ (frontmatter only)
    4. Read .claude/context/progress.md — extract "Recently Completed" and "Future Work"
    5. List directories in .claude/epics/archived/ (shipped features)
    6. For each PRD not yet shipped: glob/grep to check if feature already exists in codebase

    Cross-reference rule: A feature is SHIPPED if ANY of these is true:
    - Its epic name appears in .claude/epics/archived/
    - It is mentioned in progress.md "Recently Completed"
    - Its key components exist in the codebase (Vue views, API endpoints, models)

    Also identify 1-2 strategic opportunities:
    - What does the current architecture enable that isn't built yet?
    - What capabilities exist that could be extended to new use cases?

    Return under 30 lines:
    - SHIPPED: [list of shipped feature names]
    - BACKLOG: [list of unshipped PRD names with status and effort estimate]
    - IN-FLIGHT: [unmerged branches, if any]
    - FUTURE IDEAS: [items from progress.md "Future Work"]
    - STRATEGIC: [1-2 novel opportunities based on codebase capabilities]
    - RECENT THEMES: [2-3 word summary of last 2 weeks of commits]
```

After subagent returns, apply ranking using user's interview answers (Mode, Area, Scope
multipliers — keep existing tables unchanged).

### Files affected
- `.claude/commands/pm/next.md` — Phase 2 section (lines 62-111)

### Key design decisions
- Subagent does ALL data gathering — main context only receives summary
- Cross-referencing is 3-source (not single-source) to catch PRD status inconsistencies
- Strategic opportunities are gathered during context sweep, not as a separate phase
- Ranking multiplier tables stay in main context (not delegated)

## Test Plan

### Test Cases (Manual)
No automated tests — this is a markdown command file.

1. **Agent delegation fires**: Phase 2 spawns an Explore subagent (not inline git commands)
2. **No false positives**: Shipped features (liquidity-weighting, sp500-heatmap, etc.) excluded
3. **Unshipped PRDs included**: algo-audit, financialdatasets-ai appear as backlog candidates
4. **Context efficiency**: Phase 2 consumes <15% of context window
5. **Ranking varies by Mode**: "Build new features" produces different order than "Fix and harden"

### Edge Cases
- All PRDs shipped → only strategic suggestions and Future Work items remain
- No unmerged branches → IN-FLIGHT section empty (graceful)
- `.claude/prds/` empty → skip PRD scanning, still show strategic suggestions
