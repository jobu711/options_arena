# Research: next-command-redesign

## PRD Summary

Fix the `/pm:next` command's AskUserQuestion rendering bug (JSON-in-code-fence treated
as documentation), improve backlog mining accuracy (eliminate false positives for shipped
features), add strategic vision suggestions beyond the existing backlog, and reduce
context window consumption by ~50% through subagent delegation.

## Relevant Existing Modules

- `.claude/commands/pm/next.md` — **Primary file to modify.** The only file containing the
  broken AskUserQuestion pattern. 154 lines, 3 phases (Interview, Context Sweep, Output).
- `.claude/context/progress.md` — **Read target for backlog mining.** Contains "Recently
  Completed" (4 entries), "Future Work" (4 ideas), "In Progress" (empty), "Blockers" (none).
- `.claude/context/progress-archive.md` — **Read target for shipped features.** 27KB archive
  of 19 completed epics with PR numbers and dates.
- `.claude/prds/` — **5 PRDs** with frontmatter status values: `draft`, `planned`, `backlog`.
  No `done`/`archived` status values observed — completion tracked by epic archival instead.
- `.claude/epics/archived/` — **27 archived epics.** Each contains `epic.md`, `research.md`,
  task files, `checkpoint.json`. Presence here = shipped.

## Potential Conflicts

### 1. PRD Status Field Inconsistency
**Issue:** Shipped PRDs (liquidity-weighting, sp500-heatmap) still have `status: planned` in
their frontmatter, not `status: done`. Completion is tracked by epic archival, not PRD status.
**Mitigation:** Backlog mining must check BOTH PRD status AND archived epic presence. Cannot
rely on PRD status alone. The cross-reference should be:
1. Check `.claude/epics/archived/{prd-name}/` exists → shipped
2. Check `progress.md` "Recently Completed" → shipped
3. Check PRD `status: done|archived` → shipped
4. Only if ALL three fail → backlog candidate

### 2. Empty Backlog Scenario
**Issue:** All 26 epics are shipped. Only 2 PRDs are genuinely unshipped (`algo-audit`,
`financialdatasets-ai`). The "Future Work" section has 4 vague items. Strategic suggestions
become critical when backlog is nearly empty.
**Mitigation:** FR-3 (Strategic Vision) must produce meaningful suggestions even with an
exhausted backlog. The capability/gap/industry analysis approach handles this.

### 3. No Unmerged Branches Currently
**Issue:** `git branch --no-merged master` returns empty. The branch-checking logic has
nothing to cross-reference right now.
**Mitigation:** Branch checking is a safety net, not primary source. Works correctly when
branches exist; gracefully skips when empty.

## Open Questions

1. **Should shipped PRDs have their status updated to `done`?** Currently, shipped PRDs
   like `liquidity-weighting.md` still say `status: planned`. The redesign can work around
   this, but a one-time cleanup would simplify future backlog mining. (Non-blocking)

2. **Should strategic suggestions be generated by a subagent?** The PRD suggests capability
   analysis, gap analysis, and industry patterns. This is hard to encode in static markdown
   instructions — may need an Explore agent to scan the codebase dynamically. (Design decision
   for epic decomposition)

3. **How should the command handle `--no-interview` mode?** Some users may want to skip the
   interview and get pure-impact-ranked suggestions. Not in PRD scope, but worth noting for
   future work. (Out of scope per PRD)

## Recommended Architecture

### Phase 1: Interview (Fixed)
- Single positive directive: "Your first action is to call AskUserQuestion with these 3 questions"
- Questions described as prose bullets with field names inline
- No preceding text output, no JSON fences, no negative constraints
- Phase 2 gated by "AFTER the user answers"

### Phase 2: Context Sweep (Delegated)
- Wrap entire context gathering in an Agent tool call (Explore subagent)
- Subagent performs:
  - `git log master --oneline -10` (recent commits)
  - `git branch --no-merged master` (in-flight work)
  - Read PRD frontmatter (first 10 lines each)
  - Read `progress.md` "Recently Completed" section
  - Check `.claude/epics/archived/` directory listing
  - For each candidate: glob/grep to verify not already implemented
- Subagent returns a compact summary (~20 lines) to main context
- Main context applies ranking multipliers from interview answers

### Phase 3: Output (Enhanced)
- Separate "Backlog Items" from "Strategic Opportunities"
- Add "Filtered out (already shipped)" transparency section
- Keep existing ranking multiplier tables (Mode, Scope)
- Each recommendation 3-5 lines with actionable next command

### Single-File Change
The entire redesign is a rewrite of `.claude/commands/pm/next.md`. No other files are
modified. This is a self-contained change with no cross-module dependencies.

## Test Strategy Preview

### Manual Testing (Primary)
1. Run `/pm:next` → verify AskUserQuestion renders as interactive form
2. Answer all 3 questions → verify Phase 2 runs only after answers
3. Check recommendations → verify no shipped features appear
4. Check strategic suggestions → verify at least 1 genuinely novel idea
5. Observe context usage → verify <15% consumed by Phase 2

### Negative Testing
1. Run `/pm:next` with all PRDs shipped → verify strategic suggestions still appear
2. Run `/pm:next` with "Fix and harden" mode → verify different output than "Build new features"
3. Verify "Filtered out" section correctly lists shipped features

### No Automated Tests
This is a `.claude/commands/` markdown file — no pytest, no mypy. Testing is manual
invocation of the slash command. The command's `allowed-tools` frontmatter is the only
machine-parseable contract.
