# Task: OpenBB Service Layer

## Description

Create `services/openbb_service.py` — the core service class wrapping the OpenBB Platform SDK. Follows the exact same patterns as existing services (`MarketDataService`, `FredService`): class-based DI, async wrapping via `asyncio.to_thread()`, cache-first, never-raises contract, rate limiting.

**Before writing code**: Use Context7 to verify exact OpenBB SDK method signatures and return types for:
- `obb.equity.fundamental.overview(symbol, provider="yfinance")`
- `obb.equity.darkpool.otc(symbol, provider="stockgrid")` (or equivalent for unusual flow)
- `obb.news.world(query, provider="yfinance")`

## Technical Details

### Class Structure

```python
class OpenBBService:
    def __init__(
        self,
        config: OpenBBConfig,
        cache: ServiceCache,
        limiter: RateLimiter,
    ) -> None:
        self._config = config
        self._cache = cache
        self._limiter = limiter
        self._obb = _get_obb()  # Returns obb module or None (guarded import)

    async def fetch_fundamentals(self, ticker: str) -> FundamentalSnapshot | None: ...
    async def fetch_unusual_flow(self, ticker: str) -> UnusualFlowSnapshot | None: ...
    async def fetch_news_sentiment(self, ticker: str, limit: int = 10) -> NewsSentimentSnapshot | None: ...
    async def close(self) -> None: ...
```

### Guarded Import Pattern

```python
def _get_obb() -> Any | None:
    try:
        from openbb import obb
        return obb
    except ImportError:
        logger.info("OpenBB SDK not installed — OpenBB features disabled")
        return None
```

### Async Wrapping (CRITICAL — same pattern as `_yf_call()`)

```python
async def _obb_call(self, fn: Callable[..., T], *args: object, **kwargs: object) -> T:
    """Wrap sync OpenBB call: to_thread + wait_for."""
    return await asyncio.wait_for(
        asyncio.to_thread(fn, *args, **kwargs),
        timeout=self._config.request_timeout,
    )
```

**CRITICAL**: Pass callable + args separately: `to_thread(fn, *args)` NOT `to_thread(fn())`.

### VADER Sentiment Scoring

```python
def _score_sentiment(self, text: str) -> float:
    """Score headline text using VADER. Returns compound score -1.0 to 1.0."""
    try:
        from vaderSentiment.vaderSentiment import SentimentIntensityAnalyzer
        analyzer = SentimentIntensityAnalyzer()
        return analyzer.polarity_scores(text)["compound"]
    except ImportError:
        return 0.0  # Neutral fallback
```

Cache the analyzer instance on `self` to avoid repeated instantiation.

### Cache Key Pattern

- `"openbb:fundamentals:{ticker}"` — TTL: `config.fundamentals_cache_ttl`
- `"openbb:flow:{ticker}"` — TTL: `config.flow_cache_ttl`
- `"openbb:news:{ticker}"` — TTL: `config.news_cache_ttl`

### Never-Raises Contract

Every public method wraps its body in:
```python
try:
    ...
except Exception:
    logger.warning("OpenBB %s fetch failed for %s", data_type, ticker, exc_info=True)
    return None
```

### Edge Cases
- OpenBB SDK not installed → all methods return `None`, no crash
- vaderSentiment not installed → sentiment scores default to 0.0
- SDK returns empty/partial data → graceful mapping with `None` fields
- Timeout during fetch → caught, returns `None`
- Network error → caught, returns `None`
- Config `enabled=False` → early return `None` without SDK call
- Config `fundamentals_enabled=False` → `fetch_fundamentals()` returns `None`

### Mocking Strategy
- Mock `_get_obb()` return value to simulate SDK present/absent
- Mock `asyncio.to_thread` to return pre-built SDK response objects
- Mock `ServiceCache.get()` / `ServiceCache.set()` for cache tests
- Mock `SentimentIntensityAnalyzer` for VADER tests
- Use `monkeypatch` for import-level mocking
