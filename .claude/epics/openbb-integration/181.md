# Task: MarketContext Extension + Debate Wiring

## Description

Extend `MarketContext` with ~13 optional fields for OpenBB data, add `enrichment_ratio()` method, extend `build_market_context()` in the orchestrator to accept OpenBB data, and wire parallel OpenBB fetches into the debate path.

**Key decision**: OpenBB fields are excluded from the existing `completeness_ratio()` to avoid penalizing debates when OpenBB is disabled. A new `enrichment_ratio()` method tracks OpenBB data completeness separately.

## Acceptance Criteria

- [ ] `MarketContext` extended with ~13 optional fields (all `float | None = None` or `str | None = None` or `list[str] | None = None`)
- [ ] New fields covered by existing `validate_optional_finite` validator (float fields)
- [ ] `enrichment_ratio() -> float` method added to MarketContext
- [ ] Existing `completeness_ratio()` unchanged — returns same values as before
- [ ] `build_market_context()` signature extended with `fundamentals`, `flow`, `sentiment` kwargs (all default `None`)
- [ ] Debate path (`run_debate()` or `_debate_single()`) creates `OpenBBService`, fetches in parallel via `asyncio.gather()`, passes results to `build_market_context()`
- [ ] Debate still works identically when OpenBB is disabled or unavailable
- [ ] ~15 tests covering new fields, enrichment_ratio, wiring
- [ ] `ruff check .`, `mypy --strict src/`, `pytest tests/ -v` all pass

### `enrichment_ratio()` Method

```python
def enrichment_ratio(self) -> float:
    """Fraction of OpenBB enrichment fields populated (0.0-1.0)."""
    enrichment_fields: list[object] = [
        self.pe_ratio, self.forward_pe, self.peg_ratio,
        self.price_to_book, self.debt_to_equity, self.revenue_growth,
        self.profit_margin, self.net_call_premium, self.net_put_premium,
        self.options_put_call_ratio, self.news_sentiment,
    ]
    populated = sum(1 for f in enrichment_fields if f is not None)
    return populated / len(enrichment_fields) if enrichment_fields else 0.0
```

### Debate Path Wiring

In the debate orchestrator (where `run_debate()` or `_debate_single()` is called), add parallel OpenBB fetches:

```python
# Fetch OpenBB data in parallel (never raises — returns None on failure)
fundamentals, flow, sentiment = await asyncio.gather(
    openbb_service.fetch_fundamentals(ticker),
    openbb_service.fetch_unusual_flow(ticker),
    openbb_service.fetch_news_sentiment(ticker),
)
# Pass to build_market_context
ctx = build_market_context(..., fundamentals=fundamentals, flow=flow, sentiment=sentiment)
```

The `OpenBBService` instance should be created in the CLI/API entry point and passed through. If `OpenBBConfig.enabled` is `False`, skip creating the service entirely.
