# Task: Health Checks + Integration Testing

The health check follows the exact same pattern as existing checks (yfinance, FRED, Groq, CBOE): never-raises, returns `HealthStatus`, measures latency.

## Technical Details

### `check_openbb()` in `services/health.py`

```python
async def check_openbb(self) -> HealthStatus:
    """Check OpenBB SDK availability. Never raises."""
    start = time.monotonic()
    try:
        from openbb import obb
        # Simple probe — check SDK is importable and functional
        await asyncio.wait_for(
            asyncio.to_thread(lambda: obb.equity.price.quote("SPY", provider="yfinance")),
            timeout=self._config.yfinance_timeout,
        )
        latency_ms = (time.monotonic() - start) * 1000
        return HealthStatus(
            service_name="openbb",
            available=True,
            latency_ms=latency_ms,
            checked_at=datetime.now(UTC),
        )
    except ImportError:
        latency_ms = (time.monotonic() - start) * 1000
        return HealthStatus(
            service_name="openbb",
            available=False,
            latency_ms=latency_ms,
            error="OpenBB SDK not installed",
            checked_at=datetime.now(UTC),
        )
    except Exception as exc:
        latency_ms = (time.monotonic() - start) * 1000
        logger.warning("OpenBB health check failed: %s", exc)
        return HealthStatus(
            service_name="openbb",
            available=False,
            latency_ms=latency_ms,
            error=str(exc),
            checked_at=datetime.now(UTC),
        )
```

### Integration Test Approach

Integration tests use fully mocked external dependencies but exercise the full debate wiring:

1. Mock `OpenBBService` to return fixture snapshots
2. Mock Groq via `TestModel` (PydanticAI)
3. Call `run_debate()` (or `_debate_single()`)
4. Assert `MarketContext` contains OpenBB fields
5. Assert prompt rendering includes fundamental/flow/sentiment sections
6. Assert debate result is valid `DebateResult`

### Edge Cases
- OpenBB SDK not installed → health returns unavailable, debate works normally
- Health check timeout → returns unavailable with measured latency
- Partial OpenBB data → debate context has mix of populated and None fields
- All OpenBB fetches fail → debate identical to pre-integration

### Mocking Strategy
- Mock `openbb.obb` import for health check tests
- Mock `OpenBBService` methods for integration tests (return fixture models)
- Use `TestModel` for PydanticAI agent runs
- Mock `asyncio.to_thread` for the lambda probe call
- Use `monkeypatch` to simulate SDK absent (`builtins.__import__` or `unittest.mock.patch`)
