# Task: Three-Tier Greeks Resolution

## Description

Update `compute_greeks()` in `scoring/contracts.py` to implement three-tier Greeks resolution:
- **Tier 1**: Contract already has `greeks` (from CBOE native) → use as-is, skip `pricing/dispatch.py`
- **Tier 2**: Contract has no `greeks` → compute via `pricing/dispatch.py` (existing behavior)
- **Tier 3**: Local computation fails → `greeks=None`, contract excluded from delta selection

Also set `greeks_source` on `OptionContract` via `model_copy(update={...})` to track provenance.

## Acceptance Criteria

- [ ] When `contract.greeks is not None`: skip dispatch, preserve existing Greeks, set `greeks_source=GreeksSource.MARKET` if not already set
- [ ] When `contract.greeks is None`: compute via `pricing/dispatch.py` (existing behavior), set `greeks_source=GreeksSource.COMPUTED`
- [ ] When local computation fails: `greeks=None`, contract excluded (existing behavior)
- [ ] `greeks_source` field populated on all contracts returned by `compute_greeks()`
- [ ] No behavioral change when all contracts have `greeks=None` (yfinance-only path unchanged)
- [ ] `recommend_contracts()` pipeline works end-to-end with CBOE-provided Greeks

### Edge Cases
- All contracts have Greeks (pure CBOE path) → dispatch never called
- All contracts have `greeks=None` (pure yfinance path) → identical to current behavior
- Mixed: some with Greeks, some without → Tier 1 + Tier 2 in same batch
- Contract with `greeks` set but `greeks_source=None` → set to `MARKET`
- Contract with `greeks` set and `greeks_source=COMPUTED` → preserve (don't overwrite)
