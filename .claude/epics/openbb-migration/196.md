# Task: CBOEChainProvider Implementation

## Description

Implement `CBOEChainProvider` in a new file `services/cboe_provider.py` using OpenBB Platform SDK's `obb.equity.options.chains(provider="cboe")`. Maps CBOE response fields to `OptionContract` including native Greeks (delta, gamma, theta, vega) and bid/ask IV. Uses guarded import pattern from `openbb_service.py`.

**CRITICAL**: Before implementing, use Context7 to verify exact CBOE field names returned by `obb.equity.options.chains(provider="cboe")`. The field mapping in the PRD is estimated and may need adjustment.

## Acceptance Criteria

- [ ] `CBOEChainProvider` class in `services/cboe_provider.py` implements `ChainProvider` protocol
- [ ] Uses `_get_obb()` guarded import pattern — `available` property returns `False` if SDK missing
- [ ] Maps CBOE fields to `OptionContract` including `greeks`, `bid_iv`, `ask_iv`, `greeks_source`
- [ ] `greeks_source = GreeksSource.MARKET` when CBOE provides Greeks
- [ ] Greeks sanity-checked: `|delta| <= 1.0`, `gamma >= 0`, `vega >= 0`
- [ ] Async wrapping via `asyncio.to_thread()` + `asyncio.wait_for(timeout)` pattern
- [ ] Raises `DataSourceUnavailableError` on SDK failure (not swallowed)
- [ ] Caching with `"cboe:chain:{ticker}:{exp}"` key format (no collision with yfinance)
- [ ] Separate `"cboe:expirations:{ticker}"` cache key
- [ ] `services/__init__.py` updated to re-export `CBOEChainProvider`

### Implementation Approach

1. **Context7 verification** (first step):
   - `resolve-library-id` for OpenBB
   - `query-docs` for `obb.equity.options.chains` response schema
   - Verify field names: strike, bid, ask, volume, open_interest, implied_volatility, delta, gamma, theta, vega, bid_iv, ask_iv

2. **Guarded import** (reuse pattern from `openbb_service.py`):
   ```python
   def _get_obb() -> Any:
       try:
           from openbb import obb
           return obb
       except ImportError:
           return None
   ```

3. **Class structure**:
   ```python
   class CBOEChainProvider:
       def __init__(self, config: OpenBBConfig, cache: ServiceCache, limiter: RateLimiter) -> None:
           self._config = config
           self._cache = cache
           self._limiter = limiter
           self._obb = _get_obb()

       @property
       def available(self) -> bool:
           return self._obb is not None

       async def fetch_expirations(self, ticker: str) -> list[date]: ...
       async def fetch_chain(self, ticker: str, expiration: date) -> list[OptionContract]: ...
   ```

4. **Field mapping** (verify via Context7):
   - `_map_cboe_row()` converts CBOE response row → `OptionContract`
   - Uses `safe_decimal()`, `safe_float()`, `safe_int()` from `helpers.py`
   - Constructs `OptionGreeks` when Greeks fields are non-None and pass sanity checks
   - Sets `greeks_source = GreeksSource.MARKET` when Greeks are populated
   - Sets `bid_iv` and `ask_iv` when available

5. **Async wrapping**:
   ```python
   async def _obb_call(self, fn, *args, **kwargs):
       return await asyncio.wait_for(
           asyncio.to_thread(fn, *args, **kwargs),
           timeout=self._config.request_timeout,
       )
   ```

### Greek Sanity Checks
When CBOE returns Greeks, validate before constructing `OptionGreeks`:
- `|delta| <= 1.0` — reject otherwise (fall through to Tier 2)
- `gamma >= 0` — reject otherwise
- `vega >= 0` — reject otherwise
- `theta` and `rho` — finite check only (theta is normally negative)
- If any Greek fails sanity, set `greeks=None` and `greeks_source=None` (let Tier 2 handle)

### Edge Cases
- CBOE returns empty chain → empty list
- CBOE returns Greeks for some strikes but not others → mixed greeks/None
- CBOE bid_iv present but ask_iv missing → market_iv uses single IV
- All CBOE Greeks fail sanity → contracts have greeks=None (Tier 2 fallback in scoring)

### Mocking Strategy
- `patch("options_arena.services.cboe_provider._get_obb")` returning mock with `.equity.options.chains()` method
- Mock returns a mock OBBject with `.to_df()` producing a DataFrame with CBOE columns
- SDK unavailable: `_get_obb()` returns `None`
- SDK failure: mock raises `Exception` → mapped to `DataSourceUnavailableError`
