### Integration Test Design

Tests simulate the full data flow:
```
OptionsDataService(openbb_config=...) →
  CBOEChainProvider.fetch_chain() [mocked] →
    OptionContract with greeks, bid_iv, ask_iv, greeks_source=MARKET →
      scoring/contracts.py recommend_contracts() →
        compute_greeks() Tier 1 (skip dispatch) →
          select_by_delta() with native delta →
            Recommended contract
```

Also test the fallback path:
```
CBOEChainProvider fails →
  YFinanceChainProvider succeeds →
    OptionContract with greeks=None →
      compute_greeks() Tier 2 (local dispatch) →
        select_by_delta() →
          Recommended contract
```

### Cutover Considerations
- Changing default to `True` means CBOE is tried first for all users with OpenBB installed
- Users without OpenBB installed: zero change (CBOE provider not registered)
- `chain_validation_mode` stays `False` by default (validation is opt-in)
- Document in PR description: how to disable CBOE if issues arise (`ARENA_OPENBB__CBOE_CHAINS_ENABLED=false`)

### Edge Cases
- No OpenBB SDK installed → identical to pre-migration (pure yfinance)
- `cboe_chains_enabled=False` env override → identical to pre-migration
- CBOE returns contracts with partial Greeks → scoring handles Tier 1 + Tier 2 mix
- Batch operations (`fetch_chain_all_expirations`) → fallback per-expiration

- Mock `CBOEChainProvider._get_obb()` to simulate SDK present/absent
- Mock CBOE responses with realistic DataFrame data (verified field names)

- Use `monkeypatch.setenv` for config override tests
