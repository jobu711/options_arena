# Task: API debate enrichment fetch

## Description

Fetch OpenBB enrichment data (fundamentals, flow, sentiment) in the API debate background tasks before calling `run_debate_v2()`, so browser-initiated debates have enriched agent context.

## Acceptance Criteria

- [ ] `_run_debate_background()` fetches enrichment via `asyncio.gather()` before `run_debate_v2()` call
- [ ] `_run_batch_debate_background()` fetches enrichment per-ticker before each `run_debate_v2()` call
- [ ] Enrichment snapshots passed as `fundamentals=`, `flow=`, `sentiment=` kwargs to `run_debate_v2()`
- [ ] OpenBB service accessed from `request.app.state.openbb` (set by Task 186)
- [ ] When `app.state.openbb` is `None`, all enrichment params are `None` (no error)
- [ ] OpenBB fetch failures (returns `None`) do not crash the debate

## Technical Details

### Files to modify:
- `src/options_arena/api/routes/debate.py` — Both background task functions

### In `_run_debate_background()` (~line 105):
```python
# Fetch OpenBB enrichment (before run_debate_v2 call)
openbb_svc: OpenBBService | None = request.app.state.openbb
fundamentals = None
flow = None
sentiment = None
if openbb_svc is not None:
    fundamentals, flow, sentiment = await asyncio.gather(
        openbb_svc.fetch_fundamentals(ticker),
        openbb_svc.fetch_unusual_flow(ticker),
        openbb_svc.fetch_news_sentiment(ticker),
    )

result = await run_debate_v2(
    ...,  # existing params
    fundamentals=fundamentals,
    flow=flow,
    sentiment=sentiment,
)
```

### In `_run_batch_debate_background()` (~line 269):
Same pattern inside the per-ticker loop, before each `run_debate_v2()` call.

### Key rules:
- Never-raises: OpenBB fetch methods already return `None` on error
- No try/except needed around `asyncio.gather()` — the service handles errors internally
- Access `request.app.state.openbb` (not via Depends — background tasks don't support DI)
