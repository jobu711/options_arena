# Task: Enums & exception hierarchy

## Description
Implement all 9 `StrEnum` types in `enums.py` and the domain exception hierarchy in `utils/exceptions.py`. These are the lowest-level building blocks — every model and service imports from here.

## Acceptance Criteria
- [ ] `src/options_arena/models/enums.py` contains all 9 enums:
  - `OptionType` (CALL, PUT)
  - `PositionSide` (LONG, SHORT)
  - `SignalDirection` (BULLISH, BEARISH, NEUTRAL)
  - `ExerciseStyle` (AMERICAN, EUROPEAN)
  - `PricingModel` (BSM, BAW)
  - `MarketCapTier` (MEGA, LARGE, MID, SMALL, MICRO)
  - `DividendSource` (FORWARD, TRAILING, COMPUTED, NONE)
  - `SpreadType` (VERTICAL, CALENDAR, IRON_CONDOR, STRADDLE, STRANGLE, BUTTERFLY)
  - `GreeksSource` (COMPUTED, MARKET)
- [ ] All enums inherit from `StrEnum` (Python 3.13 `enum.StrEnum`)
- [ ] All values are lowercase strings
- [ ] `src/options_arena/utils/__init__.py` exists (empty or with re-exports)
- [ ] `src/options_arena/utils/exceptions.py` contains:
  - `DataFetchError(Exception)` — base for all data errors
  - `TickerNotFoundError(DataFetchError)`
  - `InsufficientDataError(DataFetchError)`
  - `DataSourceUnavailableError(DataFetchError)`
  - `RateLimitExceededError(DataFetchError)`
- [ ] Each exception accepts a descriptive message
- [ ] `uv run ruff check src/options_arena/models/enums.py` passes
- [ ] `uv run mypy src/options_arena/models/enums.py --strict` passes

## Technical Details
- Import `StrEnum` from `enum`, not from `typing`
- No business logic in enums — pure data definitions
- Exception classes need `__init__` only if adding structured fields beyond `message`
- Follow models/CLAUDE.md patterns exactly
