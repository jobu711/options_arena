# Task: Unit tests — enums, exceptions & config (~45 tests)

## Description
Write comprehensive unit tests for enums, the exception hierarchy, and AppSettings configuration. These are the simplest models to test and can start as soon as enums + config are implemented.

## Acceptance Criteria
- [ ] `tests/unit/models/test_enums.py` (~25 tests):
  - Each enum: member count matches expected (OptionType=2, SpreadType=6, DividendSource=4, etc.)
  - Each enum: all values are lowercase strings
  - Each enum: is a `StrEnum` subclass
  - Each enum: exhaustive iteration matches expected members
  - Each enum: string serialization works (`str(OptionType.CALL) == "call"`)
- [ ] `tests/unit/test_exceptions.py` (~5 tests):
  - `DataFetchError` is base Exception subclass
  - Each subclass (`TickerNotFoundError`, etc.) inherits from `DataFetchError`
  - Exception message accessible via `str(err)`
  - `isinstance` checks work correctly across hierarchy
- [ ] `tests/unit/models/test_config.py` (~15 tests):
  - `AppSettings()` with no args succeeds, all defaults correct
  - `settings.scan.top_n == 50` (default)
  - `settings.pricing.delta_target == 0.35` (default)
  - `settings.service.ollama_host == "http://localhost:11434"` (default)
  - Env var override: monkeypatch `ARENA_SCAN__TOP_N=30`, assert `settings.scan.top_n == 30`
  - Nested delimiter: `ARENA_PRICING__DELTA_TARGET=0.40` works
  - Type coercion: string `"30"` from env correctly parsed to `int`
  - Sub-configs are `BaseModel` instances, not `BaseSettings`
  - Constructor override: `AppSettings(scan=ScanConfig(top_n=25))` works
- [ ] All tests pass: `uv run pytest tests/unit/models/test_enums.py tests/unit/models/test_config.py tests/unit/test_exceptions.py -v`
- [ ] `conftest.py` exists in `tests/` with any shared fixtures

## Technical Details
- Use `monkeypatch.setenv` for env var tests (not `os.environ` directly)
- Use `pytest.raises(ValidationError)` for validation boundary tests
- Each test function tests ONE behavior — no mega-tests
- Name tests descriptively: `test_option_type_has_exactly_two_members`
