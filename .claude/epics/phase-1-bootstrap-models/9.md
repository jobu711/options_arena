# Task: Options models (OptionGreeks, OptionContract, SpreadLeg, OptionSpread)

## Description
Implement `options.py` with four Pydantic v2 models for options data. `OptionGreeks` tracks which pricing model produced the values. `OptionContract` carries exercise style, market IV, and computed fields for mid/spread/DTE.

## Acceptance Criteria
- [ ] `src/options_arena/models/options.py` contains `OptionGreeks`, `OptionContract`, `SpreadLeg`, `OptionSpread`
- [ ] `OptionGreeks`: frozen=True, fields: `delta`, `gamma`, `theta`, `vega`, `rho` (all float), `pricing_model: PricingModel`
  - `field_validator` on delta: must be in [-1.0, 1.0]
  - `field_validator` on gamma, vega: must be >= 0
- [ ] `OptionContract`: frozen=True, fields per models/CLAUDE.md spec including:
  - `exercise_style: ExerciseStyle`
  - `market_iv: float` (yfinance impliedVolatility passthrough)
  - `greeks: OptionGreeks | None = None`
  - Computed fields: `mid` (Decimal("2") divisor), `spread`, `dte`
  - `field_serializer` for all Decimal fields
- [ ] `SpreadLeg`: fields for individual spread components
- [ ] `OptionSpread`: combines multiple `SpreadLeg` instances with a `SpreadType`
- [ ] All validation raises `ValidationError` with clear messages
- [ ] Passes `ruff check` and `mypy --strict`

## Technical Details
- `mid` computed field divides by `Decimal("2")`, not int `2`
- `dte` uses `date.today()` — returns int days until expiration
- `greeks` is always `None` when coming from yfinance; populated after local computation
- `market_iv` is the yfinance IV passed through for use as solver seed
- `exercise_style` drives pricing dispatch (AMERICAN → BAW, EUROPEAN → BSM)
- SpreadLeg/OptionSpread may be simpler — define the shape now, flesh out in later phases
