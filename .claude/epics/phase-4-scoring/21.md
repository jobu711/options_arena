## Technical Details

### CLAUDE.md must cover:
- Module purpose: percentile-rank normalization, composite scoring, direction classification, contract filtering with Greeks dispatch
- File listing: `normalization.py`, `composite.py`, `direction.py`, `contracts.py`, `__init__.py`
- Architecture rules:
  - `normalization.py` and `composite.py` operate on `IndicatorSignals` typed model (NOT `dict[str, float]`)
  - `direction.py` returns `SignalDirection` enum
  - `contracts.py` calls `pricing/dispatch.py` for ALL Greeks — never computes locally
  - All thresholds from `ScanConfig` / `PricingConfig` — no hardcoded magic numbers
  - No API calls — receives pre-fetched data from services layer
- Key constants section (weights, thresholds, inverted indicators)
- Cherry-pick guidance: normalization, composite, direction from v3; contracts is a rewrite
- Integration rules with `models/`, `pricing/`, `indicators/`

### conftest.py should provide:
- Shared fixtures: sample `IndicatorSignals`, sample `OptionContract` list, sample `PricingConfig`/`ScanConfig`
- Factory functions for creating test data
