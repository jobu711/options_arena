# Task: Implement direction.py with signal aggregation and tiebreaker

## Description
Cherry-pick `direction.py` from v3 (`Option_Alpha/analysis/direction.py`) with minimal changes. Classifies tickers as BULLISH, BEARISH, or NEUTRAL based on ADX trend strength, RSI momentum, and SMA alignment, with SMA as tiebreaker.

## Acceptance Criteria
- [ ] `determine_direction()` returns `SignalDirection` (BULLISH / BEARISH / NEUTRAL)
- [ ] ADX below threshold → NEUTRAL regardless of other signals
- [ ] RSI-based scoring: overbought/oversold → strong signal, midpoint deviation → mild signal
- [ ] SMA alignment scoring: above threshold → bullish, below → bearish
- [ ] Tiebreaker: when bullish_score == bearish_score and both > 0, SMA direction decides
- [ ] All thresholds from `ScanConfig` parameters (not hardcoded)
- [ ] Module constants as defaults matching v3 values
- [ ] ~10 unit tests

## Technical Details

### Functions to implement:
1. **`determine_direction(adx: float, rsi: float, sma_alignment: float, config: ScanConfig | None = None) -> SignalDirection`**
   - Step 1: If `adx < config.adx_trend_threshold` (default 15.0), return `NEUTRAL`
   - Step 2: Score RSI signals:
     - `rsi > config.rsi_overbought` (70): bullish_score += 2
     - `rsi > RSI_MIDPOINT` (50): bullish_score += 1
     - `rsi < config.rsi_oversold` (30): bearish_score += 2
     - `rsi < RSI_MIDPOINT` (50): bearish_score += 1
   - Step 3: Score SMA signals:
     - `sma_alignment > SMA_BULLISH_THRESHOLD` (0.5): bullish_score += 1
     - `sma_alignment < SMA_BEARISH_THRESHOLD` (-0.5): bearish_score += 1
   - Step 4: Compare scores:
     - `bullish_score > bearish_score` → BULLISH
     - `bearish_score > bullish_score` → BEARISH
     - Tied and both > 0 → tiebreaker: `sma_alignment > 0` → BULLISH, else BEARISH
     - Both == 0 → NEUTRAL

### Module constants (defaults):
```python
RSI_MIDPOINT: float = 50.0
SMA_BULLISH_THRESHOLD: float = 0.5
SMA_BEARISH_THRESHOLD: float = -0.5
```

### Key differences from v3:
- v3 hardcodes `ADX_TREND_THRESHOLD = 15.0`, `RSI_OVERBOUGHT = 70.0`, `RSI_OVERSOLD = 30.0`
- Arena accepts `ScanConfig` for these thresholds, with same defaults
- Import path changes only

### Test cases (~10):
- Strong trend + overbought RSI + bullish SMA → BULLISH
- Strong trend + oversold RSI + bearish SMA → BEARISH
- Weak trend (ADX=10) → NEUTRAL regardless
- ADX exactly at threshold (15.0) → NEUTRAL (< not <=)
- Tiebreaker: RSI=50 (mild both), SMA=0.6 → BULLISH (SMA decides)
- Tiebreaker: RSI=50, SMA=-0.6 → BEARISH (SMA decides)
- No signals: ADX=20, RSI=50, SMA=0.0 → NEUTRAL (both scores = 0)
- Edge: RSI exactly 70 → bullish (>= boundary check)
- Edge: RSI exactly 30 → bearish (<= boundary check)
- Custom config: `ScanConfig(adx_trend_threshold=25)` → ADX=20 now NEUTRAL
