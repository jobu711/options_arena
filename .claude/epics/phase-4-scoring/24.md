# Task: Implement contracts.py with Greeks dispatch and contract selection

## Description
**Rewrite** (not cherry-pick) `contracts.py` using v3 as reference. This is the most complex file in the scoring module. It filters option contracts by liquidity, DTE, and spread, computes Greeks for all surviving contracts via `pricing/dispatch.py`, then selects the best contract by delta proximity. All thresholds come from `PricingConfig`.

## Acceptance Criteria
- [ ] `filter_contracts()` filters by OI, volume, spread, and option type (call for BULLISH, put for BEARISH)
- [ ] Zero-bid exemption: contracts with bid=0 but ask>0 skip spread check
- [ ] `select_expiration()` picks expiration closest to target DTE within [dte_min, dte_max]
- [ ] `compute_greeks()` calls `pricing/dispatch.py` for each contract — BAW for AMERICAN, BSM for EUROPEAN
- [ ] IV seeding: uses `market_iv` from yfinance when available; falls back to ATM IV estimate
- [ ] `select_by_delta()` implements primary [0.20, 0.50] + fallback [0.10, 0.80] ranges
- [ ] Closest-to-target (0.35) selection within whichever range has candidates
- [ ] `recommend_contracts()` runs full pipeline: filter → select expiration → compute Greeks → select by delta
- [ ] Accepts `exercise_style`, `dividend_yield`, `risk_free_rate`, `spot_price` as parameters
- [ ] All thresholds from `PricingConfig` — no hardcoded constants
- [ ] ~40 unit tests

## Technical Details

### Functions to implement:

1. **`filter_contracts(contracts: list[OptionContract], direction: SignalDirection, config: PricingConfig | None = None) -> list[OptionContract]`**
   - Filter by `option_type`: CALL for BULLISH, PUT for BEARISH, both for NEUTRAL
   - OI >= `config.min_oi` (default 100)
   - Volume >= `config.min_volume` (default 1)
   - Spread check: `spread / mid <= config.max_spread_pct` (default 0.10)
   - **Zero-bid exemption**: if `bid == 0` and `ask > 0`, skip spread check entirely
   - Reject contracts where both bid AND ask are 0 (truly dead)

2. **`select_expiration(contracts: list[OptionContract], config: PricingConfig | None = None) -> date | None`**
   - Filter contracts to those with DTE in [`config.dte_min`, `config.dte_max`] (default [30, 60])
   - Pick expiration closest to midpoint: `(dte_min + dte_max) / 2`
   - Return `None` if no contracts in DTE range

3. **`compute_greeks(contracts: list[OptionContract], spot: float, risk_free_rate: float, dividend_yield: float) -> list[OptionContract]`**
   - For each contract:
     - `T = contract.dte / 365.0`
     - `K = float(contract.strike)`
     - `sigma = contract.market_iv` (yfinance IV as seed)
     - If `market_iv <= 0` or `market_iv > 5.0`: attempt IV solve via `pricing.option_iv()` using mid price; if that fails, skip contract
     - Compute Greeks: `pricing.option_greeks(contract.exercise_style, spot, K, T, risk_free_rate, dividend_yield, sigma, contract.option_type)`
     - Return new `OptionContract` with `.greeks` populated (contracts are frozen, so create new instances)
   - Skip contracts where Greeks computation raises (log warning, don't crash batch)

4. **`select_by_delta(contracts: list[OptionContract], config: PricingConfig | None = None) -> OptionContract | None`**
   - All contracts must have `.greeks` populated (filter out None)
   - Primary range: `abs(delta)` in [`config.delta_primary_min`, `config.delta_primary_max`] (default [0.20, 0.50])
   - If primary range has candidates: pick closest to `config.delta_target` (default 0.35)
   - Fallback range: `abs(delta)` in [`config.delta_fallback_min`, `config.delta_fallback_max`] (default [0.10, 0.80])
   - If fallback has candidates: pick closest to target
   - If nothing: return `None`

5. **`recommend_contracts(contracts: list[OptionContract], direction: SignalDirection, spot: float, risk_free_rate: float, dividend_yield: float, config: PricingConfig | None = None) -> list[OptionContract]`**
   - Pipeline: `filter_contracts()` → `select_expiration()` → filter to that expiration → `compute_greeks()` → `select_by_delta()`
   - Returns list (0 or 1 contracts for now; future: multiple expirations)

### Key differences from v3:
| Aspect | v3 | Arena |
|--------|-----|-------|
| Spot price | Estimated via median strike | Passed as parameter (real spot from yfinance) |
| Greeks source | Local BSM fallback only | `pricing/dispatch.py` (BAW for American, BSM for European) |
| Delta range | Single range | Primary [0.20, 0.50] + fallback [0.10, 0.80] |
| Config | Hardcoded constants | `PricingConfig` injected |
| Dividend yield | Not used | Passed as `q` to pricing |
| Exercise style | Not tracked | Used for dispatch routing |
| IV handling | Direct use of yfinance IV | Validate/re-solve IV when suspect |

### Test cases (~40):

**filter_contracts (~10):**
- BULLISH → only calls pass
- BEARISH → only puts pass
- NEUTRAL → both pass
- OI below threshold → rejected
- Volume below threshold → rejected
- Wide spread → rejected
- Zero-bid exemption: bid=0, ask=1.50 → passes (skip spread check)
- Both bid and ask zero → rejected
- Config override: custom `max_spread_pct=0.05` → stricter filtering
- Empty input → empty output

**select_expiration (~5):**
- Contracts at DTE 30, 45, 60 → picks 45 (closest to midpoint)
- No contracts in DTE range → returns None
- All contracts at same DTE → returns that DTE
- Edge: DTE exactly at min (30) → included
- Edge: DTE exactly at max (60) → included

**compute_greeks (~10):**
- Valid contract with market_iv → Greeks computed via dispatch
- American exercise → BAW Greeks (pricing_model = BAW)
- European exercise → BSM Greeks (pricing_model = BSM)
- market_iv = 0 → IV solve attempted via mid price
- IV solve fails → contract skipped (not crashed)
- Multiple contracts → batch processing, partial failure OK
- Greeks values sanity: delta in [-1, 1], gamma >= 0, vega >= 0
- Deep ITM call → delta near 1.0
- Deep OTM put → delta near 0
- Frozen contract → new instance returned with greeks

**select_by_delta (~8):**
- Contracts with delta 0.25, 0.35, 0.45 → picks 0.35 (exact target)
- No contracts in primary range → falls back to [0.10, 0.80]
- No contracts in fallback range → returns None
- Multiple at same distance from target → deterministic pick (first by strike)
- Put contracts: uses abs(delta) for comparison
- Edge: delta exactly 0.20 → included in primary
- Edge: delta exactly 0.50 → included in primary
- Config override: custom `delta_target=0.40` → picks closest to 0.40

**recommend_contracts pipeline (~7):**
- Happy path: valid contracts → 1 recommendation returned
- No liquid contracts → empty list
- No contracts in DTE range → empty list
- No contracts in delta range → empty list
- Greeks computation fails for all → empty list
- Integration: verify the full pipeline from raw contracts to final pick
- Direction drives type selection correctly through pipeline
