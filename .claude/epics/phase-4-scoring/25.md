# Task: Implement normalization.py with percentile-rank scoring

## Description
Cherry-pick `normalization.py` from v3 (`Option_Alpha/analysis/normalization.py`) and adapt it to use Arena's typed models. Converts raw indicator values to percentile ranks (0–100) with proper tie handling and indicator inversion.

## Acceptance Criteria
- [ ] `percentile_rank_normalize()` converts raw indicator values to 0–100 percentile ranks
- [ ] Tied values receive averaged rank (not arbitrary assignment)
- [ ] Inverted indicators (where lower = better) are flipped: `100 - rank`
- [ ] Universally-missing indicators (all `None` across universe) are skipped and weights renormalized
- [ ] Input/output uses `IndicatorSignals` typed model (NOT `dict[str, float]`)
- [ ] `INVERTED_INDICATORS` frozenset defined as module constant
- [ ] Functions accept `dict[str, IndicatorSignals]` (ticker → signals) and return same shape
- [ ] ~15 unit tests covering all scenarios

## Technical Details

### Functions to implement:
1. **`percentile_rank_normalize(universe: dict[str, IndicatorSignals]) -> dict[str, IndicatorSignals]`**
   - For each indicator field on `IndicatorSignals`, collect non-None values across universe
   - Rank using `scipy.stats.rankdata(method="average")` or equivalent manual ranking
   - Scale to 0–100: `(rank - 1) / (n - 1) * 100` (single ticker = 50.0)
   - Write back to new `IndicatorSignals` instances

2. **`invert_indicators(normalized: dict[str, IndicatorSignals]) -> dict[str, IndicatorSignals]`**
   - For indicators in `INVERTED_INDICATORS`, apply `100.0 - value`
   - `INVERTED_INDICATORS = frozenset({"bb_width", "atr_pct", "relative_volume", "keltner_width"})`

3. **`get_active_indicators(universe: dict[str, IndicatorSignals]) -> set[str]`**
   - Returns set of indicator names that have at least one non-None value across universe
   - Used by composite scoring to skip universally-missing indicators

### Key differences from v3:
- v3 uses `dict[str, dict[str, float]]` → Arena uses `dict[str, IndicatorSignals]`
- v3 field `atr_percent` → Arena field `atr_pct`
- v3 has no universally-missing detection — Arena adds `get_active_indicators()`

### Test cases (~15):
- Basic ranking: 3 tickers, all indicators present → correct percentile ranks
- Tie handling: 3 tickers with same RSI → all get rank 50.0
- Inversion: `bb_width` high value → low percentile after inversion
- Single ticker: all indicators → 50.0 (midpoint)
- Missing data: one ticker missing RSI → rank among non-None only
- Universally missing: all tickers have `iv_rank = None` → remains None
- Empty universe: returns empty dict
- All same values: every ticker has RSI=50 → all get 50.0
- Mixed None/present: partial indicator coverage across tickers
- Edge: 2 tickers only → ranks are 0 and 100
- Inversion preserves None: inverted indicator with None stays None
