# Task: Implement composite.py with weighted geometric mean scoring

## Description
Cherry-pick `scoring.py` composite logic from v3 and adapt to Arena's typed models. Computes weighted geometric mean across indicator categories, producing a single composite score per ticker.

## Acceptance Criteria
- [ ] `composite_score()` computes weighted geometric mean of normalized indicator values
- [ ] Category weights defined as module constant matching PRD: Oscillators 0.27, Trend 0.20, Volatility 0.15, Volume 0.15, MA 0.10, Options 0.13
- [ ] Weight sum validates to 1.0 (with tolerance for float rounding)
- [ ] Floor value `1.0` substituted for percentiles ≤ 0 to avoid `log(0)`
- [ ] Output clamped to `[0.0, 100.0]`
- [ ] Universally-missing indicators (all None across universe) are skipped with weight renormalization
- [ ] `INDICATOR_WEIGHTS` maps indicator field names to `(weight, category)` tuples
- [ ] Input: `dict[str, IndicatorSignals]` (already normalized) → Output: `dict[str, float]` (ticker → composite score)
- [ ] ~15 unit tests

## Technical Details

### Functions to implement:
1. **`composite_score(normalized: dict[str, IndicatorSignals], active_indicators: set[str] | None = None) -> dict[str, float]`**
   - For each ticker, compute: `exp(sum(w_i * ln(max(x_i, 1.0))) / sum(w_i))` over active indicators
   - If `active_indicators` provided, only include those indicators (skip others, renormalize weights)
   - Clamp result to `[0.0, 100.0]`

2. **Module constants:**
   ```python
   INDICATOR_WEIGHTS: dict[str, tuple[float, str]] = {
       "rsi": (0.08, "oscillators"),
       "stochastic_rsi": (0.05, "oscillators"),
       "williams_r": (0.05, "oscillators"),
       # ... all 18 indicators
   }
   ```

3. **`score_universe(universe: dict[str, IndicatorSignals]) -> list[TickerScore]`**
   - Pipeline: `get_active_indicators()` → `percentile_rank_normalize()` → `invert_indicators()` → `composite_score()` → build `TickerScore` list
   - Note: direction classification is NOT done here — that's `direction.py` called separately

### Key differences from v3:
- v3 uses `dict[str, float]` signals → Arena uses `IndicatorSignals` typed model
- v3 `TickerScore.score` → Arena `TickerScore.composite_score`
- v3 `TickerScore.signals: dict[str, float]` → Arena `TickerScore.signals: IndicatorSignals`
- Arena adds `active_indicators` parameter for universally-missing skip

### Weight mapping (18 indicators, 6 categories, sum = 1.0):

| Oscillators | rsi (0.08), stochastic_rsi (0.05), williams_r (0.05) | — | 0.18 |

| Volume | obv (0.05), ad (0.05), relative_volume (0.05) | — | 0.15 |

**Note**: Epic says weights are Oscillators 0.27, Trend 0.20, etc. but v3 code has the individual-indicator weights above (summing to 1.0). Use the v3 individual weights — the epic category totals are approximate groupings. Verify the exact weights match v3 code during implementation.

### Test cases (~15):
- Known values: 3 tickers with known normalized scores → verify geometric mean

- All minimum (1.0 floor): composite ≈ 1.0

- Single ticker: produces valid score
- Empty universe: returns empty list
- Clamping: verify output never exceeds 100.0 or goes below 0.0
- TickerScore construction: verify all fields populated correctly
- score_universe pipeline: end-to-end from raw signals to sorted TickerScore list
