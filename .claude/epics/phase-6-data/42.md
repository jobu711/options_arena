# Task: Implement database.py — Database class with migration runner

## Description
Write `src/options_arena/data/database.py` from scratch. The `Database` class manages the async SQLite connection lifecycle: opening with WAL mode, enabling foreign keys, running sequential numbered migrations from `data/migrations/`, and closing cleanly. This is pure infrastructure — no business logic.

Write tests in `tests/unit/data/test_database.py`.

## Acceptance Criteria
- [ ] `Database` class with DI constructor: `__init__(self, db_path: Path | str, migrations_dir: Path | None = None)`
- [ ] `async def connect()` — opens aiosqlite connection, sets WAL mode, enables foreign keys, creates `schema_version` table, runs pending migrations
- [ ] `async def close()` — closes connection (idempotent, safe to call twice)
- [ ] Property or method to access the live connection (raises `RuntimeError` if not connected)
- [ ] `aiosqlite.Row` set as `row_factory` on the connection after opening
- [ ] Migration runner reads `data/migrations/*.sql` sorted by numeric prefix
- [ ] `schema_version` table created by `connect()` before reading migration files
- [ ] Migrations tracked: skips already-applied versions, applies new ones
- [ ] Each migration executed via `executescript()`
- [ ] Applied migrations recorded in `schema_version` with ISO 8601 timestamp
- [ ] `migrations_dir` defaults to `Path(__file__).resolve().parents[3] / "data" / "migrations"` when `None`
- [ ] Works with `":memory:"` for tests
- [ ] Full type annotations, `mypy --strict` passes
- [ ] `logging` module only (INFO for migrations, DEBUG for lifecycle)

### Tests (`tests/unit/data/test_database.py`) — ~10 tests:
- [ ] WAL mode enabled: `PRAGMA journal_mode` returns `"wal"`
- [ ] Foreign keys enabled: `PRAGMA foreign_keys` returns `1`
- [ ] All 6 tables created after connect (scan_runs, ticker_scores, service_cache, ai_theses, watchlists, watchlist_tickers)
- [ ] `schema_version` table exists and tracks version 1
- [ ] Idempotent connect: calling `connect()` on already-connected Database doesn't re-apply migrations
- [ ] Close is idempotent: calling `close()` twice doesn't raise
- [ ] Operation after close raises `RuntimeError`
- [ ] `:memory:` database works (no file system side effects)
- [ ] Migration applied_at is a valid ISO 8601 datetime string
- [ ] `connect()` → check tables → `close()` full lifecycle

## Technical Details
- Import: `aiosqlite`, `pathlib.Path`, `logging`, `datetime`
- Models imported: none (Database has no model dependencies)
- `executescript()` auto-commits before execution (sqlite3 behavior) — document this
- `row_factory = aiosqlite.Row` enables dict-like column access in Repository
- The `schema_version` table has columns: `version INTEGER PRIMARY KEY`, `applied_at TEXT NOT NULL`
- Migration version extracted from filename: `int(path.stem.split("_")[0])`
