# Task: Module setup — progress.py

## Description
Create `src/options_arena/scan/progress.py` with the three foundational contracts that all other scan module files depend on: `ScanPhase` StrEnum, `CancellationToken` class, and `ProgressCallback` Protocol.

Also verify that `scan/CLAUDE.md` exists (already created) and create `scan/__init__.py` as a placeholder.

## Acceptance Criteria
- [ ] `ScanPhase` StrEnum has exactly 4 members: `UNIVERSE`, `SCORING`, `OPTIONS`, `PERSIST`
- [ ] `CancellationToken` has `cancel()` method and `is_cancelled` property
- [ ] `CancellationToken` is thread-safe via `threading.Event`
- [ ] `ProgressCallback` is a `Protocol` class with `__call__(phase: ScanPhase, current: int, total: int) -> None`
- [ ] `scan/__init__.py` placeholder exists
- [ ] ~10 tests written and passing
- [ ] ruff + mypy --strict pass

## Technical Details
- `ScanPhase` follows project StrEnum convention (see `models/enums.py` for examples)
- `CancellationToken` wraps `threading.Event` — `cancel()` sets the event, `is_cancelled` checks it
- `ProgressCallback` is a `typing.Protocol` so any callable with matching signature satisfies it
- Tests should include: exhaustive member check, cancel/is_cancelled toggling, thread-safety (concurrent cancel from another thread), protocol conformance test, a `RecordingCallback` test helper
