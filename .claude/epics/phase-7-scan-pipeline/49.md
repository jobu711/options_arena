# Task: Pipeline-internal models — models.py

## Description
Create `src/options_arena/scan/models.py` with the 4 pipeline-internal typed models: `UniverseResult`, `ScoringResult`, `OptionsResult`, and `ScanResult`. These are orchestration artifacts that type the boundaries between pipeline phases — they do NOT belong in `models/` (domain-wide data shapes).

## Acceptance Criteria
- [ ] `UniverseResult(BaseModel)` with fields: `tickers: list[str]`, `ohlcv_map: dict[str, list[OHLCV]]`, `sp500_sectors: dict[str, str]`, `failed_count: int`, `filtered_count: int`
- [ ] `ScoringResult(BaseModel)` with fields: `scores: list[TickerScore]`, `raw_signals: dict[str, IndicatorSignals]`
- [ ] `OptionsResult(BaseModel)` with fields: `recommendations: dict[str, list[OptionContract]]`, `risk_free_rate: float`
- [ ] `ScanResult(BaseModel)` with fields: `scan_run: ScanRun`, `scores: list[TickerScore]`, `recommendations: dict[str, list[OptionContract]]`, `risk_free_rate: float`, `cancelled: bool = False`, `phases_completed: int = 0`
- [ ] All models use `model_config = ConfigDict(arbitrary_types_allowed=True)` if needed for pandas/numpy types
- [ ] ~12 tests written and passing
- [ ] ruff + mypy --strict pass

## Technical Details
- These models reference types from `options_arena.models`: `OHLCV`, `TickerScore`, `IndicatorSignals`, `OptionContract`, `ScanRun`
- `ScoringResult.raw_signals` stores RAW indicator values (NOT normalized) — critical for `determine_direction()` which uses absolute thresholds
- `ScanResult.phases_completed` is 0–4, tracking how far the pipeline progressed before completion or cancellation
- `ScanResult.cancelled` defaults to `False`, set to `True` when `CancellationToken` is triggered
- Tests should include: construction with valid data, default values, serialization round-trip, partial results for cancelled pipeline, `phases_completed` range validation
