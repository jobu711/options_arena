# Task: Pipeline Phase 3 + Phase 4

## Description
Implement `_phase_options()` (Phase 3) and `_phase_persist()` (Phase 4) on `ScanPipeline`, plus the `scan/__init__.py` re-exports.

Phase 3 applies liquidity pre-filter, selects top-N, fetches risk-free rate once, fetches option chains concurrently, and calls `recommend_contracts` for each ticker.

Phase 4 builds `ScanRun`, persists to database via `Repository`, and returns the final `ScanResult`.

## Acceptance Criteria
- [ ] `_phase_options()` applies liquidity pre-filter: `avg_dollar_volume >= settings.scan.min_dollar_volume` AND `latest_close >= settings.scan.min_price`
- [ ] Top-N selection by `composite_score` (default 50)
- [ ] `FredService.fetch_risk_free_rate()` called exactly once per scan
- [ ] Per-ticker option chain + ticker info fetch via `asyncio.gather` with `return_exceptions=True`
- [ ] `recommend_contracts()` called with: `direction`, `spot=float(ticker_info.current_price)`, `risk_free_rate`, `dividend_yield=ticker_info.dividend_yield`, `config=settings.pricing`
- [ ] Per-ticker error isolation — one failed ticker doesn't crash the scan
- [ ] `_phase_persist()` builds `ScanRun` with UTC timestamps, calls `repository.save_scan_run()` and `repository.save_ticker_scores()`
- [ ] `scan/__init__.py` re-exports: `CancellationToken`, `ProgressCallback`, `ScanPhase`, `ScanPipeline`, `ScanResult`
- [ ] ~20 tests written and passing
- [ ] ruff + mypy --strict pass

## Technical Details
- Liquidity pre-filter uses OHLCV data from Phase 1 (passed via `UniverseResult.ohlcv_map`):
  - `avg_dollar_volume = mean(close * volume)` over full history (use `Decimal` arithmetic or convert to float)
  - `latest_close = ohlcv_list[-1].close`
- `recommend_contracts` is from `scoring/contracts.py` — returns 0 or 1 contracts per ticker
- Phase 4: `ScanRun(started_at=..., completed_at=now_utc, preset=..., tickers_scanned=..., tickers_scored=..., recommendations=...)` — all datetime fields must be UTC
- `scan/__init__.py` must have `__all__` list
- Tests should mock all services. Verify: liquidity filter thresholds, top-N selection, FRED called once, concurrent chain fetch, recommend_contracts integration, per-ticker error isolation, ScanRun UTC timestamps, repository save calls, scan_run.id populated, cancellation between Phase 3→4, full 4-phase run.
