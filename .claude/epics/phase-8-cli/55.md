# Task: RichProgressCallback + SIGINT Handler

## Description

Implement `RichProgressCallback` class satisfying the `ProgressCallback` protocol from
`scan/progress.py`, mapping pipeline phases to Rich Progress bar tasks. Also implement
`setup_sigint_handler()` with double-press Ctrl+C logic: first press sets `CancellationToken`,
second press raises `SystemExit(130)`.

## Acceptance Criteria

- [ ] `RichProgressCallback` in `progress.py` satisfies `ProgressCallback` protocol
- [ ] Phase-to-description mapping: UNIVERSE→"Fetching universe", SCORING→"Scoring tickers", etc.
- [ ] New phases create new Rich tasks; updates call `progress.update()`
- [ ] `setup_sigint_handler(token, console)` in `commands.py` or separate module
- [ ] First Ctrl+C: `token.cancel()`, print yellow cancellation message
- [ ] Second Ctrl+C: `raise SystemExit(130)` (force exit)
- [ ] Uses `signal.signal()` (not `loop.add_signal_handler()` — Windows incompatible)
- [ ] 4 unit tests
- [ ] ruff + mypy --strict pass

### Implementation Notes
- `RichProgressCallback.__init__(self, progress: Progress)` — stores Rich Progress instance
- `RichProgressCallback.__call__(self, phase: ScanPhase, current: int, total: int)` — creates/updates tasks
- `_task_ids: dict[ScanPhase, int]` — maps phases to Rich task IDs
- Progress bar config: `SpinnerColumn()`, `TextColumn("[progress.description]{task.description}")`, `BarColumn()`, `TextColumn("[progress.percentage]{task.percentage:>3.0f}%")`, `TimeElapsedColumn()`
- Progress console: `Console(stderr=True)` — keeps progress on stderr
- `transient=False` — completed phases stay visible
- SIGINT handler: `signal.signal(signal.SIGINT, handler)` with `nonlocal sigint_count`
- Restore default handler: `signal.signal(signal.SIGINT, signal.SIG_DFL)` in `finally`
- `CancellationToken` from `scan/progress.py` (already implemented)
