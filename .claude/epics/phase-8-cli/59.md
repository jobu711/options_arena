# Task: health Command

## Description

Implement the `health` command that checks all external service availability (yfinance,
FRED, Ollama, CBOE) using `HealthService.check_all()`. Render results as a Rich table
with UP/DOWN status, latency, and errors. Exit code 0 if all services UP, 1 if any DOWN.

Also create the `render_health_table()` function in `rendering.py`.

## Acceptance Criteria

- [ ] `health` command registered on `app` in `commands.py`
- [ ] Creates `AppSettings`, `HealthService`, calls `check_all()`, closes in `finally`
- [ ] `render_health_table()` in `rendering.py` produces Rich Table with service name, status (green UP / red DOWN), latency ms, error
- [ ] Exit code 0 when all services UP, exit code 1 when any DOWN
- [ ] Uses `asyncio.run()` wrapping async internals (no `async def` on command)
- [ ] 3 unit tests in `tests/unit/cli/test_commands.py` and `tests/unit/cli/test_rendering.py`
- [ ] ruff + mypy --strict pass

## Technical Details

### Files Created
- `src/options_arena/cli/commands.py` — `health` command + `_health_async()`
- `src/options_arena/cli/rendering.py` — `render_health_table()`, `DISCLAIMER` constant

### Implementation Notes
- `health` is a simple command with no options (just `uv run options-arena health`)
- Service lifecycle: create `HealthService(settings.service)` → `check_all()` → close in `finally`
- `HealthService` already exists in `services/health.py` and returns `list[HealthStatus]`
- `HealthStatus` model already exists in `models/health.py`
- Console output to stdout, errors to stderr
- Table columns: Service (bold), Status (center, green/red), Latency (right, ms), Error
- Latency format: `{ms:.0f}ms` or `--` if None

### Tests (~3)
- `test_health_command_parsing` — CliRunner invocation succeeds
- `test_render_health_table_mixed_statuses` — UP (green) and DOWN (red) rendered correctly
- `test_health_exit_code_reflects_status` — exit 0 vs exit 1
