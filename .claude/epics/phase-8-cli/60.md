# Task: universe Subcommands

## Description

Implement `universe` as a sub-Typer with three subcommands: `refresh` (force re-fetch CBOE
universe + S&P 500), `list` (display tickers matching filters), and `stats` (universe size,
sector breakdown, S&P 500 count). All backed by `UniverseService`.

## Acceptance Criteria

- [ ] `universe_app = typer.Typer(...)` registered as sub-Typer on `app`
- [ ] `universe refresh` — force re-fetch and display counts
- [ ] `universe list` — display tickers, with `--sector` and `--preset` filter options
- [ ] `universe stats` — display universe size, sector breakdown, S&P 500 count
- [ ] Each subcommand creates `UniverseService`, uses it, closes in `finally`
- [ ] Uses `asyncio.run()` wrapping async internals
- [ ] 3 unit tests
- [ ] ruff + mypy --strict pass

## Technical Details

### Files Modified
- `src/options_arena/cli/commands.py` — add `universe_app`, `refresh`, `list_tickers`, `stats` commands

### Implementation Notes
- Sub-Typer pattern: `universe_app = typer.Typer(help="...", no_args_is_help=True)` then `app.add_typer(universe_app, name="universe")`
- Each subcommand needs: `AppSettings()` → `ServiceCache` → `RateLimiter` → `UniverseService` → use → close in `finally`
- `refresh`: call `universe_svc.fetch_cboe_tickers()` + `universe_svc.fetch_sp500_constituents()`, display counts
- `list`: fetch tickers, optionally filter by sector/preset, render as Rich table
- `stats`: fetch tickers, compute counts by sector, render summary
- `UniverseService` already exists in `services/universe.py`
- `--sector` option: `str | None`, comma-separated GICS sectors
- `--preset` option: `ScanPreset` enum

### Tests (~3)
- `test_universe_stats_produces_output` — CliRunner verification
- `test_universe_list_sector_filter` — `--sector Technology` filtering
- `test_universe_refresh_triggers_fetch` — service method called
