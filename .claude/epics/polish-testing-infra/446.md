# Task: Add CI dependency caching (uv + mypy)

## Description

Add dependency caching to the GitHub Actions CI workflow to eliminate cold `uv sync` installs
on repeat runs. Enable uv's built-in caching via `astral-sh/setup-uv@v4` and add mypy cache
persistence for the typecheck gate.

## Acceptance Criteria

- [ ] `enable-cache: true` added to all `astral-sh/setup-uv@v4` steps (lint, typecheck, tests gates)
- [ ] `actions/cache@v4` added for `.mypy_cache/` in the typecheck gate
- [ ] mypy cache key uses `uv.lock` hash + source file hash with restore-keys fallback
- [ ] CI workflow still passes all 4 gates after changes
- [ ] Cache hit confirmed on a second push (visible in Actions logs)

## Technical Details

### Files Affected
- `.github/workflows/ci.yml`

### Implementation
1. Add `enable-cache: true` to each `astral-sh/setup-uv@v4` step in gates 1-3 (lint, typecheck, tests)
2. Add `actions/cache@v4` step before mypy in the typecheck gate:
   ```yaml
   - uses: actions/cache@v4
     with:
       path: .mypy_cache
       key: mypy-${{ hashFiles('uv.lock') }}-${{ hashFiles('src/**/*.py') }}
       restore-keys: mypy-${{ hashFiles('uv.lock') }}-
   ```
3. Verify the frontend gate already has npm caching (no changes needed there)

### Key Considerations
- `astral-sh/setup-uv@v4` may already support `enable-cache` — verify at implementation time via Context7
- Cache key for mypy uses both lock file and source files because mypy cache is invalidated by code changes
- Restore-keys fallback allows partial cache hits when only source changes (lock unchanged)

## Test Plan

### Test Cases
No automated tests — verification is manual via GitHub Actions:
1. Push a commit, confirm all gates pass
2. Push a second commit, confirm uv cache hit in logs ("Cache restored from key...")
3. Confirm mypy cache hit in typecheck gate logs
4. Verify no regression in gate pass/fail behavior

### Edge Cases
- Lock file change: cache miss expected, full reinstall, new cache stored
- Source-only change: mypy partial cache hit via restore-keys
