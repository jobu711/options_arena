# Task: Create model factory functions with unit tests

## Description

Create `tests/factories.py` with factory functions for core Pydantic models. Each factory
returns a valid model instance with zero required args and supports kwarg overrides. Migrate
the existing `make_contract()` from `scoring/conftest.py` as the starting template. Add
comprehensive unit tests in `tests/unit/test_factories.py`.

## Acceptance Criteria

- [ ] `tests/factories.py` created with 9 factory functions
- [ ] Each factory produces a valid model with zero args (no `ValidationError`)
- [ ] Each factory accepts kwarg overrides for any field
- [ ] Proper `Decimal("...")` strings for price fields
- [ ] Proper `datetime.now(UTC)` for timestamp fields
- [ ] Valid `StrEnum` values for categorical fields
- [ ] Frozen models construct without error
- [ ] `tests/unit/test_factories.py` covers all factories
- [ ] `ruff check`, `mypy --strict`, and full test suite pass

## Technical Details

### Files Created
- `tests/factories.py` — 9 factory functions
- `tests/unit/test_factories.py` — unit tests for all factories

### Factory Functions

1. **`make_option_contract(**kw) -> OptionContract`** — migrated from `scoring/conftest.py:make_contract()`, realistic defaults (AAPL call, strike=150, 45 DTE)
2. **`make_quote(**kw) -> Quote`** — 6 required fields (ticker, price, volume, etc.)
3. **`make_market_context(**kw) -> MarketContext`** — 18 required + optional fields, sensible market defaults
4. **`make_ticker_score(**kw) -> TickerScore`** — composite score with nested DimensionalScores
5. **`make_dimensional_scores(**kw) -> DimensionalScores`** — all-optional breakdown
6. **`make_agent_response(**kw) -> AgentResponse`** — direction, confidence, reasoning
7. **`make_trade_thesis(**kw) -> TradeThesis`** — strategy, entry/exit levels
8. **`make_debate_result(**kw) -> DebateResult`** — full debate using sub-factories
9. **`make_scan_result(**kw) -> ScanResult`** — scan pipeline output

### Key Considerations
- Read each model's source to get accurate field names and types before writing factories
- `DebateResult` and `ScanResult` have nested models — use other factories internally
- Frozen models (`OptionContract`, `Quote`) must be fully constructed in one call
- `MarketContext` is NOT frozen — can set all fields at once in factory
- No `factory_boy` or `faker` — plain functions only
