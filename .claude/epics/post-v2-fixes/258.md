# Task: Type-safe API schema and debate route deserialization (FR-2 + FR-3)

## Description
Replace the 4 `dict[str, object] | None` fields in `DebateResultDetail` (api/schemas.py) with
typed Pydantic models (`FlowThesis`, `FundamentalThesis`, `RiskAssessment`, `ContrarianThesis`).
Then replace the 4 `json.loads()` calls in `api/routes/debate.py` with `Model.model_validate_json()`
for type-safe deserialization. Remove `import json` if no longer used.

This is the foundation task — the schema change must land first so FR-1 (export) follows the
same pattern.

## Acceptance Criteria
- [ ] `DebateResultDetail.flow_response` is `FlowThesis | None` (not `dict[str, object] | None`)
- [ ] `DebateResultDetail.fundamental_response` is `FundamentalThesis | None`
- [ ] `DebateResultDetail.risk_v2_response` is `RiskAssessment | None`
- [ ] `DebateResultDetail.contrarian_response` is `ContrarianThesis | None`
- [ ] Debate route uses `FlowThesis.model_validate_json(row.flow_json)` instead of `json.loads()`
- [ ] Same pattern for all 4 V2 fields
- [ ] `import json` removed from debate.py if no longer used
- [ ] Existing V1 fields unchanged
- [ ] `ruff check`, `mypy --strict`, and existing tests pass

## Technical Details
- **schemas.py (~lines 256-259)**: Change field type annotations + add imports from `models.analysis`
- **routes/debate.py (~lines 585-588)**: Replace `json.loads(row.flow_json) if row.flow_json else None`
  with `FlowThesis.model_validate_json(row.flow_json) if row.flow_json else None`
- Follow existing V1 pattern at debate.py lines 533-534: `AgentResponse.model_validate_json(...)`
- Add model imports: `from options_arena.models import FlowThesis, FundamentalThesis, RiskAssessment, ContrarianThesis`
