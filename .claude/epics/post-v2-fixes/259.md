# Task: Fix export endpoint to populate V2 fields (FR-1)

## Description
The export endpoint (`/api/debates/{id}/export`) reconstructs a `DebateResult` from a
`DebateRow` but omits all V2 fields, silently producing V1-only exports. Fix by deserializing
the 4 V2 JSON fields and passing them to the `DebateResult` constructor. Also pass
`debate_protocol` field.

This is the most critical fix — it's a data loss bug in production exports.

## Acceptance Criteria
- [ ] Export of V2 debate includes all 4 V2 agent sections (flow, fundamental, risk_v2, contrarian)
- [ ] Export includes `debate_protocol` field
- [ ] V2 deserialization uses `contextlib.suppress(Exception)` for resilience (matching existing pattern)
- [ ] V1 debate export unchanged (no regression)
- [ ] Markdown export of V2 debate includes V2 section renderers output

## Technical Details
- **File**: `src/options_arena/api/routes/export.py` (~line 136)
- Add imports: `FlowThesis`, `FundamentalThesis`, `RiskAssessment`, `ContrarianThesis` from `models`
- Add `contextlib.suppress` import (if not already present)
- Before the `DebateResult(...)` constructor, deserialize V2 fields:
  ```python
  flow_response = None
  with contextlib.suppress(Exception):
      if row.flow_json:
          flow_response = FlowThesis.model_validate_json(row.flow_json)
  # repeat for fundamental, risk_v2, contrarian
  ```
- Pass to `DebateResult(..., flow_response=flow_response, ...)` constructor
- Follow same `contextlib.suppress()` resilience pattern used for V1 fields in the same file
