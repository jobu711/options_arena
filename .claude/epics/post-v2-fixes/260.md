# Task: Market-timezone-aware dates in outcome collector (FR-5)

## Description
Replace `date.today()` calls in `services/outcome_collector.py` with a
`_market_today()` helper that returns the current date in US Eastern time.
This prevents premature outcome collection and incorrect expiry checks when
the system timezone is not US Eastern (e.g., UTC servers, European users).

## Acceptance Criteria
- [ ] `_market_today()` helper added using `ZoneInfo("America/New_York")`
- [ ] Both `date.today()` calls (~lines 85 and 298) replaced with `_market_today()`
- [ ] No new external dependencies (uses stdlib `zoneinfo`)
- [ ] Existing outcome collector tests still pass
- [ ] New tests verify timezone-aware behavior

## Technical Details
- **File**: `src/options_arena/services/outcome_collector.py`
- Add imports: `from zoneinfo import ZoneInfo` and `from datetime import datetime`
- Add module-level constant: `_EASTERN = ZoneInfo("America/New_York")`
- Add helper function:
  ```python
  def _market_today() -> date:
      """Return today's date in US Eastern time (market timezone)."""
      return datetime.now(_EASTERN).date()
  ```
- Replace `date.today()` at both call sites with `_market_today()`
- The helper is module-private (underscore prefix) — not exported
