# Task: Migration 020 for contract_outcomes unique constraint (FR-4)

## Description
Create migration `020_fix_outcomes_constraint.sql` that recreates the `contract_outcomes`
table with the correct `UNIQUE(recommended_contract_id, holding_days)` constraint.
SQLite does not support `ALTER TABLE DROP CONSTRAINT`, so the table must be recreated
using the standard table-recreation pattern. The migration must be idempotent for databases
that already have the correct constraint.

- [ ] Migration runs cleanly on fresh DB and on existing DB with old constraint
- [ ] Sequential numbering follows existing migration 019

## Technical Details
- **File**: `data/migrations/020_fix_outcomes_constraint.sql` (new)
- Must match exact column schema of current `contract_outcomes` table
- Pattern: `CREATE TABLE IF NOT EXISTS ... _new` → `INSERT OR IGNORE INTO _new SELECT * FROM old`
  → `DROP TABLE IF EXISTS old` → `ALTER TABLE _new RENAME TO old`
- Check existing migration 012 to get the current column definitions
- Wrap in a transaction for atomicity (SQLite auto-commits DDL, but BEGIN/COMMIT still useful)
