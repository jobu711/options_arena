# Task: Config migration and consumer updates

## Description
The critical-path task: remove 13 filter fields from `ScanConfig`, 9 contract selection fields from `PricingConfig`, and add `filters: ScanFilterSpec = ScanFilterSpec()` to `ScanConfig`. Update all consumers: pipeline orchestrator, 4 phase functions, CLI commands, API schemas/routes, scoring/contracts.py. Fix all broken tests (~200 files). Remove unused `options_batch_size` from `ScanConfig`.

This is the largest single task because field removal cascades through the entire codebase.

## Acceptance Criteria
- [ ] `ScanConfig` loses 13 filter fields + `options_batch_size`, gains `filters: ScanFilterSpec`
- [ ] `PricingConfig` loses 9 contract selection fields (dte, delta, OI, volume, spread)
- [ ] `PricingConfig` retains: `risk_free_rate_fallback`, `iv_solver_tol`, `iv_solver_max_iter`, `delta_target`
- [ ] `ScanConfig` retains: scoring thresholds, timeouts, concurrency, feature toggles
- [ ] Pipeline `run()` loses `preset` param, reads from `filter_spec.universe.preset`
- [ ] Phase 1 (`run_universe_phase`) receives `UniverseFilters` instead of `ScanConfig`
- [ ] Phase 3 (`run_options_phase`, `process_ticker_options`) receives `OptionsFilters` instead of scattered config fields
- [ ] Phase 2 unchanged (still receives `ScanConfig` for scoring thresholds only)
- [ ] Phase 4 unchanged
- [ ] CLI `scan` command builds `ScanFilterSpec` from flat args
- [ ] API `ScanRequest` builds `ScanFilterSpec` internally
- [ ] `scoring/contracts.py` reads delta/DTE/OI/volume/spread from `OptionsFilters`
- [ ] Default `ScanFilterSpec()` produces identical scan behavior to current defaults
- [ ] All existing tests pass
- [ ] `options_batch_size` fully removed (zero references)

## Technical Details

### Config changes (`models/config.py`):

**ScanConfig — fields to REMOVE:**
- `top_n`, `min_score`, `min_price`, `max_price`, `min_dollar_volume` (→ OptionsFilters/ScoringFilters)
- `ohlcv_min_bars` (→ UniverseFilters)
- `sectors`, `industry_groups`, `market_cap_tiers`, `custom_tickers` (→ UniverseFilters)
- `exclude_near_earnings_days`, `direction_filter`, `min_iv_rank` (→ OptionsFilters/ScoringFilters)
- `min_dte`, `max_dte` (→ OptionsFilters)
- `options_batch_size` (unused — zero refs outside config.py)

**ScanConfig — fields to KEEP:**
- `adx_trend_threshold`, `rsi_overbought`, `rsi_oversold` — scoring thresholds
- `options_per_ticker_timeout`, `options_concurrency` — rate limiting
- `enable_iv_analytics`, `enable_flow_analytics`, `enable_fundamental`, `enable_regime` — toggles

**ScanConfig — field to ADD:**
- `filters: ScanFilterSpec = ScanFilterSpec()`

**PricingConfig — fields to REMOVE:**
- `dte_min`, `dte_max`, `min_oi`, `min_volume`, `max_spread_pct`
- `delta_primary_min`, `delta_primary_max`, `delta_fallback_min`, `delta_fallback_max`

**PricingConfig — fields to KEEP:**
- `risk_free_rate_fallback`, `delta_target`, `iv_solver_tol`, `iv_solver_max_iter`

**Remove validators from ScanConfig/PricingConfig** that are now on the filter models:
- `validate_top_n`, `validate_ohlcv_min_bars`, `validate_min_iv_rank`, `validate_max_price`
- `validate_dte_positive`, `validate_cross_field_ranges`
- `normalize_sectors`, `normalize_industry_groups`, `deduplicate_market_cap_tiers`
- `validate_custom_tickers`

### Pipeline changes (`scan/pipeline.py`):
- `run()` signature: remove `preset: ScanPreset` param
- Read `filter_spec = self._settings.scan.filters`
- `_phase_universe()`: pass `filter_spec.universe` instead of `scan_config`
- Direction filter post-Phase 2: read from `filter_spec.scoring.direction_filter`
- `_phase_options()`: pass `filter_spec.options` alongside `scan_config` (for timeouts/concurrency)

### Phase changes:
- `phase_universe.py`: `run_universe_phase(filters: UniverseFilters, ...)` — read preset, sectors, etc. from filters
- `phase_options.py`: `run_options_phase(options_filters: OptionsFilters, ...)` and `process_ticker_options(options_filters: OptionsFilters, ...)` — read delta/DTE/OI/volume/spread from options_filters
- `phase_scoring.py`: unchanged (still receives `scan_config` for thresholds)
- `phase_persist.py`: unchanged

### Entry point changes:
- `cli/commands.py`: Build `ScanFilterSpec` from flat `--sector`, `--top-n`, `--min-dte`, etc. args
- `api/schemas.py`: `ScanRequest` builds `ScanFilterSpec` internally
- `api/routes/scan.py`: Pass `ScanFilterSpec` through to pipeline

### scoring/contracts.py changes:
- `recommend_contracts()` and `select_by_delta()` receive `OptionsFilters` alongside `PricingConfig`
- Read `min_oi`, `min_volume`, `max_spread_pct`, delta ranges, DTE from `OptionsFilters`
- Read `delta_target` from `PricingConfig` (stays there — pricing math)

### Test fixup:
- ~200 test files construct `ScanConfig` or `PricingConfig` — update field references
- Tests that pass `min_dte`, `delta_primary_min`, etc. to `PricingConfig` → pass to `OptionsFilters`
- Tests that pass `top_n`, `sectors`, etc. to `ScanConfig` → pass to `ScanConfig.filters.universe/options`
- `conftest.py` fixtures producing configs need updating
