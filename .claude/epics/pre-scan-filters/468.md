# Task: Phase optimizations (market cap pre-filter, earnings reorder)

## Description
Two optimizations that reduce wasted API calls:

1. **Phase 1 — market cap pre-filter**: Use cached `ticker_metadata` (already loaded for sector enrichment) to filter tickers by `market_cap_tiers` BEFORE fetching OHLCV data. Currently market cap filtering happens in Phase 3 after expensive chain fetches.

2. **Phase 3 — earnings check reorder**: Move the `exclude_near_earnings_days` check BEFORE chain fetch. Currently earnings date is fetched and checked AFTER chain fetch, wasting an API call per dropped ticker.

## Acceptance Criteria
- [ ] Phase 1 applies `market_cap_tiers` filter using cached `ticker_metadata` before OHLCV batch fetch
- [ ] Empty `market_cap_tiers` list → no filtering (preserve current behavior)
- [ ] Empty metadata cache → skip market cap filter with warning log (graceful degradation)
- [ ] Tickers without metadata → kept (not penalized for missing data)
- [ ] Phase 3 checks `exclude_near_earnings_days` BEFORE chain fetch
- [ ] Tickers near earnings are dropped BEFORE fetching option chains (saving API calls)
- [ ] `exclude_near_earnings_days=None` → no earnings filter (preserve current behavior)
- [ ] ~15 new tests

## Technical Details

### Phase 1 market cap pre-filter (`scan/phase_universe.py`):

The function already calls `repository.get_all_ticker_metadata()` for sector enrichment (lines ~109-119). Piggyback on this existing metadata load:

```python
# After loading metadata (existing code)
metadata_map = await repository.get_all_ticker_metadata()

# NEW: Apply market_cap_tiers filter before OHLCV fetch
if universe_filters.market_cap_tiers:
    before_count = len(tickers)
    allowed_tiers = set(universe_filters.market_cap_tiers)
    tickers = [
        t for t in tickers
        if t not in metadata_map  # keep tickers without metadata
        or metadata_map[t].market_cap_tier in allowed_tiers
    ]
    logger.info(
        "Market cap filter (%s): %d -> %d tickers",
        [t.value for t in universe_filters.market_cap_tiers],
        before_count,
        len(tickers),
    )
```

No new API calls — uses the already-loaded metadata cache.

### Phase 3 earnings reorder (`scan/phase_options.py`):

Currently in `process_ticker_options()`, earnings date is fetched and checked AFTER chain fetch (lines ~461-469). Reorder to:

1. Fetch earnings date FIRST (cheap — yfinance calendar lookup)
2. If `exclude_near_earnings_days` is set and ticker is near earnings → skip chain fetch entirely
3. Only fetch chains for tickers that pass the earnings check

```python
# BEFORE chain fetch (moved from after chain fetch):
if options_filters.exclude_near_earnings_days is not None:
    earnings_date = await market_data.fetch_earnings_date(ticker)
    if earnings_date is not None:
        days_to_earnings = (earnings_date - date.today()).days
        if 0 <= days_to_earnings <= options_filters.exclude_near_earnings_days:
            logger.info("Skipping %s: earnings in %d days", ticker, days_to_earnings)
            return ticker, [], earnings_date, None
```

### Edge Cases
- All tickers match a single market cap tier
- No tickers match any tier (all filtered out, Phase 1 returns empty)
- Metadata cache is completely empty (dict is `{}`)
- Earnings date is today (`days_to_earnings == 0`)
- Earnings date is in the past (`days_to_earnings < 0`)
- `exclude_near_earnings_days == 0` (only today's earnings)
