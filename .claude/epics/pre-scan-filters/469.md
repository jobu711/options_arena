# Task: Filter persistence (migration 031)

## Description
Add `filter_spec_json TEXT` column to `scan_runs` table via migration 031. Phase 4 stores `ScanFilterSpec.model_dump_json()` when persisting scan results. Repository provides read-back for reproducibility. This enables users to see exactly what filters were used for any historical scan.

## Acceptance Criteria
- [ ] Migration 031 adds `filter_spec_json TEXT` column to `scan_runs`
- [ ] Phase 4 (`phase_persist.py`) stores `filter_spec.model_dump_json()` on scan run
- [ ] `Repository.get_scan_run()` populates `filter_spec_json` field on returned `ScanRun`
- [ ] `ScanRun` model gains `filter_spec_json: str | None = None` field
- [ ] Existing scan runs without filter spec return `None` (backward compatible)
- [ ] `ScanFilterSpec.model_validate_json(filter_spec_json)` roundtrip works
- [ ] ~8 new tests

## Technical Details

### Migration (`data/migrations/031_add_filter_spec_json.sql`):
```sql
ALTER TABLE scan_runs ADD COLUMN filter_spec_json TEXT;
```

### Model change (`models/scan.py`):
Add `filter_spec_json: str | None = None` to `ScanRun`.

### Phase 4 change (`scan/phase_persist.py`):
Pass `filter_spec_json` when creating `ScanRun`:
```python
ScanRun(
    ...,
    filter_spec_json=filter_spec.model_dump_json() if filter_spec else None,
)
```

The `filter_spec` must be threaded from the pipeline orchestrator to Phase 4.

### Pipeline change (`scan/pipeline.py`):
Pass `filter_spec=self._settings.scan.filters` to `_phase_persist()` and `run_persist_phase()`.

### Repository change (`data/_scan.py`):
- `save_scan_run()`: include `filter_spec_json` in INSERT
- `get_scan_run()` / `get_recent_scans()`: read `filter_spec_json` from row

### Files affected:
- `data/migrations/031_add_filter_spec_json.sql` (NEW)
- `src/options_arena/models/scan.py` (add field)
- `src/options_arena/scan/phase_persist.py` (store filter spec)
- `src/options_arena/scan/pipeline.py` (thread filter_spec to Phase 4)
- `src/options_arena/data/_scan.py` (read/write filter_spec_json)
