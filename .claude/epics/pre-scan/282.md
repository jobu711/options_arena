## Technical Details
- **New file: `web/src/components/scan/PreScanFilters.vue`** (~300 lines):
  - Uses PrimeVue `Panel` with `toggleable` and `collapsed` props (project standard, not Accordion)
  - Active filter count computed in header: `"Universe (${activeCount})"`
  - Preset selector: dropdown or button group with 6 options + ticker count badges
  - Price range: `InputNumber` with `min=0`, `mode="currency"`, `currency="USD"` for prices
  - DTE range: `InputNumber` with `min=1`, `max=730` for DTE values
  - Educational `<small>` descriptions under each control
  - Emits `@update:filters` event with `PreScanFilterPayload` type
- **Refactored: `web/src/pages/ScanPage.vue`** (398 → ~130 lines):
  - Import and compose `PreScanFilters`
  - Handle `@update:filters` event, merge into scan options
  - Keep: Run Scan button, ProgressTracker, Past Scans DataTable
  - Remove: all inline filter markup (moves to PreScanFilters)
- **`web/src/types/scan.ts`**: Add interfaces:
  ```typescript
  interface PresetInfo { preset: string; label: string; description: string; estimated_count: number; }
  interface PreScanFilterPayload { preset?: string; sectors?: string[]; min_price?: number; max_price?: number; min_dte?: number; max_dte?: number; }
  ```
- **`web/src/stores/scan.ts`**: Extend `StartScanOptions` with 4 new optional fields. Update `startScan()` to include them in API payload.
- **Preserve `data-testid`**: Run `grep -r "data-testid" web/src/pages/ScanPage.vue` before refactor, verify all IDs still exist after.
