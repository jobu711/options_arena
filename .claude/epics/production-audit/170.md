# Task: Reliability + Silent Failure Fixes

## Description

Fix five reliability issues where the system fails silently or misleads users: atomic lock
acquisition for batch operations, error handling in Vue scan store, disclaimer removal,
Windows-compatible log rotation, and eager app.state initialization.

**Audit findings**: AUDIT-007, AUDIT-009, AUDIT-010, AUDIT-013, AUDIT-014

## Acceptance Criteria

### Atomic Lock (AUDIT-007)
- [ ] Remove `lock.locked()` pre-check pattern from API handlers
- [ ] Replace with atomic try-acquire: `asyncio.wait_for(lock.acquire(), timeout=0.0)`
- [ ] `TimeoutError` caught → return 409 Conflict response
- [ ] No TOCTOU race between check and acquire
- [ ] Concurrent requests to `/api/scan` correctly return 409 when scan is running

### fetchScans Error Handling (AUDIT-009)
- [ ] `fetchScans()` in `web/src/stores/scan.ts` has try/catch wrapping the API call
- [ ] Caught errors pushed to `errors.value` array (matching other store actions' pattern)
- [ ] Loading state correctly reset in finally block (already done)
- [ ] Failed fetch doesn't leave store in broken state

### Disclaimer Removal (AUDIT-010)
- [ ] `DISCLAIMER` constant removed from `reporting/debate_export.py`
- [ ] All disclaimer rendering removed from CLI output paths (`cli/commands.py`)
- [ ] No disclaimer text in exported markdown or PDF
- [ ] Tests asserting disclaimer text updated or removed

### Windows Log Rotation (AUDIT-013)
- [ ] `RotatingFileHandler` replaced with `QueueHandler` + `QueueListener` pattern in `cli/app.py`
- [ ] `QueueListener` uses `RotatingFileHandler` internally but on a separate thread (avoids
      Windows file locking issues)
- [ ] Listener started in logging setup, stopped at shutdown
- [ ] No `PermissionError` on Windows when log rotation triggers during active writes
- [ ] Log output still goes to `logs/options_arena.log` with 5MB rotation, 3 backups

### App State Initialization (AUDIT-014)
- [ ] `scan_counter`, `active_scans`, `scan_queues` initialized in `create_app()` lifespan
- [ ] Debate equivalents (`debate_counter`, etc.) also initialized in lifespan
- [ ] No lazy initialization in route handlers — all counters/queues ready at startup
- [ ] First request to any endpoint works without AttributeError

## Technical Details

### Atomic Lock Pattern
```python
# Before (TOCTOU):
if lock.locked():
    raise HTTPException(409)
async with lock:
    ...

# After (atomic):
try:
    await asyncio.wait_for(lock.acquire(), timeout=0.0)
except TimeoutError:
    raise HTTPException(409, detail="Operation already in progress")
try:
    ...
finally:
    lock.release()
```

### QueueHandler Pattern
```python
import logging
from logging.handlers import QueueHandler, QueueListener, RotatingFileHandler
from queue import Queue

log_queue: Queue[logging.LogRecord] = Queue()
queue_handler = QueueHandler(log_queue)
file_handler = RotatingFileHandler(LOG_FILE, maxBytes=5_242_880, backupCount=3)
listener = QueueListener(log_queue, file_handler, respect_handler_level=True)
listener.start()
# At shutdown: listener.stop()
```
