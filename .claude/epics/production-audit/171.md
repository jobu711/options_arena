# Task: CI + Ops Hardening

## Description

Add frontend CI gates (TypeScript type checking + production build) and dependency auditing
to the GitHub Actions workflow. Add FRED rate staleness detection with warning logs when the
cached risk-free rate is older than 48 hours.

**Audit findings**: AUDIT-002 (HIGH), AUDIT-020, AUDIT-018

## Acceptance Criteria

### Frontend CI (AUDIT-002)
- [ ] New job in `.github/workflows/ci.yml` that runs:
  - `npm ci` (install dependencies from lockfile)
  - `npx vue-tsc --noEmit` (TypeScript type checking)
  - `npm run build` (production Vite build)
- [ ] Job runs on `push` and `pull_request` to `master` (matching existing triggers)
- [ ] Job uses `working-directory: web/`
- [ ] Failing `vue-tsc` or `npm run build` blocks merge
- [ ] Uses Node.js 22.x (or current LTS matching project)

### Dependency Audit (AUDIT-020)
- [ ] `pip-audit` (or `uv` equivalent) step added to existing Python CI job
- [ ] Known CVEs in dependencies fail the build
- [ ] Step runs after `uv sync` (dependencies must be installed first)

### FRED Rate Staleness (AUDIT-018)
- [ ] `rate_fetched_at: datetime` timestamp stored alongside cached FRED rate in `services/fred.py`
- [ ] When serving a cached rate, check `datetime.now(UTC) - rate_fetched_at > timedelta(hours=48)`
- [ ] If stale, log `logging.warning("FRED risk-free rate is %d hours old; ...")`
- [ ] Rate still served (no error) — staleness is informational only
- [ ] Fresh fetch resets `rate_fetched_at`

## Technical Details

### CI Workflow Addition
```yaml
frontend:
  runs-on: ubuntu-latest
  defaults:
    run:
      working-directory: web
  steps:
    - uses: actions/checkout@v4
    - uses: actions/setup-node@v4
      with:
        node-version: '22'
        cache: 'npm'
        cache-dependency-path: web/package-lock.json
    - run: npm ci
    - run: npx vue-tsc --noEmit
    - run: npm run build
```

### Dependency Audit
- `pip-audit` can be installed via `uv tool install pip-audit` or `uv add --dev pip-audit`
- Alternative: `uv pip audit` if supported in current uv version
- Run after `uv sync` in the existing test job

### FRED Staleness
- Current FRED service caches rate as `float` — extend cache value to `(rate, fetched_at)` tuple
  or a small dataclass/NamedTuple
- `_cached_rate: tuple[float, datetime] | None` pattern
- Check staleness on every cache hit, not just on fetch

### Files Affected
- `.github/workflows/ci.yml` — new frontend job + pip-audit step
- `src/options_arena/services/fred.py` — staleness tracking + warning
- `tests/unit/services/test_fred.py` — staleness warning test
