# Task: SecretStr Migration + NaN/Inf Pricing Guards

## Description

Fix the two HIGH-severity audit findings. Migrate all API key config fields from `str | None`
to `SecretStr | None` for automatic redaction in logs/tracebacks/model_dump(). Add
`math.isfinite()` guards to pricing validation functions so NaN/Inf inputs raise `ValueError`
instead of silently producing wrong Greeks.

**Audit findings**: AUDIT-001 (HIGH), AUDIT-005, AUDIT-006

## Technical Details

### SecretStr Migration
- `from pydantic import SecretStr` ā€” already available via pydantic v2
- `SecretStr` reads from env vars natively in pydantic-settings ā€” no special handling needed
- Callers pattern: `key = config.api_key.get_secret_value() if config.api_key else None`
- `model_dump()` returns `SecretStr('**********')` by default; use `model_dump(mode="json")` for
  serialization (still redacted). Use `.get_secret_value()` only at point of HTTP request.

### NaN/Inf Guards
- `math.isfinite(v)` returns `False` for NaN, Inf, -Inf ā€” single check covers all
- Must check BEFORE positivity: `float("nan") <= 0.0` is `False` (NaN passes positivity check)
- Pattern: `if not math.isfinite(S): raise ValueError("S must be a finite number")`
