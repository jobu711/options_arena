# Task: API Input Validation + Security Hardening

## Description

Harden API input boundaries: add ticker regex validation on all API request models, add
bounds to batch limits, escape HTML in PDF export, and validate WebSocket Origin headers
to accept only loopback origins.

**Audit findings**: AUDIT-003, AUDIT-025, AUDIT-024

## Acceptance Criteria

### Ticker Validation (AUDIT-003)
- [ ] All API request models with ticker fields use
      `Field(min_length=1, max_length=10, pattern=r"^[A-Z0-9.\-^]{1,10}$")`
- [ ] `mode="before"` uppercase validator on ticker fields (`.upper().strip()`)
- [ ] Batch ticker lists have `max_length` bounds (e.g., max 50 tickers per request)
- [ ] `POST /api/scan` rejects tickers like `""; DROP TABLE` with 422
- [ ] `POST /api/debate` rejects empty or oversized ticker strings with 422
- [ ] Valid tickers like `AAPL`, `BRK.B`, `SPY`, `^VIX` accepted

### HTML Escape (AUDIT-025)
- [ ] `html.escape(md_content)` applied before embedding in `<pre>` tag in PDF export
      (`reporting/debate_export.py`)
- [ ] Markdown with `<script>` tags rendered as escaped text, not executed

### WebSocket Origin (AUDIT-024)
- [ ] `Origin` header checked before `websocket.accept()` in all WS endpoints in `api/ws.py`
- [ ] Accepted origins: `http://127.0.0.1:*`, `http://localhost:*`, `http://[::1]:*`
- [ ] Missing or non-loopback Origin → `websocket.close(code=4003)` with reason
- [ ] Dev mode (Vite proxy on :5173) origin accepted

## Technical Details

### Ticker Validation
- Apply to request models in `api/routes/` (scan, debate, ticker routes)
- Path parameters: use `Path(min_length=1, max_length=10, pattern=...)` for `{ticker}` params
- Body models: use `Field(...)` on ticker string fields
- Uppercase validator: `@field_validator("ticker", mode="before")` with `v.upper().strip()`

### HTML Escape
- `import html` (stdlib)
- Apply in `_render_pdf()` or wherever markdown content enters HTML template
- Single line change: `html.escape(content)` wrapping

### WebSocket Origin
- Extract: `origin = websocket.headers.get("origin", "")`
- Parse with `urllib.parse.urlparse(origin)`
- Allow hostname in `{"127.0.0.1", "localhost", "::1", "[::1]"}`
- Helper function `_is_loopback_origin(origin: str) -> bool` shared across WS endpoints

### Files Affected
- `src/options_arena/api/routes/scan.py` — request model validation
- `src/options_arena/api/routes/debate.py` — request model validation
- `src/options_arena/api/routes/ticker.py` — path parameter validation
- `src/options_arena/api/models.py` (or wherever API request models live) — Field constraints
- `src/options_arena/reporting/debate_export.py` — html.escape
- `src/options_arena/api/ws.py` — Origin header check
- `tests/` — validation rejection tests, HTML escape tests, WS origin tests
