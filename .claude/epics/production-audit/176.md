# Task: MarketContext Persistence + Export Fix

## Description

Persist `MarketContext` alongside debate results in the `ai_theses` SQLite table so that
debate exports render actual market prices instead of `$0.00`. Add a DB migration (009) with
a nullable JSON column. Also add a module-level weight sum assertion in `scoring/composite.py`
as a guard against future weight drift.

**Audit findings**: AUDIT-008, AUDIT-011

## Acceptance Criteria

### MarketContext Persistence (AUDIT-008)
- [ ] Migration 009 adds nullable `market_context_json TEXT` column to `ai_theses` table
- [ ] `Repository.save_thesis()` (or equivalent) serializes `MarketContext` to JSON and stores it
- [ ] `Repository.get_thesis()` (or equivalent) deserializes `market_context_json` back to
      `MarketContext` model
- [ ] Existing rows with `NULL` market_context_json handled gracefully (no crash on old data)
- [ ] Round-trip test: save debate with MarketContext → load → verify all fields match

### Export Fix (AUDIT-008 continued)
- [ ] `reporting/debate_export.py` export functions accept or fetch persisted MarketContext
- [ ] Exported markdown shows real current price, bid, ask — not `$0.00`
- [ ] Export of old debate (no persisted context) gracefully falls back (shows "N/A" or omits prices)

### Weight Assertion (AUDIT-011)
- [ ] Module-level assertion in `scoring/composite.py`:
      `assert abs(sum(INDICATOR_WEIGHTS.values()) - 1.0) < 1e-9, "Indicator weights must sum to 1.0"`
- [ ] Assertion fires at import time if weights drift

## Technical Details

### Migration 009
- File: `data/migrations/009_market_context.sql`
- SQL: `ALTER TABLE ai_theses ADD COLUMN market_context_json TEXT;`
- Pattern: follow existing migration files (001-008)

### Serialization
- `MarketContext.model_dump_json()` for serialization
- `MarketContext.model_validate_json(json_str)` for deserialization
- Store as TEXT column (SQLite has no native JSON type, but TEXT works fine)

### Export Integration
- The export path currently builds markdown from `DebateResult`
- `DebateResult` already has `market_context: MarketContext` field
- When loading from DB for export, populate `market_context` from the stored JSON
- If JSON is NULL (old record), set `market_context` to None and handle in template

### Files Affected
- `data/migrations/009_market_context.sql` — new migration file
- `src/options_arena/data/repository.py` — save/load with market_context_json
- `src/options_arena/reporting/debate_export.py` — use persisted MarketContext in export
- `src/options_arena/scoring/composite.py` — weight assertion
- `tests/` — migration test, round-trip persistence test, export with prices test
