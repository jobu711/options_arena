# Task: Extract flow, fundamental, and risk prompts

## Description

Move the inline system prompt constants from `flow_agent.py`, `fundamental_agent.py`, and
`risk.py` into dedicated files under `agents/prompts/`. Risk is the most complex: it has
`RISK_SYSTEM_PROMPT` (v1 fallback), `RISK_V2_SYSTEM_PROMPT` (active), and `RISK_STRATEGY_TREE`
— all three move to `prompts/risk.py`. Zero behavior change.

- `src/options_arena/agents/fundamental_agent.py` — remove inline prompt, add import
- `src/options_arena/agents/risk.py` — remove inline prompts and strategy tree, add imports

### Risk Module Complexity
Risk has 3 constants to extract:
1. `RISK_SYSTEM_PROMPT` (v2.1) — v1 fallback agent prompt
2. `RISK_V2_SYSTEM_PROMPT` (v1.0) — active v2 agent prompt
3. `RISK_STRATEGY_TREE` (lines 28-37) — decision tree appended to risk prompt only

All three move to `prompts/risk.py`. The risk agent module keeps:
- `risk_agent` and `risk_agent_v2` Agent instances
- `@system_prompt` decorators (dynamic for v2 — receives Phase 1 outputs)
- `@output_validator` decorators
- Dynamic injection of Phase 1 agent outputs via `<<<AGENT_NAME>>>` delimiters

### Key Constraints
- `RISK_STRATEGY_TREE` is currently defined in `risk.py` (lines 28-37), concatenated
  into `RISK_V2_SYSTEM_PROMPT`. After extraction, `prompts/risk.py` defines both and
  concatenates them together.
- Flow/Fundamental dynamic injection (`<<<BULL/BEAR_ARGUMENT>>>`) stays in agent files.
