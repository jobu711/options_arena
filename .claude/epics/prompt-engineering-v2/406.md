# Task: Prompt regression test suite

## Description

Create two test files that catch prompt regressions automatically:
1. `test_prompt_structure.py` — validates structure, version headers, token budget, appendix presence
2. `test_prompt_quality.py` — TestModel-based quality tests, citation density, output validation

Parametrized across all 8 prompt constants imported from `agents/prompts/`.

## Acceptance Criteria

- [ ] `test_prompt_structure.py` created with parametrized tests across all 8 prompts
- [ ] Version header test: each prompt has `# VERSION: vX.Y` in its source file
- [ ] Token budget test: each prompt < 8000 chars (≈ 2000 tokens)
- [ ] Appendix test: `PROMPT_RULES_APPENDIX` text present in each concatenated prompt
- [ ] `RISK_STRATEGY_TREE` present only in risk prompt constants
- [ ] `test_prompt_quality.py` created with TestModel-based tests
- [ ] Each prompt + mock context produces valid structured output
- [ ] Citation density > 0% on mock context
- [ ] No `<think>` tags in validated output
- [ ] 25+ total tests across both files
- [ ] `models.ALLOW_MODEL_REQUESTS = False` at module level

### Token Budget Heuristic
`len(prompt) / 4 < 2000` — approximately 4 chars per token for English text.
No tiktoken dependency needed.

### Citation Density Testing
Use `compute_citation_density()` from `_parsing.py` with a mock context block
containing known `LABEL:` patterns. Verify density > 0% (prompts instruct citation).

### Edge Cases
- Empty prompt string (rejected)
- Prompt exactly at 8000-char budget boundary
- Prompt with APPENDIX missing (should fail)
- Risk v1 vs v2 both have strategy tree content
- Citation density with zero context labels (edge: 0/0)

### Mocking Strategy
- `TestModel` for all agent runs — no real API calls
- `models.ALLOW_MODEL_REQUESTS = False` at module level
- Mock context block with known `LABEL:` patterns for citation density
- Reuse `conftest.py` fixtures: `mock_debate_deps`, `market_context`
