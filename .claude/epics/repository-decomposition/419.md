## Technical Details

### ScanMixin (`_scan.py`)
- **Source lines**: repository.py ~117-276 (scan/score) + ~450-589 (score history)
- **Tables**: `scan_runs`, `ticker_scores`
- **Import subset**: `ScanRun`, `TickerScore`, `IndicatorSignals`, `ScanPreset`, `SignalDirection`,
  `DimensionalScores`, `GICSSector`, `HistoryPoint`, `TrendingTicker`
- **Static helpers**: `_row_to_scan_run`, `_row_to_ticker_score`

### DebateMixin (`_debate.py`)
- **Source lines**: repository.py ~278-448 (debate persistence) + ~1512-1769 (agent calibration)
- **Tables**: `ai_theses`, `agent_predictions`, `auto_tune_weights`
- **Import subset**: `DebateRow` (from `..repository`), debate/agent models from `models/`
- **Static helpers**: `_row_to_debate_row`
- **Note**: `DebateRow` stays in `repository.py` — `_debate.py` imports it from there

### Implementation Steps
1. Read `repository.py` fully to identify exact method boundaries and imports
2. Create `_scan.py` — copy ScanMixin methods with their required imports
3. Create `_debate.py` — copy DebateMixin methods with their required imports
4. Run `uv run mypy src/options_arena/data/_scan.py src/options_arena/data/_debate.py --strict`
5. Run `uv run ruff check src/options_arena/data/_scan.py src/options_arena/data/_debate.py --fix`
