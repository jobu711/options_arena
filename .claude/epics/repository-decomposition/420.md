## Technical Details

### AnalyticsMixin (`_analytics.py`)
- **Source lines**: repository.py ~591-835 (contracts/normalization) + ~837-995 (outcomes) +
  ~997-1379 (analytics queries)
- **Tables**: `recommended_contracts`, `contract_outcomes`, `normalization_metadata`
- **Import subset**: `RecommendedContract`, `ContractOutcome`, `NormalizationStats`,
  `WinRateResult`, `ScoreCalibrationResult`, `IndicatorAttributionResult`,
  `HoldingPeriodResult`, `DeltaPerformanceResult`, `PerformanceSummary`, `OptionType`,
  `SignalDirection`
- **Static helpers**: `_row_to_recommended_contract`, `_row_to_normalization_stats`,
  `_row_to_contract_outcome`
- **Largest mixin** — ~650 LOC. Contains 6 complex analytics queries with multi-JOIN SQL.

### MetadataMixin (`_metadata.py`)
- **Source lines**: repository.py ~1381-1511
- **Tables**: `ticker_metadata`
- **Import subset**: `TickerMetadata`, `MetadataStats`, `GICSSector`
- **Static helpers**: `_row_to_ticker_metadata`
- **Smallest mixin** — ~200 LOC. Self-contained CRUD with no cross-domain joins.

### Implementation Steps
1. Read `repository.py` fully to identify exact method boundaries and imports
2. Create `_analytics.py` — copy AnalyticsMixin methods with their required imports
3. Create `_metadata.py` — copy MetadataMixin methods with their required imports
4. Run `uv run mypy src/options_arena/data/_analytics.py src/options_arena/data/_metadata.py --strict`
5. Run `uv run ruff check src/options_arena/data/_analytics.py src/options_arena/data/_metadata.py --fix`
