# Task: Persist dimensional scores in pipeline and database

## Description

Wire the existing Dimensional Scoring Engine (DSE) output into the persistence layer. The `compute_dimensional_scores()` and `compute_direction_signal()` functions already exist in `scoring/dimensional.py` and are imported by the pipeline â€” they just need to be called, assigned to `TickerScore`, and persisted. The `TickerScore` model already has `dimensional_scores: DimensionalScores | None` and `direction_confidence: float | None` fields (lines ~192-193 in `models/scan.py`). This task adds `market_regime` to the model, creates the DB migration, extends the repository, and wires Phase 2 of the pipeline.

- [ ] `save_ticker_scores()` persists dimensional_scores_json, direction_confidence, market_regime

- [ ] Pipeline Phase 2 calls `compute_direction_signal()` and assigns `direction_confidence` to TickerScore
- [ ] Pipeline Phase 2 derives `market_regime` from `signals.market_regime` using threshold mapping: >=80 CRISIS, >=60 VOLATILE, >=40 MEAN_REVERTING, <40 TRENDING
- [ ] All existing tests pass (0 regressions)

### TickerScore Model (`models/scan.py`)

- `dimensional_scores` and `direction_confidence` already exist on the model
- Add `field_validator("market_regime")` if needed for enum validation

### Repository (`data/repository.py`)
- **`save_ticker_scores()`** (~line 127): Extend INSERT to include 3 new columns. Serialize `dimensional_scores` via `score.dimensional_scores.model_dump_json() if score.dimensional_scores else None`
- **`_row_to_ticker_score()`** (~line 217): Deserialize `dimensional_scores_json` via `DimensionalScores.model_validate_json()` if non-NULL. Parse `market_regime` via `MarketRegime(value)` if non-NULL. Read `direction_confidence` as float if non-NULL.

### Pipeline Phase 2 (`scan/pipeline.py`)
- After `score_universe()` returns `list[TickerScore]`, iterate and for each ticker:
  1. Call `compute_dimensional_scores(raw_signals[ticker])` â†’ `DimensionalScores`
  2. Call `compute_direction_signal(...)` â†’ `DirectionSignal` (extract `.confidence`)
  3. Derive `MarketRegime` from `signals.market_regime` threshold mapping
  4. Assign all 3 to the `TickerScore` (may need to reconstruct since `TickerScore` could be frozen â€” check if it is)
- Note: `TickerScore` is NOT frozen (unlike snapshot models), so direct assignment should work
