# Task: Post-scan filter UI with presets and URL state

## Description

Build the frontend filter panel and quick-filter presets for post-scan dimensional filtering. Extract `ScanFilterPanel.vue` and `FilterPresets.vue` from the 915-line `ScanResultsPage.vue`. Implement debounced slider controls for dimensional scores, a market regime dropdown, earnings proximity filters, and 6 one-click quick-filter presets. All filter values sync to URL query params (extending the existing `router.replace()` pattern). Presets are TypeScript constants that populate the filter panel.

## Acceptance Criteria

- [ ] `ScanFilterPanel.vue` extracted as collapsible PrimeVue Panel with all filter controls
- [ ] Filter controls: sliders for min composite score, min direction confidence, min Trend, min IV/Vol, min Flow, min Risk
- [ ] Market regime Select dropdown (Trending / Mean Reverting / Volatile / Crisis / All)
- [ ] Earnings proximity filter (exclude within N days OR only within N days)
- [ ] All sliders debounced at 300ms before triggering API reload
- [ ] `FilterPresets.vue` with 6 quick-filter buttons: High IV Setups, Momentum Plays, Mean Reversion, Income/Theta, Earnings Plays, Low Risk
- [ ] Clicking a preset populates the filter panel with curated values (user sees + can adjust)
- [ ] Active preset visually highlighted; "Clear All" resets all filters
- [ ] All filter values sync to URL query params via `router.replace()`
- [ ] Deep links work: loading a URL with filter params restores filter state
- [ ] Browser back/forward preserves filter state

## Technical Details

### ScanFilterPanel.vue (new component)
- Collapsible PrimeVue `Panel` with header "Advanced Filters"
- Controls:
  - `Slider` for: min_confidence (0-100%), min_trend (0-100), min_iv_vol (0-100), min_flow (0-100), min_risk (0-100)
  - `Select` for: market_regime (5 options including "All")
  - `InputNumber` for: max_earnings_days, min_earnings_days
- Emit `filter-change` event with filter object on any change
- Debounce all slider inputs at 300ms using `useDebounceFn` from VueUse or manual `setTimeout`
- Show active filter count badge on panel header

### FilterPresets.vue (new component)
- 6 preset buttons as PrimeVue `Button` components
- Presets defined as TypeScript constants:
```typescript
const FILTER_PRESETS = {
  HIGH_IV: { label: 'High IV Setups', min_iv_vol: 70, min_confidence: 50 },
  MOMENTUM: { label: 'Momentum Plays', min_trend: 70, min_confidence: 60, market_regime: 'trending' },
  MEAN_REVERSION: { label: 'Mean Reversion', min_trend: 30, market_regime: 'mean_reverting' },
  INCOME: { label: 'Income/Theta', min_iv_vol: 50, min_risk: 30 },
  EARNINGS: { label: 'Earnings Plays', max_earnings_days: 14 },
  LOW_RISK: { label: 'Low Risk', min_risk: 20, max_risk: 40, min_confidence: 60 },
} as const
```
- Clicking a preset emits `preset-applied` with the filter values
- Active preset highlighted (compare current filters to preset values)
- "Clear All" button resets all filters to defaults

### URL State Sync
- Extend existing `router.replace()` pattern in ScanResultsPage
- Serialize filter values to query params: `?min_confidence=0.6&market_regime=trending&min_trend=70`
- On mount, read query params and restore filter state
- Omit default/empty values from URL (keep URLs clean)

### ScanResultsPage.vue Integration
- Import and place `FilterPresets` above `ScanFilterPanel`
- Wire `filter-change` and `preset-applied` events to API refetch
- Pass current filters as props to both components
- Reduce ScanResultsPage.vue by extracting filter logic (~150-200 lines)

### Pinia Store Extension
- Extend scan store with `filters` reactive state
- Add `setFilters()` and `clearFilters()` actions
- Fetch scores with filter params: `GET /api/scan/{id}/scores?min_trend=70&...`

### Edge Cases
- All filters at default (no filtering) → full result set
- All filters at maximum restriction → empty result set → show empty state
- Preset applied then manually adjusted → preset highlight removed
- URL params with invalid values → gracefully ignored, use defaults
- Rapid slider movement → only last debounced value triggers API call
- Browser back restores previous filter state
