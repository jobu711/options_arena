# Task: Add custom_tickers to ScanConfig and ScanRequest

## Description

Add `custom_tickers: list[str] = []` field to both `ScanConfig` (models/config.py) and
`ScanRequest` (api/schemas.py) with field validators that uppercase, strip, validate
against the ticker format regex, deduplicate, and cap at 200 entries.

## Acceptance Criteria

- [ ] `ScanConfig.custom_tickers` field with default `[]`
- [ ] `ScanRequest.custom_tickers` field with default `[]`
- [ ] Validator uppercases and strips whitespace from each ticker
- [ ] Validator rejects tickers not matching `_TICKER_RE` (`^(?=.*[A-Z0-9])[A-Z0-9^][A-Z0-9.\-^]{0,9}$`)
- [ ] Validator deduplicates via `list(dict.fromkeys(result))`
- [ ] Validator raises `ValueError` if list exceeds 200 tickers after dedup
- [ ] Empty list (default) produces no side effects — existing behavior preserved
- [ ] `mypy --strict` passes

## Technical Details

### models/config.py — ScanConfig
- Read `models/CLAUDE.md` before editing
- Add field after existing filter fields (after `theme_filters`)
- Import `re` if not already imported; define `_TICKER_RE` (or reuse if exists in the file)
- Validator pattern: `@field_validator("custom_tickers", mode="before")`
- Normalization: `[t.upper().strip() for t in v]`, then filter by regex, then dedup
- Cap: `if len(result) > 200: raise ValueError("custom_tickers exceeds 200 tickers")`

- Same validator logic as ScanConfig

### Edge Cases
- Empty string in list: `[""]` — rejected by regex
- Mixed valid/invalid: `["AAPL", "!!!"]` — invalid rejected, valid kept (or raise — TBD based on strictness; ScanConfig should reject invalid, ScanRequest should reject invalid)
- All duplicates: `["AAPL", "AAPL", "AAPL"]` → `["AAPL"]`
- Exactly 200 tickers: passes
- 201 tickers: raises ValueError
