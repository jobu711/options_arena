# Task: Pipeline Phase 1 custom tickers branch

## Description

In `scan/pipeline.py` `_phase_universe()`, add a branch that uses `custom_tickers` when
non-empty. Intersect with the optionable universe, skip preset/sector/industry/theme
filters, and proceed directly to OHLCV fetch.

## Acceptance Criteria

- [ ] When `settings.scan.custom_tickers` is non-empty, pipeline uses those tickers
- [ ] Custom tickers are intersected with optionable universe (non-optionable excluded)
- [ ] Excluded tickers logged at WARNING level
- [ ] Valid ticker count logged at INFO level
- [ ] Preset, sector, industry_group, and theme filters are bypassed
- [ ] OHLCV fetch and min-bars filter still apply (unchanged)
- [ ] When `custom_tickers` is empty, existing behavior is unchanged (regression)
- [ ] Phases 2-4 are completely unaffected

## Technical Details

- Read `scan/CLAUDE.md` before editing
- Insert custom_tickers branch after optionable universe fetch and sector_map build
- Location: after `all_tickers = await self._universe.fetch_optionable_tickers()` and
  sector_map construction, but BEFORE preset/sector/industry/theme filtering

```python

if custom:
    optionable_set = frozenset(all_tickers)
    valid = [t for t in custom if t in optionable_set]
    excluded = [t for t in custom if t not in optionable_set]

        logger.warning("Custom tickers not in optionable universe: %s", excluded)
    logger.info("Custom tickers: %d requested, %d valid", len(custom), len(valid))
```

- After the if/else, the existing OHLCV batch fetch continues as-is

- Custom tickers + sectors set → sectors ignored, custom tickers used
