# Task: Thread custom_tickers through scan API route

## Description

In `api/routes/scan.py` `start_scan()`, add `custom_tickers` to the `scan_overrides`
dict when the request body contains a non-empty list. This threads the field from the
API request through to the pipeline config via the existing `model_copy(update={...})`
pattern.

## Acceptance Criteria

- [ ] `body.custom_tickers` is threaded to `scan_overrides` when non-empty
- [ ] Empty `custom_tickers` (default) adds nothing to overrides — no behavior change
- [ ] The `model_copy(update={...})` pattern correctly passes custom_tickers to pipeline
- [ ] API returns 202 as before — no change to response shape
- [ ] `mypy --strict` passes
