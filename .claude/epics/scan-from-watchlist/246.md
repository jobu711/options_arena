# Task: Scan store param and WatchlistPage scan button

## Description

Extend the scan store's `startScan()` to accept a `customTickers` parameter, and add a
"Scan Watchlist" button to WatchlistPage that triggers a scan with the active watchlist's
tickers, shows WebSocket progress via ProgressTracker, and navigates to results on completion.

## Acceptance Criteria

- [ ] `startScan()` accepts optional `customTickers?: string[]` parameter
- [ ] `custom_tickers` included in POST body when `customTickers` is non-empty
- [ ] "Scan Watchlist" button visible on WatchlistPage when a watchlist is selected
- [ ] Button disabled when: no watchlist, 0 tickers, or `opStore.isActive`
- [ ] Clicking button starts scan with watchlist tickers as `custom_tickers`
- [ ] ProgressTracker shows scan phases inline (same as ScanPage)
- [ ] On completion, navigates to `/scan/{scanId}` (scan results page)
- [ ] On error, shows toast notification
- [ ] WebSocket cleaned up on unmount
- [ ] `vue-tsc --noEmit` passes

## Technical Details

### stores/scan.ts — startScan()
- Add `customTickers?: string[]` as last parameter
- In the request body builder:
  ```typescript
  if (customTickers && customTickers.length > 0) {
    body.custom_tickers = customTickers
  }
  ```

### pages/WatchlistPage.vue
- **Imports**: Add `useScanStore`, `useOperationStore`, `useWebSocket`, `ProgressTracker`,
  `useRouter`, `useToast`
- **State**: Add `scanning = ref(false)`, `wsClose` ref
- **Computed**: `canScan` — watchlist selected, has tickers, not `opStore.isActive`
- **Method**: `runWatchlistScan()`:
  1. `opStore.start('scan')`
  2. Extract tickers: `watchlistStore.activeWatchlist.tickers.map(t => t.ticker)`
  3. `const scanId = await scanStore.startScan('full', [], [], {}, [], tickers)`
  4. Connect WebSocket to `/ws/scan/${scanId}` with same callbacks as ScanPage
  5. On complete: `opStore.finish()`, `close()`, `router.push(\`/scan/${scanId}\`)`
  6. On error: toast warning
- **Template**: Add button + ProgressTracker (reuse ScanPage's SCAN_PHASES constant or
  define locally)
- **Cleanup**: `onUnmounted(() => wsClose?.())`

### Pattern Reference
- Copy the WebSocket + ProgressTracker pattern from `ScanPage.vue` lines ~82-155
- Same `SCAN_PHASES = ['universe', 'scoring', 'options', 'persist']`
- Same `updateProgress` / `addError` / `setComplete` callback structure
