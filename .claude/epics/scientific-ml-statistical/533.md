# Task: FRED Service Expansion + MacroContext Model

## Description
Extend `FredService` to fetch 8 FRED series (DGS10, DGS2, T10Y2Y, FEDFUNDS, VIXCLS, CPIAUCSL, INDPRO, UNRATE) with per-series TTL caching. Create `MacroContext` model in `models/macro.py` and `FredSeriesConfig` for series configuration. Add `fetch_macro_context()` batch method. Add `MLConfig(BaseModel)` submodel to `models/config.py` with feature flags.

## Acceptance Criteria
- [ ] `FredSeriesConfig` NamedTuple with id, display_name, ttl, transform type
- [ ] `MacroContext` Pydantic model with all 8 series fields (all `float | None`): treasury_10y, treasury_2y, yield_spread_10y2y, fed_funds_rate, vix, cpi_yoy, industrial_production_yoy, unemployment_rate
- [ ] `MacroRegimeResult` model with regime label, component signals, confidence
- [ ] `MLConfig(BaseModel)` submodel on `ScanConfig` with `enable_garch`, `enable_markov`, `enable_macro` flags (all `False` default)
- [ ] `fetch_macro_context()` batch-fetches all 8 series with per-series TTL via `asyncio.gather(return_exceptions=True)`
- [ ] Graceful degradation: partial FRED data returns partial `MacroContext` (unfetched fields stay `None`)
- [ ] Never raises — follows existing FRED `_fetch_with_cache` pattern
- [ ] `MacroContext.fallback()` classmethod returns all-None instance
- [ ] `MacroContext.completeness_ratio()` returns fraction of non-None fields
- [ ] All `float | None` fields have `math.isfinite()` validators

## Technical Details
- `models/macro.py`: New file. `MacroContext(BaseModel, frozen=True)`, `MacroRegimeResult(BaseModel, frozen=True)`, `FredSeriesConfig` (NamedTuple for series metadata)
- `services/fred.py`: Generalize `_fetch_from_fred(api_key, series_id)` to accept any series ID. Add `_MACRO_SERIES: list[FredSeriesConfig]` registry. Add `fetch_macro_context()` method
- `models/config.py`: Add `MLConfig(BaseModel)` with `enable_garch: bool = False`, `enable_markov: bool = False`, `enable_macro: bool = False`, `garch_p: int = 1`, `garch_q: int = 1`, `markov_n_regimes: int = 3`. Add `ml: MLConfig = MLConfig()` on `ScanConfig`
- `models/__init__.py`: Re-export `MacroContext`, `MacroRegimeResult`
- FRED series TTLs: daily series (DGS10, DGS2, T10Y2Y, FEDFUNDS, VIXCLS) → 24h; monthly series (CPIAUCSL, INDPRO, UNRATE) → 168h (7 days)
- Note: NAPM (ISM PMI) was removed from FRED in June 2016 — using INDPRO (Industrial Production Index) instead

### Edge Cases
- All 8 series return FRED missing-data marker `"."` → all None
- FRED returns negative values (e.g., negative yield spread) → valid, not rejected
- CPI/INDPRO need YoY transform (percentage change) → verify computation
- Network timeout on some series, others succeed → partial result
