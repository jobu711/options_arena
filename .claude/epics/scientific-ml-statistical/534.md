# Task: GARCH/EGARCH Volatility Forecasting + ADF Stationarity

## Description
Implement `compute_garch_forecast()`, `compute_egarch_forecast()`, and `test_stationarity()` in `indicators/vol_forecast.py` using the `arch` library. Include ADF stationarity testing via `statsmodels` as a gate before GARCH fitting. Add `arch` and `statsmodels` as optional dependencies. Add `vol_forecast_garch`, `vol_forecast_egarch`, `iv_vs_forecast_spread` fields to `IndicatorSignals`.

## Acceptance Criteria
- [ ] `test_stationarity(returns)` performs ADF test, returns `(is_stationary: bool, p_value: float)`
- [ ] `compute_garch_forecast(returns, p=1, q=1, horizon=5)` fits GARCH(1,1) and returns annualized h-step-ahead vol forecast
- [ ] `compute_egarch_forecast(returns, p=1, o=1, q=1, horizon=5)` fits EGARCH(1,1,1) and returns annualized vol forecast
- [ ] ADF stationarity test gates GARCH fitting — non-stationary returns skip GARCH
- [ ] Returns `None` on insufficient data (<252 observations, project policy)
- [ ] Returns `None` on convergence failure (`res.convergence_flag != 0`)
- [ ] Guarded `arch` import — returns `None` when `arch` not installed
- [ ] Guarded `statsmodels` import — returns `None` when `statsmodels` not installed
- [ ] `vol_forecast_garch`, `vol_forecast_egarch`, `iv_vs_forecast_spread` fields on `IndicatorSignals`
- [ ] `iv_vs_forecast_spread` = `market_iv - garch_forecast` (derived, computed when both available)
- [ ] Optional dependency group: `uv add --optional ml "arch>=7.0,<9" "statsmodels>=0.14"`

## Technical Details
- `indicators/vol_forecast.py`: New file. Functions operate on numpy arrays / pandas Series of **log returns** (not prices). No Pydantic models (indicator module purity). Returns `float | None`
- `arch` API (Context7-verified):
  - `arch_model(returns, vol='GARCH', p=1, q=1)` for GARCH
  - `arch_model(returns, vol='EGARCH', p=1, o=1, q=1)` for EGARCH — note `(p, o, q)` not `(p, q)`
  - Fit with `res = model.fit(disp='off')` — suppress optimizer output
  - Convergence check: `res.convergence_flag != 0` means failure (flag-based, not exception-based)
  - Forecast: `res.forecast(horizon=N).variance` → DataFrame with `h.1`..`h.N` columns
  - Last row of forecast DataFrame contains the out-of-sample variance forecast
  - Convert variance to vol: `sqrt(variance) * sqrt(252)` for annualization
- `statsmodels` ADF API (Context7-verified):
  - `from statsmodels.tsa.stattools import adfuller`
  - Returns `(adf_stat, p_value, usedlag, nobs, critical_values, icbest)`
  - Stationary if `p_value < 0.05`
- Returns are assumed to be **percentage log returns** (`np.log(price_t / price_{t-1}) * 100`)
- The 252-observation minimum is project policy for estimate reliability, not a library constraint
- `models/scan.py`: Add 3 new fields to `IndicatorSignals`: `vol_forecast_garch: float | None`, `vol_forecast_egarch: float | None`, `iv_vs_forecast_spread: float | None`

### Edge Cases
- Returns series with all identical values (zero variance) → convergence failure → None
- Returns series with single extreme outlier → GARCH should still converge
- Very short horizon (1) → single-step forecast
- Very long horizon (252) → annual forecast
- Flat price series → zero returns → convergence failure
