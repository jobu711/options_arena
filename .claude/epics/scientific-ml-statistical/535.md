# Task: Macro Regime Derivation + Agent Enrichment

## Description
Implement rule-based macro regime classification in `indicators/macro.py`. Enrich Fundamental Agent and Risk Agent prompts with macro context block. Add `render_macro_context()` to `agents/_parsing.py`. Add ML fields to `MarketContext` in `models/analysis.py`.

## Acceptance Criteria
- [ ] `compute_macro_regime()` in `indicators/macro.py` classifies expansionary/contractionary/transitional
- [ ] Classification rules: inverted yield curve (T10Y2Y < 0) + rising unemployment → contractionary; positive yield slope + declining unemployment → expansionary; mixed signals → transitional
- [ ] Returns `None` when `MacroContext` is incomplete (completeness_ratio < 0.5)
- [ ] `render_macro_context()` in `agents/_parsing.py` returns formatted string for agent prompts
- [ ] Fundamental Agent system prompt includes `{macro_block}` when macro data available
- [ ] Risk Agent system prompt includes macro regime + transition probabilities when available
- [ ] `MarketContext` has new optional fields: `macro_regime`, `yield_spread`, `fed_funds_rate`, `vix_level`
- [ ] All new `MarketContext` fields added to `validate_optional_finite` validator list
- [ ] Returns `None` when `MacroContext` completeness is insufficient

## Technical Details
- `indicators/macro.py`: New file. Pure math, no Pydantic models, no API calls. Input: `MacroContext` model. Output: `MacroRegimeResult | None`. Uses rule-based thresholds (inverted yield curve, unemployment trend, CPI change)
- `agents/_parsing.py`: Add `render_macro_context(ctx: MarketContext) -> str | None`. Uses existing `_render_optional()` pattern. Returns None when no macro data
- `agents/fundamental_agent.py`: Append macro context block to system prompt via `render_fundamental_context()` extension
- `agents/risk.py`: Append macro regime info to risk assessment context
- `models/analysis.py`: Add 4 optional fields to `MarketContext`: `macro_regime: str | None`, `yield_spread: float | None`, `fed_funds_rate: float | None`, `vix_level: float | None`
- Indicator module purity: `indicators/macro.py` takes typed `MacroContext` input (exception to "pandas in/out" rule since macro data is scalar, not time series)
