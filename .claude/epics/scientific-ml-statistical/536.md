# Task: Markov-Switching Regime Detection

## Description
Implement `compute_markov_regime()` in `indicators/regime_ml.py` using `statsmodels.tsa.regime_switching.markov_regression.MarkovRegression`. Returns regime labels, smoothed probabilities, and transition matrix. Complements (not replaces) existing heuristic regime detection in `indicators/regime.py`. Add `regime_markov_label`, `regime_transition_prob` fields to `IndicatorSignals`.

## Acceptance Criteria
- [ ] `compute_markov_regime(returns, k_regimes=3)` fits 3-regime Markov-switching model on daily returns
- [ ] Returns `MarkovRegimeOutput` NamedTuple: `current_regime: int`, `regime_probabilities: list[float]`, `transition_matrix: list[list[float]]`, `regime_label: str`
- [ ] Smoothed regime probabilities via `results.smoothed_marginal_probabilities`
- [ ] Transition matrix from `model.regime_transition_matrix(results.params)[:, :, 0]` (squeeze trailing dim)
- [ ] Maps 3 regimes to labels: low-vol (regime 0), normal (regime 1), high-vol (regime 2) — sorted by regime variance
- [ ] Falls back to `None` on convergence failure
- [ ] Uses `fit(search_reps=20)` for better convergence on difficult series
- [ ] Guarded `statsmodels` import — returns `None` when not installed
- [ ] Returns `None` on insufficient data (<252 observations)
- [ ] `regime_markov_label`, `regime_transition_prob` fields on `IndicatorSignals`
- [ ] Maps to existing `MarketRegime` enum where possible (no parallel taxonomy)

## Technical Details
- `indicators/regime_ml.py`: New file. Pure computation, no API calls, no Pydantic models
- `MarkovRegimeOutput`: NamedTuple (not Pydantic model — indicator module purity)
- `statsmodels` API (Context7-verified):
  - `MarkovRegression(returns, k_regimes=3)` — NOT `n_regimes`
  - Smoothed probs: `results.smoothed_marginal_probabilities` (T x k_regimes array)
  - Transition matrix: `model.regime_transition_matrix(results.params)[:, :, 0]` — must keep reference to both model AND results; squeeze from `(3,3,1)` to `(3,3)`
  - Use `fit(search_reps=20)` for robust convergence
- Regime labeling strategy: Sort regimes by estimated variance (from model parameters). Lowest variance → "low_vol", middle → "normal", highest → "high_vol"
- Mapping to `MarketRegime` enum: low_vol → `MEAN_REVERTING`, normal → `TRENDING`, high_vol → `VOLATILE` (heuristic mapping; crisis detection stays rule-based)
- Integration with `indicators/regime.py`: Markov regime is stored as ADDITIONAL fields on `IndicatorSignals`, not replacing existing heuristic regime. Existing `classify_market_regime()` untouched
- `models/scan.py`: Add 2 new fields: `regime_markov_label: float | None = None` (encoded as float for normalization), `regime_transition_prob: float | None = None`
- Shares stationarity infrastructure from Task 003 (imports `test_stationarity` from `vol_forecast.py`)
