# Task: Statistical Pipeline Integration

## Description
Wire GARCH, Markov-switching, and macro context into scan Phase 2/3. Enrich Volatility Agent with vol forecast vs IV comparison. Ensure all features are no-op when config flags are False. Verify existing test suite passes with default config (all ML disabled).

## Acceptance Criteria
- [ ] Phase 2 calls `compute_garch_forecast()` and `compute_markov_regime()` when `config.ml.enable_garch` / `config.ml.enable_markov` are True
- [ ] Phase 3 calls `fred.fetch_macro_context()` and `compute_macro_regime()` when `config.ml.enable_macro` is True
- [ ] Volatility Agent receives `iv_vs_forecast_spread` in rendered context block
- [ ] All features are complete no-ops when config flags are False (default)
- [ ] Existing test suite passes with default config (zero regressions)
- [ ] `iv_vs_forecast_spread` computed as `market_iv - garch_forecast` when both available
- [ ] ML indicator fields populated on `IndicatorSignals` during Phase 2
- [ ] Macro context fields populated on `MarketContext` during Phase 3
- [ ] Weight redistribution for new indicators in `scoring/composite.py` (maintains sum=1.0)
- [ ] `IndicatorSignals` field count comment updated to reflect new fields

## Technical Details
- `scan/phase_scoring.py`: Add `_compute_ml_indicators()` helper gated by `MLConfig` flags. Called after standard `compute_indicators()`. Uses `asyncio.to_thread()` for sync ML computations
- `scan/phase_options.py`: Add macro fetch + `compute_macro_regime()` gated by `enable_macro`. Populate `MarketContext` macro fields
- `agents/volatility.py`: Extend `render_volatility_context()` to include GARCH forecast vs IV comparison when `vol_forecast_garch` is present on context
- `agents/_parsing.py`: Add GARCH forecast rendering in `render_volatility_context()`
- `scoring/composite.py`: Add new ML indicator weights with proportional redistribution. Maintain `sum==1.0` import-time guard
- Config gating pattern: `if config.ml.enable_garch:` wraps all ML computation. When False, fields remain None on `IndicatorSignals` — scoring naturally skips None fields
- `asyncio.to_thread()` for GARCH/Markov (sync statsmodels/arch calls) with timeout via `asyncio.wait_for()`
