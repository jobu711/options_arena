# Task: Migrate OptionsDataService to ServiceBase

## Description
Migrate `OptionsDataService` to inherit from `ServiceBase[ServiceConfig]`. This is the most
complex migration: OptionsDataService has the most intricate constructor (6 parameters including
keyword-only `provider` and `openbb_config`), provider delegation pattern, and builds provider
list internally. The `_yf_call` usage is via providers (not directly), so only `_cached_fetch`
and `_retried_fetch` apply. Constructor signature must remain backward-compatible.

## Acceptance Criteria
- [ ] `OptionsDataService` inherits `ServiceBase[ServiceConfig]`
- [ ] `OptionsDataService.__init__` calls `super().__init__(config, cache, limiter)` + stores extras
- [ ] Extra params (`pricing_config`, `provider`, `openbb_config`) stay in subclass
- [ ] 2 cache blocks replaced with `self._cached_fetch()` calls
- [ ] Retry calls use `self._retried_fetch()`
- [ ] `OptionsDataService.close()` still iterates providers and calls close
- [ ] Constructor signature unchanged — backward compatible (positional args same order, kwargs preserved)
- [ ] All existing tests pass unchanged
- [ ] No consumer code changes
- [ ] Lint, format, and type check pass

## Technical Details

### OptionsDataService Migration
- Config type: `ServiceConfig`
- Constructor: `__init__(self, config, pricing_config, cache, limiter, *, provider=None, openbb_config=None)`
- `super().__init__(config, cache, limiter)` handles common triple
- `self._pricing_config`, `self._providers`, `self._openbb_config` stay in subclass
- Provider list building (CBOE primary, yfinance fallback, optional validation) stays in subclass
- 2 cache blocks → convert to `_cached_fetch()` (likely with `deserializer` callback for chain data)
- `_yf_call` usage is via `YFinanceChainProvider`, NOT directly → no `_yf_call` migration needed
- `close()` override: iterates `self._providers`, duck-type `getattr("close")` → stays as-is

### Key Risk
- Most complex constructor — careful handling of positional vs keyword args
- Provider delegation pattern must be preserved exactly
- Chain data caching may use list serde → `deserializer` callback
