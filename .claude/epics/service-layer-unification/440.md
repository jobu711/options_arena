# Task: Migrate UniverseService to ServiceBase

## Description
Migrate `UniverseService` to inherit from `ServiceBase[ServiceConfig]`. UniverseService is
moderate complexity with 6 cache methods, httpx client, and rate limiter usage. Constructor
signature must remain backward-compatible. All existing tests must pass without modification.

## Acceptance Criteria
- [ ] `UniverseService` inherits `ServiceBase[ServiceConfig]`
- [ ] `UniverseService.__init__` calls `super().__init__(config, cache, limiter)` + stores extras
- [ ] 6 cache blocks replaced with `self._cached_fetch()` calls
- [ ] Retry calls use `self._retried_fetch()`
- [ ] `UniverseService.close()` still calls `self._client.aclose()`
- [ ] Constructor signature unchanged — backward compatible
- [ ] All existing tests pass unchanged
- [ ] No consumer code changes (`cli/commands.py`, `api/app.py`, `scan/pipeline.py`)
- [ ] Lint, format, and type check pass

## Technical Details

### UniverseService Migration
- Config type: `ServiceConfig` (has `yfinance_timeout`, `request_timeout`)
- 6 cached methods use various serde patterns — most are direct model serde via `_cached_fetch`
- Some may use list/batch patterns requiring `deserializer` callback
- Uses retry pattern → delegate to `self._retried_fetch()`
- Has httpx client → override `close()` to call `self._client.aclose()`
- `self._client` stays in subclass `__init__`
- No `_yf_call` needed (uses httpx, not yfinance)
