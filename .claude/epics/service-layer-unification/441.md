# Task: Migrate FredService and OpenBBService to ServiceBase

## Description
Migrate `FredService` and `OpenBBService` to inherit from `ServiceBase`. These are the
two simplest services: FredService has no limiter and uses httpx only; OpenBBService has
no httpx client and no `_yf_call`. Constructor signatures must remain backward-compatible.
All existing tests must pass without modification.

## Acceptance Criteria
- [ ] `FredService` inherits `ServiceBase[ServiceConfig]`
- [ ] `FredService.__init__` calls `super().__init__(config, cache, limiter=None)` + stores extras
- [ ] `FredService.close()` still calls `self._client.aclose()`
- [ ] FredService inline cache blocks replaced where applicable (scalar float may stay inline)
- [ ] `OpenBBService` inherits `ServiceBase[OpenBBConfig]`
- [ ] `OpenBBService.__init__` calls `super().__init__(config, cache, limiter)`
- [ ] OpenBBService cache blocks replaced with `self._cached_fetch()` (3 methods)
- [ ] Constructor signatures unchanged — backward compatible
- [ ] All existing FredService tests pass unchanged
- [ ] All existing OpenBBService tests pass unchanged
- [ ] No changes to `cli/commands.py`, `api/app.py`, `api/deps.py`, or `scan/pipeline.py`
- [ ] Lint, format, and type check pass

## Technical Details

### FredService Migration
- Currently: standalone class with `__init__`, `close()`, cache-first fetch for risk-free rate
- FredService has unique `CachedRate` NamedTuple for staleness-aware caching of a scalar float
- Triple-tier cache is complex enough to stay inline (per PRD decision)
- `super().__init__(config, cache, limiter=None)` — FredService is the only service without limiter
- Extra param: `pricing_config: PricingConfig` stays in subclass
- `self._client` (httpx.AsyncClient) stays in subclass `__init__`
- `close()` overrides default to call `self._client.aclose()`

### OpenBBService Migration
- Currently: standalone class with guarded import, 3 cached methods
- All 3 cache blocks use direct `model_validate_json` pattern → convert to `_cached_fetch()`
- Never-raise contract: methods return `None` on error — no change to error handling
- No httpx client → default `close()` no-op is correct
- No `_yf_call` needed
- No `_retried_fetch` needed
