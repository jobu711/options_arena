# Task: CLI --sector flag, universe sectors subcommand, ETF preset wiring

## Description
Replace the existing `--sectors` CLI stub with a proper `--sector` repeatable flag using Typer's multi-value enum pattern. Add a `universe sectors` subcommand. Wire the ETF preset through the CLI.

## Acceptance Criteria
- [ ] `scan` command: `--sectors` stub replaced with `--sector` repeatable flag accepting `GICSSector` values
- [ ] `options-arena scan --sector technology` filters to Information Technology tickers
- [ ] `options-arena scan --sector technology --sector healthcare` filters to both sectors (OR logic)
- [ ] `--sector` composes with `--preset`: `--preset sp500 --sector technology` works
- [ ] Invalid sector names produce a clear error listing valid options
- [ ] Scan output header shows active sector filter when `--sector` is used
- [ ] `options-arena universe sectors` subcommand lists all 11 GICS sectors with ticker counts
- [ ] `options-arena scan --preset etfs` works (no longer logs "not yet implemented" warning)
- [ ] `options-arena universe list --preset etfs` shows detected ETF count
- [ ] All existing tests pass

## Technical Details
- **scan command** (`commands.py`): Replace `sectors: str | None = typer.Option(None, "--sectors", ...)` with `sector: list[GICSSector] = typer.Option([], "--sector", "-s", help="Filter by GICS sector (repeatable)")`. If Typer doesn't handle lowercase `list[GICSSector]`, fall back to `typing.List[GICSSector]`. Pass to `ScanConfig.sectors`.
- **_scan_async**: Remove the "not yet implemented" warning for sectors. Pass `sectors` to `ScanConfig`. Remove the "not yet implemented" warning for ETFS preset.
- **universe sectors subcommand**: New command under `universe_app`. Fetch `sp500_constituents`, group by sector, display Rich table with sector name + ticker count. Sort by count descending.
- **Scan output**: When `sectors` is non-empty, add a line to the scan header: "Sector filter: Technology, Health Care" (comma-separated canonical names).
- **list_tickers**: Update existing `--sector` single-value filter to use `GICSSector` enum instead of raw `str | None`.
