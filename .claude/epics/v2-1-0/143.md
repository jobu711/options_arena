# Task: Quick Debate from Dashboard

## Description

Add a "Quick Debate" input to the DashboardPage so power users can type a ticker symbol and
immediately start a debate without navigating through scan results first. This is pure frontend
work — the `POST /api/debate` endpoint already accepts `scan_id: null` for ticker-only debates.

## Acceptance Criteria

- [ ] InputText + "Debate" Button added to DashboardPage quick-actions bar
- [ ] Ticker input uppercases automatically and validates non-empty before submit
- [ ] Submitting calls `POST /api/debate` with `{ ticker, scan_id: null }`
- [ ] WebSocket connection opens to `/ws/debate/{id}` showing existing DebateProgressModal
- [ ] Auto-navigates to `/debate/{id}` on debate completion
- [ ] Error toast shown if debate API returns error (e.g., invalid ticker, 409 busy)
- [ ] Enter key submits the form (keyboard accessible)
- [ ] 3 Playwright E2E tests covering: submit + navigate, error handling, enter-key submit

## Technical Details

### Files Modified
- `web/src/pages/DashboardPage.vue` — add InputText + Button to quick-actions section
- `web/src/stores/debate.ts` (if needed) — reuse existing debate store methods
- `web/tests/e2e/` — new E2E test file for quick debate

### Implementation Approach
1. Add PrimeVue `InputText` (placeholder "Ticker symbol...") + `Button` (label "Debate") to the
   existing quick-actions bar in DashboardPage, positioned alongside "New Scan" etc.
2. On submit: call `POST /api/debate` with `{ ticker: inputValue.toUpperCase() }`. The API already
   handles `scan_id: null` by creating a minimal TickerScore and fetching fresh option chains.
3. On success (returns `{ debate_id }`): open WebSocket to `/ws/debate/{debate_id}`, show
   `DebateProgressModal` (already exists as a component), auto-navigate on completion event.
4. On error: show PrimeVue `Toast` with error message.

### Key Considerations
- The existing `DebateProgressModal` component handles WebSocket connection and progress display —
  reuse it rather than building new progress UI.
- The debate API returns 409 if an operation is already in progress (mutex lock) — handle gracefully.
- No backend changes needed whatsoever.
