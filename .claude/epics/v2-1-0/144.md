# Task: Watchlist Backend

## Description

Implement the full backend for watchlist CRUD: Pydantic models, repository methods, REST API
endpoints, and CLI subcommand group. The `watchlists` and `watchlist_tickers` tables already
exist in `001_initial.sql` — no schema migration needed.

## Acceptance Criteria

- [ ] `Watchlist` and `WatchlistTicker` Pydantic models added to `models/`
- [ ] 6 repository methods in `data/repository.py`:
  - `create_watchlist(name) -> Watchlist`
  - `delete_watchlist(watchlist_id) -> None`
  - `add_ticker_to_watchlist(watchlist_id, ticker) -> None`
  - `remove_ticker_from_watchlist(watchlist_id, ticker) -> None`
  - `get_watchlists() -> list[Watchlist]`
  - `get_tickers_for_watchlist(watchlist_id) -> list[WatchlistTicker]`
- [ ] `routes/watchlist.py` with 6 REST endpoints:
  - `POST /api/watchlist` — create watchlist (body: `{ name }`)
  - `DELETE /api/watchlist/{id}` — delete watchlist
  - `GET /api/watchlist` — list all watchlists
  - `GET /api/watchlist/{id}` — get watchlist with tickers + latest scores
  - `POST /api/watchlist/{id}/tickers` — add ticker (body: `{ ticker }`)
  - `DELETE /api/watchlist/{id}/tickers/{ticker}` — remove ticker
- [ ] `GET /api/watchlist/{id}` enriches each ticker with latest score, direction, last debate date
  (joins against `ticker_scores` and `ai_theses`)
- [ ] `options-arena watchlist` CLI subcommand group (`list`, `create`, `delete`, `add`, `remove`)
- [ ] ~40 Python tests: repository CRUD, API endpoint integration, CLI smoke tests
- [ ] All tests pass: `uv run pytest`, `uv run mypy src/ --strict`, `uv run ruff check .`

### Implementation Approach

1. **Models**: `Watchlist(id: int, name: str, created_at: datetime)` and
   `WatchlistTicker(id: int, watchlist_id: int, ticker: str, added_at: datetime)`.
   Both `frozen=True` with UTC validators on datetime fields.

2. **Repository**: All methods async, parameterized queries, typed returns. Use existing
   `row_factory` pattern. `create_watchlist` returns the created model with `lastrowid`.
   Handle UNIQUE constraint on `(watchlist_id, ticker)` → raise appropriate error.

3. **API**: Thin route handlers delegating to repository. `GET /api/watchlist/{id}` does
   enrichment by joining ticker list with latest scores (from `get_scores_for_scan` on
   latest scan) and last debate date (from `get_debates_for_ticker`).

4. **CLI**: Typer subcommand group registered on main app. Rich Table for list output.
   `asyncio.run()` wrapper for async repository calls.

### Key Considerations
- Read `data/CLAUDE.md` before modifying repository — strict patterns for async queries
- Read `api/CLAUDE.md` before creating routes — dependency injection, error mapping
- Read `cli/CLAUDE.md` before adding subcommands — Typer patterns, Rich output
- Watchlist name uniqueness enforced at DB level (UNIQUE constraint) — map `IntegrityError` to 409
