# Task: Watchlist Frontend

## Description

Build the web UI for watchlists: a new WatchlistPage, Pinia store, "Add to Watchlist" buttons
in TickerDrawer and scan results, and a dashboard quick-action link. Depends on Task 2
(Watchlist Backend) for the API endpoints.

## Acceptance Criteria

- [ ] New `WatchlistPage.vue` at `/watchlist` route with:
  - Watchlist selector (dropdown or tab bar for multiple watchlists)
  - PrimeVue DataTable showing: ticker, latest composite score, direction badge, last debate date
  - "Create Watchlist" button + name input dialog
  - "Delete Watchlist" button with confirmation
  - Row click navigates to debate or opens TickerDrawer
- [ ] Pinia `watchlistStore` with CRUD actions calling watchlist API endpoints
- [ ] "Add to Watchlist" button in TickerDrawer (dropdown to select which watchlist)
- [ ] "Add to Watchlist" context action available from scan results rows
- [ ] Dashboard quick-actions bar includes "Watchlists" navigation button
- [ ] Empty state when no watchlists exist ("Create your first watchlist")
- [ ] 5 Playwright E2E tests: create watchlist, add ticker, view watchlist, remove ticker, delete watchlist
- [ ] `npx vue-tsc --noEmit` passes

## Technical Details

### Files Created
- `web/src/pages/WatchlistPage.vue` — main watchlist page
- `web/src/stores/watchlist.ts` — Pinia store for watchlist state
- `web/src/types/watchlist.ts` — TypeScript interfaces (`Watchlist`, `WatchlistTicker`)
- `web/tests/e2e/watchlist.spec.ts` — E2E tests

### Files Modified
- `web/src/router/index.ts` — add `/watchlist` route (lazy-loaded)
- `web/src/components/TickerDrawer.vue` — add "Add to Watchlist" button
- `web/src/pages/DashboardPage.vue` — add "Watchlists" quick-action button
- `web/src/pages/ScanResultsPage.vue` — add watchlist action to row context

### Implementation Approach

1. **Types**: `Watchlist { id: number; name: string; created_at: string }` and
   `WatchlistTicker { ticker: string; score?: number; direction?: string; last_debate_date?: string }`.

2. **Store**: `watchlistStore` with state (`watchlists`, `activeWatchlist`, `tickers`),
   actions (`fetchWatchlists`, `createWatchlist`, `deleteWatchlist`, `addTicker`, `removeTicker`,
   `fetchTickers`). All actions call REST endpoints from Task 2.

3. **WatchlistPage**: Watchlist tabs/dropdown at top, DataTable below. "Create" opens PrimeVue
   Dialog with name input. "Delete" opens ConfirmDialog. Table columns: ticker (sortable),
   score (sortable, colored), direction (badge), last debate (relative time).

4. **TickerDrawer integration**: New "Add to Watchlist" section below existing Actions section.
   Dropdown populated from `watchlistStore.watchlists`. Shows checkmarks for watchlists the
   ticker is already in.

5. **Dashboard**: Add "Watchlists" Button alongside "New Scan", "View Universe", "Health Check".

### Key Considerations
- Follow existing PrimeVue Aura dark theme patterns from other pages
- Reuse existing direction badge component pattern from ScanResultsPage
- Read `web/CLAUDE.md` for Vue 3 composition API patterns and PrimeVue usage
