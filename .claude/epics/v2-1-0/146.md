# Task: Scan Delta View

## Description

Full-stack implementation of scan comparison: models, API endpoint, and frontend UI. Users can
select a previous scan to compare against the current one, seeing score changes, new/removed
tickers, and top movers. No new repository methods needed — diff is computed in the route
handler from two `get_scores_for_scan()` calls.

## Acceptance Criteria

- [ ] `TickerDelta` model: `ticker`, `current_score`, `previous_score`, `score_change`, `current_direction`, `previous_direction`, `is_new` flag
- [ ] `ScanDiff` model: `current_scan_id`, `base_scan_id`, `added: list[str]`, `removed: list[str]`, `movers: list[TickerDelta]` (sorted by abs score change desc)
- [ ] `GET /api/scan/{id}/diff?base_id={prev_id}` endpoint returning `ScanDiff`
- [ ] Frontend: "Compare with..." dropdown on ScanResultsPage populated from scan history
- [ ] Delta chips next to scores: green up-arrow for improved, red down-arrow for drops
- [ ] "NEW" badge for tickers appearing in current scan but not baseline
- [ ] Top movers summary section (top 5 by absolute score change)
- [ ] ~25 Python tests (models, diff computation, API endpoint)
- [ ] ~3 E2E tests (select comparison, verify delta chips, verify NEW badges)

### Implementation Approach

1. **Models**: `TickerDelta` with `frozen=True`, computed `score_change` property.
   `ScanDiff` with `frozen=True`, lists of added/removed tickers, sorted movers.

2. **API endpoint**: `GET /api/scan/{id}/diff?base_id=`. Handler calls `repo.get_scores_for_scan()`
   twice (current + base), builds dict keyed by ticker, computes set operations for
   added/removed, calculates score deltas for common tickers, sorts by abs change.

3. **Frontend**: Add `Select` dropdown above DataTable (populated from `scanStore.scans`).
   When selected, fetch diff from API. For each ticker in table:
   - If in `movers`: show colored chip with `+2.3` / `-1.5` next to score
   - If in `added`: show "NEW" PrimeVue Tag
   - Top movers summary as collapsible section above the table

4. **State management**: Diff data stored as local ref in ScanResultsPage (not a store —
   it's page-scoped). Compare dropdown value synced to URL query param `?compare=`.

### Key Considerations
- Diff computation is O(n) where n = max tickers per scan (~50 in top_n) — trivially fast
- No new repository methods needed — reuse `get_scores_for_scan()`
- Handle case where base scan has no scores (return empty diff, not error)
- Read `api/CLAUDE.md` before modifying scan routes
