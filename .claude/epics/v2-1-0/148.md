# Task: Score History Frontend

## Description

Build the web UI for score history visualization: a reusable SVG line chart component, inline
sparklines for DataTable cells, a dedicated ticker page, and Dashboard "Trending" sections.
All charts are hand-crafted SVG — no charting library dependency. Depends on Task 5 for API.

## Acceptance Criteria

- [ ] New `ScoreHistoryChart.vue` component:
  - SVG line chart with dates on X-axis, composite score (0-100) on Y-axis
  - Direction encoded as dot color: green (bullish), red (bearish), yellow (neutral)
  - Hover tooltip showing exact score, date, and direction
  - Responsive width, fixed height (~200px)
- [ ] New `SparklineChart.vue` component:
  - Inline SVG sparkline (~80px wide, ~24px tall)
  - Shows last 10 scores as a mini line chart
  - Colored by latest direction
- [ ] New `/ticker/{ticker}` page (`TickerDetailPage.vue`):
  - Full ScoreHistoryChart
  - Latest score + direction summary
  - Past debates list (reuse TickerDrawer pattern)
- [ ] TickerDrawer integration: ScoreHistoryChart shown below score section
- [ ] ScanResultsPage: SparklineChart in new "Trend" column for each ticker
- [ ] Dashboard: "Trending Up" and "Trending Down" sections (3+ consecutive scans)
  - Each shows top 5 tickers with sparkline and latest score
  - Data from `GET /api/ticker/trending?direction=bullish`
- [ ] 5 Playwright E2E tests: chart renders, sparklines in table, ticker page, trending sections

### Implementation Approach

1. **ScoreHistoryChart SVG**: Viewbox-based SVG with computed polyline points.
   X-axis: evenly spaced dates. Y-axis: score 0-100 mapped to chart height.
   Circles at each data point colored by direction. `<title>` elements for native tooltips
   (or custom tooltip div positioned on mouseover).

2. **SparklineChart SVG**: Minimal SVG `<polyline>` with no axes, labels, or decorations.
   Points computed from score array, normalized to viewbox. Stroke color from latest direction.

3. **TickerDetailPage**: Route param `:ticker`. Fetches history from
   `GET /api/ticker/{ticker}/history` on mount. Renders ScoreHistoryChart + metadata.
   Reuses debate fetching pattern from TickerDrawer.

4. **Dashboard trending**: Two sections below latest scan card. Each calls
   `GET /api/ticker/trending?direction=bullish` (and bearish). Shows PrimeVue DataTable
   with ticker, sparkline, score, consecutive scan count.

5. **ScanResultsPage sparklines**: For each ticker in the results table, need history data.
   Options: (a) batch fetch all histories when page loads, (b) lazy-load per visible row.
   Recommend (a) with a single batch API call since table is limited to ~50 tickers.

### Key Considerations
- SVG must be responsive — use `viewBox` + `preserveAspectRatio`, not fixed pixel dimensions
- Dark theme: use CSS variables from PrimeVue Aura theme for colors
- Sparkline data for scan results needs a batch endpoint or the frontend fetches per-ticker
  (consider adding `GET /api/scan/{id}/scores-with-history` if per-ticker is too many requests)
- Keep SVG simple — no animations, no complex interactions beyond hover tooltips
