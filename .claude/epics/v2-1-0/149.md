# Task: Earnings Calendar Overlay

## Description

Full-stack implementation of earnings awareness: service method to fetch earnings dates from
yfinance, population of the existing `MarketContext.next_earnings` field during scan Phase 3,
injection of earnings warnings into debate agent prompts, and frontend display in scan results
table and TickerDrawer. The `MarketContext.next_earnings: date | None` field already exists —
it just needs to be populated.

## Acceptance Criteria

### Backend
- [ ] `MarketDataService.fetch_earnings_date(ticker) -> date | None` method:
  - Uses yfinance `Ticker.calendar` via `asyncio.to_thread`
  - 24h cache TTL via existing `ServiceCache`
  - Returns `None` gracefully if yfinance has no data
- [ ] `MarketContext.next_earnings` populated during Phase 3 of scan pipeline
  (alongside existing ticker info fetch)
- [ ] Earnings date included in scan results API response
- [ ] Agent prompt injection: when `next_earnings` is set and < 7 days from now,
  append "WARNING: Earnings in {N} days. IV crush risk is elevated." to bull/bear/risk prompts
- [ ] ~30 Python tests: service method, caching, prompt injection, model validation

### Frontend
- [ ] "Earnings" column in ScanResultsPage DataTable:
  - Shows DTE to next earnings (e.g., "3d", "45d")
  - Red text + warning icon if < 7 days
  - Gray text if >= 7 days
  - "—" if no earnings data
- [ ] Earnings warning banner in TickerDrawer when < 7 days:
  - PrimeVue Message component (severity="warn")
  - "Earnings in {N} days — IV crush risk"
- [ ] ~3 E2E tests: earnings column renders, warning banner shows, no-data graceful

### Implementation Approach

1. **Service method**: Add `fetch_earnings_date()` to `MarketDataService`. Use yfinance
   `Ticker(symbol).calendar` which returns a dict with earnings dates. Wrap in
   `_yf_call()` pattern (asyncio.to_thread + wait_for timeout). Cache with 24h TTL.
   Return `date | None`.

2. **Scan pipeline integration**: In Phase 3 where `MarketContext` is built, call
   `fetch_earnings_date()` alongside existing data fetches. Set `next_earnings` field.
   Use `asyncio.gather` with existing parallel fetches.

3. **Agent prompt injection**: In the prompt building functions (bull, bear, risk agents),
   check if `market_context.next_earnings` is set and within 7 days. If so, append
   the warning string. This follows existing pattern of conditional prompt sections.

4. **Frontend earnings column**: Compute DTE from earnings date. Use PrimeVue Column
   with custom template. Color logic: `dte < 7 ? 'text-red-500' : 'text-gray-400'`.

5. **TickerDrawer warning**: Conditional PrimeVue `Message` component at top of drawer
   when earnings < 7 days. Same DTE computation.

### Key Considerations
- yfinance `Ticker.calendar` coverage varies — ~80% for S&P 500, less for small caps.
  `None` return is expected and handled gracefully everywhere.
- Cache TTL of 24h prevents hammering yfinance during repeated scans
- Read `services/CLAUDE.md` for service patterns (cache-first, asyncio.to_thread wrapping)
- Read `agents/CLAUDE.md` and `agents/prompts/CLAUDE.md` for prompt modification patterns
- The `MarketContext.next_earnings` field already exists — do NOT add it again
