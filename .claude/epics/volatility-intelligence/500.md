## Technical Details
- `smooth_iv_parity()` algorithm (from PRD FR-V1):
  1. Validate both IVs finite and positive. If only one valid, return it
  2. Compute relative spread: `spread_pct = (ask - bid) / mid` where `mid = (ask + bid) / 2`
  3. If both spreads zero/non-finite, return `(call_iv + put_iv) / 2`
  4. Weights as inverse spread: `w = 1 / max(spread_pct, 0.001)`
  5. Normalize and return weighted average
- Edge cases: zero-bid (use ask-based spread), both IVs valid but ratio > 2.0 (log warning, still average), one IV is 0 or NaN (use valid side)
- `smoothed_iv` on frozen `OptionContract` — set via `model_copy(update=...)`

### Edge Cases
- Both IVs NaN: return NaN
- One IV zero, one positive: return positive
- Zero-bid contract: spread = ask, weight accordingly
- Very wide spread vs very tight spread: weight toward tight side
- Both spreads non-finite: simple average fallback
