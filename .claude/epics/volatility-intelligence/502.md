# Task: Surface Indicators Completion

## Description
Complete the `VolSurfaceIndicators` NamedTuple and `compute_surface_indicators()` function in `indicators/vol_surface.py`. The function maps per-contract (strike, dte) keys to z-scores from the already-computed `VolSurfaceResult`. Add `fit_vol_surface: bool = True` toggle to `ScanConfig`.

## Acceptance Criteria
- [ ] `VolSurfaceIndicators` has fields: `iv_surface_residual`, `surface_fit_r2`, `surface_is_1d`
- [ ] `compute_surface_indicators()` maps contract keys to z-scores from `VolSurfaceResult`
- [ ] Returns empty/None indicators when `VolSurfaceResult.is_standalone_fallback is True`
- [ ] Returns empty/None indicators when `z_scores` is None
- [ ] Handles key lookup miss gracefully (z-score not found for a contract)
- [ ] `ScanConfig.fit_vol_surface` defaults to `True`
- [ ] All existing vol_surface tests pass unchanged

## Technical Details
- `VolSurfaceResult` already has `z_scores: np.ndarray | None`, `r_squared: float | None`, `is_1d_fallback: bool`, `is_standalone_fallback: bool`
- The z-scores array is parallel to the input arrays (strikes, ivs, dtes) passed to `compute_vol_surface()`
- `compute_surface_indicators()` needs contract keys to index into the z-scores array
- Updated signature: `compute_surface_indicators(result, strikes, dtes)` where strikes/dtes are the arrays that were passed to `compute_vol_surface()` (same order = same indices)
- Lookup: for a contract at (strike_f, dte_f), find the index where `strikes[i] == strike_f and dtes[i] == dte_f`, then `z_scores[i]` is that contract's residual
- For recommended contract, caller passes single (strike, dte) and gets z-score
