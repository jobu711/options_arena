# Task: Agent Prompt Enrichment + Scoring Tiebreaker

## Description
Enhance `render_volatility_context()` in `agents/_parsing.py` with surface mispricing fields (iv_surface_residual, surface_fit_r2). Add delta tiebreaker in `select_by_delta()` that favors underpriced vol for BULLISH direction and overpriced vol for BEARISH direction when delta distances are close.

## Acceptance Criteria
- [ ] `render_volatility_context()` includes "IV VS SURFACE" and "SURFACE R²" lines
- [ ] Mispricing classified: |z| < 0.5 → fair, z > 0.5 → overpriced, z < -0.5 → underpriced
- [ ] Significantly over/underpriced noted when |z| > 2.0
- [ ] `select_by_delta()` uses iv_surface_residual as tiebreaker when delta distances within 0.02
- [ ] BULLISH: prefer lower residual (underpriced vol is better to buy)
- [ ] BEARISH: prefer higher residual (overpriced vol is better to sell)
- [ ] Tiebreaker only applies when `smoothed_iv` or residual data is available
- [ ] No change to existing behavior when surface data is None

## Technical Details
### Rendering Enhancement
- Add to the "HV & Vol Surface" section (after line 532):
  ```python

  ```

### Scoring Tiebreaker
- In `select_by_delta()`, modify `_sort_key` to include residual-based tiebreaker:
  - Current key: `(effective_distance, strike)`
  - New key: `(effective_distance, tiebreaker, strike)`
  - `tiebreaker = 0.0` when no residual data
  - For BULLISH: `tiebreaker = residual` (lower = better, underpriced preferred)
  - For BEARISH: `tiebreaker = -residual` (lower = better = more overpriced preferred)
  - Only activates when contracts are within 0.02 delta distance of each other
- Need to add `direction: SignalDirection | None = None` parameter to `select_by_delta()`
- The residual comes from `contract.smoothed_iv` presence (if smoothing was applied) — need to access via a lookup. Alternatively, pass a `residuals: dict[tuple[Decimal, date], float]` parameter

       effective = delta_dist / max(liq, 0.01)
