# Task: Engine pre-requisites for Web API

## Description

Three additive changes to the existing engine that the FastAPI backend requires. These are
non-breaking — CLI and all 1,484 existing tests must remain green. No new external
dependencies.

### 1. Convert `DebateResult` from `@dataclass` to Pydantic `BaseModel`

**File**: `src/options_arena/agents/_parsing.py`

`DebateResult` is currently a `@dataclass`. FastAPI can't auto-serialize dataclasses with
the same fidelity as Pydantic models (especially nested Pydantic fields like `AgentResponse`,
`TradeThesis`, `MarketContext`). Convert to `BaseModel` with `ConfigDict(frozen=True)`.

**Complication**: `RunUsage` from `pydantic_ai` — verify it's Pydantic-compatible. If not,
wrap it or use `model_config = ConfigDict(arbitrary_types_allowed=True)`.

Update all tests that construct `DebateResult` instances (keyword args instead of positional).

### 2. Add `DebateProgressCallback` to debate orchestrator

**File**: `src/options_arena/agents/orchestrator.py`

Add:
- `DebatePhase(StrEnum)` with values: `BULL`, `BEAR`, `REBUTTAL`, `VOLATILITY`, `RISK`
- `DebateProgressCallback` protocol: `(phase: DebatePhase, status: str, confidence: float | None) -> None`
- Optional `progress` parameter to `run_debate()` — called on agent start/complete
- Default `None` — no progress reporting (backward-compatible with CLI)

**Files also touched**: `agents/__init__.py` (re-export new types)

### 3. Add `get_debate_by_id()` to Repository

**File**: `src/options_arena/data/repository.py`

Add method to fetch a single debate row by primary key from `ai_theses` table.
Return type: `DebateRow | None`. Currently only `get_debates_for_ticker()` and
`save_debate()` exist.

**Files also touched**: `data/__init__.py` (re-export if needed)
