# Task: Debate API — REST endpoints + WebSocket progress + export

## Description

Implement the debate API: REST endpoints for starting single-ticker debates, fetching
results, listing debate history, and exporting results. WebSocket for real-time debate
agent progress. This is the backend half of the "debate vertical slice."

### REST endpoints (`src/options_arena/api/routes/debate.py`)

```
POST /api/debate            — Start single-ticker debate (ticker + optional scan_id).
                              Returns 202 + debate_id.
GET  /api/debate             — List past debates (?ticker, ?limit=20). Returns summaries.
GET  /api/debate/{debate_id} — Full debate result (all agent responses + thesis)
```

### Export endpoint (`src/options_arena/api/routes/export.py`)

```
GET /api/debate/{debate_id}/export?format=md|pdf — Download debate as file
```

Uses existing `reporting/debate_export.py`. PDF returns 501 if `weasyprint` not installed.
Sets `Content-Disposition: attachment` header.

### WebSocket (`src/options_arena/api/ws.py` — extend from Task #126)

```
WS /ws/debate/{debate_id} — Subscribe to debate progress events
```

Uses the `DebateProgressCallback` from Task #122. Bridges callback to `asyncio.Queue`,
drains to WebSocket:
- `{type: "agent", name: "bull"|"bear"|..., status: "started"|"completed"|"failed", confidence}`
- `{type: "complete", debate_id}`

### Request/response schemas (`src/options_arena/api/schemas.py` — extend)

- `DebateRequest(BaseModel)` — `ticker: str, scan_id: int | None`
- `DebateStarted(BaseModel)` — `debate_id: int`
- `DebateResultSummary(BaseModel)` — `id, ticker, verdict, confidence, created_at`
  (lightweight for list endpoint)

### Background task pattern

`POST /api/debate` starts debate orchestrator as `asyncio.create_task()`. Single-ticker
debates do NOT acquire the operation lock (fast enough, ~60s). Stores task in `app.state`
keyed by debate_id. On completion, persists result via Repository.
