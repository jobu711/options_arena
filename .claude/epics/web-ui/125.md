# Task: Debate frontend — DebateResultPage + AgentCards + progress modal

## Description

Implement the debate frontend. After this task, a user can trigger a single-ticker debate
from the scan results page, watch agent-by-agent progress in a modal overlay, and view
the full debate result page with side-by-side agent cards.

### Pages

#### DebateResultPage (`web/src/pages/DebateResultPage.vue`)
- Agent cards in side-by-side layout (2-column grid):
  - Bull card (green accent)
  - Bear card (red accent)
  - Rebuttal card (light green, conditional on `bull_rebuttal !== null`)
  - Volatility card (purple, conditional on `vol_response !== null`)
- Risk / Thesis banner below cards:
  - Verdict (BULLISH/BEARISH/NEUTRAL) with DirectionBadge
  - Confidence bar + percentage
  - Strategy, entry/exit prices, risk/reward
- Metadata strip: model name, duration, token usage, citation density, is_fallback flag
- Export button: dropdown with Markdown / PDF → triggers file download

### Components

#### AgentCard (`web/src/components/AgentCard.vue`)
- Props: `agentName: string`, `response: AgentResponse`, `color: string`
- Layout: name + icon header, confidence bar, key points list, risks list
- Color scheme per `web/CLAUDE.md` (bull=green, bear=red, risk=blue, vol=purple, rebuttal=light green)

#### DebateProgressModal (`web/src/components/DebateProgressModal.vue`)
- PrimeVue Dialog (modal overlay)
- Shows on scan results page when debate is triggered
- Agent-by-agent progress: list of agents with started/completed/failed status icons
- Confidence shown next to each completed agent
- Auto-navigates to `/debate/:id` when complete event received

### Stores

#### debate store (`web/src/stores/debate.ts`)
- State: `debates`, `currentDebate`, `agentProgress`, `loading`
- Actions: `startDebate(ticker, scanId)`, `fetchDebate(id)`, `fetchDebates()`,
  `updateAgentProgress()`, `setDebateComplete()`
- Getters: `isDebating`

### Types

- `web/src/types/debate.ts` — `DebateResult`, `AgentResponse`, `TradeThesis`,
  `VolatilityThesis`, `DebateProgress`, `DebateResultSummary`
- `web/src/types/ws.ts` — extend with `DebateEvent` discriminated union

### Wire up debate trigger on ScanResultsPage

- "Debate" button per row → calls `debateStore.startDebate(ticker, scanId)`
- Opens `DebateProgressModal` with WebSocket connection
- On complete → navigates to `/debate/:debate_id`

## Acceptance Criteria

- [ ] "Debate" button on scan results row triggers single-ticker debate
- [ ] DebateProgressModal shows agent-by-agent progress via WebSocket
- [ ] Modal auto-navigates to debate result page on completion
- [ ] DebateResultPage shows all agent cards with correct colors
- [ ] AgentCard displays confidence, key points, risks
- [ ] Rebuttal and volatility cards render conditionally (only if present)
- [ ] Thesis banner shows verdict, confidence, strategy, entry/exit
- [ ] Export button downloads Markdown or PDF file
- [ ] Metadata strip shows model, duration, tokens, citation density
- [ ] Fallback debates show visual indicator (`is_fallback=true`)
- [ ] Null fields (`vol_response`, `bull_rebuttal`) handled gracefully with `v-if`

## Technical Details

- Read `web/CLAUDE.md` before implementing — component patterns, typing, styling
- Agent colors: bull `#22c55e`, bear `#ef4444`, risk `#3b82f6`, vol `#a855f7`, rebuttal `#86efac`
- Export: `GET /api/debate/{id}/export?format=md` → trigger browser download via `<a>` tag

- WebSocket connection: use `useWebSocket<DebateEvent>` composable from Task #128
- Debate progress modal is on ScanResultsPage, NOT a separate route
