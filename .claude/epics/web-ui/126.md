# Task: Scan API — REST endpoints + WebSocket progress

## Description

Implement the full scan API: REST endpoints for starting scans, listing past scans,
fetching scores, and a WebSocket for real-time scan progress streaming. This is the
backend half of the "scan vertical slice."

### REST endpoints (`src/options_arena/api/routes/scan.py`)

```
POST /api/scan            — Start a new scan (preset: sp500|full|etfs). Returns 202 + scan_id.
                            Acquires operation_lock. 409 if already busy.
GET  /api/scan             — List past scans (newest first, ?limit=10)
GET  /api/scan/{scan_id}   — Get single scan run metadata
GET  /api/scan/{scan_id}/scores — Paginated scores (?sort, ?order, ?direction, ?min_score,
                                   ?search, ?page, ?page_size). Returns PaginatedResponse.
GET  /api/scan/{scan_id}/scores/{ticker} — Single ticker detail (score + contracts)
DELETE /api/scan/current    — Cancel running scan. 404 if no scan in progress.
```

### WebSocket (`src/options_arena/api/ws.py`)

```
WS /ws/scan/{scan_id}  — Subscribe to scan progress events
```

Implement `WebSocketProgressBridge` — adapts the scan pipeline's sync `ProgressCallback`
to an `asyncio.Queue`. WebSocket handler drains the queue and sends JSON events:
- `{type: "progress", phase, current, total}`
- `{type: "error", ticker, message}`
- `{type: "complete", scan_id, cancelled}`

Client → Server: `{type: "cancel"}` sets `CancellationToken`.

### Request/response schemas (`src/options_arena/api/schemas.py`)

- `ScanRequest(BaseModel)` — `preset: ScanPreset`
- `ScanStarted(BaseModel)` — `scan_id: int`
- `PaginatedResponse(BaseModel, Generic[T])` — `items, total, page, pages`
- `TickerDetail(BaseModel)` — `score: TickerScore, contracts: list[OptionContract]`

### Background task pattern

`POST /api/scan` starts the pipeline as a `asyncio.create_task()`, stores the task and
`CancellationToken` in `app.state` keyed by scan_id. The WebSocket handler reads from
the bridge queue. On completion, release the operation lock.

## Acceptance Criteria

- [ ] `POST /api/scan` returns 202 with scan_id, starts pipeline in background
- [ ] `POST /api/scan` returns 409 when another operation is running
- [ ] `GET /api/scan` returns list of past scans (newest first)
- [ ] `GET /api/scan/{id}/scores` returns paginated, sorted, filtered scores
- [ ] `GET /api/scan/{id}/scores/{ticker}` returns score + contracts
- [ ] `DELETE /api/scan/current` cancels a running scan
- [ ] WebSocket streams progress events in real-time
- [ ] WebSocket accepts `{type: "cancel"}` and triggers cancellation
- [ ] WebSocket sends `{type: "complete"}` when scan finishes
- [ ] Operation lock released on scan completion (success or failure)
- [ ] All endpoints tested with FastAPI `TestClient` + mocked services
- [ ] `ruff check`, `ruff format`, `mypy --strict` all clean
