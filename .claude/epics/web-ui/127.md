# Task: Batch debate — API endpoint + batch progress modal

## Description

Add batch debate support: API endpoint to debate top N tickers from a scan, WebSocket
for batch progress, and a batch progress modal on the scan results page with per-ticker
mini-results and a summary table.

### Backend

#### REST endpoint (`src/options_arena/api/routes/debate.py` — extend)

```
POST /api/debate/batch
  Body: { "scan_id": int, "limit": 5, "tickers": ["AAPL", "MSFT", ...] | null }
  202: { "batch_id": int, "tickers": ["AAPL", "MSFT", ...] }
  409: "Another operation is in progress"
```

- If `tickers` provided, debate those specific tickers. Otherwise, take top `limit` from scan.
- Acquires operation lock (batch debates are long-running).
- Runs debates sequentially (reuses `_debate_single()` pattern from CLI).
- Each ticker: fetch quote + contracts, run debate, persist.
- Error isolation: per-ticker try/except, failures included in results.

#### WebSocket (`src/options_arena/api/ws.py` — extend)

```
WS /ws/batch/{batch_id}
```

Events:
- `{type: "batch_progress", ticker: "AAPL", index: 1, total: 5, status: "started"|"completed"|"failed"}`
- `{type: "agent", ticker: "AAPL", name: "bull", status: "started", confidence: null}` (per-agent within ticker)
- `{type: "batch_complete", results: [{ticker, debate_id, verdict, confidence, error}]}`

### Frontend

#### Batch trigger on ScanResultsPage

- Checkbox selection column on DataTable (already scaffolded in Task #128)
- "Debate Selected" button visible when rows selected → calls batch API with selected tickers
- If no rows selected, "Debate Top 5" button → calls batch API with `limit: 5`

#### DebateProgressModal — extend for batch mode

- Batch mode: shows current ticker name, overall progress bar (3/5), mini-result cards
  as each completes (ticker + verdict + confidence)
- On batch complete: show summary table with all results, link each to `/debate/:id`
- "Debate Selected" button disabled while batch is running (operation lock)

#### operation store — extend

- Track `operationType: "batch_debate"` when batch is running
- Disable "Run Scan" and "Debate Selected" buttons
