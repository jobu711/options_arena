# Task: Scan frontend — ScanPage + ScanResultsPage + TickerDrawer

## Description

Implement the scan frontend pages. After this task, a user can run a scan from the
browser, watch progress in real-time, browse sorted/filtered results in a PrimeVue
DataTable, and click a row to see ticker detail in a drawer.

### Pages

#### ScanPage (`web/src/pages/ScanPage.vue`)
- Preset dropdown (sp500 / full / etfs) + "Run Scan" button
- Progress panel (visible during scan): `ProgressTracker` component with 4-phase bar + cancel button
- Past scans list: PrimeVue DataTable with date, preset, ticker count, click to view results

#### ScanResultsPage (`web/src/pages/ScanResultsPage.vue`)
- PrimeVue `DataTable` with all TickerScore columns
- Sort: click header (synced to URL `?sort=&order=`)
- Filter: direction dropdown, min score slider, ticker search (synced to URL)
- Pagination: 50 rows/page, PrimeVue Paginator (synced to URL `?page=`)
- Virtual scroll for > 1,000 rows
- Row selection (checkbox column) for batch debate (wired in Task #127)
- Row click → opens TickerDrawer
- "Debate" button per row (wired in Task #125)

### Components

#### ProgressTracker (`web/src/components/ProgressTracker.vue`)
- Props: `phases: string[]`, `currentPhase: string`, `current: number`, `total: number`
- Multi-phase progress bar with labeled phases, filled/unfilled states
- Cancel button emits `@cancel` event

#### DirectionBadge (`web/src/components/DirectionBadge.vue`)
- Props: `direction: "bullish" | "bearish" | "neutral"`
- Colored pill: green/red/yellow with white/dark text

#### ConfidenceBadge (`web/src/components/ConfidenceBadge.vue`)
- Props: `value: number` (0.0–1.0)
- Renders as percentage with color coding (low/medium/high)

#### TickerDrawer (`web/src/components/TickerDrawer.vue`)
- PrimeVue Drawer (right-side slide-out)
- Indicator breakdown (18 signals with raw + normalized values)
- Recommended contracts table (strike, type, bid/ask, mid, Greeks, IV)
- Past debate history for this ticker (links to `/debate/:id`)

### Stores

#### scan store (`web/src/stores/scan.ts`)
- State: `scans`, `currentScanId`, `scores`, `progress`, `loading`, `errors`
- Actions: `fetchScans()`, `fetchScores(scanId)`, `updateProgress()`, `addError()`, `setComplete()`
- Getters: `latestScan`, `isScanning`

#### operation store (`web/src/stores/operation.ts`)
- State: `operationInProgress`, `operationType` (scan | batch_debate | null)
- Used to disable "Run Scan" when already busy

### Composables

#### useWebSocket (`web/src/composables/useWebSocket.ts`)
- Generic typed WebSocket composable per `web/CLAUDE.md` pattern
- Auto-reconnect with exponential backoff (max 5 attempts)
- `onUnmounted` cleanup

#### useApi (`web/src/composables/useApi.ts`)
- Typed fetch wrapper per `web/CLAUDE.md` pattern
- Base URL from `import.meta.env.VITE_API_URL`
- `ApiError` class with status code

### Types

- `web/src/types/scan.ts` — `ScanRun`, `TickerScore`, `ScanProgress`, `PaginatedResponse`
- `web/src/types/ws.ts` — `ScanEvent` discriminated union (progress | error | complete)
- `web/src/types/index.ts` — re-exports
