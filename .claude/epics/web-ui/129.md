# Task: Supporting pages — Dashboard + Universe + Health + Config

## Description

Implement the remaining supporting pages and API endpoints: Dashboard (home page),
Universe management, Health monitoring, and read-only Config endpoint. These are
lower-priority pages that complete the feature set.

### Backend

#### Health endpoint (`src/options_arena/api/routes/health.py`)

```
GET /api/health → [HealthStatus, ...]
```

Calls existing `check_all_health()` from `services/health.py`. Returns list of service
health statuses (Groq, Yahoo Finance, FRED, CBOE).

#### Universe endpoints (`src/options_arena/api/routes/universe.py`)

```
GET  /api/universe         → { optionable_count, sp500_count, last_refresh }
POST /api/universe/refresh → { optionable_count, sp500_count }
```

Uses existing `UniverseService`. Refresh triggers re-fetch of CBOE + Wikipedia data.

#### Config endpoint (`src/options_arena/api/routes/config.py`)

```
GET /api/config → { groq_api_key_set, scan_preset_default, enable_rebuttal,
                     enable_volatility_agent, agent_timeout }
```

Read-only. Exposes safe config values (never the actual API key, just `bool` for whether set).

### Frontend

#### DashboardPage (`web/src/pages/DashboardPage.vue`)
- Latest scan card: preset, date, ticker count, top 5 scorers with DirectionBadge
- Health strip: HealthDot per service, auto-refreshes every 60s
- Quick actions: "Run Scan" button → navigates to `/scan`, "View Latest Results" link
- Recent debates: last 5 debates with ticker, verdict, confidence badge

#### UniversePage (`web/src/pages/UniversePage.vue`)
- Stats bar: total optionable tickers, S&P 500 count, last refresh timestamp
- "Refresh" button with loading indicator → `POST /api/universe/refresh`
- Ticker list: searchable PrimeVue DataTable (ticker, sector for S&P 500)

#### HealthPage (`web/src/pages/HealthPage.vue`)
- Service cards: one per service (Groq, Yahoo Finance, FRED, CBOE)
- Each card: status badge (OK/Down), latency, last checked, error message if any
- "Re-check" button → `GET /api/health`, updates cards

### Components

#### HealthDot (`web/src/components/HealthDot.vue`)
- Props: `status: "ok" | "degraded" | "down"`
- Colored dot: green/yellow/red

### Stores

#### health store (`web/src/stores/health.ts`)
- State: `services` (list of health statuses), `loading`
- Actions: `fetchHealth()`, `refreshHealth()`
- Auto-refresh interval (60s) for dashboard health strip
